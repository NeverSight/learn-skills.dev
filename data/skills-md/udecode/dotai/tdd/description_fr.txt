À utiliser lors de la mise en œuvre d'une logique complexe nécessitant une couverture de test : écrivez d'abord le test, regardez-le échouer, écrivez un minimum de code à réussir ; garantit que les tests vérifient réellement le comportement en exigeant d'abord un échec ; PAS pour les composants d'interface utilisateur, le CRUD simple ou le code simple
