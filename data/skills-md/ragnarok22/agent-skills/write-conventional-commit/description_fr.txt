Créez des messages de validation Git conformes aux Commits conventionnels 1.0.0, y compris le format type/portée/description, le corps facultatif, les pieds de page de style bande-annonce et la signalisation explicite BREAKING CHANGE. À utiliser lorsque les utilisateurs demandent de rédiger des messages de validation, de valider les modifications actuelles, de réécrire un message de validation dans un format conventionnel ou d'appliquer des normes de validation conventionnelles dans un dépôt.
