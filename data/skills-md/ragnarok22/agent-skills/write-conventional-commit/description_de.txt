Erstellen Sie Git-Commit-Nachrichten, die den herkömmlichen Commits 1.0.0 entsprechen, einschließlich Typ-/Umfangs-/Beschreibungsformat, optionalem Text, Fußzeilen im Trailer-Stil und expliziter BREAKING CHANGE-Signalisierung. Wird verwendet, wenn Benutzer Commit-Nachrichten entwerfen, aktuelle Änderungen festschreiben, eine Commit-Nachricht in ein herkömmliches Format umschreiben oder herkömmliche Commit-Standards in einem Repository durchsetzen möchten.
