Crea messaggi di commit Git conformi a Conventional Commits 1.0.0, incluso il formato di tipo/ambito/descrizione, corpo facoltativo, piè di pagina in stile trailer e segnalazione esplicita di BREAKING CHANGE. Da utilizzare quando gli utenti chiedono di bozza di messaggi di commit, di commit delle modifiche correnti, di riscrivere un messaggio di commit in formato convenzionale o di applicare standard di commit convenzionali in un repository.
