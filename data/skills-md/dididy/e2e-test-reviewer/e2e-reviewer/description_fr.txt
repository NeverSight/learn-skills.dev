À utiliser lors de la révision, de l'audit ou de l'amélioration des spécifications de test E2E pour Playwright, Cypress ou Puppeteer : analyse de code statique des fichiers de test existants, sans diagnostic des échecs d'exécution. Déclencheurs sur "examiner mes tests", "auditer la qualité des tests", "trouver les tests faibles", "mes tests réussissent toujours mais manquent des bogues", "les tests réussissent CI mais manquent des régressions", "améliorer les tests de dramaturge", "améliorer les tests de cyprès", "vérifier les lacunes de couverture des tests", "mes tests sont fragiles", "les tests s'interrompent à chaque changement d'interface utilisateur", "la suite de tests est difficile à maintenir", "nous avons une couverture mais des bugs passent toujours". Détecte 10 anti-modèles : incompatibilité d'assertion de nom, puis manquant, avalement d'erreurs, assertions qui passent toujours, modèles de contournement (assertions conditionnelles + force : vrai), requêtes DOM brutes, fuite de test ciblée (test.only validé), modèles de test irréguliers (sélecteurs de position + ordre en série), mises en veille codées en dur et spécifications YAGNI + zombie (membres POM inutilisés, wrappers Util à usage unique, fichiers de spécifications zombie).
