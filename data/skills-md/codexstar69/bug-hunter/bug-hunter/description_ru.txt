Состязательный поиск ошибок с помощью последовательного конвейера (Recon, Hunter, Skeptic, Referee), который дополнительно может использовать безопасную параллельную сортировку только для чтения. Находит, проверяет и автоматически исправляет реальные ошибки по умолчанию (с отключением --scan-only), используя проверку контрольных точек и состояние возобновления для больших баз кода. Используйте этот навык всякий раз, когда пользователь хочет найти ошибки, провести аудит безопасности, регрессионную проверку или проверку кода, ориентированную на поведение во время выполнения.
