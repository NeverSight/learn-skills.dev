Checkliste für Sicherheitsaudits für Webanwendungen. Verwenden Sie diese Option, wenn Sie den Sicherheitsstatus einer Web-App überprüfen, prüfen oder verstärken. Deckt Ratenbegrenzung, Authentifizierungsheader, IP-Blockierung, CORS, Sicherheits-Middleware, Eingabevalidierung, Datei-Upload-Limits, ORM-Nutzung und Passwort-Hashing ab. Wird bei Anfragen wie „Sicherheit überprüfen“, „Diese App härten“, „Sicherheitsüberprüfung“, „Auf Schwachstellen prüfen“ oder beim Erstellen/Überprüfen von API-Endpunkten ausgelöst.
