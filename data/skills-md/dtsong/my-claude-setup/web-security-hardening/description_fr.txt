Liste de contrôle d'audit de sécurité pour les applications Web. À utiliser lors de l'examen, de l'audit ou du renforcement de la posture de sécurité d'une application Web. Couvre la limitation de débit, les en-têtes d'authentification, le blocage IP, CORS, le middleware de sécurité, la validation des entrées, les limites de téléchargement de fichiers, l'utilisation d'ORM et le hachage de mot de passe. Se déclenche sur des requêtes telles que « vérifier la sécurité », « renforcer cette application », « audit de sécurité », « vérifier les vulnérabilités » ou lors de la création/révision des points de terminaison de l'API.
