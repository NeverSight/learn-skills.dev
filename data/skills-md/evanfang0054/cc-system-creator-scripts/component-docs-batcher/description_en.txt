Automated tool for batch-generating and maintaining frontend component-library documentation. Designed for Monorepo component libraries (e.g. atom-ui-mobile), it compares the modification times of component files and their docs, automatically generates task lists, and batch-produces specification-compliant component documentation. Supports: (1) periodic batch updates of component docs, (2) single-component doc maintenance, (3) new-component doc initialization. Automatically identifies main component files, skips auxiliary files like index.ts, store, hooks, etc.; docs are placed at index.zh-CN.md inside each component directory.
