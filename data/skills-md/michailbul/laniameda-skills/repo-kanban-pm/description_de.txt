Installieren und erzwingen Sie einen einfachen Produktmanagement-Workflow in einem Code-Repository: Feature-as-Kanban-Boards, ROADMAP-Statusverfolgung, Filial-/PR-Konventionen und eine optionale tägliche OpenClaw-Cron-PM-Überprüfung. Verwenden Sie diese Fähigkeit, wenn (1) Sie mit der Arbeit in einem NEUEN Repository/Projekt beginnen, das noch nicht über ein ROADMAP+Feature-KANBAN-System verfügt, (2) Sie mit der Inbetriebnahme eines neuen Projekts beauftragt werden und vom ersten Tag an eine Multi-Agenten-Koordination wünschen oder (3) Sie aufgefordert werden, ein bestehendes Repository zu reparieren/umzustrukturieren und ein organisiertes Backlog-/Feature-Tracking-System einzuführen. Auch beim Hinzufügen einer täglichen PM-Audit-Schleife (Cron) verwenden, um Code + Dokumente + PRs synchron zu halten.
