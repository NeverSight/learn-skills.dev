Génère des interfaces de référentiel indépendantes du langage (signatures de méthode, types d'entrée/sortie et modèles d'entité) à partir d'un document de conception de table DynamoDB. À utiliser lorsque l'utilisateur dispose d'un fichier .md de conception de table (issu de la compétence dynamodb-table-design) et souhaite générer le squelette de code pour sa couche d'accès aux données. Il s'agit de l'étape 3 d'un pipeline en 3 étapes : modèles d'accès -> conception de table -> interfaces de requête.
