Genera interfaces de repositorio independientes del idioma (firmas de métodos, tipos de entrada/salida y modelos de entidad) a partir de un documento de diseño de tabla de DynamoDB. Úselo cuando el usuario tiene un archivo .md de diseño de tabla (de la habilidad dynamodb-table-design) y desea generar el esqueleto del código para su capa de acceso a datos. Este es el paso 3 de un proceso de 3 pasos: patrones de acceso -> diseño de tablas -> interfaces de consulta.
