Genera interfacce di repository indipendenti dalla lingua (firme di metodo, tipi di input/output e modelli di entità) da un documento di progettazione di tabelle DynamoDB. Da utilizzare quando l'utente dispone di un file .md di progettazione della tabella (dalla competenza dynamodb-table-design) e desidera generare la struttura del codice per il proprio livello di accesso ai dati. Questo è il passaggio 3 di una pipeline in 3 passaggi: modelli di accesso -> progettazione tabella -> interfacce di query.
