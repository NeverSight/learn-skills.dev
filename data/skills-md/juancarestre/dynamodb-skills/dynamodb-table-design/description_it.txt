Progetta una struttura a tabella singola DynamoDB da un set documentato di modelli di accesso. Da utilizzare quando l'utente dispone di un file .md dei modelli di accesso (dalla competenza dynamodb-access-patterns) e desidera progettare chiavi di partizione, chiavi di ordinamento, GSI, chiavi composite e mappare ogni modello di accesso a un'operazione di query DynamoDB concreta. Questo è il passaggio 2 di una pipeline in 3 passaggi: modelli di accesso -> progettazione tabella -> interfacce di query.
