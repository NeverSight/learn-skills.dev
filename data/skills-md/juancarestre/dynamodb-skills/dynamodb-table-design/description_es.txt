Diseña una estructura de tabla única de DynamoDB a partir de un conjunto documentado de patrones de acceso. Úselo cuando el usuario tiene un archivo .md de patrones de acceso (de la habilidad dynamodb-access-patterns) y desea diseñar claves de partición, claves de clasificación, GSI, claves compuestas y asignar cada patrón de acceso a una operación de consulta de DynamoDB concreta. Este es el paso 2 de un proceso de 3 pasos: patrones de acceso -> diseño de tabla -> interfaces de consulta.
