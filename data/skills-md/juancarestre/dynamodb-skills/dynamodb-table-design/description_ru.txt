Разрабатывает однотабличную структуру DynamoDB на основе документированного набора шаблонов доступа. Используйте, когда у пользователя есть файл шаблонов доступа .md (на основе навыка dynamodb-access-patterns) и он хочет спроектировать ключи разделов, ключи сортировки, GSI, составные ключи и сопоставить каждый шаблон доступа с конкретной операцией запроса DynamoDB. Это второй шаг трехэтапного конвейера: шаблоны доступа -> дизайн таблиц -> интерфейсы запросов.
