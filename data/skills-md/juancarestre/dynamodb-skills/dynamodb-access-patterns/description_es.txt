Habilidad interactiva para obtener, formalizar y persistir patrones de acceso a DynamoDB. Úselo cuando el usuario quiera comenzar a diseñar una tabla de DynamoDB, definir entidades o documentar cómo su aplicación leerá y escribirá datos. Este es el paso 1 de un proceso de 3 pasos: patrones de acceso -> diseño de tablas -> interfaces de consulta. El resultado es un archivo .md estructurado que alimenta la habilidad dynamodb-table-design.
