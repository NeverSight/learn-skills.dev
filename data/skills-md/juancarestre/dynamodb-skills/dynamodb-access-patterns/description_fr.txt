Compétence interactive pour obtenir, formaliser et conserver les modèles d'accès DynamoDB. À utiliser lorsque l'utilisateur souhaite commencer à concevoir une table DynamoDB, définir des entités ou documenter la manière dont son application lira et écrira des données. Il s'agit de l'étape 1 d'un pipeline en 3 étapes : modèles d'accès -> conception de table -> interfaces de requête. Le résultat est un fichier .md structuré qui alimente la compétence dynamodb-table-design.
