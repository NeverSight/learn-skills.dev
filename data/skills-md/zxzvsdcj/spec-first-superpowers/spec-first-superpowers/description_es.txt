Aplica el flujo de trabajo de especificaciones antes del código para el desarrollo impulsado por IA. Selecciona automáticamente el modo Spec-Kit u OpenSpec, clasifica la complejidad (rápida/estándar/completa), recupera el contexto de la sesión y aplica puertas de calidad (G0-G4) en cada etapa. Utilice esta habilidad siempre que el usuario diga "/super-spec", "spec first", "规范先行" o inicie cualquier función, corrección de errores o refactorización, especialmente en proyectos con directorios .spec-mode, .specify/ o .openspec/. Incluso si el usuario no solicita explícitamente un flujo de trabajo basado en especificaciones, active esta habilidad para cualquier cambio de código no trivial para evitar omitir la fase de diseño. Orquesta: Spec-Kit/OpenSpec + planificación-con-archivos + ui-ux-pro-max + Superpoderes (TDD, revisión de código, verificación, depuración).
