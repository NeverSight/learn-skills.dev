Applique le flux de travail de spécification avant le code pour le développement basé sur l'IA. Sélectionne automatiquement le mode Spec-Kit ou OpenSpec, trie la complexité (rapide/standard/approfondie), récupère le contexte de session et applique des portes de qualité (G0-G4) à chaque étape. Utilisez cette compétence chaque fois que l'utilisateur dit "/super-spec", "spec first", "规范先行" ou démarre une fonctionnalité, une correction de bug ou un refactor, en particulier dans les projets avec des répertoires .spec-mode, .specify/ ou .openspec/. Même si l'utilisateur ne demande pas explicitement un flux de travail basé sur les spécifications, activez cette compétence pour toute modification de code non triviale afin d'éviter de sauter la phase de conception. Orchestre : Spec-Kit/OpenSpec + planification-avec-fichiers + ui-ux-pro-max + Superpouvoirs (TDD, révision de code, vérification, débogage).
