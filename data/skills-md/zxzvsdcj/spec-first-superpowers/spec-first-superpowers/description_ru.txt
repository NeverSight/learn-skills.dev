Обеспечивает рабочий процесс «спецификация перед написанием кода» для разработки на основе искусственного интеллекта. Автоматически выбирает режим Spec-Kit или OpenSpec, сортирует сложность (быстрая/стандартная/тщательная), восстанавливает контекст сеанса и применяет шлюзы качества (G0–G4) на каждом этапе. Используйте этот навык всякий раз, когда пользователь говорит «/super-spec», «spec first», «规范先行» или запускает какую-либо функцию, исправление ошибок или рефакторинг — особенно в проектах с каталогами .spec-mode, .specify/ или .openspec/. Даже если пользователь явно не запрашивает рабочий процесс, основанный на спецификациях, активируйте этот навык для любого нетривиального изменения кода, чтобы не пропустить этап проектирования. Организует: Spec-Kit/OpenSpec + планирование с помощью файлов + ui-ux-pro-max + супервозможности (TDD, проверка кода, проверка, отладка).
