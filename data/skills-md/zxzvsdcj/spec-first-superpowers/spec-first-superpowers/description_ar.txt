يفرض سير عمل المواصفات قبل التعليمات البرمجية للتطوير القائم على الذكاء الاصطناعي. تحديد وضع Spec-Kit أو OpenSpec تلقائيًا، وفرز التعقيد (سريع/قياسي/شامل)، واستعادة سياق الجلسة، وتطبيق بوابات الجودة (G0-G4) في كل مرحلة. استخدم هذه المهارة عندما يقول المستخدم "/super-spec" أو "spec first" أو "规范先行" أو يبدأ أي ميزة أو إصلاح خطأ أو إعادة بناء - خاصة في المشاريع التي تحتوي على أدلة .spec-mode أو .specify/ أو .openspec/. حتى إذا لم يطلب المستخدم صراحةً سير عمل يعتمد على المواصفات، فقم بتنشيط هذه المهارة لأي تغيير غير تافه في التعليمات البرمجية لمنع تخطي مرحلة التصميم. التنسيق: Spec-Kit/OpenSpec + التخطيط مع الملفات + ui-ux-pro-max + القوى العظمى (TDD، مراجعة التعليمات البرمجية، التحقق، تصحيح الأخطاء).
