Erstellen Sie schnell ein spielbares Spiel und implementieren Sie es – ohne Assets, Design, Audio, Bereitstellung oder Monetarisierung. Bringen Sie schnell etwas auf den Bildschirm. Verwenden Sie diese Option, wenn der Benutzer „schnelles Spiel“, „schneller Prototyp“, „einfach spielbares Produkt“ sagt oder ein Spiel ohne die vollständige Pipeline möchte. Für die komplette Pipeline verwenden Sie stattdessen make-game. NICHT für Produktionsspiele verwenden (verwenden Sie make-game für die gesamte Pipeline).
