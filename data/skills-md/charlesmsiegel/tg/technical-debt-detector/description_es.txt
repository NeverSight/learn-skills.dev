Identificar y priorizar la deuda técnica en las bases de código Python. Úselo cuando el usuario solicite encontrar deuda tecnológica, analizar la calidad del código, identificar qué necesita refactorización, encontrar problemas de seguridad, verificar brechas de cobertura de pruebas, revisar dependencias, encontrar TODO/FIXME o evaluar la mantenibilidad. Activadores de frases como "encontrar deuda técnica", "qué hay de malo en este código base", "dónde debería centrarme en la refactorización", "auditar este código", "encontrar TODO", "verificar problemas de seguridad", "analizar dependencias" o "qué necesita pruebas". Complementa la habilidad simplificadora de Python (úsela para análisis de complejidad y olor de código).
