Identifiez et hiérarchisez la dette technique dans les bases de code Python. À utiliser lorsque l'utilisateur demande à trouver une dette technologique, à analyser la qualité du code, à identifier ce qui nécessite une refactorisation, à rechercher des problèmes de sécurité, à vérifier les lacunes de couverture des tests, à examiner les dépendances, à rechercher des TODO/FIXME ou à évaluer la maintenabilité. Se déclenche sur des expressions telles que « trouver une dette technique », « qu'est-ce qui ne va pas avec cette base de code », « sur quoi dois-je me concentrer sur la refactorisation », « auditer ce code », « trouver des TODO », « vérifier les problèmes de sécurité », « analyser les dépendances » ou « ce qui nécessite des tests ». Complète la compétence Python-simplifier (utilisez-la pour l'analyse de la complexité et de l'odeur du code).
