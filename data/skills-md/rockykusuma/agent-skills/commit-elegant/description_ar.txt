ينشئ التزامات git أنيقة باستخدام تنسيق الالتزام التقليدي للرموز التعبيرية. يتعامل مع التحقق قبل الالتزام، والتشغيل المرحلي التلقائي، ويحلل الاختلافات للتجميع المنطقي، ويقترح تقسيم التغييرات الكبيرة إلى التزامات مركزة، ويكتب رسائل التزام حتمية واضحة. استخدم هذه المهارة عندما يريد المستخدم تنفيذ التغييرات أو تنظيم الملفات أو صياغة رسالة التزام. يدعم --no-verify لتخطي عمليات التحقق المسبقة.
