Diagnostizieren und stellen Sie fehlerhafte Git-Status- und Worktree-Metadaten mit einem abgestuften, risikoarmen Wiederherstellungsablauf wieder her. Wird verwendet, wenn Git einen getrennten oder widersprüchlichen HEAD-Status, Phantom-Worktree-Sperren, verwaiste Worktree-Einträge, fehlende Refs, 0000000000000000000000000000000000000000000000 Hashes meldet oder Verzweigungsvorgänge mit Fehlern wie „bereits ausgecheckt“, „unbekannte Revision“, „kein gültiger Objektname“ oder „Ref kann nicht gesperrt werden“ fehlschlagen.
