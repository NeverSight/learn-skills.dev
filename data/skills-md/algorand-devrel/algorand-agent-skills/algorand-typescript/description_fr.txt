Règles et modèles de syntaxe pour les contrats intelligents Algorand TypeScript (PuyaTs). À utiliser lors de l'écriture de code de contrat TypeScript, de la rencontre d'erreurs du compilateur Puya, de questions sur les types AVM ou la sémantique des valeurs, d'avoir besoin de conseils sur les modèles de stockage (GlobalState, BoxMap) ou de questions sur clone(), les tableaux ou les transactions internes. Les déclencheurs puissants incluent « Erreur du compilateur Puya », « Comment utiliser uint64 ? », « À quoi sert clone() ? », « BoxMap ne fonctionne pas », « Erreur de type AVM », « GlobalState ne se met pas à jour ».
