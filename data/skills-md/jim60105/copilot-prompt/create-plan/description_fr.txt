Créez un plan de développement détaillé et des problèmes GitHub pour un projet. À utiliser lorsque l'utilisateur souhaite planifier une nouvelle fonctionnalité ou un nouveau correctif, créer des problèmes GitHub pour les retards ou les bogues, ou rédiger un plan de développement complet avec des répartitions de tâches, des dépendances et des stratégies de test. Les plans sont rédigés en chinois traditionnel avec des exemples de code en anglais.
