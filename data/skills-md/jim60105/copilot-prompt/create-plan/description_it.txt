Creare un piano di sviluppo dettagliato e problemi GitHub per un progetto. Da utilizzare quando l'utente desidera pianificare una nuova funzionalità o una correzione, creare problemi GitHub per arretrati o bug o redigere un piano di sviluppo completo con suddivisioni delle attività, dipendenze e strategie di test. I piani sono scritti in cinese tradizionale con esempi di codice inglese.
