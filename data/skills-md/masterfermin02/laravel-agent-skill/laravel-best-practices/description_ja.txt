Laravel で構築された PHP アプリケーションを作成、レビュー、テストするための Laravel フレームワーク スキル。 Laravel 固有のパターン (コントローラー、FormRequest、Eloquent モデル、サービス/アクション クラス、Blade テンプレート、ジョブ、イベント、ポリシー、オブザーバー、またはアーティザン コマンド) を扱うときは常に使用します。 Laravel プロジェクト内に埋め込まれた Inertia フロントエンド (React または Vue) にも適用されます。これには、useForm、usePage、リンク ナビゲーション、部分リロード、ページ コンポーネントの命名 (PostsIndexPage)、および shadcn ラッパー規約が含まれます。誰かが Laravel、Eloquent、Artisan、Blade、FormRequest、Laravel 内の慣性、Laravel アプリの N+1 クエリ、インライン コントローラー検証、ビジネス ロジックが Laravel コントローラーに存在する場所について言及したとき、または Laravel コードベースのベスト プラクティスに従うように求められたときにトリガーします。 Laravel、Rails、Django、スタンドアロン Next.js、または非 Laravel フレームワークを使用しないプレーンな PHP スクリプトには適用されません。
