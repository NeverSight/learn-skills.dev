Vérifie si la complexité est transmise aux appelants ou absorbée par les implémentations. À utiliser lorsque les appelants doivent effectuer une configuration importante, gérer les erreurs que le module pourrait résoudre ou configurer des choses qu'ils ne comprennent pas. Cette compétence se concentre spécifiquement sur la direction des flux de complexité. Pas pour évaluer la profondeur globale du module (utilisez des modules approfondis) ou pour vérifier les fuites de connaissances au-delà des frontières (utilisez la dissimulation d'informations).
