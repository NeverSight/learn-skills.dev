Diagnostica qué hace que el código sea complejo y por qué, utilizando el marco de tres síntomas y dos causas fundamentales. Úselo cuando resulte más difícil trabajar con el código de lo que debería, pero el problema específico no está claro. Esta habilidad identifica SI existe complejidad y DE DÓNDE viene. No para escanear una lista de verificación de olores de diseño conocidos (use señales de alerta) o evaluar la profundidad de un módulo específico (use módulos profundos).
