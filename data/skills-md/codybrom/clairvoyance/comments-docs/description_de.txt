Überprüft die Kommentarqualität und Dokumentationspraktiken. Verwenden Sie diese Option, wenn der Benutzer darum bittet, Kommentare oder Dokumentation zu überprüfen, wenn Kommentare nur den Code wiederholen, wenn etwas in einem Satz schwer zu beschreiben ist oder wenn Dokumentation vor dem Code geschrieben wird, um Designprobleme an die Oberfläche zu bringen. Wertet die vier Kommentartypen aus: Workflow „Kommentare zuerst“ und „Kommentar rotieren“.
