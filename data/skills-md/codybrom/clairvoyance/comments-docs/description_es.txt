Las reseñas comentan las prácticas de calidad y documentación. Úselo cuando el usuario solicite revisar comentarios o documentación, cuando los comentarios simplemente repitan el código, cuando algo sea difícil de describir en una oración o cuando escriba documentación antes del código para resaltar problemas de diseño. Evalúa los cuatro tipos de comentarios, el flujo de trabajo de comentarios primero y la descomposición de comentarios.
