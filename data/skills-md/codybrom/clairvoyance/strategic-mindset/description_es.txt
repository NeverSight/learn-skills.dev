Evalúa si el código refleja un pensamiento estratégico o táctico. Úselo cuando el usuario solicite evaluar la inversión en diseño, cuando el código se escribió bajo presión de tiempo, cuando un desarrollador produce constantemente código funcional que degrada el sistema o cuando se evalúa si una base de código invierte en diseño. Comprueba la regla de inversión del 10-20% y los patrones tácticos de tornado.
