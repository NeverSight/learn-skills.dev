엔터프라이즈 데이터베이스 개발을 위한 PostgreSQL 18+ 포괄적인 모범 사례입니다. 스키마 제공
아키텍처 패턴, 테이블 API 디자인, PL/pgSQL 코딩 표준, 마이그레이션 및 데이터 웨어하우징.

사용자가 다음과 같은 경우에 이 기술을 사용하십시오:
- PostgreSQL 스키마, 테이블, 함수, 프로시저 또는 트리거를 생성합니다.
- PostgreSQL 데이터 유형(uuid, text, timestamptz, jsonb, 숫자)에 대해 질문합니다.
- PL/pgSQL 코드를 작성하고 명명 규칙(l_, in_, io_, co_ 접두사)이 필요합니다.
- Table API 패턴 구현 (SECURITY DEFINER 함수, 스키마 분리)
- 데이터베이스 마이그레이션 또는 스키마 버전 관리 설정
- 인덱스 최적화, 제약 조건 설계 또는 쿼리 성능 도움이 필요합니다.
- PostgreSQL 18+ 기능(uuidv7, 가상 열, 임시 제약 조건)에 대해 질문합니다.
- Medallion Architecture(Bronze/Silver/Gold)로 데이터 웨어하우스 구축
- 데이터 계보 추적, ETL 패턴 또는 감사 로깅이 필요합니다.
- 모범 사례 또는 안티 패턴에 대한 데이터베이스 코드 검토
- Oracle PL/SQL에서 PostgreSQL PL/pgSQL로 마이그레이션
- 데이터베이스 변경을 위한 CI/CD 파이프라인 설정

핵심 패턴:
- 3-스키마 분리 : 데이터(테이블) → 프라이빗(내부) → API(외부)
- Table API: SET search_path를 사용한 SECURITY DEFINER 함수를 통한 모든 접근
- 기본 마이그레이션 시스템: Flyway/Liquibase를 대체하는 순수 PL/pgSQL
- Trivadis 이름 지정: l_(로컬), in_(입력), io_(inout), co_(상수)
