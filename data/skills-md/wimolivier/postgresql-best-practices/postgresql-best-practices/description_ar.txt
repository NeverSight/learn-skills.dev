PostgreSQL 18+ أفضل الممارسات الشاملة لتطوير قاعدة بيانات المؤسسة. يوفر المخطط
أنماط الهندسة المعمارية، وتصميم واجهة برمجة تطبيقات الجدول، ومعايير ترميز PL/pgSQL، وعمليات الترحيل، وتخزين البيانات.

استخدم هذه المهارة عندما يقوم المستخدم بما يلي:
- إنشاء مخططات أو جداول أو وظائف أو إجراءات أو مشغلات PostgreSQL
- يسأل عن أنواع بيانات PostgreSQL (uuid، text، timestamptz، jsonb، numeric)
- يكتب كود PL/pgSQL ويحتاج إلى اصطلاحات التسمية (البادئات l_، in_، io_، co_)
- ينفذ نمط Table API (وظائف SECURITY DEFINER، وفصل المخطط)
- إعداد عمليات ترحيل قاعدة البيانات أو إصدار المخطط
- يحتاج إلى تحسين الفهرس أو تصميم القيد أو مساعدة في أداء الاستعلام
- يسأل عن ميزات PostgreSQL 18+ (uuidv7، والأعمدة الافتراضية، والقيود الزمنية)
- بناء مستودعات البيانات باستخدام Medallion Architecture (البرونزية/الفضية/الذهبية)
- يحتاج إلى تتبع نسب البيانات أو أنماط ETL أو تسجيل التدقيق
- مراجعة كود قاعدة البيانات لأفضل الممارسات أو الأنماط المضادة
- يهاجر من Oracle PL/SQL إلى PostgreSQL PL/pgSQL
- إعداد خطوط أنابيب CI/CD لتغييرات قاعدة البيانات

الأنماط الأساسية:
- فصل ثلاثة مخططات: البيانات (الجداول) ← خاص (داخلي) ← واجهة برمجة التطبيقات (خارجية)
- واجهة برمجة تطبيقات الجدول: كل الوصول من خلال وظائف SECURITY DEFINER مع SET search_path
- نظام الترحيل الأصلي: بديل PL/pgSQL خالص لـ Flyway/Liquibase
- تسمية Trivadis: l_ (محلي)، in_ (إدخال)، io_ (إدخال)، co_ (ثابت)
