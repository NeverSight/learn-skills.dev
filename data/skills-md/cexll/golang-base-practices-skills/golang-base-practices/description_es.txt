Guía de mejores prácticas del lenguaje Go que cubre marcos (Gin/Go-Kratos), ORM (GORM), migraciones de bases de datos (Goose), estructura de proyectos DDD, manejo de errores, patrones de concurrencia, pruebas (objetivo de cobertura del 99 %) y herramientas Lint. Esta habilidad debe usarse al escribir, revisar o refactorizar código Go para garantizar que se sigan las mejores prácticas. Triggers on Go desarrollo de proyectos, revisión de código, optimización del rendimiento o diseño de arquitectura.
