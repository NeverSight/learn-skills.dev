يحلل سجل التزام git وينشئ سجلات تغيير احترافية باستخدام الإصدارات الدلالية، ودعم الالتزام التقليدي، وتنسيقات المخرجات المتعددة (Keep a Changelog، التقليدية، GitHub). يُستخدم عند تحرير ملفات CHANGELOG.md أو CHANGELOG.txt أو HISTORY.md، أو إعداد ملاحظات الإصدار، أو إنشاء الإصدارات، أو رفع الإصدارات، أو تحديث سجل التغيير، أو توثيق التغييرات، أو كتابة ملاحظات الإصدار، أو تتبع التغييرات، أو تحسين الإصدار، أو إصدار العلامات، أو عندما يذكر المستخدم "سجل التغيير"، أو "ملاحظات الإصدار"، أو "سجل الإصدارات"، أو "الإصدار"، أو "الإصدار الدلالي"، أو "الالتزامات التقليدية".
