Évaluation complète et analyse de la qualité des référentiels Java. Combine des outils d'analyse statique établis (PMD, SpotBugs, Checkstyle, JaCoCo, JDepend) avec des analyses d'historique Git (hotspots, cartes de connaissances, facteur de bus, couplage temporel) pour produire un rapport de santé détaillé. Tous les outils sont invoqués via les coordonnées complètes du plugin Maven — le pom.xml du projet n'est JAMAIS modifié. Utilisez cette compétence chaque fois que l'utilisateur souhaite analyser, évaluer, auditer ou réviser un projet, une base de code ou un référentiel Java, y compris les demandes de rapports sur la qualité du code, les revues d'architecture, les évaluations de la dette technique, la priorisation de la refactorisation, l'analyse des connaissances de l'équipe ou les évaluations de maintenabilité. Se déclenche également lorsque l'utilisateur mentionne l'état du code, l'analyse des points chauds, le facteur de bus, la propriété du code ou l'analyse des dépendances dans un contexte Java.
