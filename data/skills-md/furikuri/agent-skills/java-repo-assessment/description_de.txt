Umfassende Bewertung und Qualitätsanalyse von Java-Repositories. Kombiniert etablierte statische Analysetools (PMD, SpotBugs, Checkstyle, JaCoCo, JDepend) mit Git-Verlaufsforensik (Hotspots, Wissenskarten, Busfaktor, zeitliche Kopplung), um einen detaillierten Gesundheitsbericht zu erstellen. Alle Tools werden über vollständig qualifizierte Maven-Plugin-Koordinaten aufgerufen – die pom.xml des Projekts wird NIEMALS geändert. Verwenden Sie diese Fähigkeit immer dann, wenn der Benutzer ein Java-Projekt, eine Codebasis oder ein Repository analysieren, bewerten, bewerten, prüfen oder überprüfen möchte – einschließlich Anfragen nach Codequalitätsberichten, Architekturüberprüfungen, technischen Schuldenbewertungen, Refactoring-Priorisierung, Teamwissensanalyse oder Wartbarkeitsbewertungen. Wird auch ausgelöst, wenn der Benutzer Codezustand, Hotspot-Analyse, Busfaktor, Codebesitz oder Abhängigkeitsanalyse in einem Java-Kontext erwähnt.
