Java リポジトリの包括的な評価と品質分析。確立された静的分析ツール (PMD、SpotBugs、Checkstyle、JaCoCo、JDepend) と Git 履歴フォレンジック (ホットスポット、ナレッジ マップ、バス ファクター、時間結合) を組み合わせて、詳細な健全性レポートを作成します。すべてのツールは、Maven の完全修飾プラグイン座標を介して呼び出されます。プロジェクトの pom.xml は決して変更されません。ユーザーが Java プロジェクト、コードベース、またはリポジトリを分析、評価、評価、監査、またはレビューしたい場合はいつでも、このスキルを使用します。これには、コード品質レポート、アーキテクチャのレビュー、技術的負債の評価、リファクタリングの優先順位付け、チームの知識分析、保守性の評価などの要求が含まれます。また、ユーザーが Java コンテキストでコードの健全性、ホットスポット分析、バス要素、コードの所有権、または依存関係の分析に言及したときにもトリガーされます。
