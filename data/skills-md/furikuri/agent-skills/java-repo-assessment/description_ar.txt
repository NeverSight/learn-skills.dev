التقييم الشامل وتحليل الجودة لمستودعات Java. يجمع بين أدوات التحليل الثابتة (PMD، وSpotBugs، وCheckstyle، وJaCoCo، وJDepend) مع الطب الشرعي لتاريخ Git (النقاط الساخنة، وخرائط المعرفة، وعامل الناقل، والاقتران الزمني) لإنتاج تقرير صحي مفصل. يتم استدعاء جميع الأدوات عبر إحداثيات البرنامج المساعد المؤهلة بالكامل لـ Maven - لا يتم تعديل pom.xml الخاص بالمشروع مطلقًا. استخدم هذه المهارة عندما يريد المستخدم تحليل أو تقييم أو تقييم أو تدقيق أو مراجعة مشروع Java أو قاعدة التعليمات البرمجية أو المستودع - بما في ذلك طلبات تقارير جودة التعليمات البرمجية أو مراجعات البنية أو تقييمات الديون التقنية أو تحديد أولويات إعادة الهيكلة أو تحليل معرفة الفريق أو تقييمات قابلية الصيانة. يتم التشغيل أيضًا عندما يذكر المستخدم صحة التعليمات البرمجية، أو تحليل نقطة الاتصال، أو عامل الناقل، أو ملكية التعليمات البرمجية، أو تحليل التبعية في سياق Java.
