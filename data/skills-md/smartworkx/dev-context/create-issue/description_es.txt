Cree o actualice una incidencia de GitHub con preguntas aclaratorias. DISPARAR cuando: el usuario solicita crear, actualizar, editar o refinar un problema de GitHub. Ejemplos: "crear un problema para...", "actualizar el problema 42", "mejorar la descripción del problema 137". NO DISPARAR cuando: el usuario solo quiere ver/enumerar problemas, comentar un PR o implementar una función.
