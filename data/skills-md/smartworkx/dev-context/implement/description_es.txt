Implemente una característica a partir del problema, las especificaciones y los documentos, luego proponga un PR. DISPARAR cuando: el usuario solicita implementar, crear o codificar una función, o crear un PR a partir de un problema. Ejemplos: "implementar el problema 42", "crear la función de exportación", "implementar y abrir un PR". NO DISPARAR cuando: el usuario quiera crear un problema, escribir especificaciones o simplemente confirmar cambios.
