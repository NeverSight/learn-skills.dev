Comando de prueba integral de un extremo a otro. Lanza subagentes paralelos para investigar el código base (estructura, esquema de la base de datos, posibles errores) y luego utiliza la CLI del navegador de agentes de Vercel para probar el recorrido de cada usuario: toma capturas de pantalla, valida UI/UX y consulta la base de datos para verificar registros. Ejecútelo después de la implementación para validar todo antes de la revisión del código.
