Genera mensajes de confirmación convencionales a partir de cambios de git preparados analizando diferencias, clasificando tipos de cambios (feat, fix, refactor, docs, test, chore, ci, perf, style, build) y componiendo mensajes de confirmación estructurados con alcance opcional y notación de cambios importantes. Úselo al confirmar código, escribir mensajes de confirmación, preparar confirmaciones de git o cuando el usuario solicite ayuda con los mensajes de confirmación.
