Génère des messages de validation conventionnels à partir de modifications git par étapes en analysant les différences, en classant les types de modifications (exploit, correctif, refactor, docs, test, corvée, ci, perf, style, build) et en composant des messages de validation structurés avec une portée facultative et une notation de modification de rupture. À utiliser lors de la validation du code, de l'écriture de messages de validation, de la préparation des validations git ou lorsque l'utilisateur demande de l'aide pour les messages de validation.
