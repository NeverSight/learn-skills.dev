ينشئ رسائل التزام تقليدية من تغييرات git المرحلية من خلال تحليل الاختلافات وتصنيف أنواع التغيير (الفذ، والإصلاح، وإعادة البناء، والمستندات، والاختبار، والإجراء الروتيني، وCI، والأداء، والأسلوب، والبناء)، وتأليف رسائل التزام منظمة ذات نطاق اختياري وتدوين تغيير مجزأ. يُستخدم عند تنفيذ التعليمات البرمجية، أو كتابة رسائل الالتزام، أو إعداد التزامات git، أو عندما يطلب المستخدم المساعدة في رسائل الالتزام.
