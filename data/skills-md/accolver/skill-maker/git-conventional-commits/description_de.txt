Erzeugt herkömmliche Commit-Nachrichten aus bereitgestellten Git-Änderungen durch die Analyse von Unterschieden, die Klassifizierung von Änderungstypen (feat, fix, refactor, docs, test, chore, ci, perf, style, build) und das Verfassen strukturierter Commit-Nachrichten mit optionalem Umfang und Breaking-Change-Notation. Wird verwendet, wenn Sie Code festschreiben, Festschreibungsnachrichten schreiben, Git-Festschreibungen vorbereiten oder wenn der Benutzer um Hilfe bei Festschreibungsnachrichten bittet.
