À utiliser lors de l'examen des modifications de code avant de valider ou de fusionner : diffs par étapes, diffs de branche ou contrôles de préparation aux relations publiques. Déclencheurs : « réviser mon code », « vérifier ces modifications », « réviser cette branche », « donner des commentaires sur ma différence », « ce PR est-il prêt ? ». Pas pour l'implémentation de fonctionnalités, l'exécution de suites de tests ou les passes de formatage uniquement.
