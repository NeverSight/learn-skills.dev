Competenze di architetto software per le API Go Gin. Copre la progettazione del sistema, la valutazione della complessità, i modelli di scalabilità, la progettazione delle API, le preoccupazioni trasversali, le ADR, la gestione del debito tecnologico e l'orchestrazione delle competenze Gingo. Da utilizzare quando si prendono decisioni sull'architettura, si valuta la complessità, si progettano sistemi, si scelgono modelli, si pianifica l'evoluzione delle API o si coordinano diverse competenze gin. Attivalo anche quando l'utente menziona microservizi vs monolite, CQRS, sourcing di eventi, modellazione di dominio, contesti delimitati, controllo delle versioni API, strategia di caching, osservabilità o debito tecnologico. Questo è il "cervello" che orchestra tutte le altre abilità del gin.
