PostgreSQL DBA および Go Gin API のアーキテクト スキル。スキーマ設計の決定、移行の影響分析、インデックス戦略、クエリの最適化、パーティショニング、接続プールのサイジング、PostgreSQL 拡張エコシステム (ParadeDB、pgvector、PostGIS、TimescaleDB) をカバーします。スキーマの設計、移行の安全性の分析、インデックスの選択、クエリの最適化、拡張機能の選択、または PostgreSQL アーキテクチャの決定を行うときに使用します。これは、golang-gin-database の「接続方法」を補完する「思考方法」です。また、ユーザーが EXPLAIN ANALYZE、ロック レベル、ゼロ ダウンタイム移行、全文検索、ベクトル埋め込み、地理空間クエリ、時系列データ、またはデータベース パフォーマンス チューニングについて言及した場合にもアクティブ化します。
