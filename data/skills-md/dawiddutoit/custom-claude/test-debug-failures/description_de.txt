Evidenzbasiertes Test-Debugging zur Erzwingung einer systematischen Ursachenanalyse.
Wird verwendet, wenn Tests fehlschlagen, Pytest-Fehler auftreten, die Testsuite nicht bestanden wird,
Testfehler beheben oder fehlerhafte Tests reparieren. Verhindert Annahmen
Fehlerbehebungen durch Durchsetzung der richtigen Diagnosesequenz. Funktioniert mit Python (.py),
JavaScript/TypeScript (.js/.ts), Go, Rust-Testdateien. Unterstützt Pytest,
Jest, Vitest, Mocha, Go Test, Cargo Test und andere Frameworks.
