Выявляйте нарушения принципа единой ответственности (SRP) с помощью многомерного анализа.
Используйте при проверке кода на предмет «SRP», «единая ответственность», «божественный класс», «слишком много делать»,
«слишком много зависимостей» перед фиксацией, во время рефакторинга или в качестве контроля качества.
Анализирует файлы Python, JavaScript, TypeScript с обнаружением на основе AST, метриками (TCC, ATFD, WMC),
и шаблоны, специфичные для проекта. Предоставляет практические рекомендации по исправлению ошибок с оценками рефакторинга.
