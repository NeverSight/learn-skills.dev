Ottimizza le prestazioni della build Gradle attraverso strategie di memorizzazione nella cache, cache di configurazione, parallela
esecuzione e ottimizzazione della memoria per ottenere build più veloci del 50-80% per progetti di grandi dimensioni. Utilizzare quando
le build sono lente (>2 minuti), l'ottimizzazione delle pipeline CI/CD o la configurazione di nuovi progetti Gradle.
Si attiva su "velocizza Gradle", "build lente", "ottimizza CI/CD", "riduci tempo di build" o
"configura le prestazioni di Gradle". Funziona con gradle.properties, build.gradle.kts, GitHub Actions,
e configurazioni CI GitLab utilizzando build cache, cache di configurazione ed esecuzione parallela.
