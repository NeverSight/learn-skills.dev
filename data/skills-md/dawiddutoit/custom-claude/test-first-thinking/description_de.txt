Erzwingt die Disziplin, vor dem Schreiben von Implementierungscode über Tests, Funktionen und Wartbarkeit nachzudenken. Verwenden Sie diese Option, wenn Sie neue Klassen/Methoden starten, vorhandenen Code umgestalten oder wenn Sie aufgefordert werden, „zuerst über Tests nachzudenken“, „für Testbarkeit zu entwerfen“, „welche Tests benötige ich“, „Test-zuerst-Ansatz“ oder „TDD-Denken“. Fördert einfache, wartbare Designs, indem die Testbarkeit im Voraus berücksichtigt wird. Funktioniert mit jeder Codebasis, die Testabdeckung und Qualitätsstandards erfordert.
