Hace cumplir la disciplina de pensar en las pruebas, las características y la capacidad de mantenimiento ANTES de escribir el código de implementación. Úselo al iniciar nuevas clases/métodos, refactorizar código existente o cuando se le solicite "pensar primero en las pruebas", "diseñar para la capacidad de prueba", "qué pruebas necesito", "enfoque de prueba primero" o "pensamiento TDD". Promueve diseños simples y fáciles de mantener considerando la capacidad de prueba desde el principio. Funciona con cualquier base de código que requiera cobertura de pruebas y estándares de calidad.
