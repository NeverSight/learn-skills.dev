Выполняйте систематический самоанализ изменений кода перед фиксацией, используя структурированный контрольный список. Проверяет границы архитектуры, качество кода, тестовое покрытие, документацию и антишаблоны, специфичные для проекта. Используйте перед фиксацией, созданием PR или когда пользователь говорит «проверьте мои изменения», «самопроверка», «проверьте мой код». Адаптируется к проектам Python, JavaScript, TypeScript, Go, Rust.
