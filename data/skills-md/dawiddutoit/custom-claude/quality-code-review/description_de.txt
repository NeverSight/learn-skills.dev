Führen Sie vor dem Festschreiben mithilfe einer strukturierten Checkliste eine systematische Selbstüberprüfung von Codeänderungen durch. Validiert Architekturgrenzen, Codequalität, Testabdeckung, Dokumentation und projektspezifische Anti-Patterns. Verwenden Sie diese Option vor dem Festschreiben, dem Erstellen von PRs oder wenn der Benutzer „Meine Änderungen überprüfen“, „Selbstüberprüfung“ oder „Meinen Code überprüfen“ sagt. Passt sich an Python-, JavaScript-, TypeScript-, Go- und Rust-Projekte an.
