Cortar un lanzamiento: detectar el contexto de versiones, generar un registro de cambios a partir de confirmaciones convencionales, aumentar versiones y crear una etiqueta git. Úselo cuando el usuario diga "liberar", "cortar una versión", "etiquetar una versión", "mejorar la versión", "crear un registro de cambios" o cualquier variación del envío/publicación de una versión. Esta habilidad es intencionalmente genérica y funciona en cualquier repositorio: infiere el contexto del historial de git y la estructura del proyecto en lugar de asumir una configuración específica.
