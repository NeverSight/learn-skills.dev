Couper une version : détectez le contexte de gestion des versions, générez un journal des modifications à partir des validations conventionnelles, modifiez les versions et créez une balise git. À utiliser lorsque l'utilisateur dit « publier », « couper une version », « marquer une version », « augmenter la version », « créer un journal des modifications » ou toute variante d'expédition/publication d'une version. Cette compétence est intentionnellement générique et fonctionne sur n'importe quel dépôt : elle déduit le contexte de l'historique de Git et de la structure du projet plutôt que de supposer une configuration spécifique.
