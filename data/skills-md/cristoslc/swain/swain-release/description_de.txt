Schneiden Sie ein Release aus – erkennen Sie den Versionierungskontext, generieren Sie ein Änderungsprotokoll aus herkömmlichen Commits, erhöhen Sie Versionen und erstellen Sie ein Git-Tag. Wird verwendet, wenn der Benutzer „Release“, „Release ausschneiden“, „Release markieren“, „Version erweitern“, „Änderungsprotokoll erstellen“ oder eine andere Variante des Versendens/Veröffentlichens einer Version sagt. Diese Fähigkeit ist absichtlich allgemein gehalten und funktioniert in jedem Repo – sie leitet den Kontext aus dem Git-Verlauf und der Projektstruktur ab, anstatt ein bestimmtes Setup anzunehmen.
