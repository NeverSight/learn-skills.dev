Debuggen Sie den Quellcode interaktiv – legen Sie Haltepunkte fest, gehen Sie Zeile für Zeile durch die Ausführung, überprüfen Sie den Status von Live-Variablen, werten Sie Ausdrücke anhand des laufenden Programms aus und navigieren Sie durch den Aufrufstapel, um die Grundursachen zu ermitteln. Verwenden Sie es, wenn ein Programm abstürzt, unerwartete Ausnahmen auslöst, eine falsche Ausgabe erzeugt, wenn Sie verstehen müssen, wie die Ausführung einen bestimmten Status erreicht hat, oder wenn das Debuggen von Druckanweisungen nicht aussagekräftig genug ist.
