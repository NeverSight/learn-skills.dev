Déboguez le code source de manière interactive : définissez des points d'arrêt, parcourez l'exécution ligne par ligne, inspectez l'état des variables en direct, évaluez les expressions par rapport au programme en cours d'exécution et parcourez la pile d'appels pour rechercher les causes profondes. À utiliser lorsqu'un programme plante, génère des exceptions inattendues, produit une sortie incorrecte, lorsque vous avez besoin de comprendre comment l'exécution a atteint un certain état ou lorsque le débogage de l'instruction d'impression n'est pas suffisamment révélateur.
