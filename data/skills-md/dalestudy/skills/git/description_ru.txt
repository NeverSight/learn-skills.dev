Рекомендации по управлению версиями Git и руководство по рабочему процессу. Используйте в следующих ситуациях: (1) При написании сообщений коммитов Git (применяйте обычные правила коммитов), (2) При создании веток и управлении ими (на основе GitHub Flow), (3) При выборе стратегий создания PR и слияния, (4) При очистке истории Git (rebase, сквош, выбор вишни), (5) При разрешении конфликтов слияния, (6) При работе с ключевыми словами «git», «.git», «commit», «branch», «merge», 'перебазировать'
