Best Practices und Workflow-Leitfaden für die Git-Versionskontrolle. Verwenden Sie es in den folgenden Situationen: (1) Beim Schreiben von Git-Commit-Nachrichten (konventionelle Commits-Regeln anwenden), (2) Beim Erstellen und Verwalten von Branches (basierend auf GitHub Flow), (3) Bei der Auswahl von PR-Erstellungs- und Merge-Strategien, (4) Beim Bereinigen des Git-Verlaufs (Rebase, Squash, Cherry-Pick), (5) Beim Lösen von Merge-Konflikten, (6) Beim Arbeiten mit Schlüsselwörtern „git“, „.git“, „commit“, „branch“, „merge“, 'rebase'
