Best practice per il controllo della versione Git e guida al flusso di lavoro. Utilizzare nelle seguenti situazioni: (1) Quando si scrivono messaggi di commit Git (applicare le regole dei commit convenzionali), (2) Quando si creano e gestiscono rami (basati su GitHub Flow), (3) Quando si scelgono strategie di creazione PR e unione, (4) Quando si pulisce la cronologia Git (rebase, squash, cherry-pick), (5) Quando si risolvono conflitti di unione, (6) Quando si lavora con le parole chiave 'git', '.git', 'commit', 'branch', 'unire', 'ribasare'
