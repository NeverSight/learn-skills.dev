Habilidad de Análisis Sistemático de Requisitos basada en la metodología de "Análisis Eficaz de Requisitos" de Xu Feng. Adopta el método SERU y un proceso de análisis de tres etapas, que cubre cuatro líneas principales: requisitos de valor, requisitos funcionales, requisitos de datos y requisitos de calidad. Adecuado para la obtención, análisis y documentación de requisitos en proyectos de software. Casos de uso: iniciar el análisis de requisitos para un nuevo proyecto, organizar los requisitos de un sistema existente, redactar especificaciones de requisitos.
