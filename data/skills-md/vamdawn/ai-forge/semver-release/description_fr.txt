Workflow de publication de version automatisé. Analyse l'historique des commits git pour déduire la version sémantique, détecte automatiquement les fichiers de version dans les écosystèmes, met à jour les CHANGELOG multilingues, crée le commit et la balise git. À utiliser lorsque : (1) l'utilisateur dit "release", "publish version", "bump version", (2) l'utilisateur invoque la commande /release, (3) se prépare à publier une nouvelle version.
