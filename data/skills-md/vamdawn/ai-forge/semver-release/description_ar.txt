سير عمل إصدار الإصدار الآلي. يحلل سجل التزام git لاستنتاج الإصدار الدلالي، ويكتشف تلقائيًا ملفات الإصدار عبر الأنظمة البيئية، ويحدث سجلات التغيير متعددة اللغات، وينشئ التزام git وعلامة. يُستخدم عندما: (1) يقول المستخدم "release"، "publish version"، "bump version"، (2) يستدعي المستخدم أمر /release، (3) يستعد لإصدار إصدار جديد.
