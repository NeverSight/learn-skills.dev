Рабочий процесс разработки на основе спецификаций: требования → проектирование → задачи → цикл реализации. Используйте при планировании функций, реализации спецификаций, уточнении спецификаций или возобновлении работы. Также используйте это для спецификаций аналитики и рабочих процессов с данными, когда пользователю нужны требования/проектирование/задачи для блокнотов, экспериментов, групп Hamilton DAG, шлюзов проверки или продвижения воспроизводимого анализа. Команды: spec-plan, spec-go, spec-task, spec-audit, spec-status, spec-merge, spec-reset, spec-help. ВАЖНО: Никогда не редактируйте файлы require.md, design.md, Tasks.md или Progress.txt, не прочитав предварительно этот навык и соответствующие ссылки/документ.
