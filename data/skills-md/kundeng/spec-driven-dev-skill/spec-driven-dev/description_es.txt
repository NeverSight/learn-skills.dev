Flujo de trabajo de desarrollo basado en especificaciones: requisitos → diseño → tareas → implementar bucle. Utilícelo al planificar funciones, implementar a partir de especificaciones, perfeccionar especificaciones o reanudar el trabajo. Utilícelo también para especificaciones analíticas y de flujo de trabajo de datos cuando el usuario desee requisitos/diseño/tareas para cuadernos, barridos de experimentos, DAG de Hamilton, puertas de revisión o promoción de análisis reproducibles. Comandos: plan de especificaciones, ir de especificaciones, tarea de especificaciones, auditoría de especificaciones, estado de especificaciones, fusión de especificaciones, restablecimiento de especificaciones, ayuda de especificaciones. IMPORTANTE: nunca edite requisitos.md, diseño.md, tareas.md o progreso.txt sin leer primero esta habilidad y las referencias/documentos relevantes.
