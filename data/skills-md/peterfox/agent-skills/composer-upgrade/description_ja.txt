Composer コマンドを使用した PHP プロジェクトのアップグレードをガイドします。ユーザーが PHP パッケージをアップグレードする場合、依存関係の競合を理解する場合、「composer outdated」出力を解釈する場合、「composer Why-not」を使用してバージョン制約を診断する場合、「composer Why」を使用して依存関係を追跡する場合、「composer dump」を使用してアップグレード後にバージョン制約を強化する場合、安全なアップグレード パスを計画する場合、または、composer.json でのパッケージ バージョンの競合を解決する場合に使用します。
