Guide les mises à niveau du projet PHP à l’aide des commandes Composer. À utiliser pour aider les utilisateurs à mettre à niveau les packages PHP, à comprendre les conflits de dépendances, à interpréter la sortie « composer outdated », à utiliser « composer pourquoi pas » pour diagnostiquer les contraintes de version, à utiliser « composer pourquoi » pour tracer les dépendances, à utiliser « composer bump » pour renforcer les contraintes de version après la mise à niveau, planifier des chemins de mise à niveau sécurisés ou résoudre les conflits de version de package dans composer.json.
