SQL クエリを分析してパフォーマンスの問題 (インデックスの欠落、N+1 パターン、最適ではない結合、テーブル全体のスキャン) を検出します。 EXPLAIN 出力を解釈し、アンチパターンを検出し、インデックスを推奨し、各最適化の詳細な説明を使用してクエリを書き換えます。トリガー: 「このクエリを最適化する」、「遅いクエリ」、「クエリのパフォーマンス」、「インデックスの追加」、「インデックスの推奨事項」、「計画の説明」、「クエリ計画」、「N+1 クエリ」、「この SQL を修正する」、「SQL パフォーマンス」、「SQL の最適化」、「このクエリが遅い理由」、「インデックス戦略」。パフォーマンス分析または最適化が必要な SQL クエリが与えられた場合は、このスキルを使用します。
