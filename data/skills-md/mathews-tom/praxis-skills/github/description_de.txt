GitHub-CLI-Operationen über „gh“ für Issues, Pull Requests, CI/Aktionen, Releases, Repos, Suche, Gists und die REST/GraphQL-API. Strukturierte Ausgabe mit „--json“ und „--jq“ zum Parsen. Behandelt „gh issue create/list/view/edit/close“, „gh pr create/review/merge/checks“, „gh run list/view/rerun/watch“, „gh release create“, „gh search repos/issues/prs/code“, „gh api“ für REST- und GraphQL-Abfragen und „gh gist“-Operationen. Beinhaltet Fehlerbehandlung für HTTP 401/403/404/422/429, Scope-Fehlerbehebung und Ratenlimitverwaltung. Auslöserphrasen: „Problem erstellen“, „Fehler melden“, „Ticket öffnen“, „PR einreichen“, „Pull-Anfrage stellen“, „Pipeline prüfen“, „Testergebnisse anzeigen“, „CI fehlgeschlagen“, „Warum ist CI fehlgeschlagen“, „CI-Status prüfen“, „PR zusammenführen“, „Releases verwalten“, „GitHub-API abfragen“, „Repositorys durchsuchen“, „Triage-Workflows“, „GitHub-Vorgänge automatisieren“. Wird auch ausgelöst, wenn der Benutzer eine GitHub-URL einfügt.
