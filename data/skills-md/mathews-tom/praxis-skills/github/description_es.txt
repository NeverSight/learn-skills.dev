Operaciones de GitHub CLI a través de `gh` para problemas, solicitudes de extracción, CI/acciones, lanzamientos, repositorios, búsquedas, esencias y la API REST/GraphQL. Salida estructurada con `--json` y `--jq` para análisis. Cubre `gh issues create/list/view/edit/close`, `gh pr create/review/merge/checks`, `gh run list/view/rerun/watch`, `gh release create`, `gh search repos/issues/prs/code`, `gh api` para consultas REST y GraphQL, y operaciones `gh gist`. Incluye manejo de errores para HTTP 401/403/404/422/429, solución de problemas de alcance y administración de límites de velocidad. Frases de activación: "crear un problema", "presentar un error", "abrir un ticket", "enviar un PR", "generar una solicitud de extracción", "verificar canalización", "ver resultados de pruebas", "Error de CI", "por qué falló el CI", "verificar el estado de CI", "fusionar un PR", "administrar lanzamientos", "consultar la API de GitHub", "buscar repositorios", "flujos de trabajo de clasificación", "automatizar operaciones de GitHub". También se activa cuando el usuario pega una URL de GitHub.
