Методика отладки на основе гипотез: ранжированные гипотезы с подтверждающими/опровергающими тестами, стратегия git bisect, анализ журналов, планирование точек инструментирования и минимальный дизайн воспроизводства. Срабатывает при: «систематически отлаживать эту проблему», «анализ первопричины», «разрезать эту ошибку пополам», «ранжировать гипотезы об этой ошибке», «помогите мне изолировать эту проблему», «создать минимальную копию», «план инструментария для этой ошибки», «почему это продолжает давать сбой». Отличительной чертой является методология структурированного исследования (ранжирование гипотез, стратегия деления пополам, точки инструментирования). Используйте этот навык для неочевидных ошибок, которые требуют систематического исследования, а не для простых ошибок, которые модель диагностирует напрямую. НЕ для абстрактных рассуждений или декомпозиции проблемы без конкретной ошибки — модель изначально обрабатывает общие рассуждения.
