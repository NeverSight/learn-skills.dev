Pull-Request und Codeüberprüfung mit diff-basiertem Routing in fünf Dimensionen: Codequalität und Richtlinienkonformität, Testabdeckungsanalyse, stille Fehlererkennung, Typdesign und Invariantenanalyse sowie Prüfung der Kommentarqualität. Klassifiziert geänderte Dateien und lädt nur relevante Überprüfungsmethoden. Erstellt nach Schweregrad eingestufte Ergebnisse (Kritisch, Wichtig, Vorschlag) mit Konfidenzbewertung. Ersetzt das PR-Review-Toolkit-Plugin. Auslösersätze: „Meine PR überprüfen“, „Diesen Code überprüfen“, „Meine Änderungen überprüfen“, „Ist dies zum Zusammenführen bereit“, „Diese PR prüfen“, „Vor dem Festschreiben überprüfen“, „Codequalität prüfen“, „Probleme mit diesem Code“, „Überprüfung vor dem Zusammenführen“, „Überprüfen Sie meine Änderungen“, „Codeüberprüfung“. Verwenden Sie diese Fähigkeit, wenn Sie Code vor dem Festschreiben oder Zusammenführen überprüfen, die PR-Qualität überprüfen oder wenn der Benutzer um Feedback zu kürzlichen Änderungen bittet.
