Запрос на включение и проверка кода с маршрутизацией на основе различий по пяти измерениям: качество кода и соответствие рекомендациям, анализ тестового покрытия, автоматическое обнаружение ошибок, проектирование типов и инвариантный анализ, а также аудит качества комментариев. Классифицирует измененные файлы и загружает только соответствующие методологии проверки. Выдает результаты по степени серьезности («Критическое», «Важное», «Предложение») с оценкой достоверности. Заменяет плагин pr-review-toolkit. Триггерные фразы: «проверьте мой PR», «проверьте этот код», «проверьте мои изменения», «готово ли это к слиянию», «проверьте этот PR», «проверьте перед фиксацией», «проверьте качество кода», «любые проблемы с этим кодом», «проверка перед объединением», «просмотрите мои изменения», «проверка кода». Используйте этот навык при проверке кода перед фиксацией или слиянием, проверке качества PR или когда пользователь запрашивает отзыв о последних изменениях.
