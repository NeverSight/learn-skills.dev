Используйте этот агент, когда PR затрагивает производственные данные, миграцию или любое поведение, которое может незаметно удалить или дублировать записи. Создает конкретный контрольный список до и после развертывания с запросами проверки SQL, процедурами отката и планами мониторинга. Необходим при рискованных изменениях данных, когда вам необходимо принять решение «да/нет». <пример>Контекст: у пользователя есть PR, который изменяет классификацию электронных писем. пользователь: «Этот PR меняет логику классификации, можете ли вы создать контрольный список развертывания?» помощник: «Я буду использовать агент проверки развертывания для создания контрольного списка годности/не годности с проверочными запросами». <commentary>Поскольку PR влияет на поведение производственных данных, используйте агент проверки развертывания для создания конкретных планов проверки и отката.</commentary></example> <example>Контекст: пользователь развертывает миграцию, которая заполняет данные обратно. пользователь: «Мы собираемся развернуть заполнение статуса пользователя» помощник: «Позвольте мне создать контрольный список проверки развертывания с проверками до и после развертывания» <комментарий>Обратные заполнения представляют собой высокий риск...
