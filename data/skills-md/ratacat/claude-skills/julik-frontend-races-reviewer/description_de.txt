Verwenden Sie diesen Agenten, wenn Sie Änderungen am JavaScript- oder Stimulus-Frontend-Code mit besonderem Augenmerk auf die Rennbedingungen überprüfen müssen. Der Agent sollte aufgerufen werden, nachdem JavaScript-Funktionen implementiert, vorhandener JavaScript-Code geändert oder Stimulus-Controller erstellt oder geändert wurden. Der Agent wendet Juliks Blick für UI-Race-Bedingungen in JavaScript und Stimulus-Code an. Beispiele: - <Beispiel> Kontext: Der Benutzer hat gerade einen neuen Stimulus-Controller implementiert. Benutzer: „Ich habe einen neuen Controller zum Anzeigen und Ausblenden von Toasts erstellt.“ Assistent: „Ich habe den Controller implementiert. Lassen Sie mich nun Julik einen Blick auf mögliche Rennbedingungen und DOM-Unregelmäßigkeiten werfen lassen.“ <Kommentar> Da neuer Stimulus-Controller-Code geschrieben wurde, verwenden Sie den Agenten julik-frontend-races-reviewer, um Juliks unheimliches Wissen über UI-Datenrennen und Qualitätsprüfungen in JavaScript und Stimulus-Code anzuwenden. </commentary> </example> – <example> Kontext: Der Benutzer hat einen vorhandenen Stimulus-Controller umgestaltet. Benutzer: „Bitte refaktorieren Sie den Controller, um ihn langsam zu animieren …“
