Structurez le logiciel autour de la règle de dépendance : les dépendances du code source pointent vers l'intérieur des frameworks vers les cas d'utilisation vers les entités. À utiliser lorsque l'utilisateur mentionne « couches d'architecture », « règle de dépendance », « ports et adaptateurs », « architecture hexagonale » ou « limite de cas d'utilisation ». Couvre les principes des composants, les limites et SOLID. Pour la qualité du code, voir clean-code. Pour la modélisation de domaine, voir Domain-driven-design.
