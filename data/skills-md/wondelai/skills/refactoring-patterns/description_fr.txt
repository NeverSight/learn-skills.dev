Appliquez des transformations de refactorisation nommées pour améliorer la structure du code sans modifier le comportement. À utiliser lorsque l'utilisateur mentionne « refactoriser ceci », « le code sent », « méthode d'extraction », « remplacer le conditionnel » ou « dette technique ». Couvre la refactorisation basée sur les odeurs, les séquences de transformation sécurisées et les tests de protection. Pour les fondements de la qualité du code, voir clean-code. Pour gérer la complexité, voir philosophie de conception logicielle.
