Escriba código legible y mantenible mediante nombres disciplinados, funciones pequeñas y manejo limpio de errores. Úselo cuando el usuario mencione "revisión de código", "convenciones de nomenclatura", "función demasiado larga", "el código huele" o "código legible". Cubre SRP, disciplina de comentarios, formato y pruebas unitarias. Para conocer las técnicas de refactorización, consulte patrones de refactorización. Para arquitectura, consulte arquitectura limpia.
