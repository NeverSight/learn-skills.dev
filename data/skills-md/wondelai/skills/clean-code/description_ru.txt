Пишите читаемый и поддерживаемый код с помощью дисциплинированного именования, небольших функций и четкой обработки ошибок. Используйте, когда пользователь упоминает «проверку кода», «соглашения об именах», «слишком длинная функция», «код пахнет» или «читабельный код». Охватывает SRP, дисциплину комментариев, форматирование и модульное тестирование. Информацию о методах рефакторинга см. в разделе «Шаблоны рефакторинга». Информацию об архитектуре см. в разделе «Чистая архитектура».
