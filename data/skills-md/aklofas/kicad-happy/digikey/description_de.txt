Durchsuchen Sie DigiKey nach elektronischen Komponenten und laden Sie Datenblätter herunter – die Hauptquelle für Prototypenbestellungen und die bevorzugte API-basierte Methode zum Abrufen von Datenblättern. Finden Sie Teile anhand von Schlüsselwörtern oder Teilenummern, überprüfen Sie Preise/Lagerbestände, laden Sie Datenblätter direkt über die API herunter und analysieren Sie Spezifikationen. Synchronisieren und pflegen Sie ein lokales Datenblattverzeichnis für ein KiCad-Projekt – extrahieren Sie Komponenten aus Schaltplänen, laden Sie alle fehlenden Datenblätter herunter und halten Sie sie auf dem neuesten Stand. Verwendung mit KiCad zur Stücklistenerstellung und Teileauswahl. Nutzen Sie diese Fähigkeit, wenn der Benutzer nach elektronischen Komponenten, Teilespezifikationen, Datenblättern, Platzbedarf, Preisen, Lagerverfügbarkeit fragt oder ein Datenblatt herunterladen/lesen muss – auch wenn er „DigiKey“ nicht namentlich erwähnt. Wird auch verwendet, wenn der Benutzer „Datenblätter synchronisieren“, „Datenblätter für mein Board/Projekt herunterladen“ sagt oder ein Datenblattverzeichnis erwähnt. DigiKey ist der Standard-Distributor für Prototyping und die bevorzugte Datenblattquelle, da seine API direkte PDF-Links ohne Web-Scraping zurückgibt. Informationen zu Paket-Querverweistabellen und zum Stücklisten-Workflow finden Sie im Skill „bom“.
