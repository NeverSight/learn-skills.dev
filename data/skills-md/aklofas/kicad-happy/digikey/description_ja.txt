DigiKey で電子コンポーネントを検索し、データシートをダウンロードします。これは、プロトタイプ注文の主要なソースであり、データシートを取得するための API ベースの推奨方法です。キーワードまたは部品番号で部品を検索し、価格/在庫を確認し、API 経由でデータシートを直接ダウンロードし、仕様を分析します。 KiCad プロジェクトのローカル データシート ディレクトリを同期して維持します。回路図からコンポーネントを抽出し、不足しているデータシートをすべてダウンロードして、最新の状態に保ちます。 BOM の作成と部品の選択には KiCad と併用します。ユーザーが電子コンポーネント、部品仕様、データシート、フットプリント、価格、在庫状況について尋ねた場合、またはデータシートをダウンロード/読む必要がある場合、たとえ「DigiKey」の名前を言及していない場合でも、このスキルを使用します。ユーザーが「データシートを同期する」、「ボード/プロジェクトのデータシートをダウンロードする」、またはデータシート ディレクトリに言及する場合にも使用します。 DigiKey はプロトタイピングのデフォルトのディストリビュータであり、その API が Web スクレイピングなしで直接 PDF リンクを返すため、推奨されるデータシート ソースです。パッケージ相互参照テーブルと BOM ワークフローについては、「bom」スキルを参照してください。
