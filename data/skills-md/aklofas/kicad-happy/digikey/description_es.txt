Busque en DigiKey componentes electrónicos y descargue hojas de datos: fuente principal para pedidos de prototipos y el método basado en API preferido para obtener hojas de datos. Encuentre piezas por palabra clave o número de pieza, consulte precios/existencias, descargue hojas de datos directamente a través de API, analice especificaciones. Sincronice y mantenga un directorio de hojas de datos local para un proyecto KiCad: extraiga componentes de los esquemas, descargue todas las hojas de datos que faltan y manténgalas actualizadas. Úselo con KiCad para la creación de listas de materiales y selección de piezas. Utilice esta habilidad cuando el usuario pregunte sobre componentes electrónicos, especificaciones de piezas, hojas de datos, huellas, precios, disponibilidad de existencias o necesite descargar/leer una hoja de datos, incluso si no mencionan "DigiKey" por su nombre. Utilícelo también cuando el usuario diga "sincronizar hojas de datos", "descargar hojas de datos para mi placa/proyecto" o mencione un directorio de hojas de datos. DigiKey es el distribuidor predeterminado para la creación de prototipos y la fuente de hojas de datos preferida porque su API devuelve enlaces PDF directos sin web scraping. Para tablas de referencias cruzadas de paquetes y flujo de trabajo de BOM, consulte la habilidad `bom`.
