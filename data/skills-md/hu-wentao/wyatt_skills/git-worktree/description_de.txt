Erstellen Sie einen Git-Arbeitsbaum für das aktuelle Repository auf derselben Verzeichnisebene wie das Projektstammverzeichnis. Diese Fähigkeit automatisiert die Zweigerstellung, die Verzeichnisbenennung gemäß dem Format Projekt-T-Zweig und die anfängliche Projekteinrichtung (z. B. Abhängigkeitsinstallation). Verwenden Sie dies, wenn der Benutzer an einer neuen Funktion oder Korrektur arbeiten möchte, ohne seinen aktuellen Arbeitsbereich zu wechseln.
