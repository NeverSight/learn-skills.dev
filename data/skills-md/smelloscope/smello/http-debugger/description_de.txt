Debuggen Sie von Smello erfasste HTTP-Anfragen. Verwenden Sie diese Option, wenn der Benutzer den Datenverkehr überprüfen, API-Aufrufe debuggen, fehlgeschlagene Anfragen beheben, Antworttexte analysieren oder verstehen möchte, welche Anfragen sein Code stellt. Verwenden Sie es auch, wenn der Benutzer eine Smello-Dashboard-URL wie http://localhost:5110/#<uuid> oder http://localhost:5111/#<uuid> einfügt – extrahieren Sie die UUID nach dem Hash als Anforderungs-ID. Unterstützt gRPC-Aufrufe von Google Cloud-Bibliotheken. Erfordert einen laufenden Smello-Server.
