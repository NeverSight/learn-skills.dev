Überprüfen, validieren und führen Sie eine GitHub-Pull-Anfrage mit „gh“ und „git“ in einen Zielzweig ein. Verwenden Sie diese Option, wenn der Benutzer eine End-to-End-Zusammenführung eines PR anfordert, optional den Quellzweig löschen und optional einen temporären Arbeitsbaum verwenden möchte. Führen Sie zunächst eine autonome Überprüfung durch; Wenn Blockierungsprobleme, ungelöste Konflikte, fehlgeschlagene Prüfungen oder unzureichende Dokumente/Tests gefunden werden, kommentieren Sie die PR und hören Sie auf, anstatt sie zusammenzuführen.
