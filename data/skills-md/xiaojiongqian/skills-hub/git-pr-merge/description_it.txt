Esamina, convalida e unisci una richiesta pull GitHub in un ramo di destinazione con "gh" e "git". Da utilizzare quando l'utente richiede di unire un PR end-to-end, facoltativamente eliminare il ramo di origine e facoltativamente utilizzare un albero di lavoro temporaneo. Eseguire prima la revisione autonoma; se vengono rilevati problemi bloccanti, conflitti irrisolti, controlli falliti o documenti/test insufficienti, commentare il PR e interrompere invece di unire.
