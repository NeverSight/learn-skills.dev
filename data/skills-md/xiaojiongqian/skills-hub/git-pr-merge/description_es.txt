Revisar, validar y fusionar una solicitud de extracción de GitHub en una rama de destino con `gh` y `git`. Úselo cuando el usuario solicite fusionar un PR de un extremo a otro, opcionalmente eliminar la rama de origen y, opcionalmente, usar un árbol de trabajo temporal. Ejecute primero la revisión autónoma; Si se encuentran problemas de bloqueo, conflictos no resueltos, comprobaciones fallidas o documentos/pruebas insuficientes, comente el PR y deténgase en lugar de fusionar.
