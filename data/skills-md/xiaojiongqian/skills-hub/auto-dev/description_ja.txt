厳格な Git 安全性を備えた現在のワークツリーでの自律的な開発とデバッグ。ユーザーが「自動開発に行く」と言うか、自動開発の開始を要求したときにトリガーされます。コーディング、テスト、MCP/スキル (Playwright など)、クラウド検査 (Firestore/ストレージ/GCP)、Web ブラウジング、または現在のブランチの GitHub Actions デプロイに使用します。デフォルトのポリシーは、開発/メインおよびブランチ間のプッシュをブロックしますが、「メイン」での明示的な「スキルハブ」メンテナンスとドラフト PR ハンドオフの狭い例外を除きます。コマンド入力により、必要な認証情報またはセッション値の入力を求められます。
