Sviluppo e debug autonomi nell'albero di lavoro corrente con rigorosa sicurezza git. Si attiva quando l'utente dice "vai allo sviluppo automatico" o chiede di avviare lo sviluppo automatico. Da utilizzare per codifica, test, MCP/competenze (ad esempio, Drammaturgo), ispezione cloud (Firestore/Storage/GCP), navigazione Web o distribuzioni di azioni GitHub per il ramo corrente. La policy predefinita blocca i push dev/main e cross-branch, con una stretta eccezione per la manutenzione esplicita di "skills-hub" su "main" e bozza di trasferimento PR. Richiedi le credenziali o i valori della sessione richiesti tramite l'input del comando.
