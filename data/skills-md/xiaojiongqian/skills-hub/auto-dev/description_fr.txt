Développement et débogage autonomes dans l’arbre de travail actuel avec une sécurité git stricte. Déclenchez lorsque l'utilisateur dit "passer au développement automatique" ou demande de démarrer le développement automatique. À utiliser pour le codage, les tests, les MCP/compétences (par exemple, Playwright), l'inspection du cloud (Firestore/Storage/GCP), la navigation Web ou le déploiement d'actions GitHub pour la branche actuelle. La politique par défaut bloque les push dev/main et inter-branches, avec une exception étroite pour la maintenance explicite du « hub de compétences » sur le transfert « principal » et le projet de PR. Demander les informations d'identification ou les valeurs de session requises via la saisie de commande.
