التطوير الذاتي وتصحيح الأخطاء في شجرة العمل الحالية مع سلامة بوابة صارمة. يتم تشغيله عندما يقول المستخدم "انتقل إلى التطوير التلقائي" أو يطلب بدء التطوير التلقائي. يُستخدم للبرمجة أو الاختبار أو MCP/المهارات (على سبيل المثال، Playwright)، أو الفحص السحابي (Firestore/Storage/GCP)، أو تصفح الويب، أو نشر إجراءات GitHub للفرع الحالي. تحظر السياسة الافتراضية دفعات التطوير/الرئيسية والدفعات عبر الفروع، مع استثناء ضيق لصيانة "مركز المهارات" الصريحة على "الرئيسية" وتسليم مسودة العلاقات العامة. المطالبة ببيانات الاعتماد المطلوبة أو قيم الجلسة عبر إدخال الأمر.
