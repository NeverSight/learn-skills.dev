Desarrollo y depuración autónomos en el árbol de trabajo actual con estricta seguridad de git. Se activa cuando el usuario dice "ir a desarrollo automático" o solicita iniciar el desarrollo automático. Úselo para codificación, pruebas, MCP/habilidades (por ejemplo, Playwright), inspección de la nube (Firestore/Storage/GCP), navegación web o implementaciones de GitHub Actions para la rama actual. La política predeterminada bloquea los envíos de desarrollo/principal y entre sucursales, con una pequeña excepción para el mantenimiento explícito del `skills-hub` en `main` y el borrador de la transferencia de relaciones públicas. Solicite las credenciales requeridas o los valores de sesión mediante la entrada de comando.
