Orchestrateur de workflow multi-agents pour les projets logiciels. À utiliser lorsque l'utilisateur dit Spidy, implémente à l'aide de Spidy, planifie avec Spidy, nouveau projet Spidy, débogage Spidy ou demande d'exécuter un flux de travail Spidy. À utiliser également lorsque l'utilisateur dit commit, push, create pr, git workflow, l'expédie ou demande de créer une branche/commit/PR ses modifications. Orchestre des agents spécialisés pour : la configuration d'un nouveau projet, la planification des phases, l'exécution des phases, la vérification, le débogage, le mappage de la base de code et le workflow git.
