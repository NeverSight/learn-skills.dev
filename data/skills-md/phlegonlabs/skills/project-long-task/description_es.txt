Inicialice un proyecto de larga duración con un flujo de trabajo de documentos estructurado o actualice un proyecto existente (nuevas funciones, correcciones de errores, refactorizaciones, cambios de requisitos). Utilice esta habilidad siempre que el usuario quiera iniciar un nuevo proyecto, iniciar una tarea de compilación de larga duración, configurar la estructura de documentación del proyecto o decir cosas como "nuevo proyecto", "iniciar un proyecto", "iniciar proyecto", "configuración del proyecto" o "Quiero construir algo desde cero". Úselo también cuando el usuario mencione que desea un plan basado en hitos, un flujo de trabajo de ejecución estructurado o solicite documentación para una compilación compleja de varios pasos. Además, use esta habilidad cuando el usuario desee modificar un proyecto existente que ya tiene un directorio `docs/`; por ejemplo, "agregar una nueva característica", "corregir este error", "refactorizar X", "Quiero cambiar cómo funciona Y", "solicitud de nueva característica".
