Initialisez un projet de longue durée avec un flux de travail de documentation structuré ou mettez à jour un projet existant (nouvelles fonctionnalités, corrections de bogues, refactorisations, modifications des exigences). Utilisez cette compétence chaque fois que l'utilisateur souhaite démarrer un nouveau projet, lancer une tâche de construction de longue durée, configurer la structure de documentation du projet ou dire des choses comme "nouveau projet", "démarrer un projet", "initialiser le projet", "configuration du projet" ou "Je veux créer quelque chose à partir de zéro". À utiliser également lorsque l'utilisateur mentionne vouloir un plan basé sur des jalons, un flux de travail d'exécution structuré ou demande d'étayer la documentation pour une construction complexe en plusieurs étapes. De plus, utilisez cette compétence lorsque l'utilisateur souhaite modifier un projet existant qui possède déjà un répertoire « docs/ » — par exemple, « ajouter une nouvelle fonctionnalité », « corriger ce bug », « refactoriser X », « Je souhaite modifier le fonctionnement de Y », « demande de nouvelle fonctionnalité ».
