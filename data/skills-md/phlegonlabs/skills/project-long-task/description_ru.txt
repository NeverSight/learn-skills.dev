Инициализируйте долгосрочный проект с помощью структурированного рабочего процесса документации или обновите существующий проект (новые функции, исправления ошибок, рефакторинг, изменения требований). Используйте этот навык всякий раз, когда пользователь хочет начать новый проект, запустить длительную задачу сборки, настроить структуру документации проекта или сказать такие вещи, как «новый проект», «начать проект», «инициализировать проект», «настройка проекта» или «Я хочу создать что-то с нуля». Также используйте, когда пользователь упоминает о желании иметь план, основанный на этапах, структурированный рабочий процесс выполнения или просит подготовить документацию для сложной многоэтапной сборки. Кроме того, используйте этот навык, когда пользователь хочет изменить существующий проект, у которого уже есть каталог `docs/` — например, «добавить новую функцию», «исправить эту ошибку», «рефакторинг X», «Я хочу изменить принцип работы Y», «запрос новой функции».
