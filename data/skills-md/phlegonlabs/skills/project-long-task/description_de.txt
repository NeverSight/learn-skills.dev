Initialisieren Sie ein langlaufendes Projekt mit einem strukturierten Dokumentationsworkflow oder aktualisieren Sie ein vorhandenes Projekt (neue Funktionen, Fehlerbehebungen, Refaktorierungen, Anforderungsänderungen). Verwenden Sie diese Fähigkeit immer dann, wenn der Benutzer ein neues Projekt starten, eine lang laufende Build-Aufgabe starten, eine Projektdokumentationsstruktur einrichten oder Dinge wie „Neues Projekt“, „Projekt starten“, „Projekt initiieren“, „Projekt-Setup“ oder „Ich möchte etwas von Grund auf erstellen“ sagen möchte. Verwenden Sie dies auch, wenn der Benutzer einen meilensteinbasierten Plan oder einen strukturierten Ausführungsworkflow wünscht oder die Erstellung einer Dokumentation für einen komplexen, mehrstufigen Build wünscht. Verwenden Sie diese Fähigkeit außerdem, wenn der Benutzer ein vorhandenes Projekt ändern möchte, das bereits über ein „docs/“-Verzeichnis verfügt – z. B. „eine neue Funktion hinzufügen“, „diesen Fehler beheben“, „X umgestalten“, „Ich möchte die Funktionsweise von Y ändern“, „neue Funktionsanforderung“.
