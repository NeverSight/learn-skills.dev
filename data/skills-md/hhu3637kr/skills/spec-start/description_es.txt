Inicie un flujo de trabajo de Teams de agentes de desarrollo basado en especificaciones. Úselo cuando el usuario comienza una nueva tarea de desarrollo y necesita un equipo de colaboración estructurado y multifuncional. Corresponde al final de las especificaciones; spec-start inicia el equipo, spec-end lo cierra. Activadores: (1) el usuario dice "iniciar nueva tarea"/"crear equipo de desarrollo"/"iniciar flujo de trabajo", (2) se requiere un ciclo de desarrollo completo basado en especificaciones (explorar→escribir→implementar→probar→conclusión), (3) el usuario desea administrar una tarea de desarrollo con la arquitectura de Agent Teams. Nota: se supone que se ha ejecutado spec-init. Si el directorio spec/ está ausente, llame primero a spec-init.
