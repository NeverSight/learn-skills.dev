Launch the Agent Teams workflow for Spec-driven development. Use it when a user starts a new development task and needs to create a structured, multi-role collaborative team. Paired with spec-end, spec-start initiates the team and spec-end closes it. Trigger conditions: (1) user says “start new task” / “create dev team” / “launch workflow”, (2) a full Spec-driven development process is required (explore → write → implement → test → close), (3) user wants to manage a development task with the Agent Teams architecture. Note: this skill assumes the project has already been initialized with spec-init; if the spec/ directory does not exist, call spec-init first.
