Starten Sie den Agent Teams-Workflow für die spezifikationsgesteuerte Entwicklung. Verwenden Sie es, wenn ein Benutzer eine neue Entwicklungsaufgabe startet und ein strukturiertes, kollaboratives Team mit mehreren Rollen erstellen muss. In Kombination mit spec-end initiiert spec-start das Team und spec-end schließt es. Auslösebedingungen: (1) Benutzer sagt „Neue Aufgabe starten“ / „Entwicklerteam erstellen“ / „Workflow starten“, (2) ein vollständiger spezifikationsgesteuerter Entwicklungsprozess ist erforderlich (Erkunden → Schreiben → Implementieren → Testen → Schließen), (3) Benutzer möchte eine Entwicklungsaufgabe mit der Agent Teams-Architektur verwalten. Hinweis: Bei diesem Skill wird davon ausgegangen, dass das Projekt bereits mit spec-init initialisiert wurde. Wenn das spec/-Verzeichnis nicht existiert, rufen Sie zuerst spec-init auf.
