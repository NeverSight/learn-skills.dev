Suivez strictement plan.md pour développer de nouvelles fonctionnalités. Invoqué par le rôle spec-executor. Conditions de déclenchement : (1) le rôle spec-exécuteur doit implémenter le code conformément à plan.md, (2) l'utilisateur dit "exécuter Spec" / "démarrer l'implémentation", (3) un plan.md existe dans le répertoire spec/ mais pas de summary.md. Remarque : à partir de la version 2.0, le chemin B (équipes d'agents) et les étapes de test sont supprimés ; spec-execute n'a qu'une seule tâche : implémenter le code selon plan.md, produire summary.md et archiver. Si le répertoire contient déjà summary.md, utilisez spec-update.
