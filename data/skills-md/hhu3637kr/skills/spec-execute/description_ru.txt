Строго следуйте plan.md для разработки новых функций. Вызывается ролью spec-executor. Условия срабатывания: (1) исполнителю спецификации роли необходимо реализовать код в соответствии с plan.md, (2) пользователь говорит «Выполнить спецификацию» / «Начать реализацию», (3) plan.md существует в каталоге spec/, а summary.md — нет. Примечание: начиная с версии 2.0 удален путь B (команды агентов) и этапы тестирования; У spec-execute есть только одна обязанность: реализовать код в соответствии с plan.md, создать summary.md и заархивировать. Если summary.md уже существует в каталоге, используйте spec-update.
