Условия срабатывания: (1) Роль автора спецификации должна создать проектный документ, спецификацию API, модель данных, проект архитектуры или план рефакторинга; (2) пользователь говорит «Создать спецификацию» / «Написать проектную документацию» / «Написать техническую спецификацию» / «План проектирования»; (3) в каталоге spec/ необходимо создать plan.md.  
Примечание: начиная с версии 2.0 plan.md не включает раздел плана тестирования (создается отдельно Spec-Tester через Spec-Test); Execution_mode фиксирован как одноагентный.  
Если summary.md уже существует в каталоге, используйте вместо этого навыка spec-update.
