Directives de sécurité de type TypeScript pour écrire du code de sécurité de type maximale. Appliquez ces modèles lors de l'écriture ou de la révision de n'importe quel TypeScript : unions discriminées, rétrécissement des types, protections de type, contrôles d'exhaustivité, évitement des transtypages "tels", préférant "inconnu" à "tout" et rendant les états impossibles irreprésentables. Utilisez cette compétence chaque fois que vous écrivez du code TypeScript, révisez TypeScript pour des problèmes de sécurité des types ou lorsque l'utilisateur mentionne la sécurité des types, le rétrécissement des types, les unions discriminées ou demande de rendre les types plus stricts/plus explicites.
