このスキルは、SAP API スタイル ガイド標準に従って SAP API を文書化するための包括的なガイダンスを提供します。
REST、OData、Java、JavaScript、.NET、または C/C++ API の API ドキュメントを作成またはレビューするときに使用する必要があります。

このスキルは、命名規則、ドキュメントのコメント、OpenAPI 仕様、品質チェックリスト、非推奨ポリシー、
およびマニュアル文書テンプレート。これにより、SAP API Business Hub 標準および業界のベスト プラクティスとの一貫性が確保されます。

キーワード: SAP API、REST、OData、OpenAPI、Swagger、Javadoc、JSDoc、XML ドキュメント、API ビジネス ハブ、API 命名、
API の非推奨、x-sap-stateInfo、Entity Data Model、EDM、ドキュメント タグ、API 品質、API テンプレート
