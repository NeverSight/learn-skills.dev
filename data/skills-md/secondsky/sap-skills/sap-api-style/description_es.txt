Esta habilidad proporciona una guía integral para documentar las API de SAP siguiendo los estándares de la Guía de estilo de API de SAP.
Debe usarse al crear o revisar la documentación de API para API REST, OData, Java, JavaScript, .NET o C/C++.

La habilidad cubre convenciones de nomenclatura, comentarios de documentación, especificaciones de OpenAPI, listas de verificación de calidad, políticas de obsolescencia,
y plantillas de documentación manual. Garantiza la coherencia con los estándares de SAP API Business Hub y las mejores prácticas de la industria.

Palabras clave: API de SAP, REST, OData, OpenAPI, Swagger, Javadoc, JSDoc, documentación XML, API Business Hub, denominación de API,
Desuso de API, x-sap-stateInfo, Entity Data Model, EDM, etiquetas de documentación, calidad de API, plantillas de API
