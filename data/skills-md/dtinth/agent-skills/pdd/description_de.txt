Fähigkeiten für Puzzle Driven Development (PDD). Beziehen Sie sich auf diese Fähigkeit, wenn ein Projekt PDD verwendet oder der Betreiber PDD oder rätselgesteuerte Entwicklung erwähnt. Verwenden Sie diese Fähigkeit, wenn: (1) eine große Funktion in inkrementelle Ergebnisse aufgeteilt wird, (2) @todo-Stub-Kommentare schreiben, um nicht implementierten Code zu markieren, (3) ein vorhandenes @todo-Rätsel zur Implementierung aufgreift, (4) entscheiden, ob eine Aufgabe mit einem Stub abgeschlossen oder weitergearbeitet werden soll.
