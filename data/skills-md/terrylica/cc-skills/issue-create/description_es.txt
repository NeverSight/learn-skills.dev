Cree problemas de GitHub con buen formato con sugerencias de etiquetas inteligentes impulsadas por IA y detección de tipo de contenido. Úselo siempre que el usuario quiera crear un informe de error, solicitud de función, pregunta o problema de documentación en GitHub, o diga "presentar un problema", "crear un problema" o "crear un problema". NO lo use para gestionar problemas existentes, organizar jerarquías de problemas (use el flujo de trabajo de problemas en su lugar) ni para la creación de relaciones públicas.
