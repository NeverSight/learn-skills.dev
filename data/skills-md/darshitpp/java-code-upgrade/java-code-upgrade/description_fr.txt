Mettez à niveau le code Java des anciens idiomes vers des équivalents modernes. Analyse les modèles hérités (pré-Java 10 à Java 25) dans les fonctionnalités du langage, les collections, les chaînes, les flux, la concurrence, les entrées/sorties, la gestion des erreurs, la date/heure, la sécurité, les outils et les API d'entreprise, puis suggère des remplacements modernes avec des exemples avant/après. À utiliser lors de la modernisation des bases de code Java, de l'examen des demandes d'extraction pour les modèles obsolètes ou de la migration de Java EE vers Jakarta EE. Ne pas utiliser pour les langages non Java, la configuration de l'outil de génération ou la logique d'application spécifique au framework.
