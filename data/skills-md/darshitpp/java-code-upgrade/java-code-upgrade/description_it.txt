Aggiorna il codice Java dagli idiomi più vecchi agli equivalenti moderni. Esegue la scansione dei modelli legacy (da Java 10 a Java 25) tra funzionalità del linguaggio, raccolte, stringhe, flussi, concorrenza, input/output, gestione degli errori, data/ora, sicurezza, strumenti e API aziendali, quindi suggerisce sostituzioni moderne con esempi prima/dopo. Da utilizzare durante la modernizzazione delle basi di codice Java, la revisione delle richieste pull per modelli obsoleti o la migrazione da Java EE a Jakarta EE. Non utilizzare per linguaggi non Java, configurazione di strumenti di creazione o logica applicativa specifica del framework.
