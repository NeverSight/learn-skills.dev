Linee guida per la progettazione dell'interfaccia utente professionale per interfacce accessibili e ben strutturate. Utilizzare quando: si scrive codice CSS, HTML o componente frontend; progettazione di siti Web, app, dashboard o qualsiasi interfaccia Web; rivedere o migliorare i progetti dell'interfaccia utente esistenti; prendere decisioni su colore, tipografia, layout, spaziatura, pulsanti o moduli; garantire l’accessibilità alle WCAG 2.1 AA; creazione di sistemi di progettazione o librerie di componenti. Copre tavolozze di colori OKLCH, griglia di spaziatura 8pt, tipografia fluida con clamp(), query sui contenitori, griglia secondaria, modelli di convalida dei moduli, gerarchia dei pulsanti, modalità oscura, movimento ridotto, meta tag SEO e Core Web Vitals.
