Integrieren Sie die eigenständige Didit Face Search-API, um eine 1:N-Gesichtssuche für alle zuvor verifizierten Sitzungen durchzuführen. Verwenden Sie diese Option, wenn der Benutzer doppelte Konten erkennen, nach passenden Gesichtern suchen, überprüfen möchte, ob ein Gesicht bereits im System vorhanden ist, doppelte Registrierungen verhindern, anhand der Sperrliste suchen oder die Deduplizierung von Gesichtern mithilfe von Didit implementieren möchte.
