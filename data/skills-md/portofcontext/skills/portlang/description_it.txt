portlang: il framework dell'agente incentrato sull'ambiente. Da utilizzare durante la creazione di file field.toml, la definizione di limiti e verificatori, l'aggiunta di strumenti personalizzati (shell, Python, MCP), il debug di traiettorie, la misurazione della convergenza, la configurazione di output JSON strutturato con output_schema, l'esecuzione di valutazioni batch, la visualizzazione di dashboard di traiettorie HTML o l'analisi del comportamento dell'agente durante le esecuzioni. portlang gestisce ambienti non loop: tu definisci lo spazio di ricerca, l'agente trova il percorso.
