Implémentez, déboguez et optimisez les applications Web progressives dans Next.js (App Router) avec des techniciens de service, des notifications push, une prise en charge hors ligne, des stratégies de mise en cache et une installabilité. À utiliser lorsque les demandes mentionnent la configuration Next.js PWA, app/manifest.ts, public/sw.js, navigator.serviceWorker.register, updateViaCache, les stratégies de cache (Cache First, Network First, Stale-While-Revalidate), les pages de secours hors ligne, le modèle App Shell, la mise en cache préalable, la synchronisation en arrière-plan, le push Web avec les actions du serveur, les clés VAPID, le cycle de vie du service worker (skipWaiting, clients.claim), la notification de mise à jour UX, Tests locaux HTTPS, en-têtes de sécurité dans next.config.js, intégration Serwist/Workbox, icônes masquables, bizarreries iOS PWA ou problèmes Lighthouse PWA dans Next.js. À utiliser également lorsque les utilisateurs dépannent des caches obsolètes, des échecs d'enregistrement, des conflits de portée ou des problèmes de version de cache.
