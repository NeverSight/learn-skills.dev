Service Worker、プッシュ通知、オフライン サポート、キャッシュ戦略、インストール機能を使用して、Next.js (App Router) でプログレッシブ Web アプリを実装、デバッグ、最適化します。リクエストで Next.js PWA セットアップ、app/manifest.ts、public/sw.js、navigator.serviceWorker.register、updateViaCache、キャッシュ戦略 (Cache First、Network First、Stale-while-Revalidate)、オフライン フォールバック ページ、App Shell パターン、事前キャッシュ、バックグラウンド同期、サーバー アクションによる Web プッシュ、VAPID キー、Service Worker ライフサイクル (skipWaiting、clients.claim)、更新通知に言及する場合に使用します。 UX、HTTPS ローカル テスト、next.config.js のセキュリティ ヘッダー、Serwist/Workbox 統合、マスク可能なアイコン、iOS PWA の癖、または Next.js の Lighthouse PWA の問題。ユーザーが古いキャッシュ、登録の失敗、スコープの競合、またはキャッシュのバージョン管理の問題をトラブルシューティングするときにも使用します。
