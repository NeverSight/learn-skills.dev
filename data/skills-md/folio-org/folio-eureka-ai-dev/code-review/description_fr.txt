À utiliser lorsque l'utilisateur demande à effectuer une révision du code, à examiner les modifications du code, à analyser une différence ou à auditer la qualité du code. Exécute un examen structuré de la sortie de git diff couvrant la sécurité, l'exactitude, les performances, la maintenabilité et le style. Produit un rapport de démarque enregistré sous forme de fichier .md nommé d'après la branche actuelle.
