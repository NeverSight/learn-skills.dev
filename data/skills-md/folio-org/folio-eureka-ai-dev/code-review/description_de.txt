Verwenden Sie diese Option, wenn der Benutzer eine Codeüberprüfung durchführen, Codeänderungen überprüfen, einen Unterschied analysieren oder die Codequalität prüfen möchte. Führt eine strukturierte Überprüfung der Git-Diff-Ausgabe durch, die Sicherheit, Korrektheit, Leistung, Wartbarkeit und Stil umfasst. Erstellt einen Markdown-Bericht, der als .md-Datei mit dem Namen des aktuellen Zweigs gespeichert wird.
