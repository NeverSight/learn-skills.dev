Quando l'utente desidera verificare se una competenza, un repository GitHub, un pacchetto npm o un codice locale sono sicuri da scaricare o utilizzare. Ciò include il rilevamento di codice dannoso, malware, furto di chiavi, modifica delle variabili di ambiente, comportamento di rete sospetto e valutazione della reputazione del repository (stelle, fork, contributori, età). Utilizzare questa competenza ogni volta che l'utente menziona il controllo delle competenze per i rischi per la sicurezza, la scansione dei repository per malware, la verifica della sicurezza del codice, il controllo delle minacce nei pacchetti npm o la richiesta se un download è sicuro.
