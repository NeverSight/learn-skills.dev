Verwenden Sie diesen Skill immer dann, wenn der Benutzer über MCPorter CLI mit Vibe Kanban arbeiten möchte. Deckt jeden Workflow ab, der Vibe Kanban-Probleme, Projekte, Repositorys, Arbeitsbereiche oder Arbeitsbereichssitzungen erstellt, aktualisiert, auflistet, zuweist, markiert, verknüpft oder verwaltet
 Ausführen der Befehle „mcporter call vibe_kanban.<tool>“ im Terminal. Diese Fähigkeit
ist das Referenz- und Workflow-Handbuch zum Auflösen von Organisations-/Projekt-/Arbeitsbereichs-IDs, zum Konfigurieren von MCPorter, zum Verwalten von Repo-Skripten und zum Starten oder Verknüpfen von Arbeitsbereichssitzungen. Wird auch dann ausgelöst, wenn der Benutzer nur Dinge wie „Vibe Kanban“, „Kanban“, „mcporter“, „mcporter call vibe_kanban“, „Problem erstellen“, „Dieses Problem zuweisen“, „Arbeitsbereichssitzung starten“, „Arbeitsbereich verknüpfen“ sagt oder fragt, wie ein Agent, Claude Code, Codex, Cursor, OpenCode, Gemini, Copilot, Amp, Droid oder Qwen programmgesteuert mit Vibe Kanban verbunden werden soll.
