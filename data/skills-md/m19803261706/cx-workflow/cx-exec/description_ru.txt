Рабочий процесс CX — выполнение задач и проверка контракта. Срабатывает, когда пользователь упоминает «выполнить задачу», «начать разработку», «внедрить функцию», «написать код», «продолжить работу» или «следующую задачу». Пятиэтапный процесс: чтение задачи → реализация → проверка контракта → тестирование → фиксация. Использует локальные файлы Task-{n}.md и status.json для автоматической проверки соответствия реализации контракту. После выполнения всех задач автоматически запускается cx-summary.
