Flujo de trabajo CX: ejecución de tareas y validación de contratos. Se activa cuando el usuario dice "ejecutar tarea", "iniciar desarrollo", "implementar función", "escribir código", "continuar" o "siguiente tarea". Proceso de 5 pasos: leer tarea → implementar → validación de contrato → probar → confirmar. Utiliza la tarea local-{n}.md y status.json para verificar automáticamente la implementación con respecto al contrato. Una vez completadas todas las tareas, cx-summary se activa automáticamente.
