將編碼執行委託給 Codex CLI 來執行具有明確成功標準的任務。

**在以下情況下使用此技能：**
- 重構、重新命名、遷移、升級或轉換程式碼
- 產生測試、新增覆蓋範圍、編寫單元/整合測試
- 跨儲存庫批次編輯：“更新全部”、“應用到各處”、“掃描”
- 實現功能：“連接”、“支架”、“新增支援”
- 修復問題：“修復失敗的測試”、“修復類型/lint 錯誤”、“修復建置”、“綠色 CI”
- API 變更：“更新用法”、“採用新 API”、“codemod”
- 標準化：“保持一致”、“去重”、“標準化”
- 使用具體工件進行程式碼審查或調查（差異、日誌、重現、失敗測試）

**在以下情況下使用 CLAUDE（而非 CODEX）：**
- 策略決策：“我們應該”、“比較方法”、“推薦策略”
- 系統設計、架構、需求設計
- 教學、解釋概念、腦力激盪、RFC 寫作
- 沒有重現/日誌/堆疊追蹤的未知問題
- 不明確的任務需要先澄清

**兩階段模型：** 對於混合請求（「評估然後實施」），Claude 首先制定框架，如果計劃具體，則 Codex 執行。

CodeX性價比高，執行力強。克勞德能夠更好地處理模糊性、策略和架構。
