명확한 성공 기준이 있는 구현 작업을 위해 코딩 실행을 Codex CLI에 위임하세요.

**이 기술을 사용하는 경우:**
- 코드 리팩토링, 이름 바꾸기, 마이그레이션, 업그레이드 또는 변환
- 테스트 생성, 커버리지 추가, 단위/통합 테스트 작성
- 저장소 전반의 일괄 편집: "모두 업데이트", "모든 곳에 적용", "비우기"
- 기능 구현: "와이어 업", "비계", "지원 추가"
- 문제 해결: "실패하는 테스트 수정", "유형/린트 오류 수정", "빌드 수정", "녹색 CI"
- API 변경: "사용법 업데이트", "새 API 채택", "codemod"
- 표준화: "일관되게 만듭니다", "중복 제거", "표준화"
- 구체적인 아티팩트(차이점, 로그, 재현, 실패한 테스트)를 사용한 코드 검토 또는 조사

**다음과 같은 경우에는 CLAUDE(CODEX 아님)를 사용하세요.**
- 전략적 결정: "해야 할까요?", "접근 방식을 비교해야 할까요?", "전략을 권장합니다"
- 시스템 설계, 아키텍처, 요구사항 설계
- 교육, 개념 설명, 브레인스토밍, RFC 작성
- 재현/로그/스택 추적이 없는 알 수 없는 문제
- 먼저 명확한 설명이 필요한 모호한 작업

**2단계 모델:** 혼합 요청("평가 후 구현")의 경우 Claude가 먼저 프레임을 구성한 다음 계획이 구체적이면 Codex가 실행됩니다.

CodeX는 비용 효율적이고 실행력이 뛰어납니다. Claude는 모호함, 전략 및 아키텍처를 더 잘 처리합니다.
