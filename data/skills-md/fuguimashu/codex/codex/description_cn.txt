将编码执行委托给 Codex CLI 来执行具有明确成功标准的任务。

**在以下情况下使用此技能：**
- 重构、重命名、迁移、升级或转换代码
- 生成测试、添加覆盖范围、编写单元/集成测试
- 跨存储库批量编辑：“更新全部”、“应用到各处”、“扫描”
- 实现功能：“连接”、“支架”、“添加支持”
- 修复问题：“修复失败的测试”、“修复类型/lint 错误”、“修复构建”、“绿色 CI”
- API 更改：“更新用法”、“采用新 API”、“codemod”
- 标准化：“保持一致”、“去重”、“标准化”
- 使用具体工件进行代码审查或调查（差异、日志、重现、失败测试）

**在以下情况下使用 CLAUDE（而非 CODEX）：**
- 战略决策：“我们应该”、“比较方法”、“推荐策略”
- 系统设计、架构、需求设计
- 教学、解释概念、头脑风暴、RFC 写作
- 没有重现/日志/堆栈跟踪的未知问题
- 不明确的任务需要首先澄清

**两阶段模型：** 对于混合请求（“评估然后实施”），Claude 首先制定框架，如果计划具体，则 Codex 执行。

CodeX性价比高，执行力强。克劳德能够更好地处理模糊性、策略和架构。
