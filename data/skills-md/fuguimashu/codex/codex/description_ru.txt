Делегируйте выполнение кодирования интерфейсу командной строки Codex для задач реализации с четкими критериями успеха.

**ИСПОЛЬЗУЙТЕ ЭТОТ НАВЫК, КОГДА:**
- Рефакторинг, переименование, миграция, обновление или преобразование кода.
- Генерация тестов, добавление покрытия, написание модульных/интеграционных тестов.
- Пакетные изменения в репозитории: «обновить все», «применить везде», «очистить».
- Реализация функций: «подключить», «подмостки», «добавить поддержку».
- Исправление проблем: «исправить неудачные тесты», «исправить ошибки типа/lint», «исправить сборку», «зеленый CI».
- Изменения API: «обновить использование», «принять новый API», «codemod».
- Стандартизация: «сделать согласованным», «дедублировать», «стандартизировать».
- Обзор кода или исследование С конкретными артефактами (разница, журналы, воспроизведение, неудачный тест)

**ИСПОЛЬЗУЙТЕ CLAUDE (НЕ КОДЕКС), КОГДА:**
- Стратегические решения: «должны ли мы», «сравнить подходы», «рекомендовать стратегию»
- Проектирование системы, архитектура, разработка требований
- Преподавание, объяснение концепций, мозговой штурм, написание RFC.
- Неизвестные проблемы без следов воспроизведения, журналов и стека.
- Неоднозначные задачи, требующие предварительного разъяснения

**ДВУХЭТАПНАЯ МОДЕЛЬ:** Для смешанных запросов («оценить, затем реализовать») сначала Claude формирует, а затем выполняет Кодекс, если план конкретен.

CodeX является экономически эффективным и надежным в исполнении. Клод лучше справляется с двусмысленностью, стратегией и архитектурой.
