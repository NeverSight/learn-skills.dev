明確な成功基準を備えた実装タスクについては、コーディングの実行を Codex CLI に委任します。

**このスキルは次の場合に使用してください:**
- コードのリファクタリング、名前変更、移行、アップグレード、または変換
- テストの生成、カバレッジの追加、単体テスト/統合テストの作成
- リポジトリ全体のバッチ編集: 「すべて更新」、「どこにでも適用」、「スイープ」
- 機能の実装: 「ワイヤーアップ」、「足場」、「サポートの追加」
- 問題の修正: 「失敗したテストを修正」、「タイプ/lint エラーを修正」、「ビルドを修正」、「グリーン CI」
- API の変更: 「使用法の更新」、「新しい API の採用」、「codemod」
- 標準化: 「一貫性を持たせる」、「重複排除」、「標準化」
- 具体的な成果物 (差分、ログ、再現、失敗したテスト) を使用したコード レビューまたは調査

**次の場合にクロード (コーデックスではなく) を使用します。**
- 戦略的決定: 「すべきか」、「アプローチを比較」、「戦略を推奨」
- システム設計、アーキテクチャ、要件設計
- 教育、概念の説明、ブレインストーミング、RFC 執筆
- 再現/ログ/スタックトレースのない不明な問題
- 曖昧なタスクは最初に説明する必要があります

**2 段階モデル:** 混合リクエスト (「評価してから実装」) の場合、最初にクロード フレームが作成され、計画が具体的であれば Codex が実行されます。

CodeX はコスト効率が高く、強力な実行を実現します。クロードは、曖昧さ、戦略、アーキテクチャをより適切に処理します。
