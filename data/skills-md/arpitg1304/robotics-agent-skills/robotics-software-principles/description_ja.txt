基本的なソフトウェア設計原則は、ロボットモジュール開発に特に適用されます。このスキルは、ロボット ソフトウェア モジュールの設計、コードベースの構築、アーキテクチャの決定、ロボット コードのレビュー、または再利用可能なロボット ライブラリの構築を行うときに使用します。ユーザーがロボットの SOLID 原則、モジュール型ロボット工学ソフトウェア、ロボットのクリーン アーキテクチャ、ロボット工学の依存関係注入、ハードウェアのインターフェイス設計、リアルタイム設計の制約、ロボットのエラー処理戦略、構成管理、認識 - 計画 - 制御における関心事の分離、ロボットの動作の構成可能性、またはロボット工学のコンテキストにおけるソフトウェア クラフトマンシップに関する議論に言及するたびにトリガーされます。また、ロボット コードのコード レビュー、ロボット ソフトウェアのリファクタリング、またはロボット ライブラリの API の設計のトリガーにもなります。
