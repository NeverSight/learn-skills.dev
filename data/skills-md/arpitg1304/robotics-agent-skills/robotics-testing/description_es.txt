Estrategias de prueba, patrones y herramientas para software de robótica. Utilice esta habilidad al escribir pruebas unitarias, pruebas de integración, pruebas de simulación o pruebas de hardware en el circuito para sistemas robóticos. Se activa cada vez que el usuario menciona probar nodos ROS, pytest con ROS, launch_testing, pruebas de simulación, CI/CD para robótica, dispositivos de prueba para sensores, hardware simulado, reproducción determinista, pruebas de regresión para comportamientos de robots o validación de canales de percepción/planificación/control. También cubre pruebas basadas en propiedades para cinemática, pruebas fuzz para manejadores de mensajes y pruebas de archivos dorados para trayectorias.
