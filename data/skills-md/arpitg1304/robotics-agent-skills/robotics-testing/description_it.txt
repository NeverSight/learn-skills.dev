Strategie, modelli e strumenti di test per il software di robotica. Utilizzare questa competenza durante la scrittura di test unitari, test di integrazione, test di simulazione o test hardware-in-the-loop per sistemi robotici. Si attiva ogni volta che l'utente menziona test di nodi ROS, pytest con ROS, launch_testing, test di simulazione, CI/CD per la robotica, dispositivi di test per sensori, hardware simulato, replay deterministico, test di regressione per comportamenti dei robot o convalida di pipeline di percezione/pianificazione/controllo. Copre anche il test basato sulle proprietà per la cinematica, il test fuzz per i gestori di messaggi e il test dei file golden per le traiettorie.
