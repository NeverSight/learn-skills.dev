Muster zum Teilen von Code zwischen macOS und iOS in SwiftUI-Apps. Umfasst Projektstruktur (70 % geteilt / 15 % macOS / 15 % iOS), Plattformabstraktion über Protokolle und bedingte #if os()-Kompilierung, adaptive Navigation (NavigationSplitView auf Mac/iPad → NavigationStack auf iPhone), gemeinsam genutzte Komponenten mit Plattformstil, iOS-spezifische Erweiterungen (benutzerdefinierte Tastaturerweiterung, interaktive Widgets, Freigabeerweiterung, Aktionserweiterung, Control Center-Widget, Sperrbildschirm-Widget), App-Gruppen für die Datenfreigabe mit Erweiterungen, CloudKit-Synchronisierungsüberwachung, JSON-Export/-Import, Schema Versionierung und Migration, URL-Schema-Deep-Linking und die vollständige macOS→iOS-Migrationscheckliste. Verwenden Sie diese Option beim Erstellen von Apps, die sowohl auf macOS als auch auf iOS abzielen, beim Hinzufügen von iOS-Unterstützung zu einer macOS-App, beim Erstellen von Widgets oder Tastaturerweiterungen oder beim Einrichten der iCloud-Synchronisierung mit SwiftData.
