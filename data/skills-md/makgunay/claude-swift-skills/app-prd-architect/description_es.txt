Descubrimiento guiado de aplicaciones, generación de PRD y diseño arquitectónico para aplicaciones nativas de macOS/iOS. Lleva al usuario desde una idea aproximada de la aplicación a través de una exploración interactiva de características, experiencia de usuario y visión de diseño, luego produce tres entregables: (1) un PRD integral con requisitos funcionales, historias de usuarios y métricas de éxito, (2) un documento de arquitectura técnica con modelos de datos, diagramas de sistema y diseño de componentes, (3) una lista de características para partes interesadas no técnicas. Úselo cuando el usuario dice cosas como "Tengo una idea para una aplicación", "ayúdame a planificar una aplicación", "escribe un PRD", "diseña la arquitectura", "qué debería hacer mi aplicación", "ayúdame a pensar en las funciones" o "Quiero crear una aplicación que...". También se utiliza cuando el usuario tiene un borrador de PRD existente y desea ampliarlo, validarlo o reestructurarlo. Guía a través del descubrimiento iterativo antes de producir documentos; nunca pasa directamente a escribir.
