سير عمل منظم لتحديد مواصفات كود Python وتحسينه - نقاط اتصال وحدة المعالجة المركزية (CPU)، واستخدام الذاكرة، والتحقق من الصحة المستند إلى المعايير. استخدم هذه المهارة عندما يطلب المستخدم جعل كود Python أسرع، أو تقليل استخدام الذاكرة، أو أداء الملف الشخصي، أو العثور على الاختناقات، أو تحسين المسارات الساخنة، أو تقليل التخصيصات، أو التحقيق في سبب بطء شيء ما. يتم تشغيله أيضًا عندما يذكر المستخدم أدوات إنشاء الملفات الشخصية (cProfile، وline_profiler، وpy-spy، وmemray، وTracemalloc) أو يطلب إعداد معايير. حتى الطلبات غير الرسمية مثل "هذا بطيء" أو "هل يمكننا تسريع هذا" يجب أن تؤدي إلى تحفيز هذه المهارة.
