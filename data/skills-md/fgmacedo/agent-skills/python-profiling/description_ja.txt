Python コードのプロファイリングと最適化のための構造化されたワークフロー - CPU ホットスポット、メモリ使用量、ベンチマーク主導の検証。ユーザーが Python コードの高速化、メモリ使用量の削減、パフォーマンスのプロファイリング、ボトルネックの発見、ホット パスの最適化、割り当ての削減、または何かが遅い理由の調査を要求する場合は常に、このスキルを使用します。また、ユーザーがプロファイリング ツール (cProfile、line_profiler、py-spy、memray、tracemalloc) について言及した場合、またはベンチマークのセットアップを要求した場合にもトリガーされます。 「これは遅いです」または「これを速くできますか」などの何気ないリクエストでも、このスキルがトリガーされるはずです。
