Flux de travail structuré pour le profilage et l'optimisation du code Python : points chauds du processeur, utilisation de la mémoire et validation basée sur des benchmarks. Utilisez cette compétence chaque fois que l'utilisateur demande d'accélérer le code Python, de réduire l'utilisation de la mémoire, les performances du profil, de détecter les goulots d'étranglement, d'optimiser les chemins chauds, de réduire les allocations ou de rechercher pourquoi quelque chose est lent. Se déclenche également lorsque l'utilisateur mentionne des outils de profilage (cProfile, line_profiler, py-spy, memray, tracemalloc) ou demande de mettre en place des benchmarks. Même les requêtes occasionnelles telles que « c'est lent » ou « pouvons-nous accélérer » devraient déclencher cette compétence.
