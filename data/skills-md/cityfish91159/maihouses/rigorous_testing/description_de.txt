Strenges Testprotokoll. Setzen Sie die Regel durch: „Was nicht getestet wird, ist nicht erledigt.“

Triggerpunkte:
- Beim Beheben eines Fehlers (Regressionstest muss hinzugefügt werden)
- Bei der Implementierung einer neuen Funktion (entsprechender Test erforderlich)
- Beim Ändern von vorhandenem Code (stellen Sie sicher, dass die Tests weiterhin bestehen)
– Wenn der Benutzer „Test“, „Testen“ oder „Funktionalität überprüfen“ erwähnt
