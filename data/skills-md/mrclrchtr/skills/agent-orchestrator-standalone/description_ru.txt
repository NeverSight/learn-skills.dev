Организуйте сложную работу с помощью поэтапного многоагентного цикла (аудит → проектирование → реализация → проверка → проверка → доставка). Используйте его, когда вам нужно разделить работу на подсистемы, провести независимый аудит, согласовать полученные результаты в список подтвержденных проблем, делегировать исправления в кластеры, обеспечить соблюдение шлюзов проверки PASS/FAIL и обеспечить сквозную проверенную доставку. Не используйте для небольших задач, состоящих из одного файла.
