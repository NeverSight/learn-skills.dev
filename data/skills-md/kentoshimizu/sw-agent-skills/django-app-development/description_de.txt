Django-Anwendungsentwicklungsworkflow für App-Änderungen in Produktionsqualität. Wird verwendet, wenn Django-spezifischer Code (Modelle, Ansichten, Serialisierer/Formulare, Routing, Einstellungen, Middleware, Authentifizierungsflüsse) implementiert oder überarbeitet werden muss, wobei das Laufzeitverhalten des Frameworks im Geltungsbereich liegt; Nicht für Repository-weite Architektur-Governance oder Release-Management-Richtlinien verwenden.
