Ejecute una revisión completa del código para comprobar su corrección, mantenibilidad y riesgo de regresión cuando ninguna especialidad predomine. Úselo para revisiones amplias de preparación para la fusión con evidencia y hallazgos explícitos; Si el riesgo de seguridad o rendimiento es principal, priorice primero "seguridad-revisión de código" o "rendimiento-revisión de código".
