Ресурсно-ориентированный дизайн контракта REST/OpenAPI для семантики URI/метода, идемпотентности, нумерации страниц и поведения статуса HTTP. Запускается, когда различия контракта REST обнаруживаются в путях, методах, схемах или семантике состояния или когда выбор транспорта запроса-ответа все еще не решен до реализации. Не используйте для разработки схемы GraphQL, внесения изменений только в таксономию ошибок, управления жизненным циклом версий или реализации тестирования контракта потребителя-поставщика.
