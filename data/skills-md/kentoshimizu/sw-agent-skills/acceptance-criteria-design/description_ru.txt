Разработайте исполняемые критерии приемки для утвержденных требований, преобразовав цели/спецификации в двоичные проверки «прошел/не прошел» с наблюдаемыми результатами. Используйте, когда передача реализации, проверка качества или решения о выпуске требуют проверяемых критериев для стабильных базовых показателей `REQ-*`/`NFR-*`; не используйте для обнаружения требований, определения приоритетов или разделения спринта.
