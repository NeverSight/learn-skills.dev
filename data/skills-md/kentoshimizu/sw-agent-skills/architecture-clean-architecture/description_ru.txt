Рабочий процесс «Чистая архитектура» для обеспечения соблюдения направления зависимостей, стабильных границ домена и разработки приложений, ориентированных на варианты использования. Используйте, когда командам необходимо отделить бизнес-правила от инфраструктур и механизмов доставки; не используйте для очистки изолированного модуля без последствий для границ.
