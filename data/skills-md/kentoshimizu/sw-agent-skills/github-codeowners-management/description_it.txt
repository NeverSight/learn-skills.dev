Governa le regole di CODEOWNERS in modo che il routing delle revisioni rifletta i reali limiti di proprietà e rischio su GitHub. Da utilizzare quando è necessario definire, aggiornare o verificare la mappatura della proprietà del repository o le regole di revisione obbligatorie; non utilizzare per un'architettura runtime non GitHub o per la progettazione del livello dati.
