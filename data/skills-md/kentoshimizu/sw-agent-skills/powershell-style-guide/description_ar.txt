معايير التصميم والمراجعة وإعادة البناء للبرمجة النصية لـ PowerShell. يتم تشغيله عند إنشاء ملفات `.ps1` أو `.psm1` أو `.psd1` أو كتل سير عمل CI مع `shell: pwsh` أو `shell: powershell` أو تعديلها أو مراجعتها ويجب فرض ضوابط الجودة الخاصة بـ PowerShell (معالجة الأخطاء، والتحقق من صحة المعلمات، وسهولة القراءة، والسلامة التشغيلية). لا تستخدمه مع قواعد Bash أو POSIX العامة `sh` أو قواعد نمط التطبيق الخاصة باللغة. في طلبات السحب متعددة اللغات، يمكنك تشغيلها مع مهارات `*-style-guide` الأخرى القابلة للتطبيق.
