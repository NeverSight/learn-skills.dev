API-Versionslebenszyklus-Governance für Breaking-Change-Klassifizierung, Verfallsfenster, Migrationsplanung und Support-Matrix-Management für interne und externe Verbraucher. Wird ausgelöst, wenn Vertragsunterschiede Verbraucher unterbrechen können (z. B. Entfernung erforderlicher Felder, semantische Änderung oder Kanalwechsel), wenn eine Abkündigungs-/Ablaufplanung erforderlich ist oder wenn mehrere Verbraucherkohorten parallel unterstützt werden müssen. Nicht für den First-Pass-Endpunkt-/Schemaentwurf verwenden.
