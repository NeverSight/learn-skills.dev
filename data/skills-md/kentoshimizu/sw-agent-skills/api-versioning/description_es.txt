Gobernanza del ciclo de vida de la versión API para clasificación de cambios importantes, ventanas de obsolescencia, planificación de migración y gestión de matrices de soporte entre consumidores internos y externos. Se activa cuando las diferencias de contrato pueden afectar a los consumidores (por ejemplo, eliminación de campos obligatorios, cambio semántico o cambio de canal), cuando se requiere planificación de desaprobación o extinción, o cuando se deben admitir múltiples cohortes de consumidores en paralelo. No lo utilice para diseño de esquema/punto final de primer paso.
