Governance del ciclo di vita della versione API per la classificazione delle modifiche fondamentali, le finestre di deprecazione, la pianificazione della migrazione e la gestione della matrice di supporto tra consumatori interni ed esterni. Si attiva quando le differenze contrattuali possono interrompere i consumatori (ad esempio rimozione di campi obbligatori, modifica semantica o cambio di canale), quando è richiesta la deprecazione/pianificazione della scadenza o quando è necessario supportare più coorti di consumatori in parallelo. Non utilizzare per la progettazione di endpoint/schemi di primo passaggio.
