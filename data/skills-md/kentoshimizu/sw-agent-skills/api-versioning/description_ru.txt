Управление жизненным циклом версий API для классификации критических изменений, окон устаревания, планирования миграции и управления матрицами поддержки для внутренних и внешних потребителей. Срабатывает, когда различия в контрактах могут нарушить работу потребителей (например, удаление обязательных полей, семантическое изменение или смена канала), когда требуется планирование прекращения поддержки или прекращения поддержки или когда необходимо параллельно поддерживать несколько когорт потребителей. Не используйте для первого прохода при проектировании конечной точки/схемы.
