معايير التصميم والمراجعة وإعادة البناء للبرمجة النصية POSIX `sh`. يتم تشغيله عند إنشاء ملفات `.sh` أو ملفات تحتوي على `#!/usr/bin/env sh` أو `#!/bin/sh` أو كتل سير عمل CI مع `shell: sh` أو تعديلها أو مراجعتها ويجب فرض عناصر التحكم في قابلية النقل/السلامة لـ POSIX Shell. لا تستخدمه مع قواعد النمط الخاصة بـ Bash، أو الخاصة بـ Zsh، أو الخاصة بـ PowerShell. في طلبات السحب متعددة اللغات، يمكنك تشغيلها مع مهارات `*-style-guide` الأخرى القابلة للتطبيق.
