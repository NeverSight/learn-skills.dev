Standard di stile, revisione e refactoring per lo scripting POSIX `sh`. Si attiva quando i file `.sh`, i file con `#!/usr/bin/env sh` o `#!/bin/sh` o i blocchi del flusso di lavoro CI con `shell: sh` vengono creati, modificati o revisionati e i controlli di portabilità/sicurezza per la shell POSIX devono essere applicati. Non utilizzare per regole di stile specifiche di Bash, Zsh o PowerShell. Nelle richieste pull multilingue, eseguile insieme ad altre competenze `*-style-guide` applicabili.
