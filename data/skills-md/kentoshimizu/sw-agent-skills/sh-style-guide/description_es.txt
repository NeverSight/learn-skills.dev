Estándares de estilo, revisión y refactorización para secuencias de comandos POSIX `sh`. Se activa cuando se crean, modifican o revisan archivos `.sh`, archivos con `#!/usr/bin/env sh` o `#!/bin/sh`, o bloques de flujo de trabajo de CI con `shell: sh` y se deben aplicar controles de portabilidad/seguridad para el shell POSIX. No lo utilice para reglas de estilo específicas de Bash, Zsh o PowerShell. En solicitudes de extracción en varios idiomas, ejecútelas junto con otras habilidades de `*-style-guide` aplicables.
