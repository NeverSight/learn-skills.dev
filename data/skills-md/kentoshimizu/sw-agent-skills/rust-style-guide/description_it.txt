Standard di stile, revisione e refactoring per le basi di codice Rust. Si attiva quando i file `.rs`, `Cargo.toml` o `Cargo.lock` vengono creati, modificati o revisionati e le regole di qualità specifiche di Rust (chiarezza di proprietà/prestito, gestione degli errori, progettazione di tratti/API, confini dei crate) devono essere applicate. Non utilizzare per convenzioni della toolchain C/C++, modifiche solo della shell o problemi di stile di runtime non Rust a meno che non vengano modificati anche gli artefatti Rust. Nelle richieste pull multilingue, eseguile insieme ad altre competenze `*-style-guide` applicabili.
