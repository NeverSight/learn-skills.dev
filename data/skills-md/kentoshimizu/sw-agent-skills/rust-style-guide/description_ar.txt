معايير التصميم والمراجعة وإعادة البناء لقواعد بيانات Rust. يتم التشغيل عند إنشاء ملفات `.rs` أو `Cargo.toml` أو `Cargo.lock` أو تغييرها أو مراجعتها ويجب تطبيق قواعد الجودة الخاصة بـ Rust (وضوح الملكية/الاقتراض، ومعالجة الأخطاء، وتصميم السمة/واجهة برمجة التطبيقات (API)، وحدود الصناديق). لا تستخدمه مع اصطلاحات سلسلة أدوات C/C++، أو تغييرات الصدفة فقط، أو مخاوف نمط وقت التشغيل غير الصدأ ما لم يتم تغيير عناصر الصدأ أيضًا. في طلبات السحب متعددة اللغات، يمكنك تشغيلها مع مهارات `*-style-guide` الأخرى القابلة للتطبيق.
