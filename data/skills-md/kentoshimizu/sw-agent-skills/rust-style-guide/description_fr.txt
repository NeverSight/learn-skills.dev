Normes de style, de révision et de refactorisation pour les bases de code Rust. Déclenché lorsque les fichiers `.rs`, `Cargo.toml` ou `Cargo.lock` sont créés, modifiés ou révisés et que les règles de qualité spécifiques à Rust (clarté de propriété/emprunt, gestion des erreurs, conception de trait/API, limites de caisse) doivent être appliquées. Ne pas utiliser pour les conventions de la chaîne d'outils C/C++, les modifications du shell uniquement ou les problèmes de style d'exécution non Rust, à moins que les artefacts Rust ne soient également modifiés. Dans les demandes d'extraction multilingues, exécutez avec d'autres compétences applicables du « *-style-guide ».
