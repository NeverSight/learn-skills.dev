Rust コードベースのスタイル、レビュー、およびリファクタリングの標準。 `.rs`、`Cargo.toml`、または `Cargo.lock` ファイルが作成、変更、またはレビューされ、Rust 固有の品質ルール (所有権/借用の明確性、エラー処理、特性/API 設計、クレート境界) が適用される必要があるときにトリガーされます。 Rust アーティファクトも変更されない限り、C/C++ ツールチェーン規約、シェルのみの変更、または Rust 以外のランタイム スタイルの問題には使用しないでください。多言語プル リクエストでは、他の該当する `*-style-guide` スキルと一緒に実行します。
