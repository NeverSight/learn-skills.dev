Normes de style, de révision et de refactorisation pour les bases de code TypeScript. Déclenché lorsque des fichiers `.ts`, `.tsx`, `.d.ts` ou `tsconfig*.json` sont créés, modifiés ou révisés et que les règles de qualité spécifiques à TS (sécurité du type, rétrécissement, gestion asynchrone/des erreurs, limites des modules) doivent être appliquées. Ne pas utiliser pour les modules JavaScript uniquement qui ne relèvent pas de la portée de la vérification de type TypeScript. Dans les référentiels mixtes JS/TS, cette compétence possède des modules de portée « tsconfig ». Pour les fichiers de configuration JS/TS partagés (par exemple « package.json », configuration lint/test partagée), exécutez avec « javascript-style-guide » lorsque les artefacts JS et TS changent dans la même demande d'extraction.
