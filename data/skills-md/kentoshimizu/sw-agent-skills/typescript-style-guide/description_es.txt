Estándares de estilo, revisión y refactorización para bases de código TypeScript. Se activa cuando se crean, cambian o revisan archivos `.ts`, `.tsx`, `.d.ts` o `tsconfig*.json` y se deben aplicar reglas de calidad específicas de TS (seguridad de tipos, limitación, manejo de errores/asincronismo, límites de módulos). No lo use para módulos solo de JavaScript que estén fuera del alcance de verificación de tipo de TypeScript. En repositorios mixtos JS/TS, esta habilidad posee módulos con alcance `tsconfig`. Para archivos de configuración JS/TS compartidos (por ejemplo, `package.json`, configuración compartida de pelusa/prueba), ejecútelo junto con `javascript-style-guide` cuando los artefactos JS y TS cambien en la misma solicitud de extracción.
