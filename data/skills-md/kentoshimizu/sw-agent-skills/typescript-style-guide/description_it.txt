Standard di stile, revisione e refactoring per le basi di codice TypeScript. Si attiva quando i file `.ts`, `.tsx`, `.d.ts` o `tsconfig*.json` vengono creati, modificati o revisionati e devono essere applicate regole di qualità specifiche di TS (sicurezza del tipo, restringimento, gestione asincrona/errori, limiti del modulo). Non utilizzare per moduli solo JavaScript che non rientrano nell'ambito del controllo del tipo TypeScript. Nei repository misti JS/TS, questa competenza possiede moduli con ambito "tsconfig". Per i file di configurazione JS/TS condivisi (ad esempio `package.json`, shared lint/test config), eseguili insieme a `javascript-style-guide` quando sia gli artefatti JS che quelli TS cambiano nella stessa richiesta pull.
