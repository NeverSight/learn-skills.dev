Diseño de contrato de falla de API para mapeo de estado, códigos de error estables, semántica de reintento y cargas útiles de error rastreables en transportes sincronizados y asíncronos. Se activa cuando el comportamiento de error, la semántica de reintento o los campos de depuración cambian en las especificaciones o la fuente, o cuando esas reglas permanecen implícitas. No lo utilice para el modelado completo de recursos/esquemas ni para la definición de políticas de canales de versiones.
