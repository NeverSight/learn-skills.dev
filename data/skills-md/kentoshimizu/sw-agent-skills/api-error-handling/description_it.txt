Progettazione di contratti di errore API per mappatura dello stato, codici di errore stabili, semantica di riprovabilità e payload di errori tracciabili nei trasporti sincronizzati e asincroni. Si attiva quando il comportamento dell'errore, la semantica dei nuovi tentativi o i campi di debugging cambiano nelle specifiche o nell'origine o quando tali regole rimangono implicite. Non utilizzare per la modellazione completa di risorse/schemi o per la definizione di policy del canale di versione.
