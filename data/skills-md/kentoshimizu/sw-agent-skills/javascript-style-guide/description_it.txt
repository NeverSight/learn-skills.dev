Standard di stile, revisione e refactoring per basi di codice JavaScript con contratti di runtime espliciti, comportamento asincrono sicuro e limiti dei moduli gestibili. Da utilizzare quando gli artefatti `.js`, `.jsx`, `.mjs` o `.cjs` vengono creati, modificati o revisionati e devono essere applicate regole di qualità specifiche di JS. Non utilizzare per il codice di proprietà di `.ts`, `.tsx`, `.d.ts` o `tsconfig*.json`. Per i file di configurazione JS/TS condivisi, eseguili insieme a `typescript-style-guide` quando entrambi gli artefatti JS e TS cambiano nella stessa richiesta pull.
