Stil-, Überprüfungs- und Refactoring-Standards für JavaScript-Codebasen mit expliziten Laufzeitverträgen, sicherem asynchronem Verhalten und wartbaren Modulgrenzen. Wird verwendet, wenn „.js“, „.jsx“, „.mjs“ oder „.cjs“-Artefakte erstellt, geändert oder überprüft werden und JS-spezifische Qualitätsregeln durchgesetzt werden müssen. Nicht für „.ts“, „.tsx“, „.d.ts“ oder „tsconfig*.json“-eigenen Code verwenden. Führen Sie für gemeinsam genutzte JS/TS-Konfigurationsdateien zusammen mit „typescript-style-guide“ aus, wenn sich sowohl JS- als auch TS-Artefakte im selben Pull-Request ändern.
