Normes de style, de révision et de refactorisation pour les bases de code JavaScript avec des contrats d'exécution explicites, un comportement asynchrone sécurisé et des limites de module maintenables. À utiliser lorsque des artefacts `.js`, `.jsx`, `.mjs` ou `.cjs` sont créés, modifiés ou révisés et que des règles de qualité spécifiques à JS doivent être appliquées. Ne pas utiliser pour le code appartenant à `.ts`, `.tsx`, `.d.ts` ou `tsconfig*.json`. Pour les fichiers de configuration JS/TS partagés, exécutez avec « typescript-style-guide » lorsque les artefacts JS et TS changent dans la même demande d'extraction.
