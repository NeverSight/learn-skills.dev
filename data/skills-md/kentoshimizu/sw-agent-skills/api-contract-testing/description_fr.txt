Tests de contrat consommateur-fournisseur et conception de portes de publication pour les matrices de compatibilité, les assertions de contrat et les règles de blocage de CI entre les versions et les transports. Se déclenche lorsque des différences de contrat d'API sont détectées mais que les preuves de compatibilité sont manquantes ou obsolètes (réussite du test de contrat, cartographie de l'impact sur le consommateur ou mise à jour de la matrice de compatibilité), ou lorsque plusieurs consommateurs, versions ou transports doivent être protégés par des portes automatisées. Ne pas utiliser pour la conception d’interface API de premier passage.
