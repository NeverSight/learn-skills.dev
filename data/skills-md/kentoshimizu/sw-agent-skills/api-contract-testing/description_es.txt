Pruebas de contratos entre consumidores y proveedores y diseño de puertas de lanzamiento para matrices de compatibilidad, afirmaciones de contratos y reglas de bloqueo de CI entre versiones y transportes. Se activa cuando se detectan diferencias en los contratos de API pero falta evidencia de compatibilidad o está obsoleta (pase de prueba de contrato, mapeo de impacto en el consumidor o actualización de la matriz de compatibilidad), o cuando múltiples consumidores, versiones o transportes deben estar protegidos por puertas automatizadas. No lo utilice para el diseño de interfaz API de primer paso.
