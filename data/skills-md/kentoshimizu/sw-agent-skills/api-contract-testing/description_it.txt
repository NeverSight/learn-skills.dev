Test del contratto consumatore-fornitore e progettazione del gate di rilascio per matrici di compatibilità, asserzioni contrattuali e regole di blocco CI tra versioni e trasporti. Si attiva quando vengono rilevate differenze nei contratti API ma le prove di compatibilità sono mancanti o obsolete (test del contratto superato, mappatura dell'impatto del consumatore o aggiornamento della matrice di compatibilità) o quando più consumatori, versioni o trasporti devono essere protetti da cancelli automatizzati. Non utilizzare per la progettazione dell'interfaccia API di primo passaggio.
