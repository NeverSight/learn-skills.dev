Создавайте и поддерживайте рабочие процессы GitHub Actions с явной областью триггеров, границами безопасности и надежной оркестровкой заданий. Используйте, когда необходимо создать или изменить автоматизацию GitHub (задания сборки/тестирования/выпуска, триггеры, разрешения, кэширование, параллелизм, среды); не используйте для архитектуры среды выполнения, отличной от GitHub, или проектирования уровня данных.
