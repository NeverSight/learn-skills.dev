معايير التصميم والمراجعة وإعادة البناء للبرمجة النصية لـ Bash Shell. يتم تشغيله عند إنشاء ملفات `.sh` أو ملفات تحتوي على `#!/usr/bin/env bash` أو `#!/bin/bash` أو كتل سير عمل CI مع `shell: bash` أو تعديلها أو مراجعتها ويجب فرض ضوابط الجودة الخاصة بـ Bash (نقلاً عن الأمان ومعالجة الأخطاء وقابلية النقل وقابلية القراءة). لا تستخدمه مع POSIX `sh` أو PowerShell أو قواعد نمط التطبيق العامة. في طلبات السحب متعددة اللغات، يمكنك تشغيلها مع مهارات `*-style-guide` الأخرى القابلة للتطبيق.
