Flujo de trabajo de almacenamiento en caché de Redis para mejorar la latencia con estrategia de clave explícita, política de invalidación/TTL y límites de corrección. Úselo cuando se requieran decisiones de almacenamiento en caché respaldadas por Redis para el rendimiento de la aplicación; no lo utilice para la gobernanza de la arquitectura de todo el repositorio ni para la política de gestión de versiones.
