Рабочий процесс кэширования Redis для уменьшения задержки с помощью явной стратегии ключей, политики TTL/аннулирования и границ корректности. Используйте, когда для производительности приложения необходимы решения по кэшированию на основе Redis; не используйте для управления архитектурой всего репозитория или политики управления выпусками.
