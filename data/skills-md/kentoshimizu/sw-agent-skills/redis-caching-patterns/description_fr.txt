Workflow de mise en cache Redis pour améliorer la latence avec une stratégie de clé explicite, une politique de durée de vie/invalidation et des limites d'exactitude. À utiliser lorsque des décisions de mise en cache basées sur Redis sont nécessaires pour les performances des applications ; ne pas utiliser pour la gouvernance de l'architecture à l'échelle du référentiel ou pour la politique de gestion des versions.
