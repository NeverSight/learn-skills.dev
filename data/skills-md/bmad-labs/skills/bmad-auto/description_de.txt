Orchestriert BMAD-Implementierungsworkflows automatisch – sowohl die gesamte Epic-/Story-Pipeline der Phase 4 als auch den Quick Flow für kleine, gut verständliche Änderungen. Verwenden Sie diese Fertigkeit immer dann, wenn der Benutzer: (1) die BMAD-Phase-4-Implementierung automatisieren möchte („automatisch implementieren“, „Implementierung starten“, „Phase 4 beginnen“, „automatisches Arbeiten an Phase 4“, „alle Storys implementieren“, „die Epen verarbeiten“), (2) den Implementierungsfortschritt oder -status überprüfen möchte („Wie ist der Status?“, „Wie viele Storys sind fertig?“), (3) eine zuvor unterbrochene Sitzung fortsetzen möchte („da fortfahren, wo wir aufgehört haben“, „fortsetzen“), (4) von a implementieren möchte Tech-Spec oder Quick-Spec („Hier ist meine Tech-Spezifikation“, „Implementieren Sie diese Spezifikation“, „Quick Flow“, „Quick Dev“, „Ich habe eine Tech-Spezifikation“, „Implementieren Sie diese Änderung“), (5) Erstellen Sie eine Tech-Spezifikation für eine kleine Änderung („Quick Spec“, „Erstellen Sie eine Tech-Spezifikation“, „Spezifizieren Sie diese Änderung“, „Definieren Sie diesen Fix“). Wenn der Benutzer eine technische Spezifikationsdatei bereitstellt, auf eine technische Spezifikation verweist oder eine kleine/gut verstandene Änderung beschreibt (Bugfix, Refactoring, kleines Feature, Patch), leiten Sie ihn zum Quick Flow weiter – es sind keine vollständigen Phase-4-Artefakte erforderlich. Wenn Sie sich nicht sicher sind, ob Sie diese Fähigkeit verwenden sollen, nutzen Sie sie – sie erkennt automatisch, welcher Ablauf angemessen ist.
