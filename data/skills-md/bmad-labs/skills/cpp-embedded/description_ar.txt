إرشادات الخبراء لكتابة كود C (C99/C11) وC++ (C++17) للأنظمة المدمجة ووحدات التحكم الدقيقة. استخدم هذه المهارة عندما يعمل المستخدم مع: STM32، ESP32، Arduino، PIC، AVR، nRF52، أو أي MCU أخرى؛ FreeRTOS أو Zephyr أو ThreadX أو أي نظام RTOS؛ البرامج الثابتة المعدنية العارية؛ سجلات الأجهزة، أو DMA، أو المقاطعات، أو الإدخال/الإخراج المعين للذاكرة؛ تجمعات الذاكرة أو المخصصات أو المخازن المؤقتة ذات الحجم الثابت؛ امتثال MISRA C أو MISRA C++؛ المؤشرات الذكية أو RAII في السياقات المضمنة؛ قرارات المكدس مقابل الكومة؛ التنسيب جديد؛ صحة متقلبة. المحاذاة والتعبئة الهيكلية ؛ أنماط C99/C11؛ إمكانية التشغيل البيني C وC++؛ تصحيح أخطاء البرامج الثابتة، أو الأخطاء الصعبة، أو تجاوزات المكدس، أو تلف الكومة؛ قرارات هندسة البرامج الثابتة (superloop مقابل RTOS مقابل الأحداث)؛ أوضاع الطاقة المنخفضة (WFI/WFE/السكون)؛ إعداد مشروع CubeMX؛ اختيار برنامج التشغيل HAL مقابل LL؛ CI/CD للبرامج الثابتة؛ مراجعة التعليمات البرمجية المضمنة؛ تكوين MPU؛ استراتيجيات المراقبة؛ تصميم السلامة الحرجة (IEC 61508، SIL)؛ اختيار البروتوكول الطرفي (UART/I2C/SPI/CAN)؛ وضع ذاكرة البرنامج النصي للرابط؛ أو أنماط رد الاتصال C/C++. قم أيضًا بتشغيل الإشارات الضمنية مثل "استمرار MCU الخاص بي في التعطل"، "كتابة البرامج الثابتة"، "آمن ISR"، "المخصص المضمن"، "لا توجد ذاكرة ديناميكية"، "استهلاك الطاقة"، "أعاد CubeMX إنشاء الكود الخاص بي"، "أي نمط RTOS يجب أن أستخدمه"، "خطأ MPU"، "المراقبة تستمر في إعادة التعيين"، "أي بروتوكول يجب أن أستخدمه لجهاز الاستشعار الخاص بي"، "ESP32 نوم عميق"، "PSRAM مقابل DRAM"، "كومة ESP32 تستمر في الانكماش"، "ESP.getFreeHeap()" أو "تجاوز سعة مكدس المهام على ESP32" أو "إعادة الاتصال بشبكة WiFi بعد أن يكون النوم العميق بطيئًا".
