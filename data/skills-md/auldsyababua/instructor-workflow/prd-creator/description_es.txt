Esta habilidad debe utilizarse al crear, validar o convertir documentos de requisitos del producto (PRD) al formato nativo de LLM. Utilice esta habilidad cuando el usuario solicite escribir un PRD, revisar un PRD para verificar su cumplimiento, convertir un PRD existente a un formato legible por máquina o garantizar la calidad del PRD para el consumo de agentes de IA. Esta habilidad está optimizada para usuarios que trabajan con marcos de codificación agentes que necesitan especificaciones de requisitos estrictas para evitar el envenenamiento del contexto, las alucinaciones y la optimización de mesa.
