Flujo de trabajo de desarrollo ad hoc para implementar funciones, corregir errores y realizar cambios en el código fuera de un plan formal. Úselo cuando se le solicite implementar, agregar, arreglar, compilar o crear algo en el código base. Rutas a habilidades de dominio (postgres-expert, server-action-builder, reaccionar-form-builder, service-builder, playwright-e2e), impone el seguimiento de tareas para una recuperación compacta del contexto y sigue un bucle de compilación, prueba y verificación basado en referencias. NO lo use para funciones grandes de múltiples fases; use /create-plan + /implement en su lugar.
