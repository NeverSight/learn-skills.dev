Concevez et implémentez des points de terminaison d'API RESTful en suivant les principes REST, des conventions de dénomination cohérentes et des méthodes HTTP appropriées. Utilisez cette compétence lors de la création ou de la modification de routes, de contrôleurs ou de gestionnaires d'API dans le code backend. Lors de la mise en œuvre d'une logique de point de terminaison qui gère les requêtes HTTP (GET, POST, PUT, PATCH, DELETE). Lorsque vous travaillez avec la gestion des versions d'API, les paramètres de requête ou le formatage des réponses. Lors de la définition des structures d'URL pour les points de terminaison basés sur les ressources. Lors de la configuration de la limitation de débit ou de la gestion du code d'état HTTP. Lorsque vous travaillez avec des fichiers tels que des définitions de routes, des fichiers de contrôleur, des modules de gestionnaire d'API ou des fichiers de configuration de framework Web.
