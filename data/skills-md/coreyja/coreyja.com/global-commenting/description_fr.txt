Écrivez du code auto-documenté avec un nom et une structure clairs, en ajoutant un minimum de commentaires persistants uniquement lorsque cela est nécessaire pour expliquer une logique complexe. Utilisez cette compétence lors de l'écriture ou de la révision de tout code pouvant nécessiter des commentaires. Lors de la refactorisation du code pour le rendre plus explicite grâce à une meilleure dénomination et une meilleure structure. Lorsque vous décidez d'ajouter ou de supprimer des commentaires du code existant. Lorsque vous assurez les commentaires, expliquez pourquoi quelque chose est fait plutôt que ce qui est fait. Lors de la révision du code pour supprimer les commentaires obsolètes ou temporaires liés aux modifications. Cette compétence s'applique à tous les langages de programmation et à tous les types de fichiers de code.
