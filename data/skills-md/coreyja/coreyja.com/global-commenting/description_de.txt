Schreiben Sie selbstdokumentierenden Code mit klarer Benennung und Struktur und fügen Sie nur dann minimale, immergrüne Kommentare hinzu, wenn dies zur Erläuterung komplexer Logik erforderlich ist. Nutzen Sie diese Fähigkeit, wenn Sie Code schreiben oder überprüfen, der möglicherweise Kommentare benötigt. Beim Refactoring von Code, um ihn durch bessere Benennung und Struktur selbsterklärender zu machen. Bei der Entscheidung, ob Kommentare zum vorhandenen Code hinzugefügt oder daraus entfernt werden sollen. Wenn Sie Kommentare abgeben, erklären Sie, warum etwas getan wird und nicht, was getan wird. Beim Überprüfen von Code, um veraltete oder vorübergehende änderungsbezogene Kommentare zu entfernen. Diese Fähigkeit gilt für alle Programmiersprachen und alle Arten von Codedateien.
