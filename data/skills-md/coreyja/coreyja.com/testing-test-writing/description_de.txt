Schreiben Sie gezielte Tests für zentrale Benutzerabläufe und kritische Geschäftslogik, wobei Sie das Verhalten statt die Implementierung testen, mit klaren Namen und schneller Ausführung. Nutzen Sie diese Fähigkeit beim Schreiben von Unit-Tests, Integrationstests oder End-to-End-Tests. Beim Testen kritischer Benutzerworkflows und primärer Anwendungsfunktionen. Beim Erstellen von Tests, die sich darauf konzentrieren, was der Code tut, und nicht darauf, wie er es tut. Beim Verspotten externer Abhängigkeiten wie Datenbanken, APIs oder Dateisystemen. Bei der Benennung von Tests müssen das Szenario und das erwartete Ergebnis klar beschrieben werden. Wenn Sie Tests schnell und unabhängig durchführen möchten. Beim Abschluss der Funktionsimplementierung und beim Hinzufügen strategischer Tests an logischen Abschlusspunkten. Bei der Arbeit mit Testdateien oder Testframeworks.
