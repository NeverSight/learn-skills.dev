ユーザーフレンドリーなメッセージ、特定の例外タイプ、一元化されたエラー境界、適切な機能低下戦略を使用して、堅牢なエラー処理を実装します。このスキルは、アプリケーションの任意の部分にエラー処理ロジックを実装するときに使用します。 try-catch ブロック、エラー境界、または例外ハンドラーを作成するとき。入力を検証し、明確なエラー メッセージが表示されてすぐに失敗する場合。 API エラー、ネットワーク障害、または外部サービスのタイムアウトを処理するとき。指数バックオフを使用した再試行戦略を実装する場合。リソースが最終ブロック内で適切にクリーンアップされていることを確認する場合。技術的な詳細を公開せずにユーザーにエラー メッセージを表示する場合。このスキルは、フロントエンド コードとバックエンド コードにわたるすべてのエラー処理に適用されます。
