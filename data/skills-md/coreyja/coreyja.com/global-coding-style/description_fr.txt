Maintenez un formatage de code, des conventions de dénomination et une structure cohérents dans l'ensemble de la base de code grâce à des outils automatisés et des normes claires. Utilisez cette compétence lors de l’écriture de n’importe quel code dans n’importe quel langage ou framework. Lorsque vous nommez des variables, des fonctions, des classes ou des fichiers. Lors du formatage du code avec une indentation et des sauts de ligne appropriés. Lors de la refactorisation du code pour supprimer la duplication ou extraire des fonctions réutilisables. Lors de l'exécution de linters ou de formateurs pour garantir la cohérence. Lors de la suppression de code mort, d'importations inutilisées ou de blocs commentés. Lors de l’application des principes DRY pour éviter la duplication de code. Cette compétence s'applique universellement à toutes les tâches de programmation.
