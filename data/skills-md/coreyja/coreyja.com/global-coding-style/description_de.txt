Behalten Sie mit automatisierten Tools und klaren Standards konsistente Codeformatierung, Namenskonventionen und Struktur in der gesamten Codebasis bei. Verwenden Sie diese Fähigkeit, wenn Sie Code in einer beliebigen Sprache oder einem beliebigen Framework schreiben. Beim Benennen von Variablen, Funktionen, Klassen oder Dateien. Beim Formatieren von Code mit korrekter Einrückung und Zeilenumbrüchen. Beim Refactoring von Code, um Duplikate zu entfernen oder wiederverwendbare Funktionen zu extrahieren. Beim Ausführen von Linters oder Formattern, um die Konsistenz sicherzustellen. Beim Entfernen von totem Code, nicht verwendeten Importen oder auskommentierten Blöcken. Bei der Anwendung von DRY-Prinzipien zur Vermeidung von Codeduplizierung. Diese Fähigkeit gilt universell für alle Programmieraufgaben.
