Verhaltensspezifikations-Engine im BDD-Stil für das Skill-System. Wird verwendet, wenn: (1) das Verhalten eines neuen Skills vor der Implementierung definiert wird, (2) eine Spezifikation anhand des Schemas validiert wird, (3) ein Verhaltensvertrag (Mermaid DAG) aus einer Spezifikation generiert wird, (4) strukturelle Akzeptanztests für einen erstellten Skill ausgeführt werden. Arbeitsablauf: Spezifikation → Test → Entwickeln → Vertrag.
