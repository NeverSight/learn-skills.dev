安全な読み取り前書き込みプロセスを使用して、Teable ベース、テーブル、フィールド、ビュー、レコード、SQL 読み取りクエリ、および関連するアプリ/自動化ワークフローを操作します。ユーザー入力で Cuppy、Teable、teable CLI に言及する場合、または Teable スタイルの ID (bseXXX、tblXXX、fldXXX、recXXX、viw/viwXXX) を参照する場合は常に使用します。また、ユーザーがレコードのクエリ、テーブルの作成/更新、フィールドの管理、ダッシュボードまたは Web アプリの構築、オートメーションの作成、データのインポート/エクスポート、グラフの生成、サンドボックスでのスクリプトの実行、Teable API の検索/呼び出し、AI フィルのトリガー、または Teable 上でデータベース操作を実行したい場合にもトリガーします。これは、明示的に「Teable」とは言っていないものの、明らかに Teable ベースまたは Teable CLI を使用している場合でも同様です。
