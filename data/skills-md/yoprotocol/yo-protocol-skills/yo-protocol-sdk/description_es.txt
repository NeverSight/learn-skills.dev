Utilice SIEMPRE esta habilidad cuando el usuario escriba código TypeScript o JavaScript que importe desde `@yo-protocol/core`, utilice `createYoClient` o interactúe mediante programación con las bóvedas del Protocolo Yo (yoETH, yoUSD, yoBTC, yoEUR, yoGOLD, yoUSDT) a través del SDK. Esta habilidad cubre el SDK `@yo-protocol/core` completo: inicialización del cliente (createYoClient, publicClients multicadena, partnerId), lecturas de bóveda (getVaultState,previewDeposit,previewRedeem,isPaused), lecturas de usuario (getUserPosition, getShareBalance, getTokenBalance, getAllowance), transacciones preparadas (prepareDepositWithApproval, prepareRedeemWithApproval, prepareApprove), consultas de API REST (getVaultSnapshot, getVaultYieldHistory, getVaultTvlHistory, getUserHistory, getUserPerformance), reclamaciones de recompensas de Merkl (getClaimableRewards, prepareClaimMerklRewards) y utilidades (parseTokenAmount, formatTokenAmount, registro VAULTS, dirección de puerta de enlace). Activador para scripts de Node.js, servicios backend, bots, creadores de transacciones por lotes seguras/AA o cualquier código TypeScript que no sea React que interactúe con el protocolo Yo. También se activa cuando el usuario menciona la puerta de enlace de Yo, PreparedTransaction, las interacciones de la bóveda ERC-4626 específicas de Yo o pregunta sobre las direcciones de la bóveda y las direcciones de los tokens por cadena. NO lo use para enlaces de React (use yo-protocol-react) o comandos de shell CLI (use yo-protocol-cli).
