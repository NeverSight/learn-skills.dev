ВСЕГДА используйте этот навык, когда пользователь пишет код TypeScript или JavaScript, который импортирует из @yo-protocol/core, использует createYoClient или программно взаимодействует с хранилищами протокола Yo (yoETH, yoUSD, yoBTC, yoEUR, yoGOLD, yoUSDT) через SDK. Этот навык охватывает полный пакет `@yo-protocol/core` SDK: инициализацию клиента (createYoClient, multi-chain publicClients, PartnerId), чтение хранилища (getVaultState, PreviewDeposit, PreviewRedeem, isPaused), пользовательское чтение (getUserPosition, getShareBalance, getTokenBalance, getAllowance), подготовленные транзакции (prepareDepositWithApproval, подготовитьRedeemWithApproval, подготовитьApprove), запросы REST API (getVaultSnapshot, getVaultYieldHistory, getVaultTvlHistory, getUserHistory, getUserPerformance), заявки на вознаграждение Merkl (getClaimableRewards, подготовитьClaimMerklRewards) и утилиты (parseTokenAmount, formatTokenAmount, реестр VAULTS, адрес шлюза). Триггер для сценариев Node.js, серверных служб, ботов, построителей пакетных транзакций Safe/AA или любого кода TypeScript, отличного от React, который взаимодействует с протоколом Yo. Также срабатывает, когда пользователь упоминает шлюз Yo, ReadedTransaction, взаимодействие с хранилищем ERC-4626, специфичное для Yo, или спрашивает об адресах хранилища и адресах токенов для каждой цепочки. НЕ используйте для React перехватчики (используйте yo-protocol-react) или команды оболочки CLI (используйте yo-protocol-cli).
