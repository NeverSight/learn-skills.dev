Utilisez TOUJOURS cette compétence lorsque l'utilisateur écrit du code TypeScript ou JavaScript importé depuis `@yo-protocol/core`, utilise `createYoClient` ou interagit par programme avec les coffres-forts du protocole Yo (yoETH, yoUSD, yoBTC, yoEUR, yoGOLD, yoUSDT) via le SDK. Cette compétence couvre le SDK `@yo-protocol/core` complet : initialisation du client (createYoClient, multi-chain publicClients, PartnerId), lectures du coffre-fort (getVaultState, previewDeposit, previewRedeem, isPaused), lectures utilisateur (getUserPosition, getShareBalance, getTokenBalance, getAllowance), transactions préparées (prepareDepositWithApproval, prepareRedeemWithApproval, prepareApprove), requêtes API REST (getVaultSnapshot, getVaultYieldHistory, getVaultTvlHistory, getUserHistory, getUserPerformance), réclamations de récompense Merkl (getClaimableRewards, prepareClaimMerklRewards) et utilitaires (parseTokenAmount, formatTokenAmount, registre VAULTS, adresse de passerelle). Déclencheur pour les scripts Node.js, les services backend, les robots, les générateurs de transactions par lots Safe/AA ou tout code non-React TypeScript qui interagit avec le protocole Yo. Déclenchez également lorsque l'utilisateur mentionne la passerelle Yo, PreparedTransaction, les interactions de coffre-fort ERC-4626 spécifiques à Yo, ou pose des questions sur les adresses de coffre-fort et les adresses de jetons par chaîne. Ne pas utiliser pour les hooks React (utilisez yo-protocol-react) ou les commandes shell CLI (utilisez yo-protocol-cli).
