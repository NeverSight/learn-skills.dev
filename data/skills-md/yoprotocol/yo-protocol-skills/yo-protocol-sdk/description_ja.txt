ユーザーが「@yo-protocol/core」からインポートする TypeScript または JavaScript コードを作成する場合、「createYoClient」を使用する場合、または SDK 経由で Yo プロトコル ボールト (yoETH、yoUSD、yoBTC、yoEUR、yoGOLD、yoUSDT) とプログラム的に対話する場合は、常にこのスキルを使用してください。このスキルは、完全な `@yo-protocol/core` SDK をカバーします: クライアントの初期化 (createYoClient、マルチチェーン publicClients、partnerId)、ボールトの読み取り (getVaultState、previewDeposit、previewRedeem、isPaused)、ユーザーの読み取り (getUserPosition、getShareBalance、getTokenBalance、getAllowance)、準備されたトランザクション (prepareDepositWithApproval、 prepareRedeemWithApproval、prepareApprove)、REST API クエリ (getVaultSnapshot、getVaultYieldHistory、getVaultTvlHistory、getUserHistory、getUserPerformance)、Merkl 報酬請求 (getClaimableRewards、prepareClaimMerklRewards)、およびユーティリティ (parseTokenAmount、formatTokenAmount、VAULTS レジストリ、ゲートウェイ)住所）。 Node.js スクリプト、バックエンド サービス、ボット、Safe/AA バッチ トランザクション ビルダー、または Yo プロトコルと対話する非 React TypeScript コードのトリガー。また、ユーザーが Yo ゲートウェイ、PreparedTransaction、Yo に固有の ERC-4626 ボールト インタラクションに言及したとき、またはチェーンごとのボールト アドレスとトークン アドレスについて質問したときにもトリガーされます。 React フック (yo-protocol-react を使用) または CLI シェル コマンド (yo-protocol-cli を使用) には使用しないでください。
