当用户编写从“@yo-protocol/core”导入的 TypeScript 或 JavaScript 代码、使用“createYoClient”或通过 SDK 以编程方式与 Yo Protocol 金库（yoETH、yoUSD、yoBTC、yoEUR、yoGOLD、yoUSDT）交互时，请始终使用此技能。该技能涵盖了完整的“@yo-protocol/core” SDK：客户端初始化（createYoClient、多链 publicClients、partnerId）、金库读取（getVaultState、previewDeposit、previewRedeem、isPaused）、用户读取（getUserPosition、getShareBalance、getTokenBalance、getAllowance）、准备好的交易（prepareDepositWithApproval、prepareRedeemWithApproval、prepareApprove）、REST API 查询（getVaultSnapshot、getVaultYieldHistory、getVaultTvlHistory、getUserHistory、getUserPerformance）、Merkl 奖励声明（getClaimableRewards、prepareClaimMerklRewards）和实用程序（parseTokenAmount、formatTokenAmount、VAULTS 注册表、网关地址）。触发 Node.js 脚本、后端服务、机器人、Safe/AA 批量交易构建器或与 Yo Protocol 交互的任何非 React TypeScript 代码。当用户提到 Yo 网关、PreparedTransaction、特定于 Yo 的 ERC-4626 金库交互，或询问每个链的金库地址和代币地址时也会触发。请勿用于 React hooks（使用 yo-protocol-react）或 CLI shell 命令（使用 yo-protocol-cli）。
