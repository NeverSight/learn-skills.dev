Utilizza SEMPRE questa competenza quando l'utente scrive codice TypeScript o JavaScript che importa da `@yo-protocol/core`, utilizza `createYoClient` o interagisce a livello di codice con i vault del protocollo Yo (yoETH, yoUSD, yoBTC, yoEUR, yoGOLD, yoUSDT) tramite l'SDK. Questa competenza copre l'SDK `@yo-protocol/core` completo: inizializzazione del client (createYoClient, multi-chain publicClients, partnerId), letture del vault (getVaultState, PreviewDeposit, PreviewRedeem, isPaused), letture dell'utente (getUserPosition, getShareBalance, getTokenBalance, getAllowance), transazioni preparate (prepareDepositWithApproval, prepareRedeemWithApproval, prepareApprove), query API REST (getVaultSnapshot, getVaultYieldHistory, getVaultTvlHistory, getUserHistory, getUserPerformance), richieste di ricompensa Merkl (getClaimableRewards, prepareClaimMerklRewards) e utilità (parseTokenAmount, formatTokenAmount, registro VAULTS, indirizzo gateway). Trigger per script Node.js, servizi backend, bot, builder di transazioni batch Safe/AA o qualsiasi codice TypeScript non React che interagisce con Yo Protocol. Si attiva anche quando l'utente menziona il gateway Yo, PreparedTransaction, le interazioni del vault ERC-4626 specifiche per Yo o chiede informazioni sugli indirizzi del vault e sugli indirizzi dei token per catena. NON utilizzare per gli hook React (usa yo-protocol-react) o i comandi della shell CLI (usa yo-protocol-cli).
