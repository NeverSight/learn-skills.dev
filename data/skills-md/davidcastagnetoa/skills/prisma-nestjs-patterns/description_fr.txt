Clients d'intégration de Prisma ORM avec NestJS pour les applications de production. Utilisez PROACTIVEMENT lorsque vous travaillez avec les modèles Prisma, les migrations, PrismaService, les transactions atomiques (y compris SELECT FOR UPDATE), les référentiels comme les adaptateurs d'infrastructure en Clean Architecture, les requêtes optimisées (N+1, inclut, pagination), les graines et les tests avec Prisma Mock. Activez toujours les paramètres suivants : Prisma, PrismaService, prisma.$transaction, prisma migrate, schema.prisma, PrismaClient, Prisma.validator, include/select, upsert, createMany, ou toute opération de base de données avec Prisma dans un projet NestJS.
