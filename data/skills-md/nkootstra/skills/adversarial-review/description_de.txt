Richten Sie Multi-Agent-Pipelines für die gegnerische Verifizierung ein (Bug-Finder → Gegner → Schiedsrichter) für eine High-Fidelity-Codeüberprüfung und Fehlererkennung. Passt sich sowohl an Claude Code CLI (echte Subagenten) als auch an Claude.ai (sequentielle Simulation) an. Verwenden Sie diese Option, wenn der Benutzer eine kontradiktorische Überprüfung, eine Multi-Agent-Verifizierung, eine hochzuverlässige Fehlersuche, eine Agent-Überprüfungspipeline oder eine gründliche automatisierte Überprüfung über einen einzigen Durchgang hinaus anfordert.
