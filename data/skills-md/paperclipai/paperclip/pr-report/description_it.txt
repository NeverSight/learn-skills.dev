Esamina in modo approfondito una richiesta di pull o un contributo, spiegalo in stile tutorial per un manutentore e produci un artefatto di report raffinato come HTML o Markdown. Da utilizzare quando viene richiesto di analizzare un PR, spiegare le decisioni di progettazione di un collaboratore, confrontarlo con sistemi simili o preparare una raccomandazione per l'unione.
