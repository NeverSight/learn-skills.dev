Überprüfen Sie eine Pull-Anfrage oder einen Pull-Beitrag eingehend, erklären Sie sie einem Betreuer im Tutorial-Stil und erstellen Sie ein ausgefeiltes Berichtsartefakt wie HTML oder Markdown. Verwenden Sie es, wenn Sie gebeten werden, eine PR zu analysieren, die Designentscheidungen eines Mitwirkenden zu erläutern, es mit ähnlichen Systemen zu vergleichen oder eine Zusammenführungsempfehlung vorzubereiten.
