Pianificazione strutturata prima dell'implementazione complessa. Utilizza questa capacità ogni volta che l'utente menziona pianificazione, decisioni sull'architettura, modifiche a più file, strategie di migrazione o chiede di "riflettere", "mappare" o "capire l'approccio" prima di scrivere codice. Si attiva anche quando l'attività è a tempo indeterminato, tocca più di 3 file o richiede la valutazione di più approcci. NON utilizzare per modifiche di file singoli, semplici correzioni di bug o quando l'utente dispone già di un piano approvato.
