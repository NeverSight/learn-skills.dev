Fournit les meilleures pratiques spécifiques au domaine pour le développement Node.js avec TypeScript, couvrant la suppression de types, les modèles asynchrones, la gestion des erreurs, les flux, les modules, les tests, les performances, la mise en cache, la journalisation, etc. À utiliser lors de la configuration de projets Node.js avec prise en charge native de TypeScript, de la configuration de la suppression de types (--experimental-strip-types), de l'écriture de Node 22+ TypeScript sans étape de construction, ou lorsque l'utilisateur mentionne « TypeScript natif dans Node », « types de bande », « Node 22 TypeScript », « fichiers .ts sans compilation », « alternative ts-node », ou a besoin de conseils sur la gestion des erreurs, l'arrêt progressif, tests irréguliers, profilage ou configuration de l'environnement dans Node.js. Aide à configurer tsconfig.json pour la suppression de type, à configurer les scripts package.json, à gérer la résolution de module et l'importation d'extensions, et à appliquer des modèles robustes sur l'ensemble de la pile Node.js.
