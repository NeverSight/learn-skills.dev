TypeScript を使用した Node.js 開発のドメイン固有のベスト プラクティスを提供し、型ストリッピング、非同期パターン、エラー処理、ストリーム、モジュール、テスト、パフォーマンス、キャッシュ、ロギングなどをカバーします。ネイティブ TypeScript サポートを使用して Node.js プロジェクトをセットアップする場合、型ストリッピング (--experimental-strip-types) を構成する場合、ビルド ステップなしで Node 22 以降の TypeScript を作成する場合、またはユーザーが「Node のネイティブ TypeScript」、「タイプのストリップ」、「Node 22 TypeScript」、「コンパイルなしの .ts ファイル」、「ts ノードの代替」について言及する場合、またはエラー処理、正常なシャットダウンに関するガイダンスが必要な場合に使用します。 Node.js での不安定なテスト、プロファイリング、または環境構成。型ストリッピングのための tsconfig.json の構成、package.json スクリプトのセットアップ、モジュール解決の処理と拡張機能のインポート、および完全な Node.js スタック全体への堅牢なパターンの適用を支援します。
