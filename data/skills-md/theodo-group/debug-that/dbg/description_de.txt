Debuggen Sie Anwendungen mit dem dbg-CLI-Debugger. Unterstützt Node.js (V8/CDP), Bun (WebKit/JSC) und nativen Code über LLDB (DAP). Wird verwendet, wenn: (1) Laufzeitfehler durch schrittweises Durchlaufen des Codes untersucht werden, (2) Variablenwerte an bestimmten Ausführungspunkten überprüft werden, (3) Haltepunkte und bedingte Haltepunkte gesetzt werden, (4) Ausdrücke in einem angehaltenen Kontext ausgewertet werden, (5) Code im laufenden Betrieb gepatcht wird, ohne neu gestartet zu werden (JS/TS), (6) Testfehler durch Anhängen an einen laufenden Prozess debuggen, (7) C/C++/Rust/Swift mit LLDB debuggen, (8) jede Aufgabe, bei der das Verständnis des Laufzeitverhaltens einen Debugger erfordert. Auslöser: „dieses debuggen“, „einen Haltepunkt festlegen“, „durchgehen“, „Variablen prüfen“, „Warum ist dieser Wert falsch“, „Ausführung verfolgen“, „Debugger anhängen“, „Laufzeitfehler“, „Segfault“, „Core-Dump“.
