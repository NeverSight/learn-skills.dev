必要に応じて現在のコードの状態と Git 履歴を分析することで、マイクロサービスの生きたドキュメントを生成して維持します。ユーザーがサービスの文書化、変更後のドキュメントの更新、ブランチの比較、変更ログの生成、ドキュメントの鮮度の監査、中央リポジトリへのドキュメントの同期、レガシー ドキュメントの移行、フロー ドキュメント、ADR、またはシステム概要の作成を要求したときにトリガーされます。トリガーフレーズ: 'document [service]'、'document-branch'、'document-commits'、'sync'、'audit'、'migrate-legacy'、'document-flow'、'create-adr'、'system-overview'、'documenta'、'qué cambió'、'actualizar docs'、'generar changelog'、 「リリースノート」、「リビングドキュメント」。
