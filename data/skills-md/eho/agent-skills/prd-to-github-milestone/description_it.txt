Analizza un documento dei requisiti del prodotto (PRD) per estrarre le storie degli utenti e creare i problemi GitHub corrispondenti. Facoltativamente può collegarli a una pietra miliare GitHub. Questa competenza funge da fase di configurazione per il monitoraggio dei problemi nativi di GitHub. Assicurati di utilizzare questa abilità ogni volta che l'utente chiede di "inviare il PRD a GitHub", "creare problemi dal PRD", "impostare l'obiettivo" o menziona la trasformazione dei requisiti in problemi GitHub utilizzabili.
