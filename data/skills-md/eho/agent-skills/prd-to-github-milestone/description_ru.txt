Анализирует документ с требованиями к продукту (PRD) для извлечения пользовательских историй и создания соответствующих проблем GitHub. При желании он может связать их с этапом GitHub. Этот навык действует как этап настройки для отслеживания проблем с помощью GitHub. Обязательно используйте этот навык всякий раз, когда пользователь просит «отправить PRD в GitHub», «создать задачи из PRD», «настроить контрольную точку» или упоминает о превращении требований в практические проблемы GitHub.
