Analiza un documento de requisitos del producto (PRD) para extraer historias de usuarios y crea los problemas de GitHub correspondientes. Opcionalmente, puede vincularlos a un GitHub Milestone. Esta habilidad actúa como una fase de configuración para el seguimiento de problemas nativo de GitHub. Asegúrese de utilizar esta habilidad siempre que el usuario solicite "enviar el PRD a GitHub", "crear problemas a partir del PRD", "configurar el hito" o mencione convertir requisitos en problemas de GitHub procesables.
