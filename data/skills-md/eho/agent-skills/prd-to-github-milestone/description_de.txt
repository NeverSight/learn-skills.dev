Dieser Skill dient als Einrichtungsphase für die GitHub-native Problemverfolgung. Stellen Sie sicher, dass Sie diese Fähigkeit verwenden, wenn der Benutzer „die PRD an GitHub senden“, „Probleme aus der PRD erstellen“, „den Meilenstein einrichten“ möchte oder erwähnt, dass Anforderungen in umsetzbare GitHub-Probleme umgewandelt werden sollen.
