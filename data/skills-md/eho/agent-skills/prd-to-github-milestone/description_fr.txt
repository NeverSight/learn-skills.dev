Analyse un document d'exigences produit (PRD) pour extraire les témoignages d'utilisateurs et crée les problèmes GitHub correspondants. Il peut éventuellement les lier à un jalon GitHub. Cette compétence agit comme une phase de configuration pour le suivi des problèmes natif de GitHub. Assurez-vous d'utiliser cette compétence chaque fois que l'utilisateur demande « d'envoyer le PRD à GitHub », de « créer des problèmes à partir du PRD », de « configurer le jalon » ou mentionne la transformation des exigences en problèmes GitHub exploitables.
