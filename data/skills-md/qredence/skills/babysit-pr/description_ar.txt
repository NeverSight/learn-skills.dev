تستقبل طلب سحب GitHub من خلال الاستطلاع المستمر لفحوصات CI ومراجعة التعليقات وحالة قابلية الدمج حتى يصبح PR جاهزًا للدمج أو الإغلاق. يقوم بتشخيص حالات الفشل، وإعادة محاولة حالات الفشل غير المستقرة حتى 3 مرات، وإصلاح المشكلات المتعلقة بالفرع تلقائيًا، ويتوقف فقط عند الحاجة إلى مساعدة المستخدم. يُستخدم عندما يُطلب منك مراقبة العلاقات العامة، أو مشاهدة CI، أو التعامل مع تعليقات المراجعة، أو تتبع حالات الفشل في علاقات عامة مفتوحة.
