Die Working Backwards-Produktentwicklungsmethode von Amazon basiert auf „Working Backwards“ von Colin Bryar und Bill Carr. Verwenden Sie diese Funktion, wenn Sie Folgendes tun müssen: (1) ein PR-/FAQ-Dokument für ein neues Produkt oder eine neue Funktion schreiben, (2) eine Idee validieren, indem Sie zuerst das Kundenerlebnis definieren, (3) einen Vorschlag mit einer Pre-Mortem-Analyse einem Stresstest unterziehen, (4) Stakeholder auf eine Produktvision ausrichten, (5) entscheiden, ob es sich lohnt, eine Idee zu entwickeln, bevor Sie Code schreiben.
