Принципы, которые необходимо строго соблюдать, чтобы соответствовать критериям приемлемости пользователя. Пользователь не одобрит никакую фиксацию, которая не соответствует этим принципам. Используйте определение готовности (DoD) для завершения реализации. После внедрения выполнить сборку/тестирование/миграцию/заполнение/пересборку контейнера в соответствии с RUNBOOK.md и привести систему в проверяемое для пользователя состояние. Если какой-либо пункт не может быть выполнен, укажите причину. Используется для всех задач разработки.
