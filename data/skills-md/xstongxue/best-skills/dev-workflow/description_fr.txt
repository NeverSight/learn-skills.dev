Processus de développement en cinq étapes. Prend en charge la compréhension des exigences, la conception de solutions, la mise en œuvre du code, la correction des bogues et la révision du code. À utiliser lorsque les utilisateurs mentionnent « analyse des exigences », « conception de solution », « implémentation du code », « révision du code », « compréhension des exigences », « conception technique », « commencer le codage », « révision », « vérifier le code », « bug », « erreur », « crash », « exception » ou « quelque chose s'est mal passé ».
