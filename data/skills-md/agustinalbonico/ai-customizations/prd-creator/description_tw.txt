通用產品需求文件 (PRD) 是常用互動對話的一部分。我們將根據問題的「問題」進行調整，對問題進行分類，並總結談判的必要條件。 El 輸出是 un .md orientado a Problemas de negocio y requisitos (NO técnico)。 Usar cuando el usuario quiere create un PRD, documentar requisitos de Producto, definir un Problema a Solutionr, o dice "quiero un PRD", "documentar requisitos", "definir qué vamos a construir".
