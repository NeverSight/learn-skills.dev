Laden Sie lokale Bilder auf eine GitHub-PR hoch und betten Sie sie in die Beschreibung oder Kommentare ein. Verwenden Sie diese Option, wenn Sie aufgefordert werden, „Screenshots an PR anzuhängen“, „Bilder zu PR hinzuzufügen“, „Testergebnisse in PR hochzuladen“, „Screenshots in PR-Beschreibung einzubetten“, „Vorher-/Nachher-Bilder zu PR hinzuzufügen“, „UI-Screenshots anzuhängen“, „Testergebnisse in PR anzuzeigen“, „visuelle Beweise zu PR hinzuzufügen“ oder jede Anfrage, die Bilder und PRs betrifft. Verwenden Sie diesen Skill immer, wenn der Benutzer Änderungen in einer Pull-Anfrage visuell dokumentieren möchte, auch wenn er das Wort „hochladen“ nicht verwendet – Phrasen wie „Füge den Screenshot in die PR ein“ oder „zeige das Bild in die PR“ sollten diesen Skill auslösen. Unterstützt Playwright MCP / Chrome DevTools MCP / Agent-Browser als Browser-Automatisierungs-Backends.
