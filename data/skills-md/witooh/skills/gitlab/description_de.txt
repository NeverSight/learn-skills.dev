Interagieren Sie mit GitLab über die Glab-CLI. Unterstützt drei MR-Workflows: Lesen (Zusammenfassen), Überprüfen (vollständige Code-/Sicherheits-/QA-Überprüfung) und Fix (Überprüfen + Implementieren). Wird immer dann ausgelöst, wenn der Benutzer eine GitLab-MR-URL angibt oder etwas wie „อ่าน MR“, „ดู MR“, „check MR“, „review MR“, „ช่วย review MR นี้“, „ตรวจ MR“, „แก้ตาม“ sagt MR“, „MR reparieren“ oder fügt einfach eine GitLab MR-URL ein. Unterstützt auch das Auflisten von MRs, das Anzeigen des MR-Status, das Überprüfen von CI/CD-Pipelines, das Genehmigen von MRs und andere einfache Vorgänge. Auslösen bei „Pipeline prüfen“, „Offene MRs auflisten“ oder einer beliebigen GitLab-bezogenen Aufgabe. Wenn der Benutzer Probleme mit einem MR beheben möchte – z. „แก้ตาม MR นี้“, „fix MR“, „แก้ issue ตาม MR“, „แก้ MR <url>“, „แก้ตาม review“ – Führen Sie den MR-Fix-Workflow aus (überprüfen Sie ihn und verketten Sie ihn dann automatisch mit /neo-team für die Umsetzung).
