Interactúe con GitLab a través de la CLI de glab. Admite tres flujos de trabajo de MR: lectura (resumir), revisión (código completo/seguridad/revisión de control de calidad) y reparación (revisión + implementación). Se activa cada vez que el usuario proporciona una URL de GitLab MR o dice algo como "อ่าน MR", "ดู MR", "check MR", "review MR", "ช่วย review MR นี้", "ตรวจ MR", "แก้ตาม MR", "arreglar MR", o simplemente pega una URL de GitLab MR. También admite enumerar MR, ver el estado de MR, verificar canalizaciones de CI/CD, aprobar MR y otras operaciones glab. Activar "verificar canalización", "listar MR abiertos" o cualquier tarea relacionada con GitLab. Cuando el usuario quiere solucionar problemas de un MR, p. "แก้ตาม MR นี้", "fix MR", "แก้ issues ตาม MR", "แก้ MR <url>", "แก้ตาม review": ejecute el flujo de trabajo MR Fix (revise y luego encadene automáticamente a /neo-equipo para la implementación).
