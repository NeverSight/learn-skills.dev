Revisión integral de código utilizando subagentes especializados paralelos. Úselo cuando desee una revisión exhaustiva del código que cubra la arquitectura, la seguridad, el rendimiento, la calidad del código, el cumplimiento de los requisitos y los errores. Funciona con enlaces de relaciones públicas de GitHub O cambios de sucursales locales. Si se proporciona una URL de relaciones públicas, obtiene detalles de relaciones públicas y puede publicar comentarios. Si no se proporciona ningún PR, revisa la diferencia entre la rama actual y su rama base más cualquier cambio no confirmado. IMPORTANTE: esta habilidad es costosa, no la use a menos que el usuario solicite explícitamente usarla.
