Используйте, когда пользователь спрашивает что-либо о Gurobi или gurobipy — методы API (addConstr, addVar, addMVar,оптимизация, setParam, getAttr), параметры (TimeLimit, MIPGap, Threads, Presolve, PoolSolutions), атрибуты (ObjVal, Status, X, Pi, RC, ObjBound), функции решателя (обратные вызовы, IIS/неосуществимость, пул решений, несколько целей, нелинейность), статус/ошибка коды или изменения, специфичные для версии (Гуроби 11, 12, 13). Всегда используйте этот навык вместо того, чтобы отвечать по памяти — детали API Gurobi меняются в зависимости от версии и требуют проверки документов.
