Erzwingen Sie einen konsistenten, idiomatischen Swift-Stil beim Generieren, Überprüfen oder Refactoring von Swift-Code. Verwenden Sie es, wenn Sie neuen Swift-Code schreiben, Code auf Stilverstöße überprüfen oder Team-Stilstandards durchsetzen. Behandelt Dateistruktur, Benennung, Formatierung und Programmierpraktiken. Basierend auf dem Google Swift Style Guide (https://google.github.io/swift/), der selbst auf dem Swift-Standardbibliotheksstil von Apple basiert und in der Swift-Community weit verbreitet ist.
