Utilice esta habilidad siempre que el usuario solicite un análisis de seguridad, una evaluación de vulnerabilidad, una auditoría de seguridad o cualquier forma de Informe de evaluación de seguridad (SAR) sobre una base de código, infraestructura, API, base de datos o sistema. Los desencadenantes incluyen: "auditar mi código", "encontrar problemas de seguridad", "ejecutar una verificación de seguridad", "generar un SAR", "comprobar vulnerabilidades", "es seguro este código" o cualquier solicitud que implique evaluar la postura de seguridad de un proyecto. También se activa cuando el usuario carga o hace referencia al código fuente, archivos de configuración, variables de entorno o diagramas de arquitectura y solicita una opinión de seguridad. NO lo utilice para tareas de codificación genéricas, revisiones de código centradas en la calidad en lugar de la seguridad u optimización del rendimiento, a menos que esté explícitamente presente un ángulo de seguridad.
