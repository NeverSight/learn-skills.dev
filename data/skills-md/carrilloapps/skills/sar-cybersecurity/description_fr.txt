Utilisez cette compétence chaque fois que l'utilisateur demande une analyse de sécurité, une évaluation des vulnérabilités, un audit de sécurité ou toute forme de rapport d'évaluation de la sécurité (SAR) sur une base de code, une infrastructure, une API, une base de données ou un système. Les déclencheurs incluent : « auditer mon code », « rechercher des problèmes de sécurité », « exécuter un contrôle de sécurité », « générer un SAR », « vérifier les vulnérabilités », « ce code est-il sécurisé » ou toute demande impliquant l'évaluation de la posture de sécurité d'un projet. Se déclenche également lorsque l'utilisateur télécharge ou référence du code source, des fichiers de configuration, des variables d'environnement ou des diagrammes d'architecture et demande un avis de sécurité. Ne PAS utiliser pour des tâches de codage génériques, des révisions de code axées sur la qualité plutôt que sur la sécurité, ou pour l'optimisation des performances, à moins qu'un angle de sécurité ne soit explicitement présent.
