Warten Sie, bis eine Ausführung von GitHub Actions mit minimaler Terminalausgabe und einem zuverlässigen Exit-Code abgeschlossen ist. Verwenden Sie diese Option, wenn ein Agent darauf warten muss, dass CI bestanden wird (z. B. nach einem Push, nach dem Öffnen eines PR oder bei der Überprüfung eines bestimmten Laufs). Bevorzugen Sie gh run watch mit --exit-status und --compact, um eine Überflutung des Kontexts mit der Umfrageausgabe zu vermeiden.
