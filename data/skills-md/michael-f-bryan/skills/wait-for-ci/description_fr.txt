Attendez la fin d’une exécution d’actions GitHub avec une sortie de terminal minimale et un code de sortie fiable. À utiliser lorsqu'un agent doit attendre la réussite du CI (par exemple, après un push, après l'ouverture d'un PR ou lors de la vérification d'une exécution spécifique). Préférez gh run watch avec --exit-status et --compact pour éviter d'inonder le contexte avec la sortie du sondage.
