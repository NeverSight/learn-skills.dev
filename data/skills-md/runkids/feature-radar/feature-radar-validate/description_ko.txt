형식 규칙에 따라 SKILL.md 머리말 및 .feature-radar/ 파일을 검증합니다. 실행
verify.sh는 오류/경고를 보고하고 문제를 자동 수정합니다. 이 스킬을 사용한 후에는 반드시 사용해야 합니다.
SKILL.md 또는 .feature-radar/ 파일 편집 — 1024-char와 같은 형식 버그를 잡아냅니다.
스킬 등록이 중단되기 전 설명 제한.
다음과 같은 경우에 사용하세요.
- 사용자가 "유효성 검사", "형식 확인", "린트 기술", "유효성 검사 실행"이라고 말합니다.
- 방금 SKILL.md(설명, 이름 또는 본문)를 편집했습니다. 사전에 실행하세요.
- .feature-radar/에서 파일을 생성하거나 수정했습니다. — 사전에 실행합니다.
- 기술/ 또는 .feature-radar/에 영향을 미치는 변경 사항을 커밋하기 전
- 사용자가 "내 스킬이 유효합니까?", "스킬 확인", "스킬 형식 확인"을 묻습니다.
중요: 기술/또는 .feature-radar/ 파일을 편집한 후 이 기능을 사전에 사용하세요.
사용자가 요청하지 않더라도. 형식 유효성 검사는 자동 중단을 방지합니다.
