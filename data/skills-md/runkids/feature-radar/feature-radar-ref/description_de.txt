Zeichnen Sie externe Beobachtungen, Ökosystemtrends und kreative Inspiration auf
.feature-radar/references/. Diese Fähigkeit MUSS verwendet werden, wenn der Benutzer etwas erwähnt
Interessantes von außerhalb ihres Projekts – andere Tools, Artikel, Ansätze oder Trends.
Selbst beiläufige Erwähnungen wie „Ich habe etwas Cooles in X gesehen“ sollten diese Fähigkeit auslösen.
Verwenden Sie diese Option, wenn der Benutzer:
- Sagt: „Ich habe dieses coole Ding in der X-API gesehen“, „Sehen Sie sich an, wie X damit umgeht.“
- Teilen Sie eine URL, einen Artikel, einen Vortrag oder ein Forschungsergebnis mit relevanten Erkenntnissen
- Stellt einen Ökosystemtrend fest: neue Tools, Standards, Community-Muster
– Erwähnt ein verwandtes Projekt mit einer bemerkenswerten Funktion
- Möchte externe Inspiration mit einem Lesezeichen versehen: „interessanter Ansatz“, „Speichern“
- Sagt „Referenz hinzufügen“, „Beobachtung protokollieren“, „Dieses Projekt verfolgen“
NICHT für interne Erkenntnisse/Muster verwenden – das ist die Aufgabe von Feature-Radar-Learn.
NICHT zur Priorisierung von Funktionen verwenden – das ist die Aufgabe von Feature-Radar.
