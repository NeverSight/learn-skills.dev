外部の観察、生態系の傾向、創造的なインスピレーションを記録します。
.feature-radar/references/。ユーザーが何かを言及するときは、このスキルを使用しなければなりません
他のツール、記事、アプローチ、トレンドなど、プロジェクトの外から見ても興味深いものです。
「X でクールなものを見た」などの何気ない言及でも、このスキルがトリガーされるはずです。
ユーザーが次の場合に使用します。
- 「X の API でこの素晴らしいものを見た」、「X がこれをどのように処理するかをチェックしてください」と言います。
- 関連する洞察を含む URL、記事、講演、または調査結果を共有します
- エコシステムのトレンドに注目: 新しいツール、標準、コミュニティ パターン
- 注目すべき機能を提供する関連プロジェクトについて言及
- 外部からのインスピレーションをブックマークしたい: 「興味深いアプローチ」、「これを保存」
- 「参照を追加」、「観察をログに記録」、「このプロジェクトを追跡」と言います
内部学習/パターンには使用しないでください。それが feature-radar-learn の仕事です。
機能の優先順位付けには使用しないでください。それが機能レーダーの仕事です。
