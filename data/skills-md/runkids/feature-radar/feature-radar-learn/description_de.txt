Extrahieren Sie wiederverwendbare Muster, Architekturentscheidungen und Fallstricke aus abgeschlossenen Arbeiten
in .feature-radar/specs/. Erfasst das „Warum“ hinter Entscheidungen, damit zukünftige Sitzungen aufgebaut werden können
auf Erfahrungen aus der Vergangenheit. Diese Fähigkeit MUSS verwendet werden, wenn der Benutzer darüber nachdenkt, was funktioniert/nicht funktioniert hat.
möchte eine Entscheidung dokumentieren oder erwähnt die Erinnerung an ein Muster für die zukünftige Verwendung.
Verwenden Sie diese Option, wenn der Benutzer:
- Sagt „Diesen Ansatz merken“, „Diese Entscheidung dokumentieren“, „Dieses Muster speichern“
- Reflektiert: „Das hat gut funktioniert“, „Lektionen gelernt“, „Was haben wir gelernt“
- Möchte einfangen: „Vergiss das nicht“, „Das war ein gutes Muster“
- In eine Sackgasse geraten: „Das hat nicht funktioniert, weil …“, „Diesen Ansatz vermeiden“
- Eine technische Entscheidung getroffen, die es wert ist, für die Nachwelt festgehalten zu werden
- Sagt „Lernen extrahieren“ oder „Erfassen, was wir gelernt haben“
NICHT zur Aufzeichnung externer Beobachtungen verwenden – das ist die Aufgabe von feature-radar-ref.
NICHT zum Archivieren abgeschlossener Features verwenden – das ist die Aufgabe von feature-radar-archive.
