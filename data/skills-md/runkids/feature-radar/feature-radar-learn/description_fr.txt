Extrayez des modèles réutilisables, des décisions architecturales et des pièges du travail terminé
dans .feature-radar/specs/. Capture le « pourquoi » derrière les choix afin que les sessions futures se construisent
sur l'expérience passée. DOIT utiliser cette compétence lorsque l'utilisateur réfléchit à ce qui a fonctionné/n'a pas fonctionné,
veut documenter une décision ou mentionne se souvenir d’un modèle pour une utilisation future.
À utiliser lorsque l'utilisateur :
- Dit "souvenez-vous de cette approche", "documentez cette décision", "enregistrez ce modèle"
- Réfléchit : "cela a bien fonctionné", "les leçons apprises", "qu'avons-nous appris"
- Veut capturer : "n'oublie pas ça", "c'était un bon modèle"
- Dans une impasse : "ça n'a pas marché parce que...", "évitez cette approche"
- Prise d'une décision technique qui mérite d'être enregistrée pour la postérité
- Dit "extraire les apprentissages" ou "capturer ce que nous avons appris"
Ne PAS utiliser pour enregistrer des observations externes - c'est le travail de feature-radar-ref.
Ne PAS utiliser pour archiver des fonctionnalités terminées - c'est le travail de feature-radar-archive.
