Migliori pratiche C/C++/CAPL per sistemi embedded automobilistici. Questa competenza dovrebbe essere utilizzata durante la scrittura, la revisione o il refactoring di codice C/C++ incorporato o script CAPL destinati a ECU automobilistiche, seguendo le linee guida MISRA, AUTOSAR, ISO 26262 e ISO 21434. Trigger su attività che coinvolgono firmware integrato, comunicazione CAN/CAN FD/LIN/Ethernet, protocolli TCP/UDP/DoIP/SOME-IP, programmazione RTOS, codice critico per la sicurezza, sicurezza informatica, diagnostica (UDS), automazione dei test CAPL o integrazione della toolchain di calibrazione.
