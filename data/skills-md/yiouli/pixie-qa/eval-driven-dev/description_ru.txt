Инструментируйте приложения Python LLM, создавайте «золотые наборы данных», пишите тесты на основе оценки, запускайте их и выявляйте первопричины сбоев — охватывая полный цикл разработки на основе оценки. Обязательно используйте этот навык всякий раз, когда пользователь разрабатывает, тестирует, проводит контроль качества, оценивает или тестирует проект Python, который вызывает LLM, даже если он явно не говорит «оценивает». Используйте для проверки правильности работы приложения искусственного интеллекта, выявления регрессий после подсказок об изменениях, устранения причин, по которым агент начал вести себя по-другому, или проверки качества выходных данных перед отправкой.
