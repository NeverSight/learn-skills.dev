A security-hardening workflow for specific threat scenarios.  
Input a single concrete threat such as “prevent IDOR,” “design CSRF countermeasures,” or “strengthen RLS,” and it consistently delivers artifacts from threat modeling → mitigation design → implementation plan → testing/logging → release-gate.  
Use it for targeted fixes: IDOR, replay attacks, CSRF, XSS, path traversal, open redirect, rate-limit design, etc.  
For “scan the entire codebase,” use security-audit-quick; for “holistic app assessment,” use security-threat-review.
