Настройте поведение Litestar DTO для входящего анализа и исходящей сериализации, включая `dto`/`return_dto` на уровне уровня, политики `DTOConfig`, рабочие процессы обновления `DTOData` и пользовательские реализации `AbstractDTO`. Используйте, когда контракты полезной нагрузки API отличаются от внутренних структур модели. Не используйте, если внутренние модели можно безопасно экспонировать без преобразования.
