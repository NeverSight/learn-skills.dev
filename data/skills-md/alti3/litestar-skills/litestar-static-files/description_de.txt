Stellen Sie statische Assets in Litestar mit „create_static_files_router()“, expliziten Mount-Pfaden, HTML-Modus, Cache-Kontrollrichtlinie, URL-Umkehr, benutzerdefinierten Dateisystemen und Symlink-sicheren Verzeichnisgrenzen bereit. Verwenden Sie diese Option, wenn Sie JS, CSS, Bilder, herunterladbare Artefakte oder statisches HTML aus kontrollierten Verzeichnissen verfügbar machen. Nicht für die Upload-Aufnahme, die vom Benutzer beschreibbare Dateiveröffentlichung oder beliebige Datei-Download-Endpunkte verwenden, die eine Autorisierungslogik pro Anfrage benötigen.
