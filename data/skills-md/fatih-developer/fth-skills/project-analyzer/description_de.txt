Analysiert ein bestehendes Softwareprojekt eingehend und generiert 3–4 umfassende Berichte im Ordner docs/analyze/. Wird ausgelöst, wenn Ausdrücke wie „Projekt analysieren“, „Codeanalyse durchführen“, „Projektüberprüfung“, „Architekturbericht“, „technische Schulden“, „Projektbewertung“, „Codebasisanalyse“ vorkommen. Funktioniert, wenn ein Projektverzeichnis oder eine Repo-URL angegeben wird. Wenn API-Endpunkte erkannt werden, wird automatisch ein 4. Bericht generiert.
