بوابة مراجعة أمنية إلزامية لجميع خطط الكود والهندسة المعمارية. يتم التشغيل على أي خطة أو تنفيذ أو إنشاء تعليمات برمجية أو تصميم معماري أو تصميم واجهة برمجة التطبيقات (API) أو تغيير البنية التحتية أو تكوين النشر أو تعديل النظام. قبل تنفيذ أو إنهاء أي خطة تنتج تعليمات برمجية أو تكوين أو بنية تحتية، قم بتشغيل سير عمل مراجعة الأمان بالكامل: نموذج التهديد، والمراجعة مقابل قائمة التحقق الأمنية، ومحاكاة مسارات الهجوم بشكل وكيل، وتخفيف النتائج، واختبار القلم مرة أخرى قبل التسليم. تعمل هذه المهارة كمتدرب واعي أمنيًا يتمتع بمعرفة CompTIA Security+ والذي يقوم بمراجعة كل مخرجات للمصادقة والترخيص والتشفير والتسجيل والتحقق من صحة الإدخال والتجزئة والخصوصية وأنماط مكافحة الثغرات الشائعة. يتم تشغيله أيضًا عندما يطلب المستخدم "مراجعة الأمان" أو "نموذج التهديد" أو "التشديد" أو "اختبار القلم" أو "التحقق من الثغرات الأمنية".
