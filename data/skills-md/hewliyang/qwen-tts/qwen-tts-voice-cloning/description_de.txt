End-to-End-Pipeline zum Klonen von Stimmen mit qwen-tts auf Apple Silicon. Umfasst Datenerfassung (yt-dlp), Audioreinigung (ASR mit parakeet-tdt-0.6b-v3), Datenaufbereitung, LoRA-Feinabstimmung, Generierung und Auswertung über ASR-Verständlichkeitsprüfung (lokal, keine API) + Gemini-as-Judge (qwen-tts-Prüfung). Verwenden Sie diese Option, wenn der Benutzer eine Stimme klonen, ein TTS-Modell trainieren, Sprachtrainingsdaten vorbereiten, die Qualität der generierten Sprache bewerten oder die vollständige Pipeline „Sammeln → Bereinigen → Vorbereiten → Trainieren → Generieren → Auswerten“ ausführen möchte.
