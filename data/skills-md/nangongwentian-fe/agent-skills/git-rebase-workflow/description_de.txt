Git Rebase Branch-Synchronisierungsprozess, der verwendet wird, um den aktuellen Feature-Branch auf den neuesten Ziel-Branch (z. B. Master/Main) umzubasieren und so den Commit-Verlauf sauber zu halten. Geeignet für Szenarien, in denen sich der Feature-Branch hinter dem Ziel-Branch befindet und der neueste Code synchronisiert werden muss.
