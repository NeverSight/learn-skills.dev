Führen Sie eine mehrdimensionale Codeüberprüfung für nicht festgeschriebene Änderungen in Git durch, einschließlich der Einhaltung von Projektspezifikationen, des Scannens von Fehlern, der Einhaltung von Codekommentaren, der Überprüfung der Rationalität von Komponentenkapselung/Architekturdesign und filtern Sie Fehlalarme durch Konfidenzbewertung heraus. Wird ausgelöst, wenn der Benutzer die Überprüfung von nicht festgeschriebenem Code oder aktuellen Änderungen anfordert oder /code-review-uncommitted verwendet.
