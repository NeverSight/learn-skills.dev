Effectuez un examen multidimensionnel du code sur les modifications non validées dans git, y compris la conformité aux spécifications du projet, l'analyse des bogues, la conformité des commentaires de code, l'examen de la rationalité de la conception de l'encapsulation/de l'architecture des composants, et filtrez les faux positifs grâce à une évaluation de confiance. Déclenché lorsque l'utilisateur demande à réviser le code non validé, à réviser les modifications en cours ou à utiliser /code-review-uncommit.
