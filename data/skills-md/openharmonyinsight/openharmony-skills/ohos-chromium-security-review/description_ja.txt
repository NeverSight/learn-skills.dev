OpenHarmony/Chromium のディープ コード監査スキル – スレッド セーフティ、ライフタイム管理、ロジック欠陥に焦点を当てる

厳密なシステム レベルのプログラミング経験に基づいて、次の C/C++ ソース コードを徹底的にレビューします。
- スレッドセーフの問題 (データ競合、デッドロック、クロスシーケンスアクセス)
- 存続期間の問題 (UAF、ダングリング ポインタ、WeakPtr の誤用)
- OpenHarmony/Chromium 固有のパターンのリスク
- 静的解析ツールでは検出できない高次のセキュリティリスク

主要な機能:
- 非常に厳格な監査スタイル
- 14 の厳格なユーザー定義ルールに従います
- 構造化された JSON レポートと Markdown レポートを生成します
- チームの責任ごとに問題を分離する
