Gestisci le porte del server di sviluppo locale, coordina i flussi di lavoro multi-agente con messaggistica pub/sub, blocchi distribuiti e registro degli agenti tramite il demone Port Daddy. Da utilizzare quando si richiedono porte, si risolvono conflitti di porte, si coordinano agenti o si impostano configurazioni di porte di progetto. Attivare su "conflitto porta", "richiesta porta", "get-port", "port-daddy", "coordinazione agente", "distributed lock", "pub/sub", "dev server port". NON per la distribuzione in produzione, la rete Docker, l'infrastruttura cloud o le pipeline CI/CD.
