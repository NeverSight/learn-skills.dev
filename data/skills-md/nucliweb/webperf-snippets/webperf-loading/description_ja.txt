TTFB 調査 (DNS/接続/サーバーの故障)、レンダリング ブロッキングの検出、スクリプト パフォーマンスの詳細 (ファーストパーティとサードパーティの帰属)、フォントの最適化、およびリソース ヒントの検証のための自動ワークフローによるインテリジェントな読み込みパフォーマンス分析。遅い場合に TTFB サブパートを自動的に分析し、スクリプト読み込みのアンチパターン (非同期/遅延/プリロードの競合) を検出し、レンダリングをブロックするリソースを特定し、リソース ヒントの使用を検証するデシジョン ツリーが含まれています。完全な読み込み監査 (6 フェーズ)、バックエンドのパフォーマンス調査、優先順位の最適化のためのワークフローを備えています。 Core Web Vitals (LCP リソースの読み込み)、インタラクション (スクリプト実行のブロック)、およびメディア (遅延読み込み戦略) とのクロススキル統合。ユーザーが TTFB、FCP、レンダリング ブロッキング、読み込みの遅さ、フォントのパフォーマンス、スクリプトの最適化、またはリソースのヒントについて質問する場合に使用します。 Chrome DevTools MCP と互換性があります。
