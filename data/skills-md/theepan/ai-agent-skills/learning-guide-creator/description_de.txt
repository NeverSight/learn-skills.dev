Erstellen Sie umfassende, forschungsgestützte, ca. 50-seitige Lernleitfäden (.md) zu jedem Thema, die für erfahrene Technikexperten geschrieben wurden. Verwenden Sie diese Fähigkeit immer dann, wenn der Benutzer einen Lernleitfaden, einen Studienführer, ein Forschungsdokument, ein Leitfadendokument, ein Briefing-Dokument oder eine Anfrage zur Recherche zu einem Thema und zur Erstellung eines pädagogischen Markdown-Dokuments erstellen möchte. Lösen Sie Sätze aus wie „Erstellen Sie einen Leitfaden zu …“, „Ich muss verstehen …“, „Recherchieren und schreiben Sie über …“, „Hilf mir, mich auf den neuesten Stand zu bringen …“, „Schreiben Sie einen Lernleitfaden für …“, „Erstellen Sie Lernmaterialien zu …“, „Informieren Sie mich über ...“, „Was muss ich über ... wissen“, „Lehren Sie mich über ...“ oder „Ich möchte mehr über ... lernen“. Diese Fähigkeit kombiniert gründliche Webrecherche (mindestens 10–15 Suchanfragen mit web_fetch für vollständigen Inhalt) mit strukturiertem Schreiben, um dichte, praktische Leitfäden auf Mitarbeiterebene für Ingenieure mit Entscheidungsrahmen, architektonischem Denken, Ökosystemkarten, realen Szenarien und Kompromissanalysen zu erstellen. Die Ausgabe ist immer Markdown (.md), niemals .docx. Keine Codeausschnitte – konzentriert sich auf Konzepte, Architektur, Kompromisse, Entscheidungsrahmen und reale Anwendungsfälle. Lösen Sie diesen Skill auch bei gelegentlichen Lernanfragen immer aus – der Benutzer bevorzugt umfassende Dokumente gegenüber gesprächigen Erklärungen.
