Redactar documentos de requisitos de software profesionales (SRS, PRD, BRD, FRD, Historias de usuarios, Casos de uso). Úselo cuando un usuario solicite escribir requisitos, crear un SRS o PRD, convertir una descripción de característica en historias de usuario, generar criterios de aceptación, revisar un documento de requisitos existente o convertir una idea informal en requisitos estructurados. También se activa cuando el usuario menciona "criterios de aceptación", "requisitos funcionales", "requisitos no funcionales" o "matriz de trazabilidad de requisitos".
