Schreiben Sie professionelle Software-Anforderungsdokumente (SRS, PRD, BRD, FRD, User Stories, Use Cases). Verwenden Sie es, wenn ein Benutzer Anforderungen schreiben, ein SRS oder PRD erstellen, eine Funktionsbeschreibung in User Stories umwandeln, Akzeptanzkriterien generieren, ein vorhandenes Anforderungsdokument überprüfen oder eine informelle Idee in strukturierte Anforderungen umwandeln möchte. Wird auch ausgelöst, wenn der Benutzer „Akzeptanzkriterien“, „funktionale Anforderungen“, „nichtfunktionale Anforderungen“ oder „Anforderungsrückverfolgbarkeitsmatrix“ erwähnt.
