Esamina le richieste pull di GitHub con analisi del codice dettagliate e multi-prospettiva utilizzando agenti secondari paralleli. Utilizza questa competenza ogni volta che l'utente desidera rivedere un PR, richiede la revisione del codice su una richiesta pull, menziona "revisiona PR", "controlla questo PR", "guarda la richiesta pull" o fa riferimento a un numero PR o a un URL PR GitHub. Si attiva anche quando l'utente desidera feedback sulle modifiche al codice, desidera approvare o richiedere modifiche su un PR o chiede di rivedere il contributo di qualcuno.
