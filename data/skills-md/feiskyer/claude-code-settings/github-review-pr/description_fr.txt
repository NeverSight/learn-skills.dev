Examinez les demandes d’extraction GitHub avec une analyse de code détaillée et multiperspective à l’aide de sous-agents parallèles. Utilisez cette compétence chaque fois que l'utilisateur souhaite examiner un PR, demande une révision du code sur une demande d'extraction, mentionne « réviser le PR », « vérifier ce PR », « examiner la demande d'extraction » ou fait référence à un numéro de PR ou à une URL de PR GitHub. Déclenchez également lorsque l'utilisateur souhaite obtenir des commentaires sur les modifications du code, souhaite approuver ou demander des modifications sur un PR, ou demande de consulter la contribution de quelqu'un.
