Überprüfen Sie GitHub-Pull-Anfragen mit detaillierter, multiperspektivischer Codeanalyse mithilfe paralleler Subagenten. Verwenden Sie diesen Skill immer dann, wenn der Benutzer eine PR überprüfen möchte, eine Codeüberprüfung für eine Pull-Anfrage anfordert, „PR überprüfen“, „diese PR prüfen“, „Pull-Anfrage ansehen“ erwähnt oder auf eine PR-Nummer oder eine GitHub-PR-URL verweist. Wird auch ausgelöst, wenn der Benutzer Feedback zu Codeänderungen wünscht, Änderungen an einer PR genehmigen oder anfordern möchte oder darum bittet, den Beitrag einer anderen Person zu überprüfen.
