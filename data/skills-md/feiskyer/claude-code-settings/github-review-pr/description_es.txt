Revise las solicitudes de extracción de GitHub con un análisis de código detallado y de múltiples perspectivas utilizando subagentes paralelos. Utilice esta habilidad siempre que el usuario quiera revisar un PR, solicite una revisión del código en una solicitud de extracción, mencione "revisar PR", "verificar este PR", "ver solicitud de extracción" o haga referencia a un número de PR o una URL de PR de GitHub. También se activa cuando el usuario quiere comentarios sobre cambios de código, quiere aprobar o solicitar cambios en un PR o solicita revisar la contribución de alguien.
