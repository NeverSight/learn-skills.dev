Clean up and organize the scripts and dependencies in package.json. Use for requests like “I want to clean up package.json,” “I want to reorganize the scripts,” or “I want to remove unused dependencies.” Specifically targets: (1) deleting or merging redundant scripts (aliases/pass-throughs), (2) making implicit lifecycle hooks (pre/post) explicit, (3) unifying script naming conventions, (4) detecting and removing unused dependency packages, (5) updating documentation and CI when script names change, (6) regenerating lock files after dependency removal.
