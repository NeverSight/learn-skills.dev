Guía para @workleap/telemetry, la telemetría unificada de Workleap que conecta Honeycomb, LogRocket y Mixpanel con ID de correlación automática.

Utilice esta habilidad cuando:
(1) Inicialización de wl-telemetry en una aplicación frontend
(2) Valores de correlación (Id. de telemetría, Id. de dispositivo) y correlación de datos entre herramientas
(3) Seguimiento de panal, tramos de OpenTelemetry y monitoreo del rendimiento
(4) Reproducción de sesión de LogRocket, identificación de usuario y controles de privacidad
(5) Análisis de Mixpanel, seguimiento de eventos y seguimiento entre productos
(6) Configuración de registradores de diagnóstico de telemetría wl (LogRocketLogger)
(7) Configuración de libro de cuentos/prueba con clientes de telemetría Noop
(8) Revisión o solución de problemas de instrumentación de telemetría
