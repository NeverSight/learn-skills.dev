قم بإعداد GitHub Actions CI/CD لإعادة إنشاء مخططات سير العمل البسيطة تلقائيًا عند الدفع. يغطي إنشاء YAML لسير العمل، وبرنامج R النصي لإنشاء المخططات باستخدام علامات الحارس، والالتزام التلقائي بالمخططات المحدثة، وتكامل README الحارس لتحديثات المخططات في مكانها. يُستخدم عندما يجب أن تعكس مخططات سير العمل دائمًا الحالة الحالية للتعليمات البرمجية، أو عندما قد يقوم العديد من المساهمين بتغيير التعليمات البرمجية التي تؤثر على سير العمل، أو عند استبدال إعادة إنشاء المخطط اليدوي بخط أنابيب CI/CD تلقائي.
