Richten Sie GitHub Actions CI/CD ein, um aktuelle Workflow-Diagramme bei Push automatisch neu zu generieren. Behandelt die Workflow-YAML-Erstellung, R-Skript zur Diagrammgenerierung mit Sentinel-Markierungen, automatisches Festschreiben aktualisierter Diagramme und README-Sentinel-Integration für direkte Diagrammaktualisierungen. Verwenden Sie diese Option, wenn Workflow-Diagramme immer den aktuellen Status des Codes widerspiegeln sollen, wenn mehrere Mitwirkende Code ändern können, der sich auf den Workflow auswirkt, oder wenn die manuelle Diagrammregenerierung durch eine automatisierte CI/CD-Pipeline ersetzt wird.
