Implementieren Sie die Durchsetzung von Richtlinien als Code mithilfe von OPA Gatekeeper oder Kyverno, um Kubernetes-Ressourcen gemäß den Unternehmensrichtlinien zu validieren und zu ändern. Behandelt Einschränkungsvorlagen, Zugangskontrolle, Prüfmodus, Meldung von Verstößen und die Integration mit CI/CD-Pipelines für die Shift-Left-Richtlinienvalidierung. Verwenden Sie diese Option, wenn Sie Ressourcenkonfigurationsstandards durchsetzen, Sicherheitsfehlkonfigurationen wie privilegierte Container verhindern, Compliance vor der Bereitstellung sicherstellen, Namenskonventionen standardisieren oder vorhandene Clusterressourcen anhand von Richtlinien prüfen möchten.
