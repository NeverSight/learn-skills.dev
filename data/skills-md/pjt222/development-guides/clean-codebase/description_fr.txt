Supprimez le code mort, les importations inutilisées, corrigez les avertissements de charpie et normalisez le formatage dans une base de code sans modifier la logique ou l'architecture métier. À utiliser lorsque les avertissements de charpie se sont accumulés au cours d'un développement rapide, que les importations inutilisées et les variables encombrent les fichiers, que les chemins de code morts n'ont jamais été supprimés, que le formatage est incohérent ou que les outils d'analyse statique signalent des problèmes d'hygiène réparables.
