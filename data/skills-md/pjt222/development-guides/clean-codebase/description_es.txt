Elimine el código inactivo, las importaciones no utilizadas, corrija las advertencias de pelusa y normalice el formato en una base de código sin cambiar la lógica o la arquitectura empresarial. Úselo cuando las advertencias de pelusa se hayan acumulado durante el desarrollo rápido, las importaciones y variables no utilizadas abarroten archivos, las rutas de código inactivo nunca se hayan eliminado, el formato sea inconsistente o las herramientas de análisis estático informen problemas de higiene que se pueden solucionar.
