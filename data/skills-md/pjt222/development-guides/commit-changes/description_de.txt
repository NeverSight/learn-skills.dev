Mit herkömmlichen Commit-Nachrichten können Sie Änderungen bereitstellen, festschreiben und ändern. Behandelt die Überprüfung von Änderungen, selektives Staging, das Schreiben beschreibender Commit-Nachrichten im HEREDOC-Format und die Überprüfung des Commit-Verlaufs. Verwenden Sie diese Funktion, wenn Sie eine logische Arbeitseinheit in der Versionskontrolle speichern, einen Commit mit einer konventionellen Nachricht erstellen, den letzten Commit ändern oder gestaffelte Änderungen vor dem Commit überprüfen.
