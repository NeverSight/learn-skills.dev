Analizza una codebase arbitraria per rilevare automaticamente flussi di lavoro, pipeline di dati e dipendenze tra file utilizzando il motore put_auto() di putior. Produce un piano di annotazione che mappa i modelli I/O rilevati sui file di origine in oltre 30 lingue supportate con 862 modelli di rilevamento automatico. Da utilizzare durante l'onboarding su una codebase sconosciuta per comprendere il flusso di dati, avviare l'integrazione putior in un progetto senza annotazioni esistenti, controllare la pipeline di dati di un progetto prima della documentazione o preparare un piano di annotazioni prima di eseguire annotate-source-files.
