Analysez une base de code arbitraire pour détecter automatiquement les flux de travail, les pipelines de données et les dépendances de fichiers à l'aide du moteur put_auto() de putior. Produit un plan d'annotation qui mappe les modèles d'E/S détectés aux fichiers sources dans plus de 30 langues prises en charge avec 862 modèles de détection automatique. À utiliser lors de l'intégration sur une base de code inconnue pour comprendre le flux de données, en démarrant une intégration putior dans un projet sans annotations existantes, en auditant le pipeline de données d'un projet avant la documentation ou en préparant un plan d'annotation avant d'exécuter des fichiers source annotés.
