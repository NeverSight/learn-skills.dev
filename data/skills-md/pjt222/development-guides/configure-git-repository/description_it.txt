Configura un repository Git con .gitignore, strategia di ramo, convenzioni di commit, hook e configurazione remota adeguati. Copre la configurazione iniziale e i modelli comuni per progetti R, Node.js e Python. Da utilizzare quando si inizializza il controllo della versione per un nuovo progetto, si aggiunge un file .gitignore per un linguaggio o framework specifico, si impostano la protezione e le convenzioni dei rami o si configurano gli hook di commit.
