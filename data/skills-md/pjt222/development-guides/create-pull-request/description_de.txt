Erstellen und verwalten Sie Pull-Anfragen mit der GitHub-CLI. Umfasst die Zweigvorbereitung, das Schreiben von PR-Titeln und -Beschreibungen, das Erstellen von PRs, die Bearbeitung von Überprüfungsfeedback und Zusammenführungs-/Bereinigungsworkflows. Verwenden Sie diese Option, wenn Sie Änderungen aus einem Feature- oder Fix-Zweig zur Überprüfung vorschlagen, abgeschlossene Arbeiten im Hauptzweig zusammenführen, eine Codeüberprüfung von Mitarbeitern anfordern oder den Zweck und Umfang einer Reihe von Änderungen dokumentieren.
