Configura Nginx come server web e proxy inverso. Riguarda la fornitura di file statici, il proxy inverso ai servizi upstream, la terminazione SSL/TLS con Let's Encrypt, i blocchi di posizione, il bilanciamento del carico, la limitazione della velocità e le intestazioni di sicurezza. Da utilizzare quando si servono file statici in produzione, proxy inverso ai servizi di backend (Node.js, Python, R/Shiny), terminazione SSL/TLS, bilanciamento del carico tra istanze o aggiunta di limitazioni di velocità e intestazioni di sicurezza per rafforzare un endpoint.
