Überprüfen Sie eine Pull-Anfrage durchgängig mit der GitHub-CLI. Umfasst Diff-Analyse, Überprüfung des Commit-Verlaufs, Überprüfung der CI/CD-Prüfung, Feedback auf Schweregradebene (Blockierung/Vorschlag/Nit/Lob) und Übermittlung von GH-PR-Bewertungen. Verwenden Sie diese Option, wenn eine Pull-Anfrage zur Überprüfung zugewiesen wird, eine Selbstüberprüfung durchgeführt wird, bevor die Eingabe anderer angefordert wird, eine zweite Überprüfung durchgeführt wird, nachdem auf Feedback eingegangen wurde, oder eine zusammengeführte PR zur Qualitätsbewertung nach der Zusammenführung überprüft wird.
