Esamina una richiesta pull end-to-end utilizzando la CLI di GitHub. Copre l'analisi delle differenze, la revisione della cronologia dei commit, la verifica del controllo CI/CD, il feedback a livello di gravità (blocco/suggerimento/nit/elogio) e l'invio della revisione gh pr. Da utilizzare quando una richiesta pull viene assegnata per la revisione, eseguendo un'autorevisione prima di richiedere il contributo di altri, conducendo una seconda revisione dopo aver affrontato il feedback o controllando un PR unito per la valutazione della qualità post-unione.
