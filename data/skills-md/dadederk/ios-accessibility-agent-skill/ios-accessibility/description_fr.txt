Conseils d’experts sur les meilleures pratiques, modèles et mises en œuvre en matière d’accessibilité iOS. À utiliser lorsque les développeurs mentionnent : (1) l'accessibilité iOS, VoiceOver, Dynamic Type ou les technologies d'assistance, (2) les étiquettes, caractéristiques, conseils ou valeurs d'accessibilité, (3) les tests, audits ou tests manuels d'accessibilité automatisés, (4) le contrôle des commutateurs, le contrôle vocal ou l'accès complet au clavier, (5) la conception inclusive ou la culture d'accessibilité, (6) faire en sorte que les applications fonctionnent pour les utilisateurs handicapés.
