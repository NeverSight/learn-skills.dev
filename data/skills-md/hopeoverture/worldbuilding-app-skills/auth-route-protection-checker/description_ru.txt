Этот навык следует использовать, когда пользователь запрашивает аудит, проверку или создание защиты аутентификации и авторизации для маршрутов Next.js, серверных компонентов, маршрутов API и действий сервера. Он анализирует существующие маршруты на предмет отсутствия проверок аутентификации и генерирует логику защиты на основе ролей и разрешений пользователей. Термины триггера включают проверку аутентификации, защиту маршрута, защиту маршрутов, безопасные конечные точки, промежуточное программное обеспечение аутентификации, ролевые маршруты, проверку авторизации, безопасность API, безопасность действий сервера, защиту страниц.
