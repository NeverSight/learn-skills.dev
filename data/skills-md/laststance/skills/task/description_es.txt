Flujo de trabajo de implementación estándar para todas las tareas de codificación.
Ejecuta un ciclo sistemático de 5 fases: Investigar → Planificar → Implementar → Verificar → Completar.
Integra puntos de control de pensamiento de Serena, marcadores de introspección y puertas de calidad.
Admite el indicador --frontend-verify para la verificación visual del navegador/aplicación/CLI.

Úselo cuando:
- El usuario solicita implementar una función, corregir un error o refactorizar el código.
- El usuario proporciona una tarea que requiere cambios de código.
- El usuario dice "hacer esto", "construir esto", "arreglar esto", "agregar esto"
- Cualquier trabajo de implementación que implique edición de código.

Palabras clave: tarea, implementar, construir, arreglar, agregar, crear, refactorizar, actualizar, cambiar
