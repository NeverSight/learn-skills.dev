Flux de travail de mise en œuvre standard pour toutes les tâches de codage.
Exécute un cycle systématique en 5 phases : Enquêter → Planifier → Mettre en œuvre → Vérifier → Terminer.
Intègre les points de contrôle de réflexion Serena, les marqueurs d'introspection et les portes de qualité.
Prend en charge l'indicateur --frontend-verify pour la vérification visuelle du navigateur/application/CLI.

À utiliser lorsque :
- L'utilisateur demande à implémenter une fonctionnalité, à corriger un bug ou à refactoriser le code
- L'utilisateur fournit une tâche qui nécessite des modifications de code
- L'utilisateur dit "faire ceci", "construire ceci", "réparer ceci", "ajouter ceci"
- Tout travail d'implémentation impliquant l'édition de code

Mots-clés : tâche, implémentation, construction, correction, ajout, création, refactorisation, mise à jour, modification
