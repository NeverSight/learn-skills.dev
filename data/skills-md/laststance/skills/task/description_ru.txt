Стандартный рабочий процесс реализации для всех задач кодирования.
Выполняет систематический пятиэтапный цикл: Исследование → План → Реализация → Проверка → Завершение.
Интегрирует контрольные точки мышления Серены, маркеры самоанализа и врата качества.
Поддерживает флаг --frontend-verify для визуальной проверки браузера/приложения/CLI.

Используйте, когда:
- Пользователь просит реализовать функцию, исправить ошибку или выполнить рефакторинг кода.
- Пользователь предоставляет задачу, требующую изменения кода.
– Пользователь говорит «сделай это», «построй это», «исправь это», «добавь это».
- Любая работа по реализации, включающая редактирование кода.

Ключевые слова: задача, реализация, сборка, исправление, добавление, создание, рефакторинг, обновление, изменение.
