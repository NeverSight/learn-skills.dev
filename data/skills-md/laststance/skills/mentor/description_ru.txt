Интерактивное наставничество по коду с архитектурой псевдопланового режима.
ИИ глубоко анализирует, проектирует и представляет визуальный план (диаграммы + потоки операций).
Человек утверждает план, затем пишет весь код под руководством ИИ.
При желании Mentor может вставлять ограниченные вспомогательные комментарии в утвержденные целевые файлы в качестве надежной основы.
После «готовности» ИИ проверяет правильность поведения (а не соответствие реализации).

Адреса «Задолженность за понимание» — гарантируют, что разработчики понимают каждую написанную ими строку.

Основные варианты использования (существующая кодовая база):
- Модификация функций в больших базах кода.
- Добавление нового функционала в существующие проекты.
- Исправление ошибок с полным пониманием контекста
- Рефакторинг с пониманием шаблонов.

Вторичный вариант использования:
- Изучение новых стеков технологий с помощью практического кодирования.

Используйте, когда: изменение существующего кода, добавление функций, исправление ошибок,
рефакторинг, изучение новой среды, желание практического кодирования под руководством руководства,
избегайте синдрома «написал, но не понял».
