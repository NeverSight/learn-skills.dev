Interaktives Code-Mentoring mit Pseudo-Plan-Modus-Architektur.
KI analysiert, entwirft und präsentiert einen visuellen Entwurf (Diagramme + Sequenzflüsse).
Der Mensch genehmigt den Plan und schreibt dann den gesamten Code unter KI-Anleitung.
Der Mentor kann optional lokale Unterstützungskommentare als dauerhaftes Gerüst in genehmigte Zieldateien einfügen.
Nach „erledigt“ überprüft die KI die Verhaltenskorrektheit (nicht die Implementierungskonformität).

Behebt „Verständnisschulden“ – stellt sicher, dass Entwickler jede Zeile verstehen, die sie schreiben.

Primäre Anwendungsfälle (vorhandene Codebasis):
- Ändern von Funktionen in großen Codebasen
- Hinzufügen neuer Funktionen zu bestehenden Projekten
- Behebung von Fehlern mit vollständigem Kontextverständnis
- Refactoring mit Musterverständnis

Sekundärer Anwendungsfall:
- Erlernen neuer Tech-Stacks durch praktisches Programmieren

Wird verwendet, wenn: vorhandener Code geändert, Funktionen hinzugefügt, Fehler behoben werden,
Refactoring, Erlernen neuer Frameworks, angeleitetes praktisches Codieren wünschen,
Vermeiden Sie das „Ich habe es geschrieben, aber nicht verstanden“-Syndrom.

Schlüsselwörter: Mentor, geführte Codierung, ändern, Feature, Bugfix, Refactor,
vorhandene Codebasis, praktisch, Verständnis, Besitz, Lernen, Planmodus
