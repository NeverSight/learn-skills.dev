Mentorat de code interactif avec architecture en mode pseudo-Plan.
L'IA analyse, conçoit et présente en profondeur un plan visuel (diagrammes + flux de séquences).
L'humain approuve le plan, puis écrit tout le code avec les conseils de l'IA.
Mentor peut éventuellement injecter des commentaires d'assistance limités dans des fichiers cibles approuvés en tant qu'échafaudage durable.
Une fois « terminé », l’IA vérifie l’exactitude du comportement (et non la conformité de la mise en œuvre).

Résout la « dette de compréhension » - garantit que les développeurs comprennent chaque ligne qu’ils écrivent.

Cas d'utilisation principaux (base de code existante) :
- Modification de fonctionnalités dans de grandes bases de code
- Ajout de nouvelles fonctionnalités aux projets existants
- Correction de bugs avec une compréhension complète du contexte
- Refactoring avec compréhension de modèles

Cas d'utilisation secondaire :
- Apprendre de nouvelles piles technologiques avec un codage pratique

À utiliser lorsque : modifier du code existant, ajouter des fonctionnalités, corriger des bugs,
refactoring, apprentissage d'un nouveau framework, souhait d'un codage pratique guidé,
évitez le syndrome du « je l’ai écrit mais je ne le comprends pas ».

Mots-clés : mentor, codage guidé, modification, fonctionnalité, correction de bugs, refactorisation,
base de code existante, pratique, compréhension, appropriation, apprentissage, mode plan
