عمليات Git مع رسائل التزام ذكية وتحسين سير العمل.
يحلل التغييرات لإنشاء رسائل الالتزام التقليدية تلقائيًا.
يغطي عمليات الحالة والالتزام والدفع والسحب والتفرع والدمج.

استخدم عندما:
- يطلب المستخدم الالتزام أو الدفع أو التحقق من حالة git
- يريد المستخدم إنشاء أو تبديل الفروع
- يحتاج المستخدم إلى مساعدة في الدمج أو حل الصراع
- يقول المستخدم "ارتكب هذا"، "ادفع"، "ما الذي تغير"، "إنشاء فرع"
- أي عملية سير عمل git

الكلمات الرئيسية: git، الالتزام، الدفع، السحب، الحالة، الفرع، الدمج، rebase، stash، log، diff
