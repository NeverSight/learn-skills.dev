Fachkundige Anleitung zur Entwicklung von Progressive Web Apps (PWA) – Installierbarkeit, Servicemitarbeiter, Offline-Funktionalität und natives UX. Verwenden Sie diesen Skill immer dann, wenn der Benutzer PWA, Service Worker, Web-App-Manifest, manifest.json, Offline-First, installierbare Web-App, Zum Startbildschirm hinzufügen, Beforeinstallprompt, App-Shell, Workbox, Cache-Strategien oder Push-Benachrichtigungen für Web-Apps erwähnt. Wird auch ausgelöst, wenn der Benutzer möchte, dass seine Website offline funktioniert, sich wie eine native App anfühlt oder installierbar ist – auch wenn er den Begriff „PWA“ nicht explizit verwendet. Lösen Sie auf Phrasen wie „Ohne Internet funktionieren“, „Können Benutzer dies installieren“, „Offline-Modus“, „Startbildschirmsymbol“ oder „Warum fühlt sich meine App wie eine Website aus“ aus.
