[内容] GitHub リポジトリ、ビデオ、記事、PDF、スキル パッケージ用の構成主導型コンテンツ取り込みシステム。
[方法] フェーズ 0 では、`~/.config/lev/config.yaml` および `<project>/.lev/config.yaml` からのワークショップ オーバーレイを解決します。フェーズ 1 では、解決されたワークショップ ルートにコンテンツを取得します。フェーズ 2 は、設定された Playbook またはプロジェクト ドキュメントに対して分析します。フェーズ 3 では、インテーク レポートを作成し、統合/抽出/監視/パス/ベンダーを推奨します。
[WHEN] ユーザーが分析する URL を提供する場合、「摂取/ダウンロード」と言う場合、社外の作業を評価したい場合、またはスキル パッケージを参照する場合に使用します。
[理由] ハードコードされた前提ではなく、構成を介して従来の `~/lev/workshop` フローを維持しながら、デフォルトでマルチプロジェクトを取り込むようにします。

トリガー: 「取り込み」、「ダウンロード」、「この URL を分析」、「このリポジトリをチェックアウト」、「このビデオをレビュー」、「コンテンツを評価」、「スキルのインストール」、「スキル://」
