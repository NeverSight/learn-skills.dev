[内容] 用于 GitHub 存储库、视频、文章、PDF 和技能包的配置驱动的内容摄取系统。
[如何] 第 0 阶段解决了 `~/.config/lev/config.yaml` 和 `<project>/.lev/config.yaml` 的研讨会覆盖。第 1 阶段将内容获取到已解析的研讨会根中。第 2 阶段根据配置的剧本或项目文档进行分析。第 3 阶段编写摄入报告并建议集成/提取/监控/通过/供应商。
[何时] 当用户提供要分析的 URL、说“获取/下载”、想要评估外部工作或引用技能包时使用。
[为什么] 默认情况下引入多项目，同时保留传统的“~/lev/workshop”流通过配置而不是硬编码的假设。

触发器：“摄取”、“下载”、“分析此网址”、“查看此存储库”、“查看此视频”、“评估内容”、“安装技能”、“skill://”
