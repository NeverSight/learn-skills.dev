[ЧТО] Система приема контента на основе настроек для репозиториев GitHub, видео, статей, PDF-файлов и пакетов навыков.
[КАК] На этапе 0 устраняются наложения мастерской из `~/.config/lev/config.yaml` и `<project>/.lev/config.yaml`. На этапе 1 контент попадает в разрешенный корень мастерской. На этапе 2 проводится анализ настроенных сценариев или документации проекта. На этапе 3 составляется отчет о поступлении и рекомендуются интеграция/извлечение/мониторинг/передача/поставщик.
[КОГДА] Используйте, когда пользователь предоставляет URL-адрес для анализа, говорит «загрузить/загрузить», хочет оценить внешнюю работу или ссылается на пакет навыков.
[ПОЧЕМУ] Делает вход мультипроектным по умолчанию, сохраняя при этом устаревшие потоки `~/lev/workshop` через конфигурацию вместо жестко запрограммированных предположений.
