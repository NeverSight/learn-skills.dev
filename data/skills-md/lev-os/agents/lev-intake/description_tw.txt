[內容] 用於 GitHub 儲存庫、影片、文章、PDF 和技能套件的配置驅動的內容攝取系統。
[如何] 第 0 階段解決了 `~/.config/lev/config.yaml` 和 `<project>/.lev/config.yaml` 的研討會涵蓋。階段 1 將內容取得到已解析的研討會根。階段 2 根據配置的劇本或專案文件進行分析。階段 3 編寫攝取報告並建議整合/提取/監控/通過/供應商。
[何時] 當使用者提供要分析的 URL、說「取得/下載」、想要評估外部工作或引用技能包時使用。
[為什麼] 預設引入多項目，同時保留傳統的“~/lev/workshop”流通過配置而不是硬編碼的假設。
