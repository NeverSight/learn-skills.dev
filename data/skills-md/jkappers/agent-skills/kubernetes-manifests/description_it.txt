Scrivi e rivedi manifest Kubernetes pronti per la produzione con sicurezza, gestione delle risorse e best practice operative. Da utilizzare quando (1) si creano nuovi manifest di distribuzione, servizio o Ingress di Kubernetes, (2) si esaminano i manifest esistenti per verificarne la preparazione alla produzione, (3) si aggiungono sonde di integrità, contesti di sicurezza o limiti di risorse, (4) si configura la struttura di base/overlay di Kustomize, (5) si containerizzano app .NET o React per Kubernetes o (6) si configura PodDisruptionBudgets, diffusione della topologia o scalabilità automatica.
