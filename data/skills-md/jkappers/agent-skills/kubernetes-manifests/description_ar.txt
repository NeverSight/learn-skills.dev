قم بكتابة ومراجعة بيانات Kubernetes الجاهزة للإنتاج مع الأمان وإدارة الموارد وأفضل الممارسات التشغيلية. يُستخدم عند (1) إنشاء بيانات نشر أو خدمة أو دخول جديدة لـ Kubernetes، (2) مراجعة البيانات الحالية للاستعداد للإنتاج، (3) إضافة تحقيقات السلامة أو سياقات الأمان أو حدود الموارد، (4) إعداد بنية Kubernetes الأساسية/التراكب، (5) تخزين تطبيقات .NET أو React لـ Kubernetes، أو (6) تكوين PodDisruptionBudgets، أو انتشار الهيكل، أو القياس التلقائي.
