Rufen Sie aktuelle lokale Serverprotokolle in Repos ab und überprüfen Sie sie, die Openlogs oder die „ol“-CLI verwenden. Verwenden Sie diese Option, wenn ein Benutzer fragt, was auf dem Server passiert ist, eine aktuelle Ausgabe des Entwicklungsservers wünscht, Startfehler oder Stack-Traces benötigt oder Sie auffordert, Backend-Protokolle aus „openlogs tail“, befehlsspezifischen Protokollen oder „.openlogs/latest.txt“ zu überprüfen.
