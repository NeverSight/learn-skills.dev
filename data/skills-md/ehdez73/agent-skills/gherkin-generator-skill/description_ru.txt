**Преобразует функциональные требования в готовые к использованию сценарии Gherkin (BDD)** посредством автоматизированной итерации с двумя агентами. Генератор создает файлы `.feature`; рецензент проверяет охват, структуру и ясность — фиксируется автоматически до тех пор, пока не будет одобрено (оценка ≥85).
**ОБЯЗАТЕЛЬНО**: четкие функциональные требования + бизнес-контекст. Не хватает этих? Спросите явно.
**ИСПОЛЬЗУЙТЕ ЭТОТ НАВЫК**, когда: - Пользователь предоставляет функциональные требования + просит «сгенерировать корнишоны», «написать сценарии BDD», «создать тесты функций» - У пользователя есть документ с требованиями и ему нужны исполняемые критерии приемки/тестовые примеры - Пользователь упоминает преобразование спецификаций в формат «Дано/Когда/Тогда» – Пользователю нужны тестовые сценарии, охватывающие счастливые пути И обработку ошибок
**ДОСТАВКА**: синтаксически допустимый, полный требования файл `.feature` со 100 % функциональным охватом, четкими шагами «Дано/Когда/Тогда» (без связи с пользовательским интерфейсом), независимыми сценариями и соответствующими тегами.
