Facilite una conversación estructurada de 3 Amigos entre Producto, Desarrollo y Control de Calidad para definir la funcionalidad antes de la implementación, luego extraiga de la discusión los requisitos funcionales listos para la implementación.
Utilice esta habilidad siempre que el usuario mencione 3 Amigos, refinamiento del trabajo pendiente con Producto/Desarrollo/QA, defina requisitos antes de codificar, aclare el alcance y los criterios de aceptación, revise una transcripción de discusión de requisitos o convierta una discusión de características en requisitos de estilo de historia de usuario. Úselo también cuando el usuario solicite identificar ambigüedades, suposiciones, reglas comerciales o preguntas abiertas antes de que comience el desarrollo.
