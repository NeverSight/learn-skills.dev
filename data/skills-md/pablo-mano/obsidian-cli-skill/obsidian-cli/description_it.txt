Utilizza questa abilità ogni volta che l'utente desidera che Claude interagisca direttamente con il proprio caveau Obsidian: leggere una nota o una nota giornaliera, scrivere o aggiungere contenuti, cercare contenuti del caveau, contare o elencare note, gestire attività, spostare o rinominare file, trovare note orfane o collegamenti interrotti. Senza questa competenza, Claude non ha modo di accedere ai dati del deposito o di eseguire operazioni del deposito. Tratta qualsiasi richiesta che implichi "entra nel mio caveau e fai X" come un fattore scatenante: l'utente sta chiedendo a Claude di agire, non di spiegare. Attiva anche l'automazione del vault, lo scripting CLI o i flussi di lavoro basati su cron che coinvolgono Obsidian, la gestione della cronologia di sincronizzazione, l'esecuzione di query su Bases, il ripristino delle versioni dei file tramite la cronologia, la gestione dei segnalibri o l'esecuzione di JavaScript sull'API Obsidian. Salta per domande puramente concettuali: come funziona la GUI di Obsidian, navigazione nei menu delle impostazioni, installazione di temi o plug-in tramite l'interfaccia utente, conflitti di sincronizzazione iCloud/di terze parti, sintassi generale delle query di Dataview, scorciatoie da tastiera o analisi di file del vault con script esterni: tutto ciò in cui l'utente ha bisogno di una spiegazione piuttosto che Claude che esegue un'operazione del vault.
