ユーザーがクロードに Obsidian Vault と直接対話させたいときは常にこのスキルを使用します。メモや毎日のメモの読み取り、コンテンツの書き込みまたは追加、Vault のコンテンツの検索、メモのカウントまたはリストの作成、タスクの管理、ファイルの移動または名前変更、孤立したメモや壊れたリンクの検索などです。このスキルがなければ、クロードはボールトのデータにアクセスしたり、ボールトの操作を実行したりすることができません。 「金庫に入って X を実行する」ことを意味するリクエストはすべてトリガーとして扱います。ユーザーはクロードに説明するのではなく、行動するよう求めています。また、Obsidian に関連するボールトの自動化、CLI スクリプト、または cron ベースのワークフロー、同期履歴の管理、ベースのクエリ、履歴によるファイル バージョンの復元、ブックマークの管理、または Obsidian API に対する JavaScript の実行もトリガーします。純粋に概念的な質問はスキップしてください。Obsidian の GUI の仕組み、設定メニューの操作、UI を介したテーマやプラグインのインストール、iCloud/サードパーティの同期の競合、一般的な Dataview クエリ構文、キーボード ショートカット、または外部スクリプトを使用した Vault ファイルの解析など、クロードが Vault 操作を実行するのではなく、ユーザーに説明が必要な場合はすべてスキップしてください。
