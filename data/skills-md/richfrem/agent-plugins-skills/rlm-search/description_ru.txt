Трехэтапная стратегия поиска знаний для экосистемы RLM Factory. Вызывается автоматически, когда задачи включают поиск кода, документации или контекста архитектуры в репозитории. Обеспечивает оптимальный порядок поиска: Сводное сканирование RLM (O(1)) -> Семантический поиск векторной базы данных -> Grep/Точное совпадение. Никогда не пропускайте фазы.
