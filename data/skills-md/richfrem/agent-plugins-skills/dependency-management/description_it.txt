Dipendenza Python e gestione dell'ambiente per backend Python multiservizio o monorepo. Da utilizzare quando: (1) si aggiunge, si aggiorna o si rimuove un pacchetto Python, (2) si risponde a Dependabot o agli avvisi di vulnerabilità della sicurezza (GHSA/CVE), (3) si crea un nuovo servizio che necessita dei propri file di requisiti, (4) si esegue il debug di errori di installazione di pip o di problemi di creazione di Docker relativi alle dipendenze, (5) si rivede o si controlla l'albero delle dipendenze, (6) si esegue pip-compile. Applica il flusso di lavoro con file bloccati di compilazione pip e la gerarchia delle dipendenze a più livelli.
