マルチサービスまたはモノリポの Python バックエンドの Python 依存関係と環境管理。次の場合に使用します: (1) Python パッケージの追加、アップグレード、または削除、(2) dependabot またはセキュリティ脆弱性アラート (GHSA/CVE) への対応、(3) 独自の要件ファイルを必要とする新しいサービスの作成、(4) pip インストールの失敗または依存関係に関連する Docker ビルドの問題のデバッグ、(5) 依存関係ツリーの確認または監査、(6) pip-compile の実行。 pip-compile ロックされたファイルのワークフローと階層化された依存関係階層を強制します。
