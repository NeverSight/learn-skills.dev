Leitfaden zur Software-Design-Philosophie basierend auf John Ousterhouts „A Philosophy of Software Design“. Nutzen Sie diese Fähigkeit bei: Codeüberprüfungen, Architekturdiskussionen, API-Entwurf, Modulzerlegungsentscheidungen, Refactoring-Anleitungen, Komplexitätsanalyse, Benennungs- und Kommentarverbesserungen, Entwurf von Fehlerbehandlungsstrategien. Wird ausgelöst, wenn der Benutzer „Code ist zu komplex“, „So teilen Sie Module auf“, „Schnittstellendesign“, „Kopplung reduzieren“, „tiefe/flache Module“, „Informationsverlust“, „Fehlerbehandlung“, „Lesbarkeit des Codes“, „Designphilosophie“, „Komplexität reduzieren“, „Fehler aus der Existenz definieren“ oder ähnliche Themen erwähnt. Wird auch für jede Codeüberprüfung ausgelöst, bei der Feedback zur Designqualität angefordert wird.
