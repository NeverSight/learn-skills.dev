Руководство по философии проектирования программного обеспечения, основанное на книге Джона Оустерхаута «Философия дизайна программного обеспечения». Используйте этот навык во время: проверок кода, обсуждений архитектуры, проектирования API, принятия решений о декомпозиции модулей, руководства по рефакторингу, анализа сложности, улучшения именования и комментирования, разработки стратегии обработки ошибок. Срабатывает, когда пользователь упоминает «код слишком сложен», «как разделить модули», «проектирование интерфейса», «уменьшить связанность», «глубокие/мелкие модули», «утечку информации», «обработку ошибок», «читабельность кода», «философию дизайна», «уменьшить сложность», «определить существование ошибок» или подобные темы. Также активируется для любой проверки кода, когда запрашивается отзыв о качестве дизайна.
