دليل فلسفة تصميم البرمجيات يعتمد على "فلسفة تصميم البرمجيات" لجون أوسترهوت. استخدم هذه المهارة أثناء: مراجعة التعليمات البرمجية، ومناقشات الهندسة المعمارية، وتصميم واجهة برمجة التطبيقات (API)، وقرارات تحليل الوحدة النمطية، وتوجيهات إعادة البناء، وتحليل التعقيد، وتحسينات التسمية والتعليق، وتصميم استراتيجية معالجة الأخطاء. يتم تشغيله عندما يذكر المستخدم "التعليمات البرمجية معقدة للغاية"، أو "كيفية تقسيم الوحدات النمطية"، أو "تصميم الواجهة"، أو "تقليل الاقتران"، أو "الوحدات العميقة/السطحية"، أو "تسرب المعلومات"، أو "معالجة الأخطاء"، أو "قابلية قراءة التعليمات البرمجية"، أو "فلسفة التصميم"، أو "إزالة التعقيد"، أو "تحديد الأخطاء وإزالتها من الوجود"، أو مواضيع مماثلة. قم أيضًا بتشغيل أي مراجعة للتعليمات البرمجية حيث يتم طلب تعليقات جودة التصميم.
