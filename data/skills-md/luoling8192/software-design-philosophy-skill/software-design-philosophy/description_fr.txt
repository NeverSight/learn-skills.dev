Guide de philosophie de conception de logiciels basé sur « Une philosophie de conception de logiciels » de John Ousterhout. Utilisez cette compétence pendant : les révisions de code, les discussions sur l'architecture, la conception d'API, les décisions de décomposition de modules, les conseils de refactoring, l'analyse de la complexité, les améliorations de noms et de commentaires, la conception de stratégies de gestion des erreurs. Déclenchez lorsque l'utilisateur mentionne « le code est trop complexe », « comment diviser les modules », « conception d'interface », « réduire le couplage », « modules profonds/superficiels », « fuite d'informations », « gestion des erreurs », « lisibilité du code », « philosophie de conception », « réduire la complexité », « définir les erreurs hors de l'existence » ou des sujets similaires. Déclenchez également toute révision de code pour laquelle des commentaires sur la qualité de la conception sont demandés.
