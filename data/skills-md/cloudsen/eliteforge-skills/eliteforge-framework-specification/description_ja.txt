Radiant Workshop の統合フレームワークに基づく Java 開発では、その機能の再利用を優先し、車輪の再発明を避ける必要があります。統合フレームワークへのアクセス、アップグレード、構成、スターターの選択、およびトラブルシューティングのガイダンスについては、「Radiant Workshop 統合フレームワーク – 使用マニュアル (夜間)」に従ってください。フレームワーク機能が適用される場合は、内部スターターとその抽象 API を使用します。 Spring の公式スターターに戻すこと、またはフレームワーク抽象化レイヤーをバイパスすることは禁止されています (例: CacheService の代わりに StringRedisTemplate をキャッシュに直接使用する)。ユーザーが「統合フレームワーク」、「スターター リスト」、「アクセス ステップ」、「アップグレード ステップ」、「システム要件」、「Web MVC / サービス間呼び出し / MyBatis-Plus / 暗号化-復号化 / キャッシング / マルチデータソース / コード ジェネレーター / リソース抽象化レイヤー」について言及する場合、または「特定のスターター (キャッシュ スターターなど) を直接使用するかどうか」を決定する必要がある場合に、これを使用します。
