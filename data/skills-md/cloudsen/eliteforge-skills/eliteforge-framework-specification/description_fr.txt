Le développement Java sous le framework unifié Radiant Workshop doit donner la priorité à la réutilisation de ses capacités et éviter de réinventer la roue. Suivez le « Radiant Workshop Unified Framework – Manuel d'utilisation – tous les soirs » pour obtenir des conseils sur l'intégration du framework, les mises à niveau, la configuration, la sélection du démarreur et le dépannage. Chaque fois qu'une fonctionnalité du framework s'applique, utilisez le démarreur intégré et ses API abstraites ; il est interdit de revenir aux démarreurs officiels de Spring ou de contourner la couche d'abstraction du framework (par exemple, utiliser StringRedisTemplate directement au lieu de CacheService pour la mise en cache). Invoquez cette règle lorsque les utilisateurs mentionnent « framework unifié », « liste de démarrage », « étapes d'intégration », « étapes de mise à niveau », « configuration système requise », « Web MVC / appels inter-services / MyBatis-Plus / cryptage et déchiffrement / mise en cache / multi-sources de données / générateur de code / couche d'abstraction de ressources » ou pour juger « si une fonctionnalité doit utiliser directement un démarreur spécifique (par exemple, un démarreur de cache). »
