يجب أن يعطي تطوير Java ضمن Constrain الأولوية لإعادة استخدام إمكانات Unified Framework الخاصة بـ CuiCan Workshop وتجنب إعادة اختراع العجلة. اتبع "إطار عمل CuiCan الموحد - دليل المستخدم - ليلاً" للحصول على إرشادات حول إعداد إطار العمل، والترقيات، والتكوين، واختيار المبتدئين، واستكشاف الأخطاء وإصلاحها. عندما يتم تطبيق إمكانية إطار العمل، يجب عليك استخدام واجهة برمجة التطبيقات المبدئية والملخصة المقابلة داخل الإطار؛ يُحظر الرجوع إلى البداية الرسمية لـ Spring أو تجاوز طبقة تجريد إطار العمل (على سبيل المثال، في سيناريوهات التخزين المؤقت، لا تتجاوز CacheService واستخدم StringRedisTemplate مباشرةً). عندما يذكر المستخدمون "إطار العمل الموحد"، أو "قائمة المبتدئين"، أو "خطوات الإعداد"، أو "خطوات الترقية"، أو "متطلبات النظام"، أو "Web MVC / المكالمات بين الخدمات / MyBatis-Plus / التشفير وفك التشفير / التخزين المؤقت / مصدر بيانات متعدد / مولد التعليمات البرمجية / طبقة تجريد الموارد"، أو يحتاجون إلى تحديد "ما إذا كان يجب أن تستخدم الميزة بشكل مباشر بداية معينة (على سبيل المثال، بداية ذاكرة التخزين المؤقت)،" قم بتطبيق هذه القاعدة.
