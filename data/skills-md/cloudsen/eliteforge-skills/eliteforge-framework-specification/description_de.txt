Java-Entwickler müssen der Wiederverwendung der einheitlichen Framework-Funktionen von Brilliance Workshop Priorität einräumen und vermeiden, das Rad neu zu erfinden. Weitere Informationen zu Onboarding, Upgrades, Konfiguration, Starterauswahl und Fehlerbehebung für das Unified Framework finden Sie im „Brilliance Workshop Unified Framework – Benutzerhandbuch – nightly“. Wenn eine Framework-Funktion gilt, verwenden Sie die entsprechende Starter- und Abstract-API innerhalb des Frameworks. Das Zurückgreifen auf den offiziellen Starter von Spring oder das Umgehen der Framework-Abstraktionsschicht ist verboten (z. B. das Umgehen von CacheService und die direkte Verwendung von StringRedisTemplate in Caching-Szenarien). Verwenden Sie dieses Handbuch, wenn Benutzer „einheitliches Framework“, „Starterliste“, „Onboarding-Schritte“, „Upgrade-Schritte“, „Systemanforderungen“, „Web MVC/Inter-Service-Aufrufe/MyBatis-Plus/Verschlüsselung und Entschlüsselung/Caching/Multi-Datenquelle/Codegenerator/Ressourcenabstraktionsschicht“ erwähnen oder wenn sie bestimmen, „ob eine Funktion direkt einen bestimmten Starter (z. B. Cache-Starter) verwenden soll“.
