Los desarrolladores de Java deben priorizar la reutilización de las capacidades del marco unificado de Brilliant Workshop y evitar reinventar la rueda. Siga el "Marco unificado de Brilliant Workshop – Manual del usuario – todas las noches" para obtener orientación sobre la integración del marco, actualizaciones, configuración, selección de inicio y solución de problemas. Siempre que se aplique una capacidad del marco, utilice el iniciador integrado y su API abstracta; Está prohibido volver al iniciador oficial de Spring o pasar por alto la capa de abstracción del marco (por ejemplo, no use StringRedisTemplate directamente en lugar de CacheService en escenarios de almacenamiento en caché). Utilice este manual siempre que el usuario mencione "marco unificado", "lista de inicio", "pasos de integración", "pasos de actualización", "requisitos del sistema", "Web MVC / llamadas entre servicios / MyBatis-Plus / cifrado-descifrado / almacenamiento en caché / fuente de datos múltiples / generador de código / capa de abstracción de recursos", o necesite decidir "si una característica debe usar directamente un iniciador específico (por ejemplo, iniciador de caché)".
