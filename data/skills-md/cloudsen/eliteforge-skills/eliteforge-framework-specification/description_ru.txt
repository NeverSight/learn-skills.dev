При разработке на Java в соответствии с мандатом «Ограничения» отдавайте приоритет повторному использованию возможностей единой среды CuiCan-Workshop и избегайте изобретения велосипеда. В своей работе используйте «Унифицированную платформу CuiCan-Workshop — Руководство пользователя (ежедневно)» для ознакомления с унифицированной платформой, ее обновлений, настройки, выбора начального уровня и рекомендаций по устранению неполадок. Всякий раз, когда применяется возможность платформы, вы должны использовать соответствующий стартер и его абстрактный API; возврат к официальному стартеру Spring или обход уровня абстракции платформы запрещены (например, в сценариях кэширования не обходите CacheService и не используйте StringRedisTemplate напрямую). Вызовите это правило всякий раз, когда пользователь упоминает «унифицированную структуру», «стартовый список», «этапы адаптации», «шаги обновления», «системные требования», «Web MVC / межсервисные вызовы / MyBatis-Plus / шифрование-дешифрование / кэширование / несколько источников данных / генератор кода / уровень абстракции ресурсов» или ему нужно решить, «стоит ли использовать конкретный начальный элемент напрямую (например, кэш-стартер)» для функции.
