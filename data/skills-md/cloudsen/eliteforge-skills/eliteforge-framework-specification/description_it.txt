Lo sviluppo Java nell'ambito del framework unificato Radiant Workshop deve dare priorità al riutilizzo delle sue capacità ed evitare di reinventare la ruota. Segui il "Radiant Workshop Unified Framework - Manuale dell'utente (serale)" per l'onboarding del framework, gli aggiornamenti, la configurazione, la selezione dello starter e le indicazioni per la risoluzione dei problemi. Ogni volta che si applica una funzionalità del framework, utilizzare lo starter corrispondente e l'API astratta all'interno del framework; è vietato ripristinare gli avviatori ufficiali di Spring o ignorare il livello di astrazione del framework (ad esempio, negli scenari di memorizzazione nella cache, non ignorare CacheService e utilizzare direttamente StringRedisTemplate). Applica questa regola ogni volta che gli utenti menzionano "framework unificato", "elenco di avvio", "passaggi di onboarding", "passaggi di aggiornamento", "requisiti di sistema", "Web MVC / chiamate interservizi / MyBatis-Plus / crittografia e decrittografia / caching / origine multi-dati / generatore di codice / livello di astrazione delle risorse" o quando determinano "se una funzionalità deve utilizzare direttamente uno starter specifico (ad esempio, cache-starter)".
