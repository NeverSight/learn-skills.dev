Ejecute un canal de revisión de errores estructurado y adversario de múltiples agentes en una base de código. Utilice esta habilidad siempre que el usuario quiera encontrar errores, auditar la calidad del código, revisar una base de código en busca de problemas o ejecutar cualquier tipo de flujo de trabajo de análisis de código o búsqueda de errores. También se activa cuando el usuario solicita "revisar mi código en busca de errores", "encontrar todos los problemas en este repositorio", "auditar esta base de código" o cualquier solicitud similar. El proceso utiliza tres fases secuenciales: un buscador de errores que maximiza el descubrimiento de problemas, un adversario de errores que desafía los falsos positivos y un árbitro que emite veredictos finales, lo que produce un informe de errores limpio y de alta confianza.
