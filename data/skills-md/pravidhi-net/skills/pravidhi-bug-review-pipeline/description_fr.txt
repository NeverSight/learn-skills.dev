Exécutez un pipeline d’examen des bogues multi-agents structuré et contradictoire sur une base de code. Utilisez cette compétence chaque fois que l'utilisateur souhaite rechercher des bogues, auditer la qualité du code, examiner une base de code à la recherche de problèmes ou exécuter tout type de flux de travail de recherche de bogues ou d'analyse de code. Se déclenche également lorsque l'utilisateur demande de « vérifier mon code pour détecter les bogues », « rechercher tous les problèmes dans ce dépôt », « auditer cette base de code » ou toute demande similaire. Le pipeline utilise trois phases séquentielles : un outil de recherche de bogues qui maximise la découverte des problèmes, un adversaire de bogues qui conteste les faux positifs et un arbitre qui émet des verdicts finaux, produisant ainsi un rapport de bogues clair et de haute confiance.
