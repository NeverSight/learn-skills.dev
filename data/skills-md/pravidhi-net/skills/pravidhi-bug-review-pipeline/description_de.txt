Führen Sie eine strukturierte, kontroverse Multi-Agent-Bug-Review-Pipeline auf einer Codebasis aus. Verwenden Sie diese Fähigkeit immer dann, wenn der Benutzer Fehler finden, die Codequalität prüfen, eine Codebasis auf Probleme überprüfen oder einen Workflow zur Fehlersuche oder Codeanalyse ausführen möchte. Wird auch ausgelöst, wenn der Benutzer „Meinen Code auf Fehler überprüfen“, „Alle Probleme in diesem Repo finden“, „Diese Codebasis prüfen“ oder eine ähnliche Anfrage stellt. Die Pipeline verwendet drei aufeinanderfolgende Phasen: einen Bug Finder, der die Problemerkennung maximiert, einen Bug Adversary, der Fehlalarme bekämpft, und einen Arbiter, der endgültige Urteile fällt – und so einen sauberen, hochzuverlässigen Bugbericht erstellt.
