在代码库上运行结构化的、对抗性的多代理错误审查管道。 Use this skill whenever the user wants to find bugs, audit code quality, review a codebase for issues, or run any kind of bug-finding or code analysis workflow.当用户要求“检查我的代码是否有错误”、“查找此存储库中的所有问题”、“审核此代码库”或任何类似请求时也会触发。该管道使用三个连续的阶段：最大限度地发现问题的 Bug Finder、挑战误报的 Bug Adversary 以及发布最终裁决的仲裁器 - 生成干净、高可信度的 Bug 报告。
