# 分镜脚本设计技能

## 核心原则

**分镜脚本是整个视频制作的基础，必须在录制前获得用户明确确认。**

## 分镜脚本要素

### 1. 场景定义

每个场景必须包含：

```json
{
  "scene_id": "场景唯一标识",
  "scene_name": "场景名称",
  "duration": "时长（秒）",
  "start_time": "开始时间",
  "end_time": "结束时间",
  "description": "画面描述",
  "operations": ["操作步骤列表"],
  "voiceover": "配音文字",
  "notes": "备注/特殊要求"
}
```

### 2. 标准场景类型

| 类型 | 建议时长 | 用途 |
|------|----------|------|
| opening | 8-12秒 | 片头动画 |
| features | 6-10秒 | 功能亮点展示 |
| demo | 根据内容 | 功能演示录屏 |
| closing | 8-12秒 | 片尾动画 |

### 3. 配音文字要求

- 每段配音控制在 4-10 秒
- 中文语速约 4 字/秒
- 配音时长 ≤ 场景时长 - 0.5秒（留缓冲）
- 避免长句，使用短句更易控制节奏

## 分镜脚本模板

```markdown
# [产品名] 产品介绍视频分镜脚本

## 视频概要
- 目标时长: X分X秒
- 目标受众: [描述]
- 核心信息: [一句话]

## 场景列表

### 场景1: 片头 (0:00-0:10)
**画面**: Logo动画 + 产品名称 + 副标题
**配音**: "欢迎使用 [产品名]，[一句话定位]"

### 场景2: 功能亮点 (0:10-0:18)
**画面**: 功能卡片依次展示
**配音**: "我们为您提供[功能概述]"

### 场景3: [功能名] 演示 (0:18-0:XX)
**画面**: [具体操作描述]
**操作**:
1. [步骤1]
2. [步骤2]
**配音**: "[功能说明]"

### 场景N: 片尾 (X:XX-X:XX)
**画面**: Logo + 口号 + 行动按钮
**配音**: "[口号]，立即开始使用吧！"

## 时间线汇总

| 场景 | 开始 | 结束 | 时长 | 配音字数 |
|------|------|------|------|----------|
| 片头 | 0:00 | 0:10 | 10秒 | 20字 |
| ... | ... | ... | ... | ... |
```

## 分镜确认检查清单

在请求用户确认前，确保：

- [ ] 所有场景时间线连续无缝隙
- [ ] 配音字数与场景时长匹配（4字/秒）
- [ ] 每个演示场景的操作步骤清晰可执行
- [ ] 配音文字专业、简洁、无歧义
- [ ] 总时长在目标范围内

## 用户确认流程

1. 展示完整分镜脚本
2. 说明关键时间点和内容
3. 询问是否需要调整
4. 获得明确的"确认开始制作"指令
5. 记录确认时间，开始录制

## 常见问题处理

### 用户要求修改分镜

1. 记录修改内容
2. 更新分镜脚本
3. 重新展示修改后的版本
4. 再次确认

### 时长超出预期

- 优先精简配音文字
- 其次减少演示操作步骤
- 最后考虑删除非核心场景

### 配音文字过长

计算公式：`配音字数 ≤ 场景时长(秒) × 4`

如果超出，需要：
1. 精简文字
2. 或加快语速（最多 +35%）
3. 或延长场景时长
