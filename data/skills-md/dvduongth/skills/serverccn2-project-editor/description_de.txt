Redakteur und Projektmanager für den CCN2-Gameserver (serverccn2/). Verwenden Sie diese Fertigkeit immer dann, wenn der Benutzer Serverkonfigurationen verwalten, Bereitstellungsumgebungen erstellen oder ändern, Serverarchitektur überprüfen, technische Serverdokumentation erstellen oder aktualisieren, Serverfunktionsideen vor dem Codieren bearbeiten, Konsistenz zwischen Serverdesigndokumenten und Quellcode überprüfen, Servereigenschaften umgebungsübergreifend verwalten, Konfigurationen auf Produktionsbereitschaft prüfen oder neue Servermodule/-funktionen planen möchte. Wird auch ausgelöst, wenn der Benutzer Folgendes erwähnt: „Serverkonfiguration“, „Servereigenschaften“, „Umgebung erstellen“, „Konfiguration bereitstellen“, „Serverarchitektur“, „Server-Tech-Dokument“, „Server-Scan“, „Konfigurationsüberprüfung“, „Umgebungssetup“, „dao_type“, „db_prefix_key“, „configByMode“ oder jede Anfrage, die serverccn2/Projektmanagement betrifft. Diese Fähigkeit erzwingt einen Design-First-Workflow: Die Dokumentation wird immer aktualisiert, bevor Code geschrieben wird. Jede Befehlsausgabe wird über „validate_result“ automatisch validiert, um die Richtigkeit sicherzustellen, bevor den Ergebnissen vertraut wird.
