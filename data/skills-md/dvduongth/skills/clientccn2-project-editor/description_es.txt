Editor y gestor de proyectos para el cliente del juego CCN2 (clientccn2/). Utilice esta habilidad siempre que el usuario desee: administrar módulos de cliente, agregar nuevas acciones o efectos de juego, planificar o diseñar funciones del lado del cliente, generar o actualizar documentación técnica del cliente, editar ideas de funciones del cliente antes de codificar, verificar la coherencia entre el código del cliente y el servidor/GDD, agregar nuevas escenas o componentes de la interfaz de usuario, administrar el sistema de cola de acciones, trabajar con el bus de eventos, revisar o refactorizar la arquitectura del cliente (heredada → nueva), administrar cargadores de configuración o planificar nuevos paneles de interfaz de usuario. También se activa cuando el usuario menciona: "código de cliente", "módulo de cliente", "arquitectura de cliente", "cola de acciones", "ActionQueue", "BaseAction", "EventBus", "EventKeys", "BaseModule", "GameHUD", "SceneGame", "BaseScene", "BaseGUI", "lógica de juego", "escaneo de cliente", "documentación técnica del cliente", "Cocos2d-x", "cliente cocos", "configuración del cliente", "ConfigManager", "ServiceContainer", "AppContext", "NavigationCoordinator", "gv.bus", "gv.signalMgr", o cualquier solicitud que involucre clientccn2/gestión de proyectos. Esta habilidad impone un flujo de trabajo en el que el diseño es lo primero: la documentación siempre se actualiza antes de escribir el código. Cada salida de comando se valida automáticamente a través de `validate_result` para garantizar la corrección antes de confiar en los resultados.
