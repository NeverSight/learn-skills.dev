محرر ومدير مشروع لعميل لعبة CCN2 (clientccn2/). استخدم هذه المهارة عندما يريد المستخدم: إدارة وحدات العميل، أو إضافة إجراءات أو تأثيرات لعبة جديدة، أو تخطيط أو تصميم ميزات من جانب العميل، أو إنشاء أو تحديث الوثائق الفنية للعميل، أو تحرير أفكار ميزات العميل قبل البرمجة، أو التحقق من الاتساق بين رمز العميل والخادم/GDD، أو إضافة مشاهد جديدة أو مكونات واجهة المستخدم، أو إدارة نظام قائمة انتظار الإجراءات، أو العمل مع ناقل الأحداث، أو مراجعة بنية العميل أو إعادة تصميمها (القديمة → الجديدة)، أو إدارة أدوات تحميل التكوين، أو تخطيط لوحات واجهة مستخدم جديدة. يتم تشغيله أيضًا عندما يذكر المستخدم: "رمز العميل"، "وحدة العميل"، "بنية العميل"، "قائمة الانتظار"، "ActionQueue"، "BaseAction"، "EventBus"، "EventKeys"، "BaseModule"، "GameHUD"، "SceneGame"، "BaseScene"، "BaseGUI"، "game logic"، "client scan"، "client tech doc"، "Cocos2d-x"، "cocos" client)، أو "client config"، أو "ConfigManager"، أو "ServiceContainer"، أو "AppContext"، أو "NavigationCoordinator"، أو "gv.bus"، أو "gv.signalMgr"، أو أي طلب يتضمن إدارة مشروع Clientccn2/. تفرض هذه المهارة سير عمل التصميم أولاً: يتم تحديث الوثائق دائمًا قبل كتابة التعليمات البرمجية. يتم التحقق من صحة كل مخرجات أمر تلقائيًا عبر "validate_result" لضمان صحتها قبل الثقة في النتائج.
