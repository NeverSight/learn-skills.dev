Redakteur und Projektmanager für den CCN2-Spielclient (clientccn2/). Verwenden Sie diese Fähigkeit, wann immer der Benutzer: Client-Module verwalten, neue Spielaktionen oder -effekte hinzufügen, clientseitige Funktionen planen oder entwerfen, technische Client-Dokumentation erstellen oder aktualisieren, Ideen für Client-Funktionen vor dem Codieren bearbeiten, die Konsistenz zwischen Client-Code und Server/GDD prüfen, neue Szenen oder UI-Komponenten hinzufügen, das Aktionswarteschlangensystem verwalten, mit dem Event-Bus arbeiten, die Client-Architektur (alt → neu) überprüfen oder umgestalten möchte, Konfigurationslader verwalten oder neue UI-Panels planen möchte. Wird auch ausgelöst, wenn der Benutzer Folgendes erwähnt: „Client-Code“, „Client-Modul“, „Client-Architektur“, „Aktionswarteschlange“, „ActionQueue“, „BaseAction“, „EventBus“, „EventKeys“, „BaseModule“, „GameHUD“, „SceneGame“, „BaseScene“, „BaseGUI“, „Spielelogik“, „Client-Scan“, „Client-Tech-Dokument“, „Cocos2d-x“, „Cocos-Client“, „Client-Konfiguration“, „ConfigManager“, „ServiceContainer“, „AppContext“, „NavigationCoordinator“, „gv.bus“, „gv.signalMgr“ oder jede Anfrage, die clientccn2/Projektmanagement betrifft. Diese Fähigkeit erzwingt einen Design-First-Workflow: Die Dokumentation wird immer aktualisiert, bevor Code geschrieben wird. Jede Befehlsausgabe wird über „validate_result“ automatisch validiert, um die Richtigkeit sicherzustellen, bevor den Ergebnissen vertraut wird.
