كبير مهندسي الألعاب ومحرر المشروع لمستودع الألعاب. استخدم هذه المهارة عندما يريد المستخدم: تخطيط أو تصميم ميزة جديدة للعبة، أو تحرير أفكار اللعبة قبل البرمجة، أو تحديث مستند تصميم اللعبة (GDD)، أو إنشاء وثائق فنية أو تحديثها، أو التحقق من الاتساق بين مستندات التصميم والتعليمة البرمجية المصدر، أو إعادة بناء قاعدة التعليمات البرمجية بناءً على تغييرات التصميم، أو إنشاء تعليمات برمجية من تصميم معتمد. يتم تشغيله أيضًا عندما يذكر المستخدم: "مشروع المسح"، أو "إنشاء مستند تقني"، أو "تحرير الفكرة"، أو "تحديث GDD"، أو "التحقق من اتساق التصميم"، أو "إنشاء تعليمات برمجية من التصميم"، أو "قاعدة تعليمات إعادة البناء"، أو مراجعة البنية، أو سير عمل التصميم أولاً، أو أي طلب يتضمن تغيير آليات اللعبة أو القواعد أو بنية المشروع. تفرض هذه المهارة سير عمل التصميم أولاً: يتم تحديث الوثائق دائمًا قبل كتابة التعليمات البرمجية. يتم التحقق من صحة كل مخرجات أمر تلقائيًا عبر "validate_result" لضمان صحتها قبل الثقة في النتائج.
