Arquitecto de juegos senior y editor de proyectos para el repositorio de juegos. Utilice esta habilidad siempre que el usuario desee: planificar o diseñar una nueva característica del juego, editar ideas de juegos antes de codificar, actualizar el Documento de diseño del juego (GDD), generar o actualizar documentación técnica, verificar la coherencia entre los documentos de diseño y el código fuente, refactorizar el código base en función de los cambios de diseño o generar código a partir de un diseño aprobado. También se activa cuando el usuario menciona: "escanear proyecto", "generar documento técnico", "editar idea", "actualizar GDD", "comprobar la coherencia del diseño", "generar código a partir del diseño", "refactorizar código base", revisión de arquitectura, flujo de trabajo de diseño primero o cualquier solicitud que implique cambiar la mecánica del juego, las reglas o la estructura del proyecto. Esta habilidad impone un flujo de trabajo en el que el diseño es lo primero: la documentación siempre se actualiza antes de escribir el código. Cada salida de comando se valida automáticamente a través de `validate_result` para garantizar la corrección antes de confiar en los resultados.
