Leitender Spielearchitekt und Projektredakteur für das Spiele-Repository. Verwenden Sie diese Fähigkeit, wann immer der Benutzer eine neue Spielfunktion planen oder entwerfen, Spielideen vor dem Codieren bearbeiten, das Game Design Document (GDD) aktualisieren, technische Dokumentation erstellen oder aktualisieren, die Konsistenz zwischen Designdokumenten und Quellcode überprüfen, die Codebasis basierend auf Designänderungen umgestalten oder Code aus einem genehmigten Design generieren möchte. Wird auch ausgelöst, wenn der Benutzer Folgendes erwähnt: „Projekt scannen“, „Technisches Dokument generieren“, „Idee bearbeiten“, „GDD aktualisieren“, „Designkonsistenz prüfen“, „Code aus Design generieren“, „Codebasis umgestalten“, Architekturüberprüfung, Design-First-Workflow oder jede Anfrage, die das Ändern von Spielmechaniken, Regeln oder Projektstruktur beinhaltet. Diese Fähigkeit erzwingt einen Design-First-Workflow: Die Dokumentation wird immer aktualisiert, bevor Code geschrieben wird. Jede Befehlsausgabe wird über „validate_result“ automatisch validiert, um die Richtigkeit sicherzustellen, bevor den Ergebnissen vertraut wird.
