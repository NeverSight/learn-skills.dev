標準の開発ツール、フック、ワークフローを使用して、任意のリポジトリを冪等にブートストラップします。新しいリポジトリで作業を開始する場合、既存のプロジェクトにオンボーディングする場合、またはリポジトリに適切な CI/CD セットアップがあることを確認する場合に使用します。構成: git フック (lefthook)、コミット メッセージ テンプレート、PR テンプレート、および lint/format/type-check/build の GitHub アクション。変更前にユーザーに確認を求めます。トリガー: 「ブートストラップ リポジトリ」、「フックのセットアップ」、「CI の構成」、「リグ」、「リポジトリの標準化」。
