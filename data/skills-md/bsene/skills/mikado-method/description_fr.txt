Appliquez et guidez la méthode Mikado lorsqu'un développeur refactorise, restructure ou traite du code existant. Utilisez cette compétence chaque fois que l'utilisateur mentionne la refactorisation, la dette technique, le code hérité, la restructuration du code, le démêlage des dépendances ou « tout casser » lors des modifications. Déclenchez également lorsque l'utilisateur souhaite effectuer une modification importante en toute sécurité, demande comment diviser une tâche de refactorisation importante, souhaite travailler sur une branche principale sans branche de fonctionnalités de longue durée ou demande comment améliorer progressivement une base de code. La compétence applique la boucle Mikado complète : objectif → tentative naïve → prérequis de la carte → revenir → implémenter les feuilles → valider → répéter.
