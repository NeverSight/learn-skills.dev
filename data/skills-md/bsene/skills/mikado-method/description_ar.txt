قم بفرض وتوجيه طريقة Mikado عندما يقوم المطور بإعادة البناء أو إعادة الهيكلة أو التعامل مع التعليمات البرمجية القديمة. استخدم هذه المهارة عندما يذكر المستخدم إعادة البناء، أو الديون الفنية، أو التعليمات البرمجية القديمة، أو إعادة هيكلة التعليمات البرمجية، أو فك التبعية، أو "كسر كل شيء" عند إجراء التغييرات. يتم تشغيله أيضًا عندما يريد المستخدم إجراء تغيير كبير بأمان، أو يسأل عن كيفية تقسيم مهمة إعادة هيكلة كبيرة، أو يريد العمل على الفرع الرئيسي دون فرع ميزات طويل الأمد، أو يسأل عن كيفية تحسين قاعدة التعليمات البرمجية بشكل تدريجي. تفرض المهارة حلقة ميكادو الكاملة: الهدف ← محاولة ساذجة ← خريطة المتطلبات الأساسية ← الرجوع ← تنفيذ الأوراق ← الالتزام ← التكرار.
