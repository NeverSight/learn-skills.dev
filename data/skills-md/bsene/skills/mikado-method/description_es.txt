Haga cumplir y guíe el Método Mikado cuando un desarrollador esté refactorizando, reestructurando o trabajando con código heredado. Utilice esta habilidad siempre que el usuario mencione refactorización, deuda técnica, código heredado, reestructuración de código, desenredado de dependencias o "romper todo" al realizar cambios. También se activa cuando el usuario quiere realizar un cambio importante de forma segura, pregunta cómo dividir una gran tarea de refactorización, quiere trabajar en la rama principal sin una rama de características de larga duración o pregunta cómo mejorar incrementalmente una base de código. La habilidad impone el ciclo completo de Mikado: objetivo → intento ingenuo → requisitos previos del mapa → revertir → implementar hojas → confirmar → repetir.
