Aplique el principio Tell Don't Ask (TDA) al revisar, escribir o refactorizar código orientado a objetos. Utilice esta habilidad siempre que el usuario pregunte sobre el diseño de programación orientada a objetos, mencione captadores/definidores, quiera revisar una clase para detectar problemas de encapsulación, pregunte cómo acercar la lógica a los datos o pregunte por qué el código se siente "procedimental" a pesar de usar clases. También se activa cuando el usuario solicita refactorizar el código que consulta el estado de un objeto antes de tomar decisiones externamente. Esta habilidad debería utilizarse para cualquier revisión de código o pregunta de diseño que involucre patrones de acceso a datos, encapsulación o cómo deberían colaborar los objetos.
