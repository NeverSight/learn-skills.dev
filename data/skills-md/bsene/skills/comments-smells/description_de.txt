Erkennen, erklären und beheben Sie „Kommentar“-Codefehler – Fälle, in denen übermäßige oder erklärende Kommentare darauf hinweisen, dass der zugrunde liegende Code einer Umgestaltung statt einer Annotation bedarf. Verwenden Sie diese Fähigkeit immer dann, wenn der Benutzer Code mit umfangreichen Kommentaren teilt, nach Best Practices für Kommentare fragt, eine Codeüberprüfung mit Schwerpunkt auf Lesbarkeit wünscht, fragt „Sind meine Kommentare zu umfangreich?“, Codegerüche erwähnt oder die Klarheit des Codes verbessern möchte. Wird auch ausgelöst, wenn der Benutzer Code einfügt und fragt, wie er sauberer, selbstdokumentierender oder verständlicher gestaltet werden kann – auch wenn er den „Kommentargeruch“ nicht ausdrücklich erwähnt.
