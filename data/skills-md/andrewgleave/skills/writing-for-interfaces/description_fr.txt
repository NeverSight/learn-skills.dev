À utiliser lorsque quelqu'un demande à écrire, réécrire, réviser ou améliorer le texte qui apparaît à l'intérieur d'un produit ou d'une interface. Exemples : "examiner la copie UX", "existe-t-il une meilleure façon de formuler cela", "réécrire ce message d'erreur", "écrire une copie pour cet écran/flux/page", examiner les étiquettes des boutons, améliorer les messages de sortie CLI, rédiger une copie d'intégration, des descriptions de paramètres ou des boîtes de dialogue de confirmation. Déclenchez chaque fois que la demande implique un libellé affiché aux utilisateurs finaux dans le logiciel : applications, Web, CLI, notifications par e-mail, modaux, info-bulles, états vides ou alertes. Déclenchez également des demandes vagues telles que « réviser l'UX » où la révision de la copie de l'interface est implicite. Ne déclenchez PAS le marketing de contenu, les articles de blog, les listes de magasins d'applications, les documents API, les guides de marque, les lettres de motivation ou les questions d'entretien : il s'agit d'une compétence de rédaction technique pour le langage de l'interface.
