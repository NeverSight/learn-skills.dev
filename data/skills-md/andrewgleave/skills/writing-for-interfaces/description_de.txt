Verwenden Sie diese Option, wenn jemand darum bittet, Text zu schreiben, neu zu schreiben, zu überprüfen oder zu verbessern, der in einem Produkt oder einer Benutzeroberfläche erscheint. Beispiele: „Überprüfen Sie die UX-Kopie“, „Gibt es eine bessere Möglichkeit, dies zu formulieren“, „Schreiben Sie diese Fehlermeldung neu“, „Schreiben Sie eine Kopie für diesen Bildschirm/Flow/diese Seite“, überprüfen Sie die Schaltflächenbeschriftungen, verbessern Sie die CLI-Ausgabemeldungen, schreiben Sie Onboarding-Texte, Einstellungsbeschreibungen oder Bestätigungsdialoge. Wird immer dann ausgelöst, wenn die Anfrage Formulierungen beinhaltet, die Endbenutzern in der Software angezeigt werden – Apps, Web, CLI, E-Mail-Benachrichtigungen, Modalitäten, Tooltips, leere Status oder Warnungen. Wird auch für vage Anfragen wie „Überprüfen Sie die UX“ ausgelöst, bei denen eine Überprüfung der Schnittstellenkopie impliziert ist. Triggern Sie NICHT für Content-Marketing, Blog-Beiträge, App-Store-Einträge, API-Dokumente, Markenleitfäden, Anschreiben oder Interviewfragen – dies ist eine technische Schreibfähigkeit für die Schnittstellensprache.
