Создайте рабочее дерево git и управляйте им для параллельной разработки функций, а затем откройте новое рабочее дерево в редакторе (код/курсор) для второго сеанса Кодекса. Используйте, когда пользователь просит одновременно работать над другой функцией, параллельно запускать несколько функций, разворачивать параллельную рабочую область или открывать новое рабочее дерево, даже если рабочее дерево явно не упоминается.
