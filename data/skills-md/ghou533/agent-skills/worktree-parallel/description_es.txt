Cree y administre un árbol de trabajo de git para el desarrollo de funciones en paralelo, luego abra el nuevo árbol de trabajo en el editor (código/cursor) para una segunda sesión de Codex. Úselo cuando el usuario solicite trabajar en otra función simultáneamente, ejecutar varias funciones en paralelo, activar un espacio de trabajo paralelo o abrir un nuevo árbol de trabajo incluso si no menciona el árbol de trabajo explícitamente.
