Erstellen und verwalten Sie einen Git-Arbeitsbaum für die parallele Funktionsentwicklung und öffnen Sie dann den neuen Arbeitsbaum im Editor (Code/Cursor) für eine zweite Codex-Sitzung. Verwenden Sie diese Option, wenn der Benutzer gleichzeitig an einer anderen Funktion arbeiten, mehrere Funktionen parallel ausführen, einen parallelen Arbeitsbereich erstellen oder einen neuen Arbeitsbaum öffnen möchte, auch wenn er den Arbeitsbaum nicht explizit erwähnt.
