Créez et gérez un arbre de travail git pour le développement de fonctionnalités parallèles, puis ouvrez le nouvel arbre de travail dans l'éditeur (code/curseur) pour une deuxième session Codex. À utiliser lorsque l'utilisateur demande à travailler sur une autre fonctionnalité simultanément, à exécuter plusieurs fonctionnalités en parallèle, à créer un espace de travail parallèle ou à ouvrir un nouvel arbre de travail même s'il ne mentionne pas explicitement l'arbre de travail.
