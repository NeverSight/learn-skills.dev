Спецификация управления правами доступа к данным многопользовательского проекта leniu-tengyun-core. На основе архитектуры двойной базы данных с физической изоляцией базы данных, включая переключение источников данных, контекст клиента и операции между арендаторами.

Триггерные сценарии:
- Используйте @UseSystem для переключения операций с базой данных системы.
- Используйте Executors.doInTenant/readInTenant для запросов между арендаторами.
— Используйте TenantContextHolder для получения контекста арендатора.
- Настройка маршрутизации источника данных с двумя базами данных.
- Инициализация данных между арендаторами

Ключевые слова-триггеры: мультитенант, разрешение на данные, @UseSystem, Executors.doInTenant, readInSystem, TenantContextHolder, MERCHANT-ID, изоляция двойной базы данных, переключение источника данных.
