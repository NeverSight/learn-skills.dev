Навыки отладки запросов к базе данных MySQL. Запрашивайте локальную базу данных MySQL напрямую через CLI mysql и используйте журналы для отслеживания ошибок.  
Во время исправления ошибок автоматически оценивайте необходимость запроса к базе данных, извлекайте имя базы данных (идентификатор клиента) из журналов и поддерживайте указание базы данных вручную.

Триггерные сценарии:
- Запрос к базе данных для проверки данных при отладке с помощью журналов.
- Проверить фактическое значение записи в базе данных
- Проверьте, соответствуют ли результаты выполнения SQL ожиданиям.
- Расследование несогласованности данных, потери данных или аномалий.
- Необходимо просмотреть структуру таблицы, индексы и распределение данных.

Триггерные слова: проверка базы данных, проверка таблицы, выполнение SQL, запись запроса, запрос mysql, проверка базы данных, проверка данных, проверка данных, запрос к базе данных, отладка базы данных.
