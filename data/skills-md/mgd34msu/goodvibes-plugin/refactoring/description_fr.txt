Chargez de manière PROACTIVE lorsque la tâche implique d’améliorer la structure du code sans changer le comportement. À utiliser lorsque l'utilisateur dit "refactoriser ceci", "nettoyer ce code", "supprimer le code mort", "réduire la duplication", "réorganiser les fichiers" ou "extraire ceci dans une fonction". Couvre l'extraction de fonctions/composants, le renommage du symbole, la simplification des conditions, l'amélioration de la sécurité des types, l'inversion des dépendances, la suppression du code mort, la résolution des dépendances circulaires et la refactorisation des requêtes de base de données avec validation de sécurité automatisée (vérification de type, tests) à chaque étape.
