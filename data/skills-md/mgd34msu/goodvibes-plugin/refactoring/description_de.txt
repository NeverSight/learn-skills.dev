Laden Sie PROAKTIV, wenn die Aufgabe darin besteht, die Codestruktur zu verbessern, ohne das Verhalten zu ändern. Wird verwendet, wenn der Benutzer sagt: „Umgestalten“, „Diesen Code bereinigen“, „Toten Code entfernen“, „Duplikate reduzieren“, „Dateien neu organisieren“ oder „In eine Funktion extrahieren“. Behandelt das Extrahieren von Funktionen/Komponenten, das Umbenennen von Symbolen, das Vereinfachen von Bedingungen, die Verbesserung der Typsicherheit, die Umkehrung von Abhängigkeiten, das Entfernen von totem Code, die Auflösung zirkulärer Abhängigkeiten und das Refactoring von Datenbankabfragen mit automatischer Sicherheitsvalidierung (Typprüfung, Tests) bei jedem Schritt.
