Outils Monorepo, orchestration des tâches et architecture d'espace de travail pour les référentiels JavaScript/TypeScript. À utiliser lors de la configuration des espaces de travail Turborepo, Nx, pnpm ou npm ; concevoir les limites des packages ; configuration de la mise en cache à distance ; optimisation de CI pour les packages concernés ; gérer le versioning avec Changesets ; ou démêler les dépendances circulaires. Activez sur "monorepo", "turborepo", "nx", "pnpm workspace", "task pipeline", "remote cache", "changesets", "CODEOWNERS", "circular dependency", "affected packages", "workspace". PAS pour les sous-modules git ou les stratégies de fédération multi-dépôts, les monorepos non JavaScript (Bazel, Pants, Buck) ou la configuration d'un référentiel à package unique.
