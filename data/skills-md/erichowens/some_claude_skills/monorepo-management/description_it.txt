Strumenti Monorepo, orchestrazione delle attività e architettura dell'area di lavoro per repository JavaScript/TypeScript. Da utilizzare durante la configurazione di aree di lavoro Turborepo, Nx, pnpm o npm; progettazione dei confini del pacchetto; configurazione della memorizzazione nella cache remota; ottimizzazione della CI per i pacchetti interessati; gestire il controllo delle versioni con Changeset; o districare le dipendenze circolari. Attivare su "monorepo", "turborepo", "nx", "pnpm workspace", "task pipeline", "remote cache", "changesets", "CODEOWNERS", "circolare dipendenza", "pacchetti interessati", "workspace". NON per sottomoduli git o strategie di federazione multi-repo, monorepos non JavaScript (Bazel, Pants, Buck) o configurazione di repository a pacchetto singolo.
