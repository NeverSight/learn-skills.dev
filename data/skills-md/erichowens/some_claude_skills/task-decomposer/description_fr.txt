Divise les descriptions de problèmes en langage naturel en sous-tâches adaptées aux nœuds DAG. Le point d’entrée du méta-DAG. Identifie les phases, les dépendances, les opportunités de parallélisation et les nœuds vagues/pluripotents qui ne peuvent pas encore être spécifiés. Utilise les méta-compétences du domaine lorsqu'elles sont disponibles. Activez sur "décomposer la tâche", "décomposer le problème", "planifier le workflow", "quelles sont les étapes", "sous-tâches", "répartition des tâches". PAS pour exécuter les tâches décomposées (utilisez dag-runtime), construire la structure DAG (utilisez dag-planner) ou faire correspondre les compétences aux nœuds (utilisez dag-skills-matcher).
