アプリケーション アーキテクチャ パターンとコード構成 — クリーン アーキテクチャ、ヘキサゴナル アーキテクチャ、機能ベースの構造、依存関係の逆転、モジュール境界、モジュール スケールでの SOLID 原則。 「クリーン アーキテクチャ」、「ヘキサゴナル アーキテクチャ」、「フィーチャベースの構造」、「依存関係の反転」、「モジュール境界」、「アーキテクチャ パターン」、「フォルダ構造」、「ファット コントローラ」、「循環依存関係」、「IoC コンテナ」でアクティブ化します。マイクロサービスの分解 (マイクロサービス パターンを使用) またはデータベース スキーマ設計 (データベース デザイン パターンを使用) には使用できません。
