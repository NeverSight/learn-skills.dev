実証済みのシンプルな Web デザイン原則を適用して、Web サイトの UI と UX を批評、監査、改善、構築します。ユーザーがデザインのフィードバック、UX レビュー、ランディング ページの批評、または Web インターフェイスのデザインの支援を求めたときは常に、このスキルをトリガーします。たとえ「この見た目は良いですか?」、「サイトのどこが問題ですか?」、「UX を改善するにはどうすればよいですか?」、「ホームページを見直してください」、「アプリをすっきりさせてください」、または「ページのデザインを手伝ってください」などの何気ない言葉で表現された場合でも同様です。ユーザーがデザインレビューのためにHTML/CSSコードまたは画像データを提供する場合にも適用されます。このスキルは、Anthony Hobday の「Simple Web Design」フレームワークに基づいています。これは、視覚的な複雑さやトレンドの追従よりも、コンテンツ、明瞭さ、長期的な使いやすさを優先する、実戦でテストされた 15 の原則です。
