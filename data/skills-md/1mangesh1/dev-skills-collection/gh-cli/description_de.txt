GitHub CLI (gh)-Beherrschung für Issues, PRs, Releases, Aktionen, Gists und Repo-Management. Wird verwendet, wenn der Benutzer von der Befehlszeile aus auffordert, „eine PR zu erstellen“, „Probleme aufzulisten“, „den CI-Status zu überprüfen“, „ein Release zu erstellen“, „Workflow-Ausführungen anzuzeigen“, „einen Kern zu erstellen“, „ein Repository zu klonen“, „ein Repository zu forken“ oder GitHub-Vorgänge auszuführen.
