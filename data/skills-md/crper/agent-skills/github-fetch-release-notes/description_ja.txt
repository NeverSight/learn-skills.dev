📦 GitHub リポジトリの更新の概要を取得します。 CHANGELOG の読み取りを優先し、見つからない場合はリリースにフォールバックします。ユーザーが GitHub リポジトリの更新、リリース ノート、CHANGELOG、バージョン比較、日次/週次レポート、複数のプロジェクトのサブスクライブ、GitHub リポジトリ URL の貼り付け、または GitHub 更新の cron/automation への統合について言及する場合は常に、このスキルを使用します。スクリプトと LLM に適した安定した構造化 JSON をデフォルトで出力し、ローカルの「gh auth login」セッションを再利用します。小規模バッチ クエリ向けに設計されており、一度に最大 10 リポジトリを推奨します。
