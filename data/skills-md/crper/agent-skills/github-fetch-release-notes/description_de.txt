📦 Erhalten Sie eine Zusammenfassung der GitHub-Repository-Updates. Priorisieren Sie das Lesen von CHANGELOG; Greifen Sie auf Releases zurück, wenn diese nicht verfügbar sind. Verwenden Sie diesen Skill immer dann, wenn der Benutzer GitHub-Repo-Updates, Versionshinweise, CHANGELOG, Versionsvergleiche, tägliche/wöchentliche Berichte, das Abonnieren mehrerer Projekt-Feeds, das direkte Einfügen einer GitHub-Repo-URL oder die Integration von GitHub-Updates in Cron/Automation erwähnt. Standardmäßig wird eine stabile, strukturierte JSON-Ausgabe verwendet, die für Skripte und den LLM-Verbrauch geeignet ist, und die lokale „gh auth login“-Sitzung wiederverwendet. Entwickelt für kleine Abfragen; empfehlen bis zu 10 Repos gleichzeitig.
