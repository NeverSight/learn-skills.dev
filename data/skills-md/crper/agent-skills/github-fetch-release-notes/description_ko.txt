📦 GitHub 리포지토리 업데이트 요약을 받아보세요. CHANGELOG를 읽는 것이 좋습니다. 찾을 수 없으면 릴리스로 대체하세요. 사용자가 GitHub 저장소 업데이트, 릴리스 노트, CHANGELOG, 버전 비교, 일일/주간 보고서, 여러 프로젝트 업데이트 구독, GitHub 저장소 URL 직접 게시 또는 GitHub 업데이트를 cron/자동화에 통합하려고 할 때마다 이 기술을 사용하세요. 기본적으로 스크립트 및 대규모 모델에 적합한 안정적인 구조의 JSON을 출력하고 로컬 `gh auth login` 세션을 재사용하는 것을 선호합니다. 이 기술은 소규모 배치 쿼리를 위해 설계되었으며 한 번에 최대 10개의 리포지토리가 권장됩니다.
