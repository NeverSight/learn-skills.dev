Umfassendes Audit-Toolkit mit 14 Audit-Typen. Beinhaltet ein Pre-Push-Branch-Audit (8-Kategorien-Checkliste mit Clean/Minor/Blocking-Urteilen) sowie 13 Deep-Dive-Audits, die über Subagenten durchgeführt werden: Codequalität, Dokumentationsbrüchigkeit, Dokument-Code-Synchronisierung, Sprach-Best Practices, Parallelität, Ressourcenmanagement, Testqualität, Funktionsvollständigkeit, Leistung, Fehlermuster, Einhaltung der Designphilosophie, Sicherheitslücken und UI-Design (CRAP-Prinzipien). Verwenden Sie diese Fähigkeit IMMER, wenn der Benutzer prüfen, vor dem Pushen überprüfen, seinen Zweig überprüfen, nach Duplikaten oder totem Code suchen, Dokumente überprüfen, die Testqualität überprüfen, Parallelitätsfehler finden, Ressourcenlecks überprüfen, Sicherheit oder Leistung analysieren, das UI-Design überprüfen, die Vollständigkeit von Funktionen überprüfen oder Code anhand von Best Practices oder Designprinzipien überprüfen möchte.
