Procese todos los problemas vinculados de un proyecto de GitHub e impleméntelos como RP separados. Úselo cuando el usuario proporciona una URL de proyecto de GitHub (github.com/owner/repo/projects/N o github.com/orgs/NAME/projects/N) y desea implementar todos los problemas en ese proyecto. Cada tema se convierte en su propio PR. Maneja dependencias mediante relaciones públicas en cascada donde los problemas dependientes se ramifican de la rama de relaciones públicas de su dependencia.
