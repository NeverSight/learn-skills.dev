Verarbeiten Sie alle verknüpften Probleme aus einem GitHub-Projekt und implementieren Sie sie als separate PRs. Verwenden Sie diese Option, wenn der Benutzer eine GitHub-Projekt-URL (github.com/owner/repo/projects/N oder github.com/orgs/NAME/projects/N) bereitstellt und alle Probleme in diesem Projekt implementieren möchte. Jede Ausgabe wird zu einer eigenen PR. Behandelt Abhängigkeiten mithilfe kaskadierender PRs, bei denen abhängige Probleme vom PR-Zweig ihrer Abhängigkeit abzweigen.
