Erstellen, verwalten und organisieren Sie GitHub Gists mit der gh CLI. Verwenden Sie diese Fähigkeit immer dann, wenn ein Benutzer einen Gist erstellen, Codeausschnitte teilen, vorhandene Gists verwalten, Dateien zu einem Gist hinzufügen, Gists auflisten oder durchsuchen, Gist-Inhalte oder -Beschreibungen bearbeiten, einen Gist klonen oder einen Gist löschen möchte. Jeder erstellte oder geänderte Kern muss eine README.md enthalten, die seinen Zweck und Inhalt dokumentiert.
