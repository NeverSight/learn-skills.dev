Überprüfung des Sicherheitscodes auf Expertenebene, die echte Schwachstellen in jeder Sprache und jedem Framework findet. Identifiziert die Top 10 von OWASP, CWE-zugeordnete Probleme, Injektionsfehler, Authentifizierungsumgehungen, die Offenlegung von Geheimnissen, unsichere Standardeinstellungen und Risiken in der Lieferkette – und stellt dann spezifische Korrekturen durch Kopieren und Einfügen bereit. Verwenden Sie diese Fähigkeit immer dann, wenn der Benutzer Code auf Sicherheit überprüfen, eine Codebasis prüfen, auf Schwachstellen prüfen, eine Anwendung härten, eine Sicherheitsbewertung durchführen, Sicherheitsfehler finden oder Fragen zu Anwendungssicherheit, AppSec, Bedrohungsmodellierung oder sicheren Codierungspraktiken stellen möchte. Wird auch bei der Überprüfung von Pull-Requests ausgelöst, wenn Sicherheit erwähnt wird oder wenn der Benutzer Dinge sagt wie „Ist das sicher?“, „Kann das ausgenutzt werden?“ oder „Diesen Code pentestieren“.
