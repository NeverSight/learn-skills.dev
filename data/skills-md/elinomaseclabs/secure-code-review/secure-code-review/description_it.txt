Revisione del codice di sicurezza a livello di esperto che rileva vulnerabilità reali in qualsiasi linguaggio o framework. Identifica i 10 principali problemi OWASP mappati da CWE, difetti di iniezione, bypass di autenticazione, esposizione di segreti, impostazioni predefinite non sicure e rischi della catena di fornitura, quindi fornisce soluzioni specifiche tramite copia e incolla. Utilizza questa competenza ogni volta che l'utente chiede di rivedere il codice per la sicurezza, controllare una codebase, verificare la presenza di vulnerabilità, rafforzare un'applicazione, eseguire una valutazione della sicurezza, trovare bug di sicurezza o chiedere qualsiasi cosa relativa alla sicurezza dell'applicazione, AppSec, modellazione delle minacce o pratiche di codifica sicure. Si attiva anche quando si esaminano le richieste pull se viene menzionata la sicurezza o quando l'utente dice cose come "è sicuro?", "può essere sfruttato?" o "pentest questo codice".
