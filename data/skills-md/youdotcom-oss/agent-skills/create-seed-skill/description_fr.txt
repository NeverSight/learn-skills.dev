Élaborez une nouvelle « compétence de départ » de type intégration : un fichier SKILL.md avec des ressources de référence et des entrées prompts.jsonl qui apprennent à un agent à générer un code d'intégration fonctionnel et testable à partir de zéro.
- À utiliser lorsque : le développeur souhaite créer une nouvelle compétence pour un SDK ou une intégration de framework que les agents devraient être capables de mettre en œuvre et de tester.
- Demandez d'abord : s'agit-il d'une compétence d'intégration (génération de code + tests) ou d'une compétence d'outil (wrapper CLI sans tests) ? N'utilisez cette compétence que pour les compétences de type intégration.
