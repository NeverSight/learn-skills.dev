Révision complète du code dans un style rôti. Analyse la sécurité, l'architecture, la complexité, la conformité TDD, la sécurité des types, la gestion des erreurs, la dénomination, le code mort, les performances, les dépendances, la conception de l'API, la qualité du frontend et l'hygiène de Git. Fournit des résultats sous forme de rôtis divertissants avec des correctifs exploitables. Utilisez /roast pour revoir votre base de code. Prend en charge le niveau --(doux/épicé/sauvage/sensei).
