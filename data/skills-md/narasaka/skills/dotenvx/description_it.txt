DEVE UTILIZZARE per qualsiasi attività che coinvolga lo strumento CLI dotenvx: crittografia di file .env, esecuzione di comandi con variabili env inserite, gestione dei segreti tra ambienti e decrittografia in fase di runtime. Utilizza questa abilità ogni volta che l'utente menziona dotenvx, dotenv crittografia, DOTENV_PRIVATE_KEY, file .env crittografati o i comandi dotenvx encrypt/run/set/get/decrypt/keypair. Si attiva anche quando l'utente desidera: eseguire il commit dei file .env in modo sicuro su git, interrompere la condivisione di segreti su Slack/chat, crittografare le variabili di ambiente con crittografia a chiave pubblica, impostare configurazioni .env multi-ambiente (produzione/staging/ci), gestire i segreti in un monorepo con il flag -fk, migrare da python-dotenv o dotenv semplice a env crittografati, iniettare env vars in qualsiasi processo in qualsiasi linguaggio (Node, Python, Ruby, Go, Rust, ecc.) o configurare pipeline CI/CD (GitHub Actions, Docker) con file env crittografati. Questa abilità contiene il riferimento autorevole alla CLI: senza di esso, le risposte darebbero allucinazioni a comandi e flag inesistenti.
