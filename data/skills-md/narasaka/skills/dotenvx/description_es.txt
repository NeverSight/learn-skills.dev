DEBE USARSE para cualquier tarea que involucre la herramienta CLI dotenvx: cifrar archivos .env, ejecutar comandos con variables env inyectadas, administrar secretos en todos los entornos y descifrar en tiempo de ejecución. Utilice esta habilidad siempre que el usuario mencione dotenvx, cifrado dotenv, DOTENV_PRIVATE_KEY, archivos .env cifrados o los comandos dotenvx encrypt/run/set/get/decrypt/keypair. También se activa cuando el usuario quiere: enviar archivos .env de forma segura a git, dejar de compartir secretos a través de Slack/chat, cifrar variables de entorno con criptografía de clave pública, configurar configuraciones .env multientorno (producción/puesta en escena/ci), administrar secretos en un monorepo con el indicador -fk, migrar de python-dotenv o dotenv simple a envs cifrados, inyectar vars env en cualquier proceso en cualquier idioma (Node, Python, Ruby, Go, Rust, etc.) o configure canalizaciones de CI/CD (GitHub Actions, Docker) con archivos env cifrados. Esta habilidad contiene la referencia CLI autorizada; sin ella, las respuestas alucinarán con comandos y banderas inexistentes.
