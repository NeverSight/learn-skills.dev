A workflow skill that analyzes CSS, Tailwind settings, and components of an existing project to understand design tokens, visually improves the UI, and sends it to Figma via the Figma MCP server. Rather than merely enhancing the UI, it ensures improvements align with the project's design system. It proposes new colors and spacing without hardcoding, staying within existing tokens and naming conventions. Use it in situations like: "Look at the UI and send it to Figma," "Improve this page's design and reflect it in Figma," "Analyze the UI and create a design proposal in Figma," or whenever web-app UI improvements and design hand-off to Figma are needed—especially when the user says "ui-to-figma," "send UI to Figma," or "improve design and send to Figma." Prerequisites: Playwright (webapp-testing skill) and the Figma MCP server must be available.
