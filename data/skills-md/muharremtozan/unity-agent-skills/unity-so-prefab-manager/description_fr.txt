Gère la relation structurée entre ScriptableObjects (données) et préfabriqués (logique/visuels) dans Unity 6. Suit le modèle de pont « SO-to-Mono » pour garantir l'indépendance des instances (par exemple, santé individuelle pour des robots identiques) tout en conservant une architecture propre et basée sur les données. À utiliser lorsque : (1) Création de nouveaux types d'unités/éléments, (2) Câblage de données SO à des MonoBehaviors préfabriqués, (3) Résolution de bogues de partage de données où la modification d'un SO affecte toutes les instances.
