Gestiona la relación estructurada entre ScriptableObjects (datos) y Prefabs (lógica/visuales) en Unity 6. Sigue el patrón Bridge 'SO-to-Mono' para garantizar la independencia de la instancia (por ejemplo, salud individual para robots idénticos) mientras mantiene una arquitectura limpia basada en datos. Úselo cuando: (1) cree nuevos tipos de unidades/elementos, (2) conecte datos de SO a monocomportamientos prefabricados, (3) resuelva errores de intercambio de datos en los que el cambio de un SO afecta a todas las instancias.
