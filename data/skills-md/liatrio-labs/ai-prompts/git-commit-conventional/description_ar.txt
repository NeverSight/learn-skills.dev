قم بإنشاء رسائل الالتزام التقليدية وتنفيذ التزامات git الآمنة من تغييرات شجرة العمل. يُستخدم عندما يطلب المستخدمون إنشاء التزام، أو كتابة رسالة التزام تقليدية، أو تقسيم التغييرات العامة إلى التزامات متعددة، أو تنظيم أجزاء فقط من الملفات، أو تشغيل الالتزام المسبق قبل الالتزام، أو إجراء مراجعة سريعة لجودة وقت الالتزام باستخدام حواجز حماية الخطورة.
