Konvertieren Sie eine vorhandene Codebasis im aktuellen Arbeitsverzeichnis in ein ShinkaEvolve-Aufgabenverzeichnis, indem Sie einen Snapshot des relevanten Codes erstellen, Evolve-Blöcke hinzufügen und „evaluate.py“ plus Shinka-Runner-/Konfigurationsdateien generieren. Verwenden Sie diese Option, wenn der Benutzer vorhandenen Code mit Shinka optimieren möchte, anstatt eine brandneue Aufgabe aus einer Beschreibung in natürlicher Sprache zu erstellen.
