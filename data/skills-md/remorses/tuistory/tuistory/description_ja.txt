ターミナルアプリの劇作家。 tmux に似ていますが、エージェント向けに設計されており、仮想端末で次のことが可能です。
に入力し、キーを押し、テキストを待ち、スナップショットを撮り、画像としてスクリーンショットを撮ります。

tuistory には **2 つのモード**があります:

- **CLI** (`tuistory` コマンド) — シェルベース。セッションの起動、入力、キーの押し、スナップショット、スクリーンショット。
  コマンド間でセッションを維持するバックグラウンド デーモンを実行します。グローバルにインストールするか、「npx」/「bunx」を使用します。
  **最新のコマンドとオプションを確認するには、CLI を使用する前に `tuistory --help` を実行する必要があります**。
- **JS/TS API** (`import { launchterminal } from 'tuistory'`) — プログラムによるもの。 vitest/bun:test で使用する
  インライン スナップショットを使用して CLI および TUI 用の Playwright スタイルのテストを作成します。

次のような場合に tuistory を使用します。
- インライン スナップショットを使用して CLI/TUI アプリ (vitest、bun:test) の e2e テストを作成します
- 端末の対話を自動化します (REPL、デバッガー、または TUI を起動して駆動します)
- ユーザーに送信する画像としてターミナルのスクリーンショットを作成 (Discord ボット、kimaki/openclaw などのエージェント UI)
- 正確な手順をスクリプト化することで、インタラクティブ CLI でバグを再現します。
- 観察 - 行為 - 観察ループを使用して TUI アプリを段階的に探索します
