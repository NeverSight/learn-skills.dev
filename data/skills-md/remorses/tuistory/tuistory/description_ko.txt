입력하고, 키를 누르고, 텍스트를 기다리고, 스냅샷을 찍고, 스크린샷을 이미지로 찍으세요.

tustory에는 **2가지 모드**가 있습니다:

- **CLI** (`tuistory` 명령) — 쉘 기반. 세션을 시작하고, 입력하고, 키를 누르고, 스냅샷, 스크린샷을 찍습니다.
  명령 전반에 걸쳐 세션을 유지하는 백그라운드 데몬을 실행합니다. 전역적으로 설치하거나 `npx`/`bunx`를 사용하세요.
  **CLI를 사용하기 전에 `tuistory --help`를 실행해야 최신 명령과 옵션을 볼 수 있습니다.
- **JS/TS API**(`import { launchTerminal } from 'tuistory'`) — 프로그래밍 방식. vitest/bun:test에서 사용
  인라인 스냅샷을 사용하여 CLI 및 TUI에 대한 Playwright 스타일 테스트를 작성합니다.

다음이 필요할 때 tustory를 사용하세요:
- 인라인 스냅샷을 사용하여 CLI/TUI 앱(vitest, bun:test)에 대한 e2e 테스트 작성
- 터미널 상호 작용 자동화(REPL, 디버거 또는 TUI 실행 및 구동)
- 사용자에게 보낼 이미지로 스크린샷 터미널(Discord 봇, kimaki/openclaw와 같은 에이전트 UI)
- 정확한 단계를 스크립팅하여 대화형 CLI에서 버그를 재현합니다.
- 관찰-행위-관찰 루프를 통해 TUI 앱을 점진적으로 탐색합니다.
