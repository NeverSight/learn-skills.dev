Bonnes pratiques d'intégration du module @nuxtjs/sanity pour les applications Nuxt 3 connectées à Sanity CMS. Couvre useSanityQuery, useLazySanityQuery, useSanity, SanityImage, SanityContent (Portable Text), l'édition visuelle/aperçu en direct avec stega, TypeScript typegen, les clients nommés et les routes du serveur Nitro. Couvre également l'architecture de démarrage avec les points de terminaison Nitro mis en cache, les composables de commutateur de prévisualisation, la mise en cache Netlify CDN et l'invalidation des balises de cache. À utiliser lorsque l'utilisateur mentionne @nuxtjs/sanity, nuxt-sanity, useSanityQuery, SanityImage dans Nuxt ou crée une application Nuxt qui récupère les données de Sanity.
