Rédaction de tests backend NestJS suivant les conventions Sellernote. À utiliser lors de la création de tests unitaires, de tests d'intégration ou de tests e2e pour les services, contrôleurs, référentiels ou modules NestJS. Se déclenche sur des tâches impliquant l'écriture de tests, des fichiers de spécifications, des plaisanteries, des tests de services, des tests de contrôleurs, des tests de référentiels, des tests e2e, une couverture de tests ou tout travail de test back-end dans un projet NestJS. À utiliser également lorsqu'on vous demande d'écrire des tests, d'ajouter des tests unitaires, de créer des tests e2e, d'ajouter des fichiers de spécifications ou de vérifier la qualité des tests pour l'architecture à 3 couches de Sellernote (Contrôleur → Service → Référentiel).
