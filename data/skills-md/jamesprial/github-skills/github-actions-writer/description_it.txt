Questa competenza deve essere utilizzata quando gli utenti devono creare, modificare, ottimizzare o risolvere i problemi relativi ai flussi di lavoro CI/CD di GitHub Actions. Da utilizzare quando gli utenti chiedono informazioni sull'automazione di build, test, distribuzioni o qualsiasi attività correlata a GitHub Actions. I trigger includono richieste come "crea un flusso di lavoro CI", "distribuisci su AWS", "automatizzazione dei test", "configura azioni GitHub" o durante il debug degli errori del flusso di lavoro.
