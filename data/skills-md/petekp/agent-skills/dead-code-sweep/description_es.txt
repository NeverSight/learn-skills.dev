Esta habilidad se debe utilizar al limpiar bases de código que han acumulado código inactivo, implementaciones redundantes y artefactos huérfanos, especialmente bases de código mantenidas por agentes de codificación. Activadores de "encontrar código inactivo", "limpiar código no utilizado", "eliminar código redundante", "podar esta base de código", "barrido de código inactivo", "limpieza de código" o cuando una base de código ha pasado por múltiples refactores controlados por agentes y probablemente contiene restos pasados ​​por alto. Identifica sistemáticamente los errores, clasifica los hallazgos y elimina el código muerto confirmado con la aprobación del usuario.
