Marco basado en fragmentos y basado en evidencia para auditar y migrar bases de código con mucha deuda. Úselo para refactorizaciones grandes, consolidación de bases de código, pago de deudas tecnológicas, estandarización de dependencias, limpieza de arquitectura o cualquier migración de múltiples sesiones. Úselo también cuando se le solicite auditar el código base, migrar de X a Y, estandarizar en un enfoque, consolidar código duplicado o ejecutar una refactorización sistemática con limpieza integrada, barrido de residuos y una puerta final de preparación para el envío. Asegúrese de utilizar esta habilidad siempre que el usuario desee una migración controlada con divisiones, barreras de seguridad, transferencias, limpieza o cierre de versiones, incluso si no dicen explícitamente "migración". Las menciones a trinquetes, cortes, guías de migración, disciplina anti-vestigios, convergencia o limpieza de liberación deberían desencadenarlo fuertemente.
