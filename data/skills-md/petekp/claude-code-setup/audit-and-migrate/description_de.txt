Slice-basiertes, evidenzbasiertes Framework zur Prüfung und Migration schuldenintensiver Codebasen. Verwendung für große Refaktoren, Codebasis-Konsolidierung, technische Schuldentilgung, Abhängigkeitsstandardisierung, Architekturbereinigung oder jede Multisitzungsmigration. Verwenden Sie es auch, wenn Sie aufgefordert werden, die Codebasis zu prüfen, von Stellen Sie sicher, dass Sie diese Fähigkeit immer dann verwenden, wenn der Benutzer eine kontrollierte Migration mit Slices, Leitplanken, Übergaben, Bereinigung oder Release-Closeout wünscht, auch wenn er nicht ausdrücklich „Migration“ sagt. Erwähnungen von Ratchets, Slices, Migrations-Playbooks, Anti-Rest-Disziplin, Konvergenz oder Release-Bereinigung sollten dies stark auslösen.
