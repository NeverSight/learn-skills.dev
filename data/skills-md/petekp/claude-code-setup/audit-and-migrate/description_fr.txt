Cadre basé sur des tranches et fondé sur des preuves pour l'audit et la migration de bases de code lourdement endettées. À utiliser pour les grands refactors, la consolidation de la base de code, le remboursement de la dette technologique, la standardisation des dépendances, le nettoyage de l'architecture ou toute migration multi-session. À utiliser également lorsqu'on vous demande d'auditer la base de code, de migrer de X vers Y, de standardiser sur une approche, de consolider le code dupliqué ou d'exécuter une refactorisation systématique avec un nettoyage intégré, des balayages de résidus et une porte finale de préparation à l'expédition. Assurez-vous d'utiliser cette compétence chaque fois que l'utilisateur souhaite une migration contrôlée avec des tranches, des garde-fous, des transferts, un nettoyage ou une clôture de version, même s'il ne dit pas explicitement « migration ». Les mentions de cliquets, de tranches, de manuels de migration, de discipline anti-vestigiale, de convergence ou de nettoyage de versions devraient fortement le déclencher.
