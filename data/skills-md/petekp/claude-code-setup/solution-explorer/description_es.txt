Explore exhaustivamente un problema y un espacio de solución antes de implementar una característica o tomar una decisión arquitectónica. Úselo cuando se le solicite implementar una característica, diseñar un sistema, elegir entre enfoques o cuando haya varias formas válidas de resolver un problema. Obliga a la exploración sistemática de alternativas para encontrar el óptimo global en lugar de decidirse por la primera idea razonable. Utilice esta habilidad siempre que el usuario diga cosas como "implementar X", "cómo deberíamos construir", "la mejor manera de hacerlo", "diseñar", "arquitectar", "agregar una característica para" o describa una característica no trivial que desea construir. Incluso si el usuario no solicita explícitamente una exploración, utilice esta habilidad cuando reconozca que la implementación tiene múltiples paradigmas viables que vale la pena comparar.
