Erkunden Sie ein Problem und einen Lösungsraum eingehend, bevor Sie eine Funktion implementieren oder eine architektonische Entscheidung treffen. Verwenden Sie diese Option, wenn Sie aufgefordert werden, eine Funktion zu implementieren, ein System zu entwerfen, zwischen Ansätzen zu wählen oder wenn es mehrere gültige Möglichkeiten zur Lösung eines Problems gibt. Erzwingt die systematische Erforschung von Alternativen, um das globale Optimum zu finden, anstatt sich auf die erste vernünftige Idee festzulegen. Verwenden Sie diese Fähigkeit immer dann, wenn der Benutzer Dinge wie „Implementieren Sie X“, „Wie sollen wir bauen“, „Beste Vorgehensweise“, „Design“, „Architekt“, „Fügen Sie eine Funktion hinzu für“ sagt oder eine nicht triviale Funktion beschreibt, die er erstellen möchte. Auch wenn der Benutzer nicht ausdrücklich um Erkundung bittet, nutzen Sie diese Fähigkeit, wenn Sie erkennen, dass die Implementierung mehrere praktikable Paradigmen aufweist, die einen Vergleich wert sind.
