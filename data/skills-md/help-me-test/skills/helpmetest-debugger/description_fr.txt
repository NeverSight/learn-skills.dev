Lorsqu'un test échoue et que vous avez besoin de savoir pourquoi, utilisez cette compétence. Il gère : un test nommé échouant avec un message d'erreur (élément introuvable, sélecteur introuvable, délai d'attente), déterminant si l'échec est un bug dans l'application ou un test interrompu, un test qui réussit localement mais échoue sur CI et plusieurs tests régressant après un déploiement ou une fusion PR. Le signal distinctif : l'utilisateur a un test qui échoue et souhaite la cause première, pas seulement une nouvelle exécution. Ne pas utiliser pour : écrire de nouveaux tests, exécuter des suites de tests, explorer des fonctionnalités ou réviser la logique de test dans le résumé.
