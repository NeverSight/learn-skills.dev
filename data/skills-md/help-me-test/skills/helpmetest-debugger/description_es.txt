Cuando una prueba falla y necesitas saber por qué, usa esta habilidad. Maneja: una prueba con nombre que falla con un mensaje de error (elemento no encontrado, selector no encontrado, tiempo de espera), determina si la falla es un error en la aplicación versus una prueba fallida, una prueba que pasa localmente pero falla en CI y múltiples pruebas que retroceden después de una implementación o fusión de PR. La señal distintiva: el usuario tiene una prueba fallida y quiere la causa raíz, no solo volver a ejecutarla. No lo utilice para: escribir nuevas pruebas, ejecutar conjuntos de pruebas, explorar funciones o revisar la lógica de las pruebas en abstracto.
