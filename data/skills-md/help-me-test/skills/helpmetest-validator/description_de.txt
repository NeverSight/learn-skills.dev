Rufen Sie diese Fähigkeit auf, wenn ein Benutzer Testcode teilt und fragt, ob er tatsächlich wie beabsichtigt funktioniert – nicht um den Test auszuführen oder zu reparieren, sondern um zu bewerten, ob der Test einen echten Wert hat. Wird ausgelöst bei: „Ist dieser Test gut?“, „Würde er einen echten Fehler erkennen?“, „Dieser Test besteht immer – ist das normal?“, „Überprüfen Sie diese Tests, bevor ich einen Commit durchführe“ oder „Überprüft dieser Test etwas Sinnvolles?“. Wird auch ausgelöst, wenn jemand den Verdacht hat, dass ein Test nutzlos ist, ein vorab festgelegtes Qualitätsgate wünscht oder sich nicht sicher ist, ob es sich lohnt, einen automatisch generierten Test aufzubewahren. Die Kernfrage, die dieser Skill beantwortet, lautet: „Würde dieser Test fehlschlagen, wenn die Funktion kaputt gehen würde?“ Wenn nicht, wird der Test abgelehnt. NICHT zum Generieren neuer Tests, zum Beheben fehlerhafter Tests oder zum Erkunden von Anwendungsfunktionen verwenden.
