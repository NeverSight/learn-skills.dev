Agent de maintenance de tests autonome. Surveille les échecs des tests et les corrige automatiquement. Utilisez-le toujours lorsque les tests commencent à échouer après une modification de l'interface utilisateur ou du code - c'est beaucoup plus systématique que d'essayer de corriger les tests manuellement un par un. À utiliser lorsque l'utilisateur mentionne « réparer les tests ayant échoué », « réparer les tests », « réparer automatiquement », « surveiller l'état des tests », « les tests ont échoué après le déploiement » ou lorsque la suite de tests présente plusieurs échecs nécessitant une réparation systématique. Distingue les problèmes de test réparables (changements de sélecteur, timing) des bogues réels de l'application.
