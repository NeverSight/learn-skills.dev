Java Spring WebFlux リアクティブ プログラミングに関する専門家によるガイダンス。純粋なリアクティブ フロー、Mono.defer() による遅延エラー処理、 flatMap/map/filter の適切な使用、Mono.zip() による並列操作、および命令的パターンの排除を強制します。 WebFlux を使用したリアクティブ Java アプリケーションの開発、Mono/Flux ストリームの処理、命令型コードのリアクティブ スタイルへのリファクタリング、アンチパターンのリアクティブ コードのレビュー、リアクティブな問題 (ストリームが実行されない、呼び出しのブロック、メモリ リーク) のデバッグ、エラー処理戦略の実装、リアクティブ パフォーマンスの最適化、またはブロッキング コードからノンブロッキングへの変換を行う場合に使用します。 Mono、Flux、WebFlux、リアクティブ ストリーム、または Project Reactor が言及されている場合は、ユーザーがリアクティブ パターンのガイダンスを明示的に求めていない場合でも、常にこのスキルを使用してください。
