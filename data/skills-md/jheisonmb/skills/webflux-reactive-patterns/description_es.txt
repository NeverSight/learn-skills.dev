Orientación experta para la programación reactiva Java Spring WebFlux. Implica flujos reactivos puros, manejo diferido de errores con Mono.defer(), uso adecuado de flatMap/map/filter, operaciones paralelas con Mono.zip() y eliminación de patrones imperativos. Úselo al desarrollar aplicaciones Java reactivas con WebFlux, manejar transmisiones Mono/Flux, refactorizar código imperativo a estilo reactivo, revisar código reactivo en busca de antipatrones, depurar problemas reactivos (transmisiones que no se ejecutan, bloquear llamadas, pérdidas de memoria), implementar estrategias de manejo de errores, optimizar el rendimiento reactivo o convertir código de bloqueo a sin bloqueo. Utilice siempre esta habilidad cuando vea que se mencionan Mono, Flux, WebFlux, flujos reactivos o Project Reactor, incluso si el usuario no solicita explícitamente orientación sobre patrones reactivos.
