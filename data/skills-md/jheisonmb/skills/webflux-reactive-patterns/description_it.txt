Guida esperta per la programmazione reattiva Java Spring WebFlux. Impone flussi reattivi puri, gestione pigra degli errori con Mono.defer(), uso corretto di flatMap/map/filter, operazioni parallele con Mono.zip() ed eliminazione di modelli imperativi. Da utilizzare durante lo sviluppo di applicazioni Java reattive con WebFlux, la gestione di flussi Mono/Flux, il refactoring del codice imperativo in uno stile reattivo, la revisione del codice reattivo per anti-pattern, il debug di problemi reattivi (flussi non eseguiti, chiamate di blocco, perdite di memoria), l'implementazione di strategie di gestione degli errori, l'ottimizzazione delle prestazioni reattive o la conversione del codice bloccante in non bloccante. Utilizza sempre questa abilità quando vedi menzionati Mono, Flux, WebFlux, flussi reattivi o Project Reactor, anche se l'utente non richiede esplicitamente indicazioni sui modelli reattivi.
