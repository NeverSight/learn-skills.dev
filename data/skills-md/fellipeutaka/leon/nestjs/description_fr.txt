Bonnes pratiques NestJS pour créer des API REST, des API GraphQL et des microservices prêts pour la production avec TypeScript.
À utiliser lors de l'écriture, de la révision ou de la refactorisation du code NestJS : contrôleurs, modules, fournisseurs,
injection de dépendances, gardes, intercepteurs, tuyaux, filtres d'exception, middlewares, personnalisés
décorateurs, DTO, validation, authentification, autorisation, JWT, configuration, tests,
base de données (TypeORM/Prisma), mise en cache, files d'attente (BullMQ), OpenAPI/Swagger, WebSockets,
GraphQL (résolveurs, mutations, abonnements, ObjectType, InputType, ArgsType, code-first,
schéma d'abord, PubSub), casque, CORS, CSRF, hooks de cycle de vie, arrêt progressif et
architecture des modules de fonctionnalités. À utiliser également lorsque
travailler avec @nestjs/jwt, @nestjs/throttler, @nestjs/terminus, @nestjs/config,
@nestjs/swagger, @nestjs/typeorm, @nestjs/mongoose, @nestjs/graphql, @nestjs/apollo,
@apollo/server, bullmq, class-validator, graphql ou graphql-ws.
