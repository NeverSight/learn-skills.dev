Best practice NestJS per creare API REST pronte per la produzione, API GraphQL e microservizi con TypeScript.
Da utilizzare durante la scrittura, la revisione o il refactoring del codice NestJS: controller, moduli, provider,
inserimento delle dipendenze, protezioni, intercettori, pipe, filtri delle eccezioni, middleware, personalizzazioni
decoratori, DTO, convalida, autenticazione, autorizzazione, JWT, configurazione, test,
database (TypeORM/Prisma), caching, code (BullMQ), OpenAPI/Swagger, WebSocket,
GraphQL (resolver, mutazioni, abbonamenti, ObjectType, InputType, ArgsType, code-first,
schema-first, PubSub), Casco, CORS, CSRF, hook del ciclo di vita, arresto graduale e
architettura del modulo di funzionalità. Utilizzare anche quando
lavorando con @nestjs/jwt, @nestjs/throttler, @nestjs/terminus, @nestjs/config,
@nestjs/swagger, @nestjs/typeorm, @nestjs/mongoose, @nestjs/graphql, @nestjs/apollo,
@apollo/server, bullmq, class-validator, graphql o graphql-ws.
