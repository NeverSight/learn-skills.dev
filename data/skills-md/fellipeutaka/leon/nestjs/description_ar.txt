أفضل ممارسات NestJS لإنشاء واجهات برمجة تطبيقات REST وواجهات برمجة تطبيقات GraphQL والخدمات الصغيرة الجاهزة للإنتاج باستخدام TypeScript.
يُستخدم عند كتابة كود NestJS أو مراجعته أو إعادة هيكلته: وحدات التحكم، والوحدات النمطية، والموفرون،
حقن التبعية، الحراس، المعترضات، الأنابيب، مرشحات الاستثناء، البرامج الوسيطة، العرف
الديكورات، DTOs، التحقق من الصحة، المصادقة، الترخيص، JWT، التكوين، الاختبار،
قاعدة البيانات (TypeORM/Prisma)، التخزين المؤقت، قوائم الانتظار (BullMQ)، OpenAPI/Swagger، WebSockets،
GraphQL (المحللات، الطفرات، الاشتراكات، ObjectType، InputType، ArgsType، الكود الأول،
المخطط الأول، PubSub)، Helmet، CORS، CSRF، خطافات دورة الحياة، إيقاف التشغيل الرشيق، و
بنية الوحدة النمطية. استخدم أيضًا متى
العمل مع @nestjs/jwt، @nestjs/throttler، @nestjs/terminus، @nestjs/config،
@nestjs/swagger، @nestjs/typeorm، @nestjs/mongoose، @nestjs/graphql، @nestjs/apollo،
@apollo/server، أو Bullmq، أو class-validator، أو graphql، أو graphql-ws.
