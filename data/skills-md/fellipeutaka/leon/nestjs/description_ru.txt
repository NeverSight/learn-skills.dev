Лучшие практики NestJS для создания готовых к использованию API REST, API GraphQL и микросервисов с помощью TypeScript.
Используйте при написании, проверке или рефакторинге кода NestJS: контроллеры, модули, поставщики,
внедрение зависимостей, средства защиты, перехватчики, каналы, фильтры исключений, промежуточное ПО, пользовательские
декораторы, DTO, проверка, аутентификация, авторизация, JWT, настройка, тестирование,
база данных (TypeORM/Prisma), кеширование, очереди (BullMQ), OpenAPI/Swagger, WebSockets,
GraphQL (резольверы, мутации, подписки, ObjectType, InputType, ArgsType, код с приоритетом,
сначала схема, PubSub), Helmet, CORS, CSRF, перехватчики жизненного цикла, корректное завершение работы и
архитектура функционального модуля. Также используйте, когда
работа с @nestjs/jwt, @nestjs/throttler, @nestjs/terminus, @nestjs/config,
@nestjs/swagger, @nestjs/typeorm, @nestjs/mongoose, @nestjs/graphql, @nestjs/apollo,
@apollo/server, Bullmq, класс-валидатор, Graphql или Graphql-ws.
