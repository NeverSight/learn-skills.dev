使用 TypeScript 构建生产就绪的 REST API、GraphQL API 和微服务的 NestJS 最佳实践。
在编写、审查或重构 NestJS 代码时使用：控制器、模块、提供程序、
依赖注入、守卫、拦截器、管道、异常过滤器、中间件、自定义
装饰器、DTO、验证、身份验证、授权、JWT、配置、测试、
数据库 (TypeORM/Prisma)、缓存、队列 (BullMQ)、OpenAPI/Swagger、WebSockets、
GraphQL（解析器、突变、订阅、ObjectType、InputType、ArgsType、代码优先、
架构优先、PubSub）、Helmet、CORS、CSRF、生命周期挂钩、正常关闭以及
功能模块架构。也使用时
使用@nestjs/jwt、@nestjs/throttler、@nestjs/terminus、@nestjs/config、
@nestjs/swagger、@nestjs/typeorm、@nestjs/mongoose、@nestjs/graphql、@nestjs/apollo、
@apollo/server、bullmq、类验证器、graphql 或 graphql-ws。
