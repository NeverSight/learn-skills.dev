Mejores prácticas de NestJS para crear API REST, API GraphQL y microservicios listos para producción con TypeScript.
Úselo al escribir, revisar o refactorizar código NestJS: controladores, módulos, proveedores,
inyección de dependencia, guardias, interceptores, canalizaciones, filtros de excepción, middlewares, personalizados
decoradores, DTO, validación, autenticación, autorización, JWT, configuración, pruebas,
base de datos (TypeORM/Prisma), almacenamiento en caché, colas (BullMQ), OpenAPI/Swagger, WebSockets,
GraphQL (resolvedores, mutaciones, suscripciones, ObjectType, InputType, ArgsType, código primero,
esquema primero, PubSub), casco, CORS, CSRF, ganchos de ciclo de vida, cierre ordenado y
Arquitectura del módulo de funciones. Úselo también cuando
trabajando con @nestjs/jwt, @nestjs/throttler, @nestjs/terminus, @nestjs/config,
@nestjs/swagger, @nestjs/typeorm, @nestjs/mongoose, @nestjs/graphql, @nestjs/apollo,
@apollo/server, bullmq, validador de clases, Graphql o Graphql-ws.
