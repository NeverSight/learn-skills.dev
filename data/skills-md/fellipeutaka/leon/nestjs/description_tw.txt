使用 TypeScript 建立生產就緒的 REST API、GraphQL API 和微服務的 NestJS 最佳實踐。
在編寫、審查或重構 NestJS 程式碼時使用：控制器、模組、提供者、
依賴注入、守衛、攔截器、管道、異常過濾器、中間件、自訂
裝飾器、DTO、驗證、身份驗證、授權、JWT、配置、測試、
資料庫 (TypeORM/Prisma)、快取、佇列 (BullMQ)、OpenAPI/Swagger、WebSockets、
GraphQL（解析器、突變、訂閱、ObjectType、InputType、ArgsType、程式碼優先、
架構優先、PubSub）、Helmet、CORS、CSRF、生命週期掛鉤、正常關閉以及
功能模組架構。也使用時
使用@nestjs/jwt、@nestjs/throttler、@nestjs/terminus、@nestjs/config、
@nestjs/swagger、@nestjs/typeorm、@nestjs/mongoose、@nestjs/graphql、@nestjs/apollo、
@apollo/server、bullmq、類別驗證器、graphql 或 graphql-ws。
