Analyse et explique les décisions d'architecture de gestion des API Azure pour les implémentations du marché des API d'entreprise à l'aide du mode interne du réseau virtuel, de la porte d'entrée, de l'authentification hybride et des stratégies multi-environnements. À utiliser pour discuter de la sélection des composants APIM, de la topologie du réseau, de l'optimisation des coûts ou pour comparer des alternatives telles que les espaces de travail par rapport aux instances, le mode interne du réseau virtuel par rapport au mode externe ou Front Door par rapport à Application Gateway.
