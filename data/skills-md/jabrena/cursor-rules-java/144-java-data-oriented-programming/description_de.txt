Verwenden Sie diese Option, wenn Sie Best Practices für die datenorientierte Programmierung in Java anwenden müssen – einschließlich der Trennung von Code (Verhalten) von Datenstrukturen mithilfe von Datensätzen, der Gestaltung unveränderlicher Daten mit reinen Transformationsfunktionen, der Flachhaltung und Denormalisierung von Daten mit ID-basierten Referenzen, beginnend mit der Konvertierung generischer Datenstrukturen in bestimmte Typen bei Bedarf, der Gewährleistung der Datenintegrität durch reine Validierungsfunktionen und der Erstellung flexibler generischer Datenzugriffsschichten.
