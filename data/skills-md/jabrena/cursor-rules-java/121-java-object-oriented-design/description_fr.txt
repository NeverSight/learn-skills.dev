À utiliser lorsque vous devez réviser, améliorer ou refactoriser le code Java pour une qualité de conception orientée objet, notamment en appliquant les principes SOLID, DRY et YAGNI, en améliorant la conception des classes et des interfaces, en corrigeant les utilisations abusives du concept POO (encapsulation, héritage, polymorphisme), en identifiant et en résolvant les odeurs de code (God Class, Feature Envy, Data Clumps) ou en améliorant les modèles de création d'objets, la conception des méthodes et la gestion des exceptions. Une partie du projet skills-for-java
