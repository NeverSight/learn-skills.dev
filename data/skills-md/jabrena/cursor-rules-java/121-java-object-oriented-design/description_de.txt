Verwenden Sie es, wenn Sie Java-Code für objektorientierte Designqualität überprüfen, verbessern oder umgestalten müssen – einschließlich der Anwendung von SOLID-, DRY- und YAGNI-Prinzipien, der Verbesserung des Klassen- und Schnittstellendesigns, der Behebung des Missbrauchs von OOP-Konzepten (Kapselung, Vererbung, Polymorphismus), der Identifizierung und Auflösung von Code-Smells (God Class, Feature Envy, Data Clumps) oder der Verbesserung von Objekterstellungsmustern, Methodendesign und Ausnahmebehandlung.
