Этот навык следует использовать, когда пользователь просит «написать документацию», «задокументировать это», «добавить README», «создать журнал изменений», «написать ежедневный журнал», «задокументировать эту услугу», «написать ADR», «создать технический отчет», «добавить JSDoc», «добавить строки документации», «задокументировать этот API», «написать вскрытие», «создать документацию по архитектуре», «обновить документацию», «написать руководство по миграции», «задокументировать это решение», «создать обзор сервиса», «напишите TESTS.md» или спрашивает, как документировать любой код, сервис, функцию или архитектурное решение. Охватывает как постоянную документацию (ежедневные журналы, журналы изменений, документы API), так и разовую документацию (отчеты, ADR, анализы, руководства по миграции) для любого технологического стека.
