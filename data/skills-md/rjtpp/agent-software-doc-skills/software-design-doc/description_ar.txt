قم بصياغة ومراجعة وتحديث أوصاف تصميم البرامج باستخدام بنية مستوحاة من IEEE 1016-2009 مع إضفاء الطابع الرسمي الواضح على البنية/العروض/العناصر والتحقق من صحة بنية المخرجات. استخدم هذا عندما يطلب المستخدم كتابة SDD، أو تقييم جودة/اكتمال SDD، أو محاذاة مستندات التصميم مع مفاهيم IEEE 1016، أو تعيين متطلبات PRD للتصميم، أو إنتاج أقسام تصميم البنية/الواجهة/البيانات، أو إنشاء تقرير فجوات مع إجراءات المعالجة، أو إجراء تحليل الفجوات لمراجعة SDD فقط، أو تحديث SDD بعد تغييرات البنية.
