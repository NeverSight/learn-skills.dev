Redactar, revisar y actualizar descripciones de diseño de software utilizando una estructura inspirada en IEEE 1016-2009 con formalización explícita de arquitectura/vistas/elementos y validación de la estructura de salida. Utilice esto siempre que un usuario solicite escribir un SDD, evaluar la calidad/integridad del SDD, alinear los documentos de diseño con los conceptos IEEE 1016, mapear los requisitos de PRD para el diseño, producir secciones de diseño de arquitectura/interfaz/datos, generar un informe de brechas con acciones correctivas, realizar un análisis de brechas de solo revisión de SDD o actualizar un SDD después de cambios en la arquitectura.
