Redigere, rivedere e aggiornare le descrizioni della progettazione del software utilizzando una struttura ispirata a IEEE 1016-2009 con formalizzazione esplicita di architettura/viste/elementi e convalida della struttura di output. Utilizzalo ogni volta che un utente chiede di scrivere un SDD, valutare la qualità/completezza dell'SDD, allineare i documenti di progettazione ai concetti IEEE 1016, mappare i requisiti PRD per la progettazione, produrre sezioni di progettazione di architettura/interfaccia/dati, generare un rapporto sulle lacune con azioni correttive, eseguire analisi delle lacune di sola revisione dell'SDD o aggiornare un SDD dopo le modifiche dell'architettura.
