Entwerfen, überprüfen und aktualisieren Sie Software-Designbeschreibungen mithilfe einer von IEEE 1016-2009 inspirierten Struktur mit expliziter Architektur-/Ansichts-/Elementformalisierung und Ausgabestrukturvalidierung. Verwenden Sie dies immer dann, wenn ein Benutzer eine SDD schreiben, die SDD-Qualität/-Vollständigkeit beurteilen, Designdokumente an IEEE 1016-Konzepten ausrichten, PRD-Anforderungen für das Design abbilden, Architektur-/Schnittstellen-/Datendesignabschnitte erstellen, einen Lückenbericht mit Abhilfemaßnahmen erstellen, eine SDD-Lückenanalyse nur zur Überprüfung durchführen oder eine SDD nach Architekturänderungen aktualisieren möchte.
