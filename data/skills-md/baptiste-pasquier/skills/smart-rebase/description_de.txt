Helfen Sie dem Benutzer, einen Feature-Zweig sicher auf „main“, „develop“ oder einen anderen Zielzweig umzubasieren und gleichzeitig das Feature-Verhalten bei Konflikten beizubehalten. Verwenden Sie diese Fähigkeit immer dann, wenn der Benutzer einen Rebase oder eine Aktualisierung eines langlebigen Zweigs, eine Wiedergabe einer Funktion auf einer neueren Basis oder die Lösung von Rebase-Konflikten nach Refaktorierungen, umbenannten APIs, verschobenen Dateien, Typänderungen oder neuen Konventionen im Zielzweig wünscht.
