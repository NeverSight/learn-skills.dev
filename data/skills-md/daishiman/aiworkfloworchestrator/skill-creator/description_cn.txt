用于创建、更新和改进提示的元技能。  
从抽象的想法到具体的实现，以**协作**的方式与用户交互共创，灵活适应任何阶段。  
在 **编排** 模式下，选择任务执行引擎（Claude Code / Codex / 集成）。

锚点：  
• 持续交付 (Jez Humble) / 应用：自动化管道 / 目的：确定性执行  
• 精益创业（Eric Ries）/应用：构建-测量-学习/目的：迭代改进  
• 领域驱动设计（Eric Evans）/应用：战略设计、通用语言、有界上下文/目的：领域结构的清晰性  
• 简洁架构（Robert C. Martin）/应用：依赖规则、分层分离/目的：稳健、高精度的技能，能够适应变化  
• 设计思维（IDEO）/应用：以用户为中心的设计/目的：同理心和共同创造

触发：  
在创建新技能、更新现有技能或改进提示时使用。
