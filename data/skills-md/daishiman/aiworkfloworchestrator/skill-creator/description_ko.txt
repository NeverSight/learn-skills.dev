프롬프트를 생성, 업데이트, 개선하기 위한 메타 기술입니다.  
**협업** 모드에서는 대화를 통해 사용자와 공동 창작하며 추상적인 아이디어에서 구체적인 구현으로 유연하게 이동합니다.  
**오케스트레이트** 모드에서는 작업 실행 엔진(Claude Code / Codex / Integration)을 선택합니다.

앵커:  
• Continuous Delivery(Jez Humble) / 적용: 자동화된 파이프라인 / 목적: 결정론적 실행  
• 린 스타트업(Eric Ries) / 적용: 빌드-측정-학습 / 목적: 반복 개선  
• 도메인 중심 디자인(Eric Evans) / 적용: 전략적 디자인, 유비쿼터스 언어, 제한된 컨텍스트 / 목적: 도메인 구조 명확화  
• Clean Architecture (Robert C. Martin) / 적용: 종속성 규칙, 계층적 분리 / 목적: 강력하고 정확도가 높은 기술  
• 디자인씽킹(IDEO) / 적용 : 사용자 중심 디자인 / 목적 : 공감과 공동창작

트리거:  
새로운 기술을 생성하거나, 기존 기술을 업데이트하거나, 프롬프트를 다듬을 때 사용됩니다.
