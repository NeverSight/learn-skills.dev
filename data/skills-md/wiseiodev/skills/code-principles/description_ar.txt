مبادئ هندسة البرمجيات لمراجعة التعليمات البرمجية والتحقق من صحة التخطيط والتعلم. يغطي SOLID وDRY وYAGNI واستدلالات التصميم والاختبار وتصميم واجهة برمجة التطبيقات (API) وأنماط الهندسة المعمارية وأفضل ممارسات React. يُستخدم عند مراجعة التعليمات البرمجية، أو التحقق من صحة الخطط/المواصفات، أو إدخال معايير المراجعة في جلسات المجلس، أو عندما يسأل المستخدم عن مبدأ محدد. مشغلات على "المبادئ"، أو "قائمة مراجعة التعليمات البرمجية"، أو "SOLID"، أو "DRY"، أو "استدلالات التصميم"، أو "معايير المراجعة"، أو "أفضل الممارسات"، أو أي مبدأ مسمى مثل "قانون ديميتر"، أو "Liskov"، أو "YAGNI"، وما إلى ذلك.
