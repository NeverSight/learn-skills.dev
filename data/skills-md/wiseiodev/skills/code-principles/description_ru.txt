Принципы разработки программного обеспечения для проверки кода, проверки планирования и обучения. Охватывает SOLID, DRY, YAGNI, эвристику проектирования, тестирование, проектирование API, шаблоны архитектуры и лучшие практики React. Используйте при проверке кода, проверке планов/спецификаций, внедрении критериев проверки в заседания совета или когда пользователь спрашивает о конкретном принципе. Триггеры по «принципам», «контрольному списку проверки кода», «SOLID», «DRY», «эвристике проектирования», «критериям проверки», «лучшим практикам» или любому названному принципу, например «Закон Деметры», «Лисков», «YAGNI» и т. д.
