Synchronisez les commits de la branche de développement actuelle (généralement dev) avec la branche principale via git rebase, poussez vers la branche distante, puis revenez à la branche d'origine. Déclenchez cette compétence lorsque l'utilisateur mentionne des expressions telles que « sync to main », « rebase to main », « update main », « push to main » ou « fusionner dev into main ». Même si l'utilisateur ne dit pas explicitement « rebase », tant que l'intention est de synchroniser le contenu de la branche de développement avec celui de main, cette compétence doit être activée.
