Erstellen Sie hochwertige Git-Commits: Überprüfen/stufen Sie beabsichtigte Änderungen ein, teilen Sie sie in logische Commits auf und schreiben Sie klare Commit-Nachrichten (im Anschluss an herkömmliche Commits, mit Emoji-Unterstützung). Verwenden Sie diese Fähigkeit, wenn der Benutzer Code festschreiben, Festschreibungsnachrichten schreiben, Änderungen durchführen oder die Arbeit in mehrere Festschreibungen aufteilen möchte. Wenn der Benutzer das Schlüsselwort „Breaking Change“ erwähnt, muss die Commit-Nachricht der BREAKING CHANGE-Spezifikation dieses Skills folgen und das Ausrufezeichenformat verwenden. Priorisieren Sie das Festschreiben von Dateien im Git-Staging-Bereich. Berücksichtigen Sie den gesamten Arbeitsbaum nur, wenn der Staging-Bereich leer ist. Wenn der Benutzer das Schlüsselwort „kategorisieren“ erwähnt, befolgen Sie die Multi-Commit-Aufteilungsspezifikation dieses Skills und teilen Sie sorgfältig in mehrere Commits aus den vier Dimensionen Dateityp, Geschäftsmodul, Änderungstyp und Änderungsbereich auf.
