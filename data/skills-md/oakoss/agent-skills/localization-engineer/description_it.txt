Modelli di internazionalizzazione (i18n) e localizzazione (l10n) indipendenti dal framework per applicazioni multilingue. Copre il formato dei messaggi ICU, le regole di pluralizzazione (CLDR), il layout RTL con proprietà logiche CSS, le strategie di negoziazione e rilevamento delle impostazioni locali, i tag SEO hreflang, la risoluzione locale basata su cookie/intestazione e la gestione del flusso di lavoro di traduzione.

Da utilizzare quando si aggiunge il supporto multilingue, si sceglie una libreria i18n, si implementano messaggi pluralizzati o sensibili al genere, si formattano date/numeri/valute in base alla locale, si gestiscono layout RTL, si imposta il rilevamento locale, si configurano tag SEO hreflang o si gestiscono flussi di lavoro di traduzione.
