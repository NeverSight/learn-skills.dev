Discipline de refactorisation rouge-vert pour l'écriture de code piloté par des tests. Couvre le cycle TDD, la qualité des tests, les limites de simulation et la conception pour la testabilité.

À utiliser lorsque : implémentez des fonctionnalités, corrigez des bogues, refactorisez du code existant, concevez des interfaces de module ou travaillez dans des tranches verticales à partir d'un flux de travail de traceurs.
