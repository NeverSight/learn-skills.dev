Disciplina de refactor rojo-verde para escribir código impulsado por pruebas. Cubre el ciclo TDD, la calidad de las pruebas, los límites simulados y el diseño para la capacidad de prueba.

Úselo cuando: implemente funciones, corrija errores, refactorice código existente, diseñe interfaces de módulos o trabaje dentro de cortes verticales de un flujo de trabajo de trazadores.
