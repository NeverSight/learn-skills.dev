文檔驅動的專案開發工作流程。智慧判斷任務類型，執行已有計劃或使用 feature-dev 產生新計劃。

**使用場景：**
- 已有完整專案文檔，需要執行開發任務
- 使用者指定執行某個計劃（如"執行 plan 001"）
- 使用者要求開發新功能（自動使用 feature-dev 產生計劃）

**前置條件：**
- 專案應該已有 docs/ 文件結構（PRD、SAD 等）
- 如果是新項目，建議先使用 project-docs-setup skill 建立完整文檔

**觸發方式：**
- "執行 plan 001" / "繼續 001-user-authentication"
- "開始開發" / "執行計劃" / "繼續上次的任務"
- "實作使用者認證功能" / "新增 XXX 模組"
