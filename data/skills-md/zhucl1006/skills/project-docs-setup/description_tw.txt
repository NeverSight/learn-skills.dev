專案文檔結構建立助理。身為資深產品經理，透過多輪對話深入了解專案需求，分析合理性並提供最佳實務建議，最終產生完整的技術文件系統。

**使用時機：**
- 新專案啟動，需要建立文件體系（建議第一步驟）
- 現有項目缺乏規範文檔，需補充完善
- 已有程式碼但缺少文檔，需要根據程式碼回補文檔
- 需要專業的產品和架構分析建議

**支援兩種工作模式：**
- 模式 A：新專案文件建立（從需求到文件）
- 模式 B：程式碼回補文件（從程式碼到文件）

**與 project-workflow 的關係：**
- 本 skill 負責建立完整文件（PRD、SAD、開髮指南等）
- project-workflow 負責基於文件執行開發任務
- 建議流程：project-docs-setup → project-workflow

**觸發方式：**
- "建立專案文件" / "初始化專案文件" / "設定專案文件"
- "根據程式碼產生文件" / "回補專案文件" / "補充專案文件"
- 直接描述專案需求，如"幫我創建一個線上教育平台的文件"
