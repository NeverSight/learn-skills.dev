Cette compétence doit être utilisée lors de l'audit d'une base de code pour vérifier l'état de préparation des agents d'IA, ou pour guider les améliorations visant à faire fonctionner une base de code avec des outils de codage agent. Cela s'applique lorsque les utilisateurs demandent à évaluer la couverture des tests, la structure des fichiers, l'utilisation du système de type, la vitesse de l'environnement de développement ou l'application automatisée - les cinq piliers qui déterminent l'efficacité avec laquelle les agents de codage peuvent fonctionner dans un projet. Déclencheurs sur « auditer ma base de code », « préparer cet agent », « améliorer pour les agents IA », « convivialité pour les agents » ou des questions sur les raisons pour lesquelles les agents ont des difficultés avec une base de code.
