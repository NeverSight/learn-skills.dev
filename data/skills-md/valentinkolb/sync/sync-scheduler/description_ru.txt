Используйте этот навык при реализации распределенного планирования cron с помощью планировщика @валентинколб/синхронизации: регистрация идемпотентных расписаний между модулями, диспетчеризация с ограничением по лидерам через job.submit, политика ошибок (skip/catch_up_one/catch_up_all), триггерNow для ручной отправки, отмены регистрации, показателей/работоспособности и выборов лидера для нескольких модулей. Зависит от задания синхронизации для выполнения и синхронизации мьютекса для блокировки лидера.
