Полный рабочий процесс разработки git: определение ветки по умолчанию, создание веток функций с интеллектуальным именованием, выполнение атомарных коммитов в соответствии с соглашениями о репо и открытие запросов на включение с помощью GitHub Issues или ссылок Jira. Используйте, когда вас просят «начать работу», «создать ветку», «зафиксировать», «открыть PR», «отправить изменения» или любую задачу полного цикла разработки.
