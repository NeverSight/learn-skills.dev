Typgesteuertes Designprinzip: Unstrukturierte Daten an Systemgrenzen in strukturierte Typen umwandeln, wodurch illegale Zustände nicht mehr darstellbar sind. Wird zum Schreiben oder Überprüfen von Code verwendet, der Eingaben validiert, Datentypen entwirft, Funktionssignaturen definiert, Fehler behandelt oder Domänenlogik modelliert. Verwenden Sie diese Option, wenn Sie Validierungsfunktionen sehen, die ungültig/undefiniert zurückgeben, redundante Nullprüfungen, stringtypisierte Daten, boolesche Flags, die das Verhalten steuern, oder Funktionen, die Daten empfangen können, die sie nicht sollten. Löst aus bei: „Parsen, nicht validieren“, „Typgesteuertes Design“, „Unzulässige Zustände nicht darstellbar machen“, „Eingabevalidierung“, „Datenmodellierung“, „Typen umgestalten“, „Typen stärken“, „Intelligenter Konstruktor“, „Neuer Typ“, „Markentyp“.
