Verhaltenstestmethodik – testen Sie, was Benutzer erleben, nicht wie der Code strukturiert ist. Verwenden Sie es, wenn Sie Tests schreiben, die Testqualität überprüfen, eine Teststrategie für neue Funktionen planen oder wenn vorhandene Tests spröde/ausführlich/an Implementierungsdetails gekoppelt sind. Auslöser: Schreiben von Tests, TDD, Testüberprüfung, „Tests gehen ständig kaputt“, „zu viele Mocks“, „Tests sind ausführlich“, Planung der Testabdeckung, verhaltensgesteuerte Entwicklung.
