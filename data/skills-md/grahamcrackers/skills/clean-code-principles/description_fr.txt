Principes de code propre pour des bases de code TypeScript et React lisibles et maintenables. Couvre la dénomination, les fonctions, l'abstraction, la composition, la gestion des erreurs, les commentaires et les odeurs de code. À utiliser lors de l'écriture d'un nouveau code, de la refactorisation, de la révision de la qualité du code ou lorsque l'utilisateur pose des questions sur le code propre, la lisibilité ou la maintenabilité.
