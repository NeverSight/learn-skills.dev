Générez des classes Apex prêtes pour la production pour Salesforce en suivant les meilleures pratiques de l'entreprise. Couvre les classes de service, les sélecteurs, les classes de domaine, les lots/files d'attente/planifiables, les DTO, les utilitaires, les interfaces, les classes abstraites et les exceptions personnalisées. Pas pour les déclencheurs ou les tests unitaires. Utilisez cette compétence chaque fois que l'utilisateur demande à créer, écrire, construire, générer ou échafauder une classe Apex, y compris lorsqu'il décrit des fonctionnalités qui nécessiteraient une classe Apex sans indiquer explicitement « classe Apex ». Déclenchez des expressions telles que « créer un service pour », « créer un gestionnaire », « J'ai besoin d'une classe dans laquelle », « écrire Apex dans », « échafauder un travail par lots », « créer une file d'attente » ou toute requête impliquant la logique côté serveur Salesforce. À utiliser également lorsque l'utilisateur demande à refactoriser, améliorer ou restructurer les classes Apex existantes. Même si la demande est vague, comme « J'ai besoin de quelque chose qui traite les opportunités la nuit », utilisez cette compétence si la solution implique une classe Apex.
