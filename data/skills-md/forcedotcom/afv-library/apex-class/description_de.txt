Generieren Sie produktionsbereite Apex-Klassen für Salesforce gemäß den Best Practices für Unternehmen. Deckt Dienstklassen, Selektoren, Domänenklassen, Batch-/Warteschlangen-/planbare Klassen, DTOs, Dienstprogramme, Schnittstellen, abstrakte Klassen und benutzerdefinierte Ausnahmen ab. Nicht für Trigger oder Unit-Tests. Verwenden Sie diese Fähigkeit immer dann, wenn der Benutzer eine Apex-Klasse erstellen, schreiben, erstellen, generieren oder ein Gerüst erstellen möchte – auch wenn er Funktionen beschreibt, die eine Apex-Klasse erfordern würden, ohne explizit „Apex-Klasse“ zu sagen. Lösen Sie auf Phrasen wie „Erstellen Sie einen Dienst für“, „Erstellen Sie einen Handler“, „Ich benötige eine Klasse, die“, „Schreiben Sie Apex in“, „Gerüst für einen Batch-Job“, „Erstellen Sie eine Warteschlangendatei“ oder jede Anforderung aus, die serverseitige Salesforce-Logik beinhaltet. Verwenden Sie es auch, wenn der Benutzer vorhandene Apex-Klassen umgestalten, verbessern oder umstrukturieren möchte. Auch wenn die Anfrage vage ist, etwa „Ich brauche etwas, das Opportunities jede Nacht verarbeitet“, verwenden Sie diese Fähigkeit, wenn die Lösung eine Apex-Klasse beinhalten würde.
