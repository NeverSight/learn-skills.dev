사용자가 Gemini CLI(gemini-cli) 도구를 설치, 구성 또는 사용하려고 할 때마다 이 기술을 사용하십시오. npm/npx/brew를 통한 gemini-cli 설치, 인증 설정(API 키, Google OAuth, Vertex AI), -p 플래그를 사용하여 비대화형/헤드리스 프롬프트 실행, settings.json 구성, GEMINI.md 컨텍스트 파일 생성, 사용자 정의 슬래시 명령(.toml 파일) 작성, MCP 서버 연결, 확장 생성, 쉘 스크립트로 작업 자동화, --output-format json/stream-json 사용, 채팅 세션 관리, /memory 명령, --auto-approve 모드, ADC(애플리케이션 기본 자격 증명) 또는 gemini-cli와 관련된 스크립팅/자동화를 사용합니다. 또한 사용자가 Gemini 모델을 CLI 워크플로, CI/CD 파이프라인에 통합하거나 CLI 도구를 통해 Gemini API를 프로그래밍 방식으로 사용하는 방법을 묻는 경우에도 트리거됩니다.
