Habilidad de prueba automatizada de aplicaciones móviles. Utilice esta habilidad siempre que el usuario quiera probar una aplicación móvil (Android o iOS), escribir casos de prueba, analizar la estructura de la aplicación, ejecutar pruebas de IU automatizadas mediante un emulador o generar informes de prueba. Se activa cuando el usuario menciona: "probar mi aplicación", "ejecutar pruebas", "pruebas de UI", "escribir casos de prueba", "verificar la funcionalidad de la aplicación", "probar en el emulador", "control de calidad móvil", "cobertura de pruebas", "pruebas de casos de uso", "pruebas de escenarios de usuario" o cualquier combinación de móvil + prueba/verificación/verificación/validación. También se activa cuando el usuario carga o hace referencia a un APK, .ipa o una carpeta de proyecto móvil (React Native, Flutter, Android, iOS) y pregunta qué hacer a continuación. Utilice siempre esta habilidad para cualquier tarea de control de calidad de una aplicación móvil, incluso tareas parciales como "simplemente escriba algunos casos de uso" o "muéstreme qué pruebas debo ejecutar".
