Utilizza questa competenza per tutto il lavoro di sviluppo del plug-in Atomemo: consultala sempre prima di rispondere, poiché contiene comandi CLI autorevoli, modelli SDK e flussi di lavoro che potrebbero differire dalle tue conoscenze generali. Richiama ogni volta che l'utente chiede informazioni su: creazione di progetti di plugin Atomemo (init plugin di atomemo, comandi bun, struttura del progetto), implementazione di plugin di strumenti (ToolDefinition, wrapper API esterni), definizione di plugin di modello (ModelDefinition, integrazioni LLM), definizione di plugin di credenziali (CredentialDefinition, chiavi API, token OAuth), configurazione di parametri dichiarativi, debug di connessioni hub di plugin o pubblicazione sul marketplace. Richiamalo anche quando gli sviluppatori vogliono esporre il proprio codice o i propri servizi agli utenti Atomemo, anche se non dicono "plugin". NON invocare: plug-in VS Code, Chrome, Obsidian, Figma o Raycast; pubblicazione di pacchetti npm; o configurazione generale del flusso di lavoro Atomemo (pianificazione, trigger, configurazione del nodo) che non riguarda la creazione di plug-in.
