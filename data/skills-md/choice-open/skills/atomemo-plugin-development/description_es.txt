Utilice esta habilidad para todo el trabajo de desarrollo del complemento Atomemo; consúltela siempre antes de responder, ya que contiene comandos CLI autorizados, patrones de SDK y flujos de trabajo que pueden diferir de su conocimiento general. Invoque cada vez que el usuario pregunte sobre: ​​crear proyectos de complementos Atomemo (complemento atomemo init, comandos bun, estructura del proyecto), implementar complementos de herramientas (ToolDefinition, envoltorios de API externos), definir complementos de modelo (ModelDefinition, integraciones LLM), definir complementos de credenciales (CredentialDefinition, claves API, tokens OAuth), configurar parámetros declarativos, depurar conexiones de centros de complementos o publicar en el mercado. También invoca cuando los desarrolladores quieran exponer su código o servicios a los usuarios de Atomemo, incluso si no dicen "complemento". NO invoque para: complementos VS Code, Chrome, Obsidian, Figma o Raycast; publicación de paquetes npm; o configuración general del flujo de trabajo de Atomemo (programación, activadores, configuración de nodos) que no se trata de crear complementos.
