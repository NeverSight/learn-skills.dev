Kritische Analyse- und Ausführungsfähigkeiten für React/TypeScript-Code-Refactoring.  
Wenn der Benutzer einen Komponentenpfad und seine Probleme angibt, wird jedes Problem aus der Perspektive des „sauberen Codes“ kritisch überprüft, sequenzielles Denken wird zur schrittweisen Analyse angewendet und ein Refactoring-Plan wird erstellt und ausgeführt.  

Auslöser: Wird verwendet, wenn Schlüsselwörter wie „Refactoring“, „Verbessern wollen“, „Codestrukturänderung“, „Hook-Trennung“, „Kohäsion“, „Kopplung“ usw. zusammen mit einem React-Komponentenpfad erwähnt werden.
