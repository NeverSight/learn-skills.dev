Capacità di analisi critica ed esecuzione per il refactoring del codice React/TypeScript.  
Quando l'utente fornisce il percorso di un componente e i relativi problemi, ogni problema viene esaminato criticamente da una prospettiva di "codice pulito", viene applicato il pensiero sequenziale per analizzare passo dopo passo e viene creato ed eseguito un piano di refactoring.  

Trigger: utilizzato quando parole chiave come "refactoring", "voglio migliorare", "modifica della struttura del codice", "separazione degli hook", "coesione", "accoppiamento", ecc. vengono menzionate insieme al percorso di un componente React.
