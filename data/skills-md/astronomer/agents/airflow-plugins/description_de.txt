Erstellen Sie Airflow 3.1+-Plugins, die FastAPI-Apps, benutzerdefinierte UI-Seiten, React-Komponenten, Middleware, Makros und Operator-Links direkt in die Airflow-Benutzeroberfläche einbetten. Verwenden Sie diese Fähigkeit immer dann, wenn der Benutzer ein Airflow-Plugin erstellen, eine benutzerdefinierte UI-Seite oder einen Navigationseintrag zu Airflow hinzufügen, FastAPI-gestützte Endpunkte in Airflow erstellen, statische Assets über ein Plugin bereitstellen, eine React-App in die Airflow-Benutzeroberfläche einbetten, Middleware zum Airflow-API-Server hinzufügen, benutzerdefinierte Operator-Extra-Links erstellen oder die Airflow-REST-API aus einem Plugin heraus aufrufen möchte. Wird auch ausgelöst, wenn der Benutzer AirflowPlugin, fastapi_apps, external_views, React_apps, Plugin-Registrierung oder die Einbettung einer Web-App in Airflow 3.1+ erwähnt. Wenn jemand in Airflow 3.1+ etwas Benutzerdefiniertes erstellt, das Python und eine browserbasierte Oberfläche beinhaltet, ist diese Fähigkeit mit ziemlicher Sicherheit anwendbar.
