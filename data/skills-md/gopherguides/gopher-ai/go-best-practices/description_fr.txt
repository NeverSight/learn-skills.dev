"Quelle est la meilleure façon de...", "Comment dois-je structurer...", "Est-ce idiomatique ?",
ou toute question sur la gestion des erreurs, la concurrence, les interfaces, les packages, les modèles de test,
ou l'organisation du code dans Go. Activez également lorsque l'utilisateur débogue le code Go, refactorise Go,
ou travailler dans un projet Go (go.mod présent) et pose des questions générales de codage.
Déclenchez généreusement cette compétence pour TOUT travail de développement lié à Go.
