Навыки проверки кода на основе локальных различий git. Используется для самопроверки перед отправкой запроса на запрос, строгого повторного использования трехмерных стандартов PR-проверки (стандарты качества кода, рекомендации по стилю FastGPT, контрольный список общих проблем), поддержки проверки неустановленных изменений, поэтапных изменений, конкретного коммита или всех изменений текущей ветки относительно базовой ветки.
