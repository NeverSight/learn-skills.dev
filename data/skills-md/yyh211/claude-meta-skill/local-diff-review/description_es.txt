Habilidades de revisión de código basadas en git diff local. Se utiliza para la autorrevisión antes de enviar un PR, reutilizando estrictamente los criterios tridimensionales de la revisión previa (estándares de calidad del código, pautas de estilo FastGPT, lista de verificación de problemas comunes), respaldando la revisión de cambios no preparados, cambios preparados, una confirmación específica o todos los cambios en la rama actual en relación con la rama de referencia.
