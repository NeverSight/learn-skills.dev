Cette compétence doit être utilisée lorsque l'utilisateur demande de « créer une fonctionnalité », « implémenter une fonctionnalité », « ajouter des fonctionnalités », « démarrer le développement », « écrire des tests d'acceptation », « écrire des spécifications », « utiliser ATDD », « utiliser TDD avec des tests d'acceptation » ou commence tout travail d'implémentation de fonctionnalité. Également déclenché par la commande /atdd. Applique le workflow de développement piloté par les tests d'acceptation : rédigez les spécifications Given/When/Then avant le code, générez un pipeline de test spécifique au projet et gérez deux flux de test.
