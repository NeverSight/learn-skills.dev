6 つのカテゴリ (スキーマ、セキュリティ、セマンティック、ベスト プラクティス、スタイル、アクションlint) にわたる 46 のルールを使用して、GitHub Actions ワークフロー ファイルを分析して、セキュリティの脆弱性、セマンティック エラー、ベスト プラクティス違反、スキーマの問題、およびスタイルの問題を分析します。ワークフローを 0 ～ 100 のスケールで評価し、文字のグレード (A+ ～ F) を付けます。 YAML の前後の例を含む詳細な修正推奨事項を生成します。ユーザーがレビューのために GitHub Actions ワークフローを共有する場合、CI/CD YAML を貼り付ける場合、GitHub Actions のベスト プラクティスやセキュリティ強化について質問する場合、ワークフローのセキュリティ体制を改善したい場合、.github/workflows について言及する場合、または CI/CD パイプライン設定について議論する場合には、たとえ「検証」や「分析」と明示的に言わなくても、このスキルを積極的に使用してください。
