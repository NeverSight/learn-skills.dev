Analysez les manifestes Kubernetes pour détecter les vulnérabilités de sécurité, les problèmes de fiabilité, les violations des meilleures pratiques, les erreurs de schéma, les problèmes de référence entre ressources et les erreurs de configuration RBAC à l'aide de 67 règles réparties dans 5 catégories (sécurité, fiabilité, meilleures pratiques, schéma, ressources croisées/RBAC). Les scores se manifestent sur une échelle de 0 à 100 avec des notes alphabétiques (A+ à F) et un résumé de conformité PSS de base/restreint. Génère des recommandations de correctifs détaillées avec des exemples YAML avant/après. Utilisez cette compétence de manière PROACTIVE chaque fois qu'un utilisateur partage un manifeste Kubernetes pour examen, colle le YAML de K8 avec les champs apiVersion/kind, pose des questions sur les meilleures pratiques de K8, les normes de sécurité des pods, les références CIS, la sécurité RBAC ou souhaite améliorer la sécurité de son cluster, même s'il ne dit pas explicitement « valider » ou « analyser ». Déclenchez sur les déploiements, les StatefulSets, les DaemonSets, les services, les entrées, les ressources RBAC et tout YAML K8 multi-documents.
