End-to-End-Koordinator für PR-Rezensionen. Ruft GitHub PR-Unterschiede über .diff-URLs ab, um Codeänderungen zusammenzufassen, löst eine CI-Überprüfung mithilfe des „review-ci“-Skills aus und führt, wenn CI eine Vorschau-URL verfügbar macht, „review-ui“ für diese URL und relevante Szenarien aus. Verwenden Sie diese Option, wenn der Benutzer eine ganzheitliche Überprüfung eines GitHub-Pull-Requests einschließlich Code, CI-Pipelines und (falls möglich) der bereitgestellten Benutzeroberfläche wünscht.
