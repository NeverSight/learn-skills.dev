Вызывайте только тогда, когда пользователь явно запрашивает «проверить/оптимизировать цитаты в тексте обзора `{topic}_review.tex`» или «запустить check-review-alignment».

Используя семантическое понимание принимающего ИИ, построчно проверяйте каждую цитату на соответствие литературе; **только при обнаружении фатальной ошибки цитирования** выполнить минимальную переписывание «предложения, содержащего цитату», и повторно использовать сценарий рендеринга `systematic-literature-review` для вывода PDF/Word (скрипт не вызывает локальный API LLM).

Основной принцип: **не меняйтесь ради изменения**. Если невозможно определить, является ли это фатальной ошибкой, оставьте все как есть и пометьте предупреждение в отчете.

⚠️ Не распространяется на:
- Пользователи, которые просто хотят создать текст систематического обзора (используйте систематический обзор литературы).
- Пользователи, которые хотят только добавлять/проверять записи BibTeX (используйте специальный рабочий процесс управления нагрудниками)
