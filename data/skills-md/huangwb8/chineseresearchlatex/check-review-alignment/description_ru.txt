Используется, когда пользователь явно просит «проверить/оптимизировать внутритекстовое цитирование обзора `{Topic}_review.tex`» или «запустить check-review-alignment».

Семантическое понимание ИИ хоста используется для проверки соответствия цитат содержанию документа одна за другой. Только при обнаружении фатальных ошибок цитирования «предложения, содержащие цитаты», минимально переписываются, а сценарий рендеринга `systematic-literature-review` повторно используется для вывода PDF/Word (скрипт не вызывает напрямую LLM API локально).

Основной принцип: **Не меняйтесь ради изменений**. Если ошибку невозможно определить как фатальную, оставьте ее как есть и предупредите в отчете.

⚠️Не применяются следующие ситуации:
- Пользователь просто хочет сгенерировать текст систематического обзора (необходимо использовать systematic-literature-review)
- Пользователь просто хочет добавить/проверить записи BibTeX (следует использовать специальный процесс управления нагрудниками)
