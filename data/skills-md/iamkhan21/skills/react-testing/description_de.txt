Testen von React-Komponenten, Hooks und Interaktionen mit der React Testing Library. Verwenden Sie diese Option, wenn Sie neue Tests schreiben, fehlgeschlagene Tests debuggen, Testcode umgestalten, eine Testinfrastruktur einrichten, APIs in React-Tests verspotten oder wenn der Benutzer Jest/Vitest mit React, „render“, „screen“, „userEvent“, „waitFor“, „findBy“, MSW oder Testmustern erwähnt. Verwenden Sie es auch, wenn der Benutzer fragt, wie eine bestimmte React-Komponente oder ein bestimmter React-Hook getestet werden soll, auch wenn er „Testbibliothek“ nicht explizit erwähnt.
