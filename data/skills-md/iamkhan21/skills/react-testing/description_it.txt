Testare componenti, hook e interazioni di React con la libreria di test di React. Da utilizzare durante la scrittura di nuovi test, il debug di test non riusciti, il refactoring del codice di test, l'impostazione dell'infrastruttura di test, la simulazione di API nei test React o quando l'utente menziona Jest/Vitest con React, `render`, `screen`, `userEvent`, `waitFor`, `findBy`, MSW o modelli di test. Da utilizzare anche quando l'utente chiede come testare uno specifico componente o hook di React, anche se non menziona esplicitamente la "libreria di test".
