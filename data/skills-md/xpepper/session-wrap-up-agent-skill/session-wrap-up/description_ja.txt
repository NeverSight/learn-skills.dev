コーディング セッションの終了時、ハンドオフ前、またはユーザーが学習内容を取得したいときに実行します。会話と最近の git 変更を確認し、修正、決定、新しいワークフロー、フォローアップ タスク、自動化の機会、および明らかではない落とし穴を特定し、適切なプロジェクト ドキュメントとエージェント指示ファイル (AGENTS.md、CLAUDE.md、コパイロット指示、カーソル ルール、またはプロジェクトの同等物) を更新し、将来のセッションのために永続的なレッスンを保持します。
「セッションのまとめ」と明示的に言わなくても、ユーザーがまとめをしたいとき、作業を引き継ぎたいとき、学んだことを保存したいとき、セッションに基づいてドキュメントやエージェントの指示を更新したいとき、レッスンを記録したいとき、落とし穴を記録したいとき、決定事項を保存したいとき、次のステップをメモしたいとき、何をコミットすべきかを尋ねたいとき、または次回回避するために繰り返される間違いを文書化したいときは、このスキルを使用します。
