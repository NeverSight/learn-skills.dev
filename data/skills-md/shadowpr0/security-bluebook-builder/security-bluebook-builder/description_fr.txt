Créez ou affinez une politique de sécurité concise et normative (« Blue Book ») pour les applications sensibles. À utiliser lorsque les utilisateurs ont besoin d'un modèle de menace, de règles de classification des données, d'une politique d'authentification/de session, d'exigences de journalisation et d'audit, d'attentes de conservation/suppression, de réponse aux incidents ou de barrières de sécurité pour les applications gérant les données PII/PHI/financières.
