Umfassende GitHub-Pull-Request-Überprüfung mit der gh-CLI. Überprüft PR-Unterschiede zu verknüpften Problemen, Repo-Konventionen und Codebasiskontext. Prüft auf Fehler, Sicherheitsrisiken, Stilkonsistenz und Zielausrichtung. Führt verfügbare Lint-, Typprüfungs- und Testbefehle aus. Erstellt einen priorisierten Bericht mit einem Genehmigungs-/Blockierungsurteil. Verwenden Sie diese Fähigkeit immer dann, wenn der Benutzer „eine PR überprüfen“ oder „PR überprüfen“ fragt
