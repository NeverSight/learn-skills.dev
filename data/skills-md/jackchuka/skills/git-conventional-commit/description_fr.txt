Créez des commits git en suivant la spécification Conventional Commits v1.0.0 (conventionalcommits.org). À utiliser lorsque l'utilisateur demande de valider les modifications, dit "/conventional-commit" ou souhaite un message de validation bien structuré. Se déclenche sur des requêtes telles que « valider ceci », « valider mes modifications », « créer une validation » ou tout autre workflow de validation git. Analyse les modifications planifiées/non planifiées et produit des messages de validation conformes avec le type, la portée, la description, le corps et les pieds de page appropriés.
