قم بإنشاء التزامات git وفقًا لمواصفات الالتزامات التقليدية v1.0.0 (conventionalcommits.org). يُستخدم عندما يطلب المستخدم تنفيذ التغييرات، أو يقول "/conventional-commit"، أو يريد رسالة التزام جيدة التنظيم. يتم تشغيل طلبات مثل "الالتزام بهذا"، أو "الالتزام بالتغييرات الخاصة بي"، أو "إنشاء التزام"، أو أي سير عمل للالتزام في git. يحلل التغييرات المرحلية/غير المرحلية وينتج رسائل التزام متوافقة مع النوع والنطاق والوصف والنص والتذييلات المناسبة.
