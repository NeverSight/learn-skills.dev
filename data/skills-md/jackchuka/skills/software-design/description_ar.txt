دليل رأي لمبادئ تصميم البرمجيات والأنماط المعمارية. يُستخدم عند مراجعة تصميم التعليمات البرمجية، أو تخطيط بنية الميزات، أو طرح السؤال "هل هذا هو التصميم الصحيح؟"، أو "كيف يمكنني هيكلة هذا؟"، أو طلب إرشادات فلسفة التصميم. يثير أسئلة حول SOLID، أو DRY، أو KISS، أو YAGNI، أو الهندسة المعمارية النظيفة، أو DDD، أو الهندسة المعمارية السداسية، أو التكوين مقابل الميراث، أو الاقتران، أو التماسك، أو أي مناقشة للمقايضة في تصميم البرامج.
