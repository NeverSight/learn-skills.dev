Génère un sous-agent critique sceptique pour tester sous pression le plan, la proposition ou l’approche actuelle avant son exécution. Le critique inspecte activement la base de code pour vérifier les affirmations, puis renvoie les préoccupations classées par gravité. L’agent principal doit prendre une décision précise sur chaque point et montrer ce qui a réellement changé. Utilisez-le pour les travaux à risque moyen/élevé (architecture, migrations, authentification, concurrence, performances, modifications sensibles à la sécurité, refactors inter-fichiers). Ignorez les petites modifications à faible risque. Si demandé sur une petite tâche, exécutez le mode léger (max 3 résultats).
