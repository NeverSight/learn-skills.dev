Genera un subagente crítico escéptico para probar el plan, propuesta o enfoque actual antes de su ejecución. El crítico inspecciona activamente el código base para verificar las afirmaciones y luego devuelve inquietudes clasificadas por gravedad. El agente principal debe tomar una decisión concreta sobre cada punto y mostrar qué cambió realmente. Úselo para trabajos de riesgo medio/alto (arquitectura, migraciones, autenticación, concurrencia, rendimiento, cambios sensibles a la seguridad, refactorizaciones entre archivos). Omita las pequeñas ediciones de bajo riesgo. Si se solicita en una tarea pequeña, ejecute el modo ligero (máximo 3 resultados).
