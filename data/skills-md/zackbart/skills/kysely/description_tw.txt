Kysely 的型別安全 SQL 查詢產生器助手。每當使用者使用 Kysely 編寫資料庫查詢、設定 Kysely 資料庫實例、定義資料庫類型、編寫遷移或詢問如何在 Kysely 中執行 SQL 操作（SELECT、INSERT、UPDATE、DELETE、JOIN、CTE、MERGE、交易）時，請使用此技能。當使用者在程式碼中提及 kysely、從「kysely」匯入、具有帶表類型的資料庫介面或詢問 TypeScript 中的類型安全性 SQL 時也會觸發。也會觸發常見的拼字錯誤，例如“kaisley”和“kysley”。即使他們只是說「查詢建構器」或「我如何在 kysely 中執行 X」或貼上使用 selectFrom/insertInto/updateTable/deleteFrom 模式的程式碼，也可以啟動。此技能涵蓋關係式 (jsonArrayFrom/jsonObjectFrom)、表達式建構器、透過 sql 標籤的原始 SQL、條件查詢、可重複使用幫助器、模式管理、遷移、執行管道/調試、插件和所有 Kysely 模式。
