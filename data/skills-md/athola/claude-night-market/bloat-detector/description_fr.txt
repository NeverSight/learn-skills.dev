Détectez l’encombrement de la base de code grâce à une analyse progressive : code mort, duplication, complexité, encombrement de la documentation. À utiliser lorsque l'utilisation du contexte est élevée, la maintenance trimestrielle, le nettoyage avant la publication, avant la refactorisation. Ne pas utiliser en cas de développement de fonctionnalités actif, de bogues urgents, de base de code < 1 000 lignes.
