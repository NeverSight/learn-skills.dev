Untersuchen Sie ein Open-Source-Projekt, bevor Sie mit ihm interagieren – PRs, Probleme oder Kommentare. Verwenden Sie VORHER: Einreichen eines Problems, Einreichen einer PR oder Kommentieren eines bestehenden Threads. Auslöser bei: „Upstream-Aufklärung“, „Sollte ich dies PR machen“, „Werden sie zusammengeführt“, „Projekt prüfen“, „Repo untersuchen“, „PR-Strategie“, „Ein Problem einreichen“, „Vorhandene Probleme prüfen“, „Soll ich einen Kommentar abgeben“ oder jederzeit, wenn der Benutzer mit einem Repo interagieren möchte, das er nicht verwaltet. Verwenden Sie es auch proaktiv, wenn Sie ein Problem oder eine PR einreichen möchten – die Überprüfung vorhandener Threads verhindert Duplikate und verschwendeten Aufwand.
