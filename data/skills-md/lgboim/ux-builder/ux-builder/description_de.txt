Verwenden Sie diese Fähigkeit immer dann, wenn der Benutzer ein Produkt (Website, App, Formular, Zielseite, Checkout, Dashboard) erstellen, UX/UI-Designs prüfen oder überprüfen, Konversionsraten (CRO) verbessern, Barrierefreiheitskonformität erreichen oder Formulare, Navigation, Suche oder Produktseiten optimieren möchte. Lösen Sie bei jeder Erwähnung von: UX, UI, Design, Wireframe, Mockup, Prototyp, Landingpage, Checkout, Formular, Konvertierung, Zugänglichkeit, Benutzerfreundlichkeit oder Anfragen zur Verbesserung einer digitalen Schnittstelle aus. Wird auch ausgelöst, wenn der Benutzer einen Screenshot oder eine URL für Design-Feedback teilt. Wendet über 150 binäre Pass/Fail-Regeln mit präzisen quantifizierten Daten an (z. B. „oben ausgerichtete Etiketten 28 % schneller“, „35,2 % aggregierte Conversion-Steigerung“). Verwenden Sie diese Fähigkeit immer für UX/UI-Aufgaben.
