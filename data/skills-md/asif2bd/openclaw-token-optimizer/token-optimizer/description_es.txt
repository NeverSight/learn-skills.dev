Reduzca el uso de tokens de OpenClaw y los costos de API mediante enrutamiento de modelos inteligentes, optimización de latidos, seguimiento de presupuesto y funciones nativas de 2026.2.15 (poda de sesión, límites de tamaño de arranque, alineación de TTL de caché). Úselo cuando los costos de los tokens sean altos, se alcancen los límites de tasa de API o se alojen varios agentes a escala. Los 4 scripts ejecutables (context_optimizer, model_router, heartbeat_optimizer, token_tracker) son solo locales: sin solicitudes de red, sin llamadas a subprocesos, sin modificaciones del sistema. Los archivos de referencia (PROVIDERS.md, config-patches.json) documentan estrategias multiproveedor opcionales que requieren claves API externas y acceso a la red si decide utilizarlas. Consulte SECURITY.md para obtener un desglose completo.
