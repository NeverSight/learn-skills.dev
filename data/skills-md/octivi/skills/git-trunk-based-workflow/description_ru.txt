Планируйте и выполняйте рабочие процессы разработки на основе Git Trunk в репозитории, используя main в качестве магистрали. Используйте всякий раз, когда пользователь спрашивает о стратегии ветки, именовании ветки, перебазировании на главную, конфликтах слияния во время перебазирования, потоке PR, стратегии слияния, безопасной перезаписи истории или очистке ветки, даже если они явно не упоминают «на основе магистрали». Для форматирования сообщения о коммите и формулировки обычных коммитов делегируйте полномочия git-commits.
