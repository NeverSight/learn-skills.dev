إدارة المهام وتتبع التقدم وتسجيل التحديثات اليومية عبر ai-todo CLI. المحفزات: 1) صريح - يذكر المستخدم المهام أو المهام أو تتبع التقدم أو الاستعداد أو التخطيط السريع أو إدارة المشروع أو سجلات العمل. 2) ما بعد الإجراء - بعد تنفيذ/دفع git، أو النشر، أو دمج العلاقات العامة، أو الإصدار، أو أي حدث رئيسي، قم بتسجيل التقدم بشكل استباقي وتحديث/إكمال المهام ذات الصلة. عند انتهاء أدوات git أو vercel:deploy، قم دائمًا بتشغيل هذه المهارة. 3) الإجراء المسبق — عندما يناقش المستخدم خطط التنفيذ، أو التصميم المعماري، أو تفاصيل الميزات، أو التخطيط السريع، اقترح إنشاء المهام. عند اكتمال عملية العصف الذهني، قم بتحويل النتائج إلى مهام. 4) SESSION-END — بعد إجراء تغييرات جوهرية في التعليمات البرمجية، اقترح ملخصًا للتقدم.
