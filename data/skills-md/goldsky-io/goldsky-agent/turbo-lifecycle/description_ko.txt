Goldsky Turbo의 파이프라인 상태 관리 - 규칙 및 안전 동작을 사용하여 명령을 일시 중지, 재개, 다시 시작 및 삭제합니다. 사용자가 질문할 때 이 기술을 사용하십시오. 파이프라인을 삭제하면 postgres/clickhouse 테이블에 이미 있는 데이터가 손실됩니까? 데이터베이스 유지 관리를 수행하는 동안 파이프라인을 일시 중지하는 방법, 모든 기록 데이터를 다시 처리하기 위해 블록 0에서 다시 시작하는 방법, 실행 중인 스트리밍 파이프라인을 업데이트할 수 있습니까, 아니면 삭제 및 재배포해야 합니까, 일시 중지된 파이프라인을 중단된 위치(체크포인트)부터 다시 시작합니까, 완료된 작업 파이프라인을 처음부터 다시 실행하는 방법, 작업 모드 파이프라인을 일시 중지하거나 다시 시작할 수 있습니까? 또한 삭제 시 체크포인트 상태에 어떤 일이 발생하는지, 종료 후 1시간 후에 작업이 자동 삭제되는지에 대해 설명합니다. 파이프라인이 손상되거나 오류가 발생하는 이유를 적극적으로 진단하려면 대신 /turbo-doctor를 사용하세요.
