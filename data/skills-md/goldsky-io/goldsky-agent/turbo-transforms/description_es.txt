Escriba SQL, TypeScript y transformaciones de tablas dinámicas para canalizaciones Goldsky Turbo. Utilice esta habilidad para: decodificar registros de eventos de EVM con _gs_log_decode (requiere ABI) o entradas de transacciones con _gs_tx_decode, filtrar y transmitir datos de blockchain en SQL, combinar múltiples tipos de eventos decodificados en una tabla con UNION ALL, escribir transformaciones de TypeScript/WASM usando la firma de la función invoke(data), configurar tablas de búsqueda dinámica para filtrar transferencias por una lista de billetera que actualiza en tiempo de ejecución (dynamic_table_check), encadenar SQL y TypeScript pasos juntos o depurar valores nulos en campos decodificados. Para una estructura YAML de canalización completa, use /turbo-pipelines en su lugar. Para construir una tubería completa de un extremo a otro, use /turbo-builder en su lugar.
