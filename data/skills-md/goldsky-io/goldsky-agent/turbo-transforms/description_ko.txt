Goldsky Turbo 파이프라인을 위한 SQL, TypeScript 및 동적 테이블 변환을 작성합니다. 이 기술을 사용하는 용도: _gs_log_decode(ABI 필요)로 EVM 이벤트 로그 디코딩 또는 _gs_tx_decode로 트랜잭션 입력, SQL에서 블록체인 데이터 필터링 및 캐스팅, UNION ALL을 사용하여 여러 디코딩된 이벤트 유형을 하나의 테이블로 결합, 호출(데이터) 함수 시그니처를 사용하여 TypeScript/WASM 변환 작성, 런타임에 업데이트하는 지갑 목록(dynamic_table_check)으로 전송을 필터링하도록 동적 조회 테이블 설정, SQL 및 TypeScript 단계를 함께 연결 또는 디코딩된 필드에서 null 값을 디버깅합니다. 전체 파이프라인 YAML 구조의 경우 /turbo-pipelines를 대신 사용하세요. 전체 파이프라인을 엔드 투 엔드로 구축하려면 대신 /turbo-builder를 사용하세요.
