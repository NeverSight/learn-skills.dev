Scrivi trasformazioni di tabelle SQL, TypeScript e dinamiche per le pipeline Goldsky Turbo. Utilizzare questa competenza per: decodificare registri eventi EVM con _gs_log_decode (richiede ABI) o input di transazione con _gs_tx_decode, filtrare e trasmettere dati blockchain in SQL, combinare più tipi di eventi decodificati in un'unica tabella con UNION ALL, scrivere trasformazioni TypeScript/WASM utilizzando la firma della funzione invoke(data), impostare tabelle di ricerca dinamica per filtrare i trasferimenti in base a un elenco di portafogli aggiornato in fase di runtime (dynamic_table_check), concatenare passaggi SQL e TypeScript insieme o eseguire il debug di valori null nei campi decodificati. Per la struttura YAML completa della pipeline, utilizzare invece /turbo-pipelines. Per creare un'intera pipeline end-to-end, utilizzare invece /turbo-builder.
