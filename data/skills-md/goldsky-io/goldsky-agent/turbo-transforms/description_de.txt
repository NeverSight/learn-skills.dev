Schreiben Sie SQL-, TypeScript- und dynamische Tabellentransformationen für Goldsky Turbo-Pipelines. Verwenden Sie diese Fähigkeit für: Dekodieren von EVM-Ereignisprotokollen mit _gs_log_decode (erfordert ABI) oder Transaktionseingaben mit _gs_tx_decode, Filtern und Umwandeln von Blockchain-Daten in SQL, Kombinieren mehrerer dekodierter Ereignistypen in einer Tabelle mit UNION ALL, Schreiben von TypeScript/WASM-Transformationen mithilfe der invoke(data)-Funktionssignatur, Einrichten dynamischer Nachschlagetabellen zum Filtern von Übertragungen anhand einer Wallet-Liste, die Sie zur Laufzeit aktualisieren (dynamic_table_check), Verketten von SQL und Gemeinsame Schritte von TypeScript oder Debuggen von Nullwerten in decodierten Feldern. Für eine vollständige Pipeline-YAML-Struktur verwenden Sie stattdessen /turbo-pipelines. Um eine gesamte Pipeline durchgängig zu erstellen, verwenden Sie stattdessen /turbo-builder.
