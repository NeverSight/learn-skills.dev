Goldsky Turbo パイプライン用の SQL、TypeScript、および動的テーブル変換を作成します。このスキルは次の目的で使用します: _gs_log_decode (ABI が必要) による EVM イベント ログのデコード (ABI が必要) または _gs_tx_decode によるトランザクション入力、SQL でのブロックチェーン データのフィルタリングとキャスト、UNION ALL による複数のデコードされたイベント タイプの 1 つのテーブルへの結合、invoke(data) 関数シグネチャを使用した TypeScript/WASM 変換の作成、実行時に更新するウォレット リスト (dynamic_table_check) による転送をフィルタリングするための動的ルックアップ テーブルの設定、 SQL と TypeScript のステップを一緒に連鎖させたり、デコードされたフィールドの null 値をデバッグしたりできます。完全なパイプライン YAML 構造の場合は、代わりに /turbo-pipelines を使用してください。パイプライン全体をエンドツーエンドで構築するには、代わりに /turbo-builder を使用します。
