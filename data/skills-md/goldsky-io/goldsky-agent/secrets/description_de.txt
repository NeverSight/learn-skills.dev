Verwenden Sie diese Fähigkeit, wenn ein Benutzer Goldsky-Geheimnisse speichern, verwalten oder damit arbeiten möchte – die benannten Anmeldeinformationsobjekte, die von Pipeline-Senken verwendet werden. Dazu gehören: Erstellen eines neuen Geheimnisses aus einer Verbindungszeichenfolge oder Anmeldeinformationen, Auflisten oder Überprüfen vorhandener Geheimnisse, Aktualisieren oder Rotieren von Anmeldeinformationen nach einer Kennwortänderung und Löschen nicht mehr benötigter Geheimnisse. Auslöser für jede Abfrage, bei der der Benutzer „Goldsky-Geheimnis“ erwähnt, Datenbankanmeldeinformationen für eine Pipeline sicher speichern möchte oder mit Senkenauthentifizierung für PostgreSQL, Neon, Supabase, ClickHouse, Kafka, S3, Elasticsearch, DynamoDB, SQS, OpenSearch oder Webhooks arbeitet.
