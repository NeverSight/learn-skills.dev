Utilice esta habilidad cuando un usuario desee almacenar, administrar o trabajar con secretos de Goldsky: los objetos de credenciales con nombre utilizados por los receptores de canalización. Esto incluye: crear un nuevo secreto a partir de una cadena de conexión o credenciales, enumerar o inspeccionar secretos existentes, actualizar o rotar credenciales después de un cambio de contraseña y eliminar secretos que ya no son necesarios. Activador de cualquier consulta en la que el usuario mencione "goldsky secret", quiera almacenar de forma segura las credenciales de la base de datos para una canalización o esté trabajando con autenticación de receptor para PostgreSQL, Neon, Supabase, ClickHouse, Kafka, S3, Elasticsearch, DynamoDB, SQS, OpenSearch o webhooks.
