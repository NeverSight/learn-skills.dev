Verwenden Sie diese Fähigkeit, wenn der Benutzer die Syntax der Goldsky Turbo-CLI nachschlagen muss – Flags und Optionen für „inspect“, „logs“ oder „list“-Befehle – oder Tastaturkürzel für die „turbo inspect“-TUI kennen möchte. Auch zum Dekodieren der Bedeutung einer bestimmten Fehlermeldung (Rückstau, verweigerte Verbindung, Authentifizierungsfehler usw.), zum Verwenden des Skripts „analysate-logs.sh“ oder zum Verstehen der Funktionsweise von inspect/logs-Befehlen verwenden. Unterscheidungsmerkmal: Diese Fähigkeit liefert Referenzinformationen und Erklärungen. Wenn der Benutzer eine defekte Pipeline hat und eine schrittweise interaktive Diagnose wünscht, verwenden Sie stattdessen /turbo-doctor.
