Erstellen und verwalten Sie Amazon S3-Buckets mit der EKS-Pod-Identitätsauthentifizierung. Wird verwendet, wenn: (1) S3-Buckets für jeden auf EKS ausgeführten Dienst erstellt werden, (2) die EKS-Pod-Identität eingerichtet wird, damit Pods ohne statische Anmeldeinformationen auf S3 zugreifen können, (3) IAM-Rollen/-Richtlinien für den S3-Zugriff konfiguriert werden, (4) Kubernetes-Dienste mit S3-Speicher auf EKS verbunden werden, (5) Fehlerbehebung beim S3-Zugriff von Kubernetes-Pods. Erfordert: aws CLI v2, kubectl, eksctl.
