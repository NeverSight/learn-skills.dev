Используйте при создании или повторении подробной спецификации технического проекта для каждой подсистемы из спецификации системы перед запуском рабочего процесса OpenSpec. Триггеры: «спецификация проекта», «спецификация подсистемы», «написание спецификации для S1», «разбивка этапов», «этапы реализации», «спецификация среднего уровня», «технический проект». Кодирует упрямую прогрессивную фазовую дисциплину с прогрессом FP ​​и границами контракта. НЕ используйте для системных спецификаций высокого уровня (используйте мозговой штурм) или для артефактов OpenSpec (используйте openspec напрямую).
