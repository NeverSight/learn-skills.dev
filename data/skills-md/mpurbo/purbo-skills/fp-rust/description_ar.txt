البرمجة الوظيفية في الصدأ - إرشادات مدروسة لكتابة رمز الصدأ الاصطلاحي FP-first. استخدم هذه المهارة عند كتابة كود Rust أو مراجعته أو إعادة هيكلته أو دعمه. يغطي: النواة الوظيفية / بنية الصدفة الحتمية، وأنماط الثبات أولاً، والوظائف النقية، وأنواع البيانات الجبرية، والبرمجة الموجهة لخطوط الأنابيب مع التكرارات وسلاسل النتائج، ومعالجة الأخطاء كبيانات، ورفض التبعية عبر حقن التبعية، والتصميم القائم على النوع (لا يتم التحقق من صحة التحليل، وحالة الكتابة)، والتزامن عبر تمرير الرسائل. يُستخدم أيضًا عندما يسأل المستخدم عن بنية مشروع Rust، أو أنماط معالجة أخطاء Rust، أو خطوط أنابيب مكرر Rust، أو إعداد Cargo.toml لمشروع Rust جديد، أو مراجعة كود Rust. قم بتطبيق هذه الإرشادات حتى عندما لا يذكر المستخدم صراحةً "البرمجة الوظيفية" - فهذا هو نمط الترميز الافتراضي لجميع أعمال Rust.
