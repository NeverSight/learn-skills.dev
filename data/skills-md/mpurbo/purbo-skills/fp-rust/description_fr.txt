Programmation fonctionnelle dans Rust - directives avisées pour l'écriture de code Rust idiomatique et FP-first. Utilisez cette compétence chaque fois que vous écrivez, révisez, refactorisez ou échafaudez du code Rust. Couvre : l'architecture de base fonctionnelle/shell impérative, les modèles axés sur l'immuabilité d'abord, les fonctions pures, les types de données algébriques, la programmation orientée pipeline avec des itérateurs et des chaînes de résultats, la gestion des erreurs en tant que données, le rejet des dépendances par rapport à l'injection de dépendances, la conception basée sur le type (l'analyse ne valide pas, l'état de type) et la concurrence via la transmission de messages. À utiliser également lorsque l'utilisateur pose des questions sur la structure du projet Rust, les modèles de gestion des erreurs Rust, les pipelines d'itérateur Rust, la configuration de Cargo.toml pour un nouveau projet Rust ou la révision du code Rust. Appliquez ces directives même lorsque l'utilisateur ne mentionne pas explicitement la « programmation fonctionnelle » — il s'agit du style de codage par défaut pour tous les travaux de Rust.
