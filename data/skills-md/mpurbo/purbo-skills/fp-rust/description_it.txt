Programmazione funzionale in Rust: linee guida supponenti per scrivere codice Rust idiomatico, FP-first. Usa questa abilità ogni volta che scrivi, rivedi, refactoring o impalcature del codice Rust. Copre: nucleo funzionale/architettura shell imperativa, modelli incentrati sull'immutabilità, funzioni pure, tipi di dati algebrici, programmazione orientata alla pipeline con iteratori e catene di risultati, gestione degli errori come dati, rifiuto delle dipendenze tramite inserimento delle dipendenze, progettazione basata sui tipi (analisi non convalidata, stato tipografico) e concorrenza tramite passaggio di messaggi. Da utilizzare anche quando l'utente chiede informazioni sulla struttura del progetto Rust, sui pattern di gestione degli errori di Rust, sulle pipeline dell'iteratore Rust, sulla configurazione di Cargo.toml per un nuovo progetto Rust o sulla revisione del codice Rust. Applica queste linee guida anche quando l'utente non menziona esplicitamente la "programmazione funzionale" — questo è lo stile di codifica predefinito per tutto il lavoro su Rust.
