Programación funcional en Rust: pautas obstinadas para escribir código Rust idiomático, primero en FP. Utilice esta habilidad siempre que escriba, revise, refactorice o cree andamiaje de código Rust. Cubre: núcleo funcional/arquitectura de shell imperativo, patrones de inmutabilidad primero, funciones puras, tipos de datos algebraicos, programación orientada a canalizaciones con iteradores y cadenas de resultados, manejo de errores como datos, rechazo de dependencia sobre inyección de dependencia, diseño basado en tipos (analizar, no validar, escribir estado) y concurrencia a través de paso de mensajes. Úselo también cuando el usuario pregunte sobre la estructura del proyecto Rust, los patrones de manejo de errores de Rust, las canalizaciones de iteradores de Rust, la configuración de Cargo.toml para un nuevo proyecto de Rust o la revisión del código de Rust. Aplique estas pautas incluso cuando el usuario no mencione explícitamente la "programación funcional": este es el estilo de codificación predeterminado para todo el trabajo de Rust.
