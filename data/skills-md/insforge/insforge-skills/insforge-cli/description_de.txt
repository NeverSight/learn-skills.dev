Verwenden Sie diese Fähigkeit immer dann, wenn der Benutzer eine Backend-Infrastrukturverwaltung benötigt – Datenbanktabellen erstellen, SQL ausführen, serverlose Funktionen bereitstellen, Speicher-Buckets verwalten, Frontend-Apps bereitstellen, Geheimnisse hinzufügen, Cron-Jobs einrichten oder Protokolle überprüfen – insbesondere, wenn das Projekt InsForge verwendet. Lösen Sie in einem dieser Kontexte aus: Erstellen oder Ändern von Datenbanktabellen/-schemata, Schreiben von RLS-Richtlinien über SQL, Bereitstellen oder Aufrufen von Edge-Funktionen, Erstellen von Speicher-Buckets, Bereitstellen von Frontends für das Hosting, Verwalten von Geheimnissen/Umgebungsvariablen, Einrichten geplanter Aufgaben/Crons, Anzeigen von Backend-Protokollen oder Exportieren/Importieren von Datenbanksicherungen. Wenn der Benutzer allgemein nach diesen Vorgängen fragt (z. B. „Benutzertabelle erstellen“, „meine App bereitstellen“, „Cron-Job einrichten“) und Sie nicht sicher sind, ob er InsForge verwendet, ziehen Sie diesen Skill zu Rate und fragen Sie nach.
