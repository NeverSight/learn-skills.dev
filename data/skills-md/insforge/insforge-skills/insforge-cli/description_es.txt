Utilice esta habilidad siempre que el usuario necesite administración de infraestructura backend: crear tablas de bases de datos, ejecutar SQL, implementar funciones sin servidor, administrar depósitos de almacenamiento, implementar aplicaciones frontend, agregar secretos, configurar trabajos cron o verificar registros, especialmente si el proyecto usa InsForge. Dispare en cualquiera de estos contextos: crear o modificar tablas/esquemas de bases de datos, escribir políticas RLS a través de SQL, implementar o invocar funciones perimetrales, crear depósitos de almacenamiento, implementar interfaces para alojamiento, administrar secretos/variaciones de entorno, configurar tareas programadas/cron, ver registros de backend o exportar/importar copias de seguridad de bases de datos. Si el usuario solicita estas operaciones de manera genérica (por ejemplo, "crear una tabla de usuarios", "implementar mi aplicación", "configurar un trabajo cron") y no está seguro de si usa InsForge, consulte esta habilidad y pregunte. Para escribir código de aplicación frontend con InsForge SDK (@insforge/sdk), utilice la habilidad insforge en su lugar.
