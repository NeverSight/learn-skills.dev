Utilisez cette compétence chaque fois que l'utilisateur a besoin de gérer l'infrastructure back-end : création de tables de base de données, exécution de SQL, déploiement de fonctions sans serveur, gestion des compartiments de stockage, déploiement d'applications frontales, ajout de secrets, configuration de tâches cron ou vérification des journaux, en particulier si le projet utilise InsForge. Déclenchez sur l'un de ces contextes : création ou modification de tables/schémas de base de données, écriture de politiques RLS via SQL, déploiement ou appel de fonctions Edge, création de compartiments de stockage, déploiement d'interfaces vers l'hébergement, gestion de secrets/vars d'environnement, configuration de tâches planifiées/cron, affichage des journaux backend ou exportation/importation de sauvegardes de base de données. Si l'utilisateur demande ces opérations de manière générique (par exemple, « créer une table d'utilisateurs », « déployer mon application », « configurer une tâche cron ») et que vous ne savez pas s'il utilise InsForge, consultez cette compétence et demandez. Pour écrire du code d’application frontend avec le SDK InsForge (@insforge/sdk), utilisez plutôt la compétence insforge.
