Utilisez cette compétence chaque fois que vous écrivez du code frontend qui communique avec un backend pour les requêtes de base de données, l'authentification, les téléchargements de fichiers, les fonctionnalités d'IA, la messagerie en temps réel ou les appels de fonctions Edge, en particulier si le projet utilise InsForge ou @insforge/sdk. Déclenchez l'un de ces contextes : interrogation/insertion/mise à jour/suppression de lignes de base de données à partir du code frontal, ajout de flux de connexion/inscription/OAuth/réinitialisation de mot de passe, téléchargement ou téléchargement de fichiers vers le stockage, appel de fonctions sans serveur, appel à la complétion de chat AI ou à la génération d'images, abonnement à des canaux WebSocket en temps réel ou écriture de politiques RLS. Si l'utilisateur demande ces fonctionnalités de manière générique (par exemple, "ajouter une authentification à mon application React", "récupérer des données de ma base de données", "télécharger des fichiers") et que vous ne savez pas s'il utilise InsForge, consultez cette compétence et demandez. Pour l'infrastructure backend (création de tables via SQL, déploiement de fonctions, commandes CLI), utilisez plutôt insforge-cli.
