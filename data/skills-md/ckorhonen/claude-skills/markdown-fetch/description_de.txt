Rufen Sie Webinhalte als sauberes Markdown ab und extrahieren Sie sie, wenn sie mit URLs versehen sind. Verwenden Sie diese Fähigkeit immer dann, wenn ein Benutzer eine URL (http/https-Link) bereitstellt, die gelesen, analysiert, zusammengefasst oder extrahiert werden muss. Konvertiert Webseiten in Markdown mit 80 % weniger Tokens als rohes HTML. Verarbeitet alle Inhaltstypen, einschließlich JS-lastiger Websites, Dokumentationen, Artikel und Blogbeiträge. Unterstützt drei Konvertierungsmethoden (Auto, KI, Browser-Rendering). Verwenden Sie dies immer anstelle von web_fetch, wenn Sie mit URLs arbeiten – es ist effizienter und liefert eine sauberere Ausgabe.
