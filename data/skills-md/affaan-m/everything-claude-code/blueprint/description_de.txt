Verwandeln Sie ein einstufiges Ziel in einen schrittweisen Bauplan für Engineering-Projekte mit mehreren Sitzungen und mehreren Agenten. Jeder Schritt verfügt über eine eigenständige Kontextbeschreibung, sodass ein neuer Agent ihn kalt ausführen kann. Beinhaltet ein kontradiktorisches Review-Gate, ein Abhängigkeitsdiagramm, eine parallele Schritterkennung, einen Anti-Pattern-Katalog und ein Planmutationsprotokoll. TRIGGER, wenn: Der Benutzer einen Plan, eine Blaupause oder eine Roadmap für eine komplexe Multi-PR-Aufgabe anfordert oder eine Arbeit beschreibt, die mehrere Sitzungen erfordert. NICHT AUSLÖSEN, wenn: die Aufgabe in einem einzigen PR oder in weniger als 3 Tool-Aufrufen abgeschlossen werden kann oder der Benutzer sagt: „Mach es einfach“.
