一行の目標を、マルチセッション、マルチエージェントのエンジニアリング プロジェクトの段階的な構築計画に変えます。各ステップには自己完結型のコンテキスト概要が含まれているため、新しいエージェントはコールドで実行できます。敵対的レビュー ゲート、依存関係グラフ、並列ステップ検出、アンチパターン カタログ、計画変更プロトコルが含まれます。トリガーのタイミング: ユーザーが複雑なマルチ PR タスクの計画、青写真、またはロードマップを要求する場合、または複数のセッションが必要な作業について説明する場合。次の場合はトリガーしないでください: タスクが 1 回の PR または 3 回未満のツール呼び出しで完了できる場合、またはユーザーが「とにかくやってください」と言った場合。
