Analysieren Sie rohe Eingabeaufforderungen, identifizieren Sie Absichten und Lücken, gleichen Sie ECC-Komponenten (Skills/Befehle/Agenten/Hooks) ab und geben Sie eine optimierte Eingabeaufforderung aus, die sofort eingefügt werden kann. Nur beratende Rolle – führt niemals die Aufgabe selbst aus. TRIGGER, wenn: Der Benutzer sagt „Eingabeaufforderung optimieren“, „Meine Eingabeaufforderung verbessern“, „Wie schreibe ich eine Eingabeaufforderung für“, „Hilf mir eine Eingabeaufforderung“, „Diese Eingabeaufforderung neu schreiben“ oder fragt ausdrücklich nach einer Verbesserung der Eingabeaufforderungsqualität. Löst auch chinesische Entsprechungen aus: „优化prompt“, „改进prompt“, „怎么写prompt“, „帮我优化这个指令“. NICHT AUSLÖSEN, wenn: Der Benutzer möchte, dass die Aufgabe direkt ausgeführt wird, oder sagt „Mach es einfach“ / „直接做“. NICHT AUSLÖSEN, wenn der Benutzer „优化代码“, „优化性能“, „Leistung optimieren“ oder „Diesen Code optimieren“ sagt – dabei handelt es sich um Refactoring-/Leistungsaufgaben, nicht um sofortige Optimierung.
