Identifie les risques de migration d'Oracle vers PostgreSQL en croisant le code avec les différences de comportement connues (chaînes vides, refcurseurs, coercition de type, tri, horodatages, transactions simultanées, etc.). À utiliser lors de la planification d'une migration de base de données, de l'examen des artefacts de migration ou de la validation du fait que les tests d'intégration couvrent les différences Oracle/PostgreSQL.
