Lisez cette compétence pour écrire, générer ou ajouter du code de test. Déclenché lorsque : (1) l'utilisateur demande d'écrire, d'ajouter, de créer ou de générer des tests de toute nature (tests unitaires, tests d'intégration, tests API, tests E2E, tests Playwright, tests Vitest) ; (2) l'utilisateur a du code avec des tests manquants ou nuls et souhaite une couverture ; (3) l'utilisateur vient d'implémenter un nouveau service, point de terminaison, fonctionnalité ou module et a besoin de tests pour celui-ci ; (4) l'utilisateur a refactorisé le code et souhaite vérifier que rien n'a été rompu avec les tests ; (5) l'utilisateur souhaite améliorer ou étendre la couverture de test existante. Cette compétence produit des suites de tests complètes en forme de pyramide : lit le code, sélectionne uniquement les couches nécessaires (unité/intégration/API/E2E) et génère tous les fichiers nécessaires (tests de schéma, tests de service, usines, aides, utilitaires de nettoyage, fichiers de spécifications) en suivant les modèles Vitest et Playwright stricts. Ignorez cette compétence lors du débogage de tests ayant échoué, en demandant comment utiliser les API ou les outils de test, en expliquant les concepts de test, en configurant les exécuteurs de tests, en révisant le code de test existant ou en migrant entre des frameworks de test.
