اقرأ هذه المهارة لكتابة أو إنشاء أو إضافة كود الاختبار. يتم التشغيل عندما: (1) يطلب المستخدم كتابة اختبارات من أي نوع أو إضافتها أو إنشائها أو إنشاءها (اختبارات الوحدة، اختبارات التكامل، اختبارات واجهة برمجة التطبيقات، اختبارات E2E، اختبارات الكاتب المسرحي، اختبارات Vitest)؛ (2) لدى المستخدم رمز به اختبارات مفقودة أو صفر ويريد التغطية؛ (3) قام المستخدم للتو بتنفيذ خدمة أو نقطة نهاية أو ميزة أو وحدة نمطية جديدة ويحتاج إلى اختبارات لها؛ (4) كود مُعاد هيكلته بواسطة المستخدم ويريد التحقق من عدم وجود أي خلل في الاختبارات؛ (5) يريد المستخدم تحسين أو توسيع تغطية الاختبار الحالية. تنتج هذه المهارة مجموعات اختبار كاملة على شكل هرم - تقرأ التعليمات البرمجية، وتحدد الطبقات الضرورية فقط (الوحدة/التكامل/API/E2E)، وتولد كل ملف مطلوب (اختبارات المخطط، واختبارات الخدمة، والمصانع، والمساعدين، وأدوات التنظيف المساعدة، وملفات المواصفات) باتباع أنماط Vitest وPlaywright الصارمة. تخطي هذه المهارة عند تصحيح أخطاء الاختبارات الفاشلة، أو السؤال عن كيفية استخدام واجهات برمجة التطبيقات أو الأدوات الخاصة بالاختبار، أو شرح مفاهيم الاختبار، أو تكوين مشغلي الاختبار، أو مراجعة كود الاختبار الحالي، أو الترحيل بين أطر الاختبار.
