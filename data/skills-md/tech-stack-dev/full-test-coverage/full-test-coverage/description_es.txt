Lea esta habilidad para escribir, generar o agregar código de prueba. Se activa cuando: (1) el usuario solicita escribir, agregar, crear o generar pruebas de cualquier tipo (pruebas unitarias, pruebas de integración, pruebas API, pruebas E2E, pruebas Playwright, pruebas Vitest); (2) el usuario tiene código al que le faltan pruebas o no tiene pruebas y desea cobertura; (3) el usuario acaba de implementar un nuevo servicio, punto final, característica o módulo y necesita pruebas para ello; (4) el usuario refactorizó el código y quiere verificar que nada haya roto con las pruebas; (5) el usuario desea mejorar o ampliar la cobertura de prueba existente. Esta habilidad produce conjuntos de pruebas completos en forma de pirámide: lee el código, selecciona solo las capas necesarias (unidad/integración/API/E2E) y genera todos los archivos necesarios (pruebas de esquema, pruebas de servicio, fábricas, ayudas, utilidades de limpieza, archivos de especificaciones) siguiendo estrictos patrones de Vitest y Playwright. Omita esta habilidad al depurar pruebas fallidas, preguntar cómo usar las API o herramientas de prueba, explicar conceptos de prueba, configurar ejecutores de pruebas, revisar el código de prueba existente o migrar entre marcos de prueba.
