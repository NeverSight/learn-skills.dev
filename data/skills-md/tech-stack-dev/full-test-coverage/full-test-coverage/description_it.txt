Leggi questa Skill per scrivere, generare o aggiungere codice di test. Si attiva quando: (1) l'utente chiede di scrivere, aggiungere, creare o generare test di qualsiasi tipo (test unitari, test di integrazione, test API, test E2E, test Playwright, test Vitest); (2) l'utente ha un codice con test mancanti o zero e desidera la copertura; (3) l'utente ha appena implementato un nuovo servizio, endpoint, funzionalità o modulo e necessita di test per esso; (4) l'utente ha rifattorizzato il codice e desidera verificare che nulla si sia rotto con i test; (5) l'utente desidera migliorare o espandere la copertura dei test esistenti. Questa abilità produce suite di test complete e a forma piramidale: legge il codice, seleziona solo i livelli necessari (unità/integrazione/API/E2E) e genera tutti i file necessari (test degli schemi, test dei servizi, fabbriche, helper, utilità di pulizia, file delle specifiche) seguendo rigorosi modelli Vitest e Playwright. Salta questa competenza quando si esegue il debug di test falliti, si chiede come utilizzare le API o gli strumenti di test, si spiegano i concetti di test, si configurano i test runner, si rivede il codice di test esistente o si esegue la migrazione tra framework di test.
