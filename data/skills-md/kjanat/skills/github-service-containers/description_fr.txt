Configurez les conteneurs de services Docker (Redis, PostgreSQL, etc.) en tant que services side-car dans les workflows GitHub Actions pour les tests d'intégration. À utiliser lors de l'ajout de bases de données, de caches ou de files d'attente de messages aux flux de travail CI, ou lors du débogage de la mise en réseau et des vérifications de l'état des conteneurs de services.
