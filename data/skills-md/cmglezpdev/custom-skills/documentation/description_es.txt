Cómo crear, actualizar y gestionar documentación para cualquier proyecto de software. Utilice esta habilidad siempre que se le solicite agregar documentación, actualizar documentos, crear un runbook, escribir un ADR, documentar una característica, agregar una especificación de producto, escribir guías de incorporación o cualquier tarea relacionada con la documentación. También se activa cuando se realizan cambios de código que cambian el comportamiento (características, correcciones, refactorizaciones, integraciones, cambios de infraestructura) que requieren actualizaciones de documentación adjuntas, incluso si el usuario no menciona explícitamente "documentos".
