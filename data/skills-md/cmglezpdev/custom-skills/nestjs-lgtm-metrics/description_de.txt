Fügen Sie Prometheus-Metriken auf Anwendungsebene zu einer NestJS-App hinzu, indem Sie das OpenTelemetry SDK und einen OTel Collector verwenden. Behandelt HTTP RED-Metriken, Node.js-Laufzeitmetriken, Geschäftsmetriken und erweiterte Observability-Muster. Verwenden Sie diesen Skill immer dann, wenn der Benutzer Metriken, Dashboards oder Überwachung zu einer NestJS-Anwendung hinzufügen möchte, Prometheus, Grafana-Metriken, OpenTelemetry-Metriken, OTel Collector erwähnt oder nach SLIs/SLOs oder Überwachung auf Anwendungsebene in einem NestJS-Kontext fragt. Wird auch ausgelöst, wenn der Benutzer benutzerdefinierte Zähler, Histogramme, Messgeräte oder Zusammenfassungen in NestJS wünscht. Dieser Skill konzentriert sich ausschließlich auf Metriken, die die ANWENDUNG ausgeben muss. Protokollierung, Ablaufverfolgung und Infra-Level-Metriken von cAdvisor, Node-Exporter, Postgres-Exporter oder Redis-Exporter werden nicht behandelt.
