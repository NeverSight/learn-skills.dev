Guidelines for redesigning a messy codebase into a clear package/module structure.  
Targets: (1) reorganizing messy code, (2) splitting a single huge module, (3) reviewing package boundaries, (4) designing the module hierarchy for new projects.  
Trigger: Initiated by structural-improvement requests such as “I want to review the package structure,” “module dependencies are too complex,” “I want to tidy up file placement,” or “I want to eliminate circular dependencies.”
