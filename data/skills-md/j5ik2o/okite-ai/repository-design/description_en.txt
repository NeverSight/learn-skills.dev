Provides design rules and anti-patterns for repositories in DDD. Checks aggregate-level naming conventions, method design based on CQS (Command Query Separation), and type constraints for inputs and outputs. Used to detect repository design issues during code reviews, new implementations, and refactoring. Target languages: language-agnostic (Java, Kotlin, Scala, TypeScript, Go, Rust, Python, etc.). Trigger: Activates on repository design-related requests such as “review repository design,” “repository name seems off,” “findById return value,” “repository returning DTOs,” “created repository based on table name,” “aggregate-level repository,” “repository anti-patterns,” “repository CQS.”
