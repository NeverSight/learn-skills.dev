Предоставляет правила проектирования и антишаблоны для репозиториев в DDD. Проверяет соглашения об именах для каждого агрегата, разработку методов на основе CQS (разделение командных запросов) и ограничения типа ввода/вывода. Используется для обнаружения проблем с дизайном репозитория во время проверки кода, новых реализаций или рефакторинга. Целевые языки: языко-независимые (Java, Kotlin, Scala, TypeScript, Go, Rust, Python и т. д.). Триггеры: запросы на проверку дизайна репозитория, такие как «проверка дизайна репозитория», «имя репозитория кажется неправильным», «тип возвращаемого значения findById», «репозиторий, возвращающий DTO», «созданный репозиторий для каждой таблицы», «репозиторий для каждого агрегата», «анти-шаблоны репозитория», «CQS репозитория».
