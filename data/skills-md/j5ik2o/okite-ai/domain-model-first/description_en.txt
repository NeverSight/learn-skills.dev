A domain-model-centric development procedure guide. It explains a method where you design and implement the domain model test-first, then proceed in the order of in-memory repository → use cases → infrastructure. Trigger: launched by development-process-related requests such as “I want to start from the domain model,” “TDD with DDD,” “how to build an in-memory repository,” “how to test use cases,” and “how to drive DDD development.”
