A skill that detects structures where backward compatibility mass-produces garbage code and supports governance that treats compatibility as a “contract and retirement plan.” Includes clarifying public API boundaries, institutionalizing deprecation cycles, localizing compatibility layers (Adapter/Strangler Fig), verifying compatibility via contract tests (CDC), and designing compatibility gates for AI-generated code. Used during code reviews, API design, refactoring, and legacy migration to prevent technical debt caused by compatibility. Target languages: language-agnostic (Java, TypeScript, Go, Python, Rust, etc.).  
Triggered by compatibility-management requests such as “I want to maintain backward compatibility,” “what to do with deprecated APIs,” “compatibility layers are growing,” “legacy migration strategy,” “API design review,” “too much compatibility code,” “I want to create a deprecation policy,” or “managing breaking changes.”
