Clarifiez la relation entre les agrégats et les limites des transactions, détectez et corrigez l'anti-modèle consistant à inclure plusieurs agrégats dans une seule transaction. Appliquez le principe selon lequel un agrégat est une limite de cohérence forte : lorsqu'un cas d'utilisation met à jour plusieurs agrégats, utilisez plutôt la cohérence éventuelle. À utiliser lors des révisions de code, de la conception de cas d'utilisation et de la refactorisation pour détecter les problèmes liés aux limites des transactions.  
Langages cibles : indépendants du langage (Java, Kotlin, Scala, TypeScript, Go, Rust, Python, etc.).  
Déclencheurs : requêtes sur les limites des transactions telles que « mettre à jour plusieurs agrégats dans la même transaction », « @Transactional sur un cas d'utilisation », « comment maintenir la cohérence entre les agrégats », « dois-je utiliser le modèle Saga », « concevoir les limites des transactions », « un agrégat par transaction », « mettre en œuvre une cohérence éventuelle », « transactions qui traversent les agrégats ».
