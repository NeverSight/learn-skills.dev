Habilidad para analizar código existente (no construido sobre DDD) y proponer un modelo de dominio (agregados, entidades locales, objetos de valor, servicios de dominio) con pseudocódigo. El diseño incluye propiedades, métodos, invariantes y límites agregados. Idioma de destino: independiente del idioma. Activador: activado por solicitudes como "Extraer un modelo de dominio del código existente", "Rediseñar este código con DDD", "Proponer un modelo de dominio", "Buscar los agregados", "Migrar a DDD", "Analizar el código existente y crear un modelo de dominio" o "Refactorizar el código heredado a DDD".
