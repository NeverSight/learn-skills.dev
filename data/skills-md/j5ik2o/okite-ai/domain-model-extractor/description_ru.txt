Проанализируйте существующий код (не созданный с помощью DDD), предложите модель предметной области (агрегаты, локальные сущности, объекты значений, сервисы предметной области) с помощью псевдокода. Предлагайте дизайн, включающий свойства, методы, инварианты, границы агрегатов. Целевой язык: языково-агностический. Триггер: запросы типа «извлечь модель предметной области из существующего кода», «хотеть перепроектировать этот код с помощью DDD», «предложить модель предметной области», «найти агрегаты», «перейти на DDD», «проанализировать существующий код и создать модель предметной области», «переместить устаревший код в DDD».
