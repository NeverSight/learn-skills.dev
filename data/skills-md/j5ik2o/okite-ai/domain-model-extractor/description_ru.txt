Проанализируйте существующий код (не созданный с учетом DDD), предложите модель предметной области (агрегаты, локальные сущности, объекты значений, сервисы предметной области) с помощью псевдокода. Обеспечьте проектирование, включая свойства, методы, инварианты и границы агрегатов. Целевой язык: языково-агностический.  
Триггер: активируется такими запросами, как «извлечь модель предметной области из существующего кода», «перепроектировать этот код с помощью DDD», «предложить модель предметной области», «найти агрегаты», «перейти на DDD», «проанализировать существующий код и создать модель предметной области», «переместить устаревший код в DDD».
