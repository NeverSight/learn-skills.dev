Analysez le code existant (non construit avec DDD à l'esprit), proposez un modèle de domaine (agrégats, entités locales, objets de valeur, services de domaine) avec pseudocode. Fournir une conception comprenant les propriétés, les méthodes, les invariants et les limites globales. Langue cible : indépendante de la langue.  
Déclencheur : activé par des requêtes telles que « extraire le modèle de domaine du code existant », « repenser ce code avec DDD », « proposer un modèle de domaine », « rechercher des agrégats », « migrer vers DDD », « analyser le code existant et créer un modèle de domaine », « déplacer le code existant vers DDD ».
