Un guide d'empaquetage de couche de domaine basé sur le modèle de module de DDD. Détecte les emballages axés sur la technologie (entités/, objets de valeur/, services/, référentiels/, etc.) et incite à une refactorisation vers un emballage basé sur des termes de domaine. Conduit à des noms de modules qui font partie du langage omniprésent dans la couche de domaine.  
Déclencheurs : requêtes liées aux modules de domaine telles que « structure de package de couche de domaine », « conception de module DDD », « souhait de corriger le packaging axé sur la technologie », « souhait de cesser d'utiliser des entités/dossier », « réviser les packages de domaine ».
