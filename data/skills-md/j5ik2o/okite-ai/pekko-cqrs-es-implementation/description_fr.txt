Un guide d'implémentation CQRS/Event Sourcing utilisant Apache Pekko + Scala 3. Fournit aux acteurs agrégés PersistenceEffector, la séparation du modèle de domaine et des acteurs, les transitions d'état de type sécurisé, la conception d'événements, la sérialisation des tampons de protocole, la couche de cas d'utilisation basée sur ZIO et les modèles de mise à jour du modèle de lecture. Langue cible : Scala 3 uniquement. À utiliser uniquement lorsque l'architecture CQRS/Event Sourcing est supposée. Condition de déclenchement : Activer uniquement lorsque la requête contient « Scala » et (« CQRS » ou « Event Sourcing » ou « Pekko »). N'utilisez pas cette compétence pour des langages autres que Scala ou des architectures autres que CQRS/ES. Déclencheurs : « Je souhaite implémenter CQRS/ES avec Pekko », « Je souhaite écrire des acteurs agrégés dans Scala », « Comment utiliser PersistenceEffector », « Source d'événements Pekko », « Implémentation du côté commande de CQRS dans Scala », « Gestion des transitions d'état avec Pekko » — activés par les requêtes d'implémentation Pekko + Scala + CQRS/ES. Non-déclencheurs : « Compromis CQRS », « Qu'est-ce que le sourcing d'événements », « CQRS en Java », « Sourçage d'événements dans Go », « Expliquer les concepts CQRS » – ne s'activent pas pour les questions conceptuelles ou les sujets non Scala/Pekko.
