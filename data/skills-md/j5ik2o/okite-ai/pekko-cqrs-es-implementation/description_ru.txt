Руководство по реализации CQRS/источника событий с использованием Apache Pekko + Scala 3. Предоставляет агрегированные актеры с PersistenceEffector, разделение модели предметной области и актеров, типобезопасные переходы между состояниями, проектирование событий, сериализацию буферов протоколов, уровень вариантов использования на основе ZIO и шаблоны обновления модели чтения.  
Целевой язык: только Scala 3. Используйте только в том случае, если предполагается архитектура CQRS/Event Sourcing.  
Условие триггера: активируется только в том случае, если запрос содержит одновременно «Scala» и «CQRS», «Event Sourcing» или «Pekko». Не используйте этот навык для языков, отличных от Scala, или архитектур, отличных от CQRS/ES.  
Триггеры: такие запросы, как «Я хочу реализовать CQRS/ES с помощью Pekko», «Я хочу написать агрегатные актеры в Scala», «Как использовать PersistenceEffector», «Источник событий Pekko», «Реализация командной части CQRS в Scala», «Управление переходами между состояниями с помощью Pekko», которые включают реализацию Pekko + Scala + CQRS/ES.  
Нетриггеры: не активируйте для «Компромиссов CQRS», «Что такое источник событий», «CQRS в Java», «Источник событий в Go», «Объяснение концепций CQRS» или любых концептуальных вопросов за пределами Scala/Pekko.
