The ability to create custom lint rules by leveraging each programming language’s existing linter ecosystem. This linter is not for humans but for AI agents; error messages function as prompts that instruct the AI on how to fix the code. Using standard tooling for each language—Rust (dylint), TypeScript/JavaScript (ESLint), Python (pylint), Go (golangci-lint), etc.—custom rules are created in a `lints/` directory. Use it when: (1) you want to enforce project-specific coding rules on AI, (2) you want a rule that, on violation, outputs an AI-readable correction prompt, (3) you want AI-driven enforcement of naming conventions, structural patterns, or consistency rules. Triggers: “create a linter rule”, “add a lint rule”, “enforce this pattern”, “AI linter”, “custom lint”, “code rules”, “naming rules”, “structure rules”.
