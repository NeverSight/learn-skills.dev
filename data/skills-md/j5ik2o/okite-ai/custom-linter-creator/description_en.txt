The skill of creating custom lint rules by leveraging existing linter ecosystems for each programming language. A linter intended not for humans but for AI agents, where error messages serve as correction prompt instructions for the AI. Create custom rules in the `lints/` directory using the standard approach for each language: Rust (dylint), TypeScript/JavaScript (ESLint), Python (pylint), Go (golangci-lint), etc. Use this when: (1) you want to enforce project-specific coding rules on the AI, (2) you want to write lint rules that output AI-readable correction instructions on violation, (3) you want to enforce naming conventions, structural patterns, or consistency rules through AI-driven linting.
