Навыки создания собственных правил линтера, используя существующие экосистемы линтера на различных языках программирования. Это линтер для агентов ИИ, а не для людей; Сообщения об ошибках функционируют как подсказки инструкций по исправлению для ИИ. Создавайте собственные правила в каталоге `lints/`, используя стандартные методы для каждого языка: Rust (dylint), TypeScript/JavaScript (ESLint), Python (pylint), Go (golangci-lint) и т. д. Используйте, когда: (1) вы хотите обеспечить соблюдение специфичных для проекта правил кодирования для ИИ, (2) вам нужны правила lint, которые выводят инструкции по исправлению ошибок, читаемые ИИ при нарушениях, (3) вы хотите обеспечить соблюдение соглашений об именах, структурных шаблонов или правил согласованности с помощью Линтинг на основе искусственного интеллекта. Триггеры: «создать правило линтера», «добавить правило линта», «применить этот шаблон», «ИИ-линтер», «пользовательский линт», «правила кода», «правила именования», «структурные правила», «создать правило линтера», «добавить правило линта», «применить этот шаблон», «ИИ-линтер».
