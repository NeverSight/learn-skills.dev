Навыки создания пользовательских правил линтера путем использования существующих экосистем линтера для каждого языка программирования. Линтер, предназначенный не для людей, а для агентов ИИ, где сообщения об ошибках служат инструкциями по исправлению для ИИ. Создайте собственные правила в каталоге `lints/`, используя стандартный подход для каждого языка: Rust (dylint), TypeScript/JavaScript (ESLint), Python (pylint), Go (golangci-lint) и т. д. Используйте это, когда: (1) вы хотите применить к ИИ правила кодирования, специфичные для проекта, (2) вы хотите написать правила lint, которые выдают читаемые AI инструкции по исправлению нарушений, (3) вы хотите обеспечить соблюдение соглашений об именах, структурных шаблонов или правил согласованности посредством Линтинг на основе искусственного интеллекта. Триггеры: «создать правило линтера», «добавить правило линта», «применить этот шаблон», «ИИ-линтер», «пользовательский линт», «правила кода», «правила именования», «правила структуры», «создать правило линтера», «добавить правило линта», «применить этот шаблон», «ИИ-линтер».
