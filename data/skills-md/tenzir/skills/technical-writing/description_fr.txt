Rédigez une documentation technique claire en suivant les conventions de style de Tenzir. À utiliser lors de l'écriture ou de la modification de documents, de fichiers README, de documentation API, de guides d'utilisation, de descriptions de journaux de modifications ou de messages d'erreur. À utiliser également lorsque l'utilisateur pose des questions sur le style d'écriture, le ton, le formatage, les règles de capitalisation, le langage inclusif ou la manière de structurer le contenu technique, même s'il ne mentionne pas explicitement la « documentation ».
