Redactar documentación técnica clara siguiendo las convenciones de estilo de Tenzir. Úselo al escribir o editar documentos, archivos README, documentación de API, guías de usuario, descripciones de registros de cambios o mensajes de error. Úselo también cuando el usuario pregunte sobre el estilo de escritura, el tono, el formato, las reglas de uso de mayúsculas, el lenguaje inclusivo o cómo estructurar el contenido técnico, incluso si no menciona la "documentación" explícitamente.
