Überprüfen Sie Codeänderungen in Tenzir-Projekten. Verwenden Sie diese Option, wenn Sie Diffs oder Pull-Requests auf Fehler, Sicherheitsprobleme, fehlende Tests, Dokumentationsabweichungen, Lesbarkeitsprobleme, Leistungseinbußen, Benutzererfahrungsprobleme prüfen oder entscheiden, wie auf GitHub-Bewertungskommentare reagiert werden soll. Verwenden Sie diese Fähigkeit auch immer dann, wenn der Benutzer „Überprüfen“, „Sehen Sie sich diese PR an“, „Meine Änderungen überprüfen“, „Diesen Unterschied prüfen“, „Was halten Sie von diesem Code“ sagt oder um Feedback zu Code bittet, den er geschrieben oder geändert hat – auch wenn er nicht ausdrücklich „Codeüberprüfung“ sagt.
