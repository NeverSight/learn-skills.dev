Tenzir プロジェクトのコード変更を確認します。バグ、セキュリティ問題、テストの欠落、ドキュメントのずれ、可読性の問題、パフォーマンスの低下、ユーザー エクスペリエンスの問題についての差分またはプル リクエストを監査する場合、または GitHub レビュー コメントに応答する方法を決定する場合に使用します。また、ユーザーが「レビュー」、「この PR を見て」、「変更をチェック」、「この差分を監査」、「このコードについてどう思いますか」と言ったり、自分が書いたコードや変更したコードについてフィードバックを求めたりするときに、たとえ「コード レビュー」と明示的に言わなくても、このスキルを使用します。
