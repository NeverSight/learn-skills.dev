Coordonnez les mises à jour de docs.tenzir.com parallèlement aux modifications de code. À utiliser lors de la préparation de la caisse `.docs/`, de la création d'une branche docs correspondante, de l'ouverture d'une pull request `tenzir/docs` ou de la liaison croisée de documents et de PR de code. À utiliser également lorsque l'utilisateur mentionne « mettre à jour la documentation », « docs PR », « .docs/ » ou lorsqu'une modification de code affecte le comportement face à l'utilisateur qui doit être reflété sur docs.tenzir.com.
