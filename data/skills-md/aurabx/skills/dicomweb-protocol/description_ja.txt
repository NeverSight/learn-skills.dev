WADO-RS (取得)、STOW-RS (ストア)、および QIDO-RS (クエリ) プロトコルを使用して、DICOMweb クライアントと統合を構築します。マルチパート MIME エンコーディング、コンテンツ ネゴシエーション、クエリ パラメータ、および認証を処理します。 DICOMweb クライアントの構築、DICOMweb サーバーへのクエリ、STOW-RS 経由の DICOM アップロード、WADO-RS 経由のイメージングの取得、または Aurabox、Orthanc、dcm4chee、Google Cloud Healthcare API などの DICOMweb 準拠システムとの統合の場合に使用します。
