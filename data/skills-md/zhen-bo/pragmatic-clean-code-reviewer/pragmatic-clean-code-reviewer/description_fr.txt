Révision stricte du code selon les principes Clean Code, Clean Architecture et The Pragmatic Programmer. À utiliser lorsque : (1) l'examen du code ou des demandes d'extraction, (2) la détection d'odeurs de code ou de problèmes de qualité, (3) l'audit des décisions d'architecture, (4) la préparation du code pour la fusion, (5) la refactorisation du code existant ou (6) la vérification du respect des principes SOLID, DRY, YAGNI, KISS. Comprend un système de questionnaires 3+4+2 pour calibrer la rigueur de L1 (laboratoire) à L5 (critique). Se déclenche également sur : "ce code est-il bon ?", "vérifier la qualité du code", "prêt à fusionner ?", "dette technique", "odeur de code", "meilleures pratiques", "nettoyer le code", "révision du refactor".
