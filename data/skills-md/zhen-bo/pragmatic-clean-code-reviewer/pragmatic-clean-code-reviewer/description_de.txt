Strenge Codeüberprüfung nach den Grundsätzen von Clean Code, Clean Architecture und The Pragmatic Programmer. Wird verwendet, wenn: (1) Code- oder Pull-Requests überprüft werden, (2) Codegerüche oder Qualitätsprobleme erkannt werden, (3) Architekturentscheidungen geprüft werden, (4) Code für die Zusammenführung vorbereitet wird, (5) vorhandener Code umgestaltet wird oder (6) die Einhaltung der SOLID-, DRY-, YAGNI- und KISS-Prinzipien überprüft wird. Verfügt über ein 3+4+2-Fragebogensystem zur Kalibrierung der Strenge von L1 (Labor) bis L5 (kritisch). Wird auch ausgelöst bei: „Ist dieser Code gut?“, „Codequalität prüfen“, „Bereit zum Zusammenführen?“, „Technische Schulden“, „Codegeruch“, „Best Practices“, „Code bereinigen“, „Refactor-Überprüfung“.
