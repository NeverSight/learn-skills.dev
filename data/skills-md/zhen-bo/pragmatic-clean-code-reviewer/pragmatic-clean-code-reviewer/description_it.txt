Revisione rigorosa del codice seguendo i principi Clean Code, Clean Architecture e The Pragmatic Programmer. Da utilizzare quando: (1) rivedere il codice o le richieste pull, (2) rilevare odori di codice o problemi di qualità, (3) verificare le decisioni sull'architettura, (4) preparare il codice per l'unione, (5) eseguire il refactoring del codice esistente o (6) verificare l'aderenza ai principi SOLID, DRY, YAGNI e KISS. Presenta un sistema di questionari 3+4+2 per calibrare il rigore da L1 (laboratorio) a L5 (critico). Si attiva anche su: "questo codice è buono?", "controlla la qualità del codice", "pronto per l'unione?", "debito tecnico", "odore di codice", "migliori pratiche", "pulizia del codice", "revisione del refactoring".
