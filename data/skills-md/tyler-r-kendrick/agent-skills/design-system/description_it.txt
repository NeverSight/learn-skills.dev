Da utilizzare durante la creazione o la manutenzione di un sistema di progettazione: l'insieme coordinato di token di progettazione, librerie di componenti, documentazione e strumenti che garantiscono la coerenza visiva e comportamentale tra i prodotti.
UTILIZZO PER: progettazione dell'architettura del sistema, scelta dei formati token rispetto ai framework dei componenti, connessione di Figma al codice, flussi di lavoro dalla progettazione allo sviluppo, coerenza multipiattaforma
NON UTILIZZARE PER: creazione di token specifici (usare design-token), flussi di lavoro Figma (usare figma), catalogazione di componenti (usare storybook), trasformazione di token (usare dizionario di stile), componenti cross-framework (usare mitosi)
