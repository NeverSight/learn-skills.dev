Da utilizzare quando si sceglie una strategia di test, si dimensiona correttamente la copertura del test o si comprendono le categorie di test. Copre il modello Test Trophy, i compromessi sul tipo di test e le indicazioni sul bilanciamento di analisi statica, unità, integrazione e test end-to-end.
UTILIZZO PER: strategia di test, Trofeo di test, selezione del tipo di test, copertura del test del giusto dimensionamento, bilanciamento delle categorie di test, scelta degli strumenti di test, architettura di automazione dei test
NON UTILIZZARE PER: implementazione di categorie di test specifiche (utilizzare analisi statica, test unitari, test di integrazione, e2e-testing, ecc.), creazione di specifiche BDD (utilizzare specifiche/documentazione/gherkin)
