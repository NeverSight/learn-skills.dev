GoldRush x402: accesso ai dati blockchain con pagamento per richiesta utilizzando il protocollo x402 (HTTP 402 Payment Required). Utilizza questa competenza ogni volta che l'utente sta costruendo un agente AI che necessita di dati blockchain senza chiavi API, desidera micropagamenti basati su portafoglio per dati on-chain, necessita di accesso autonomo o senza account all'API GoldRush, menziona il protocollo x402 o desidera l'accesso ai dati blockchain senza registrazione/nessun onboarding. Questa è la competenza giusta per agenti autonomi, applicazioni serverless e prototipazione senza onboarding. Fornisce accesso a oltre 60 endpoint API Foundational tramite un proxy inverso trasparente con pagamenti in stablecoin su Base. Se l'utente necessita di una chiave API tradizionale con fatturazione mensile, utilizza invece goldrush-foundational-api. Se l'utente necessita di dati di streaming in tempo reale tramite WebSocket, utilizza invece goldrush-streaming-api.
