GoldRush x402 – Pay-per-Request-Blockchain-Datenzugriff über das x402-Protokoll (HTTP 402-Zahlung erforderlich). Verwenden Sie diese Fähigkeit immer dann, wenn der Benutzer einen KI-Agenten erstellt, der Blockchain-Daten ohne API-Schlüssel benötigt, Wallet-basierte Mikrozahlungen für On-Chain-Daten wünscht, autonomen oder kontolosen Zugriff auf die GoldRush-API benötigt, das x402-Protokoll erwähnt oder keinen Anmelde-/Onboarding-Blockchain-Datenzugriff wünscht. Dies ist die richtige Fähigkeit für autonome Agenten, serverlose Anwendungen und Prototyping ohne Onboarding. Bietet Zugriff auf über 60 Foundational-API-Endpunkte über einen transparenten Reverse-Proxy mit Stablecoin-Zahlungen auf Base. Wenn der Benutzer einen herkömmlichen API-Schlüssel mit monatlicher Abrechnung benötigt, verwenden Sie stattdessen goldrush-foundational-api. Wenn der Benutzer Echtzeit-Streaming-Daten über WebSocket benötigt, verwenden Sie stattdessen goldrush-streaming-api.
