GoldRush x402: acceso a datos de blockchain de pago por solicitud utilizando el protocolo x402 (se requiere pago HTTP 402). Utilice esta habilidad siempre que el usuario esté creando un agente de IA que necesite datos de blockchain sin claves API, quiera micropagos basados ​​en billetera para datos en cadena, necesite acceso autónomo o sin cuenta a la API GoldRush, mencione el protocolo x402 o quiera acceso a datos de blockchain sin registro o incorporación. Esta es la habilidad adecuada para agentes autónomos, aplicaciones sin servidor y creación de prototipos sin incorporación. Proporciona acceso a más de 60 puntos finales de API fundamentales a través de un proxy inverso transparente con pagos de monedas estables en Base. Si el usuario necesita una clave API tradicional con facturación mensual, utilice goldrush-foundational-api en su lugar. Si el usuario necesita transmisión de datos en tiempo real a través de WebSocket, utilice goldrush-streaming-api en su lugar.
