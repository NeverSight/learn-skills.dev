GoldRush x402 — accès aux données blockchain au paiement par demande à l'aide du protocole x402 (paiement HTTP 402 requis). Utilisez cette compétence chaque fois que l'utilisateur crée un agent d'IA qui a besoin de données blockchain sans clés API, souhaite des micropaiements basés sur un portefeuille pour les données en chaîne, a besoin d'un accès autonome ou sans compte à l'API GoldRush, mentionne le protocole x402 ou souhaite un accès aux données blockchain sans inscription ni intégration. Il s'agit de la compétence idéale pour les agents autonomes, les applications sans serveur et le prototypage sans intégration. Fournit un accès à plus de 60 points de terminaison d’API fondamentaux via un proxy inverse transparent avec des paiements stables sur Base. Si l'utilisateur a besoin d'une clé API traditionnelle avec facturation mensuelle, utilisez plutôt goldrush-foundational-api. Si l'utilisateur a besoin de données en streaming en temps réel via WebSocket, utilisez plutôt goldrush-streaming-api.
