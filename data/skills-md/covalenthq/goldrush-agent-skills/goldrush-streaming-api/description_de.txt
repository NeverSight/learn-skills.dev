GoldRush Streaming API – Echtzeit-Blockchain-Daten über GraphQL-Abonnements über WebSocket. Verwenden Sie diese Fähigkeit immer dann, wenn der Benutzer Live-Preis-Feeds (OHLCV-Kerzen), Echtzeit-DEX-Paarüberwachung (neue Paare, Liquiditätsaktualisierungen), Wallet-Aktivitäts-Streaming, entschlüsselte Swap-/Transfer-Ereignisse, Token-Suche, Händler-PnL-Analyse oder einen Blockchain-Ereignis-Push mit einer Latenz von unter einer Sekunde benötigt. Dies ist die richtige Fähigkeit für Trading-Bots, Live-Dashboards, Warnsysteme, Copy-Trading, DEX-Sniping und Echtzeitanalysen. Deckt auch einmalige GraphQL-Abfragen zur Token-Erkennung und Rentabilitätsanalyse ab. Wenn der Benutzer historische Daten, Batch-Abfragen oder paginierte REST-Ergebnisse benötigt, verwenden Sie stattdessen goldrush-foundational-api. Wenn der Benutzer einen Pay-per-Request-Zugriff ohne API-Schlüssel benötigt, verwenden Sie stattdessen goldrush-x402.
