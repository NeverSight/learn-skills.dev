Rufen Sie mit dem Dienst defuddle.md lesbaren Textinhalt von jeder URL ab. Verwenden Sie diese Fähigkeit immer dann, wenn Sie Inhalte aus URLs extrahieren müssen, die schwer direkt abzurufen sind – z. B. x.com/twitter-Beiträge, Seiten hinter der JavaScript-Wiedergabe oder Websites, die Standardabruffunktionen blockieren. Verwenden Sie es auch als allgemeinen Fallback, wenn WebFetch fehlschlägt oder unbrauchbare Ergebnisse zurückgibt. Auslöser bei: Lesen von Tweets/X-Posts, Abrufen von Artikelinhalten aus heiklen Domains, „Text von diesem Link abrufen“ oder jeder URL, bei der das direkte Abrufen unzuverlässig ist.
