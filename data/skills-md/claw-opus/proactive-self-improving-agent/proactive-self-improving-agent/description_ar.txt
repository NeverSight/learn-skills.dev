التقاط الخبرة تلقائيًا والتطور بأمان. شروط الزناد:  
(1) فشل الأمر/الإجراء → قم بتسجيل الدخول إلى ERRORS.md  
(2) يقوم المستخدم بتصحيحك ("خطأ" / "يجب أن يكون") → قم بتسجيل الدخول إلى LEARNINGS.md  
(3) يحتاج المستخدم إلى قدرة مفقودة → قم بتسجيل الدخول إلى FEATURE_REQUESTS.md  
(4) أخطاء واجهة برمجة التطبيقات/الأداة الخارجية → قم بتسجيل الدخول إلى ERRORS.md  
(5) ابحث عن المعرفة القديمة/غير الصحيحة ← قم بتسجيل الدخول إلى LEARNINGS.md  
(6) اكتشف ممارسة أفضل ← قم بتسجيل الدخول إلى LEARNINGS.md  
(7) تنتهي كل مهمة ← عملية المراجعة؛ قم بتسجيل دروس جديدة في LEARNINGS.md  

إلغاء البيانات المكررة: قم بالتخطي إذا لم تكن هناك رؤية جديدة أو تمت تغطيتها بالفعل.  
في كل عملية كتابة، قم بإلحاق إدخال JSONL بـ .learnings/CHANGELOG.md.  
بعد تكرار ≥3، قم بتشجيع الدخول إلى AGENTS.md / TOOLS.md / SOUL.md.  
انظر النص الرئيسي للحصول على التفاصيل.
