Automatically capture experience and safely evolve skills.  
Trigger conditions:  
(1) Command/operation fails → log to ERRORS.md  
(2) Corrected by user (“wrong” / “should be”) → log to LEARNINGS.md  
(3) User needs a non-existent capability → log to FEATURE_REQUESTS.md  
(4) External API/tool error → log to ERRORS.md  
(5) Find own knowledge outdated/incorrect → log to LEARNINGS.md  
(6) Discover a better practice → log to LEARNINGS.md  
(7) Each task completed → review process, log new experience to LEARNINGS.md.  

Deduplication: skip if no new experience or already covered.  
Every write appends a JSONL entry to .learnings/CHANGELOG.md.  
Experience recurring ≥3 times is promoted to AGENTS.md/TOOLS.md/SOUL.md.
