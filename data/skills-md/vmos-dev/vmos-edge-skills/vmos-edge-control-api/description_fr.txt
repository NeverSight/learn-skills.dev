Utilisez cette compétence lorsque vous contrôlez un téléphone cloud Android VMOS Edge via l'API de contrôle Android. Si la session en cours dispose également de « vmos-edge-container-api », utilisez « host_ip » et résolvez d'abord la cible « db_id » via les API du conteneur. Si cette compétence est installée seule, préférez `cloud_ip` ; si `host_ip` et `db_id` sont déjà connus, la route hôte peut également être utilisée. Si aucune IP n'est fournie et que « cbs_go » local est en cours d'exécution, « host_ip » par défaut est « 127.0.0.1 ».
